Blackroot — Sprite Atlas Baker

Bakes the procedural character/prop renderers into PNG sprite atlases. The runtime (SpriteAtlasSystem) loads assets/atlases/entities.json and blits frames with a single drawImage per actor instead of re-drawing ~140 vector operations every frame. While that folder holds only the placeholder manifest, everything falls back to procedural drawing.

Must be served over http

Opening this from file:// blocks fetch() of the script list. From the project root run python3 -m http.server 8899, then open http://localhost:8899/tools/sprite-baker.html.

1 · Load Injects the game's renderer scripts (skips game.js so no game boots).
2 · Smoke test (optional)
3 · Full bake

Bakes every definition (~10k frames). The tab will freeze for a while — that is expected. Your browser will then save several PNG pages plus entities.json; you may need to allow automatic multiple downloads for this site.

4 · Install
  1. Copy every downloaded entities_*.png into assets/atlases/.
  2. Copy entities.json there too, replacing the placeholder.
  3. Reload the game. Verify with DarkWoodsGame.spriteAtlasSystem.getStats() — framesLoaded should be non-zero and proceduralFallbacks should stop climbing.
