# V0.18.71 — Item Obtainability Audit (Roadmap Item 12)

Foundation for the itemization expansion (Item 4): before adding new items, make sure every item actually has a **reachable** way to get it. This adds the audit tool and fixes the one real gap it found.

## The audit — `core/registry.js` `auditItemObtainability()`
Exposed on `DR.Registry.auditItemObtainability()`. Cross-references every acquisition source that names items and reports items with **no reachable source** (the roadmap's "no item exists only as an unused data record" check).

Sources covered:
- **Loot tables** (incl. boss / mini-boss / chest / dungeon-reward tables) + their `rarePool` and `guaranteedPool` — by `itemId`.
- **Crafting** recipe outputs — by `itemId`.
- **Quests** — fixed `rewards.items` and player-choice `rewards.choiceItems` — by `id`.
- **Gathering** resource-node drops — by item **name** → resolved to id.
- **Fishing** catch tables (see below).
- **Class starting gear** — the `item_starter_` id convention.

Its summary is folded into `runBootValidation()`'s report (`bootReport.obtainability`) and logged once at boot as a `console.info` (not a warning). **Vendor stock is tag-based** (no explicit item list), so it's a *known uncaptured source* — the audit is a review report, not a hard failure.

To make it accurate, **`systems/fishing-system.js` now exposes its fish catch tables as `DR.FISH_TABLES`** (read-only), so a fish reads as reachable-by-fishing.

## Result on current content
**252 / 262 items sourced; 10 flagged.** The 10:
- 2 vendor **bags** (`item_linen_pouch`, `item_travelers_pack`)
- 5 gathering **tools/rods** (`item_crude_pickaxe`, `item_worn_hatchet`, `item_herbalist_knife`, `item_basic_fishing_rod`, `item_worn_fishing_rod`)
- 3 **quest-script** items (`item_hunters_journal`, `item_tarnished_coin`, `item_torn_lookout_ledger`)

All obtainable via vendor / quest-script / starting mechanisms the data-level audit doesn't resolve — flagged for review, not bugs.

## The gap it found (fixed)
`item_cooked_forest_stew` — a green **food** item that existed with **no recipe or any other source**. Fixed:
- `data/crafting.js` → **`recipe_cooked_forest_stew`** (campfire, `gloomleaf ×2 + gloomcap mushroom` → the stew).
- The item is upgraded to a proper food: `useEffect { hp:18, mana:8, buff: 'Warm Meal' (+6 hp, 150s, food) }` + a 12s cooldown, instead of a bare heal.

Re-audit confirms it now resolves to `craft:recipe_cooked_forest_stew`; flagged count dropped 11 → 10.
