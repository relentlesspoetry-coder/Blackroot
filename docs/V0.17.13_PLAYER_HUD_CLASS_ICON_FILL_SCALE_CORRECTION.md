## Old Class Icon Size

- The layout box and renderer requested 84×84px.
- `background-size: contain` fit the entire transparent 512×512 source canvas into that box.
- Canonical visible alpha bounds occupy only 58–65% of source height, making visible Warden art approximately 52px tall despite the nominal 84px box.

## New Class Icon Size

- The 84×84px layout box remains unchanged.
- The emblem background now renders at 140% within that box.
- Visible class art occupies approximately 81–91% of available height; Warden is approximately 73px tall, or 87%.

## Padding / Container Adjustments

- Removed the redundant later `contain !important` rule that overrode the owning player-emblem fill rule.
- Compensated for transparent source-image padding through `background-size`, not transforms or asset distortion.
- Preserved centered positioning, aspect ratio, transparent edges, and the unframed right-side presentation.
- The right column, container, HUD width, HUD height, gap, and outer padding were not expanded.

## Bar Width Adjustments

- None. The existing dedicated 84px right column already isolates class art from player text and bars.
