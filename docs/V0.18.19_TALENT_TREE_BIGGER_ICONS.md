# V0.18.19 — Talent tree: remove tier labels, bigger icons

User request: "remove the Tier text on the sides and make the talent icons bigger so
they kinda fill in each spec tree and are easier to see and read." (`systems/talent-system.js`)

## Changes
- **Removed the tier labels.** The `T1/T2/…` + gate-number column (`.talTierGate`)
  on the left of every tier row is gone (render + CSS). The gate requirement is
  still shown in the node hover tooltip ("Requires N points in <spec>"), and locked
  tiers are still greyed out. Removing the label column also frees the full column
  width for the icons.
- **Bigger, wider icons.** Talent icons **48 → 60 px** (capstones **52 → 66 px**),
  tiles **62 → 70 px**, node spacing **16 → 30 px**, and node-name font **10 → 11 px**
  — so the icons fill each tree, and are easier to see and read. The route spine +
  sibling rungs were re-measured to the new icon centres (rung `(n-1)×100`, rung
  vertical position at the 60 px icon centre; the spine is JS-positioned so it
  auto-adapts).

## Keeping it in the window (the tricky part)
Bigger icons make the 5-tier tree taller, so it risked overflowing short windows.
- Reclaimed vertical space (tier padding `3/1 → 2/0`, tighter name line-height) and
  let the panel use more of the screen (`max-height: calc(100vh - 40px)`, `top:28`).
- **Result:** at the user's 963×669 window the whole tree (3 specs × 5 tiers, 60 px
  icons) fits with **no scroll and fully on-screen** (top 28, bottom 650, left 13,
  right 950); node names uniform; tier labels gone; spine/rungs aligned.

### Note on window position
The talent panel is a draggable game window; a **dragged position is saved** and
restored (`applyUiWindowStoredPosition`). During testing a stale saved position had
been pushing it off-screen — that's per-user drag state, not a code default. The
built-in default (centred, `top:28`) fits; a fresh install has no saved position.

## Verified (in-browser, 0 console errors)
- Tier-gate labels removed; icons render at 60 px; names uniform.
- Panel fits fully in a 963×669 window with no scroll/clip.
- Spine spans first→last node centre and rungs align to the larger icons.
