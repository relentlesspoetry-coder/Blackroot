# V0.18.47 — Silk Web Caverns: web columns rebuilt to look like spider webs

Follow-up to V0.18.46. Your feedback:

> "the webs that stretch to the ceiling still do not look like spider webs it looks too stringy like white lines theres no web design to it at all."

Showcase of the rebuilt columns: [`docs/V0.18.47_columns.png`](V0.18.47_columns.png) — left→right: two thick columns, medium, small, royal thick.

## Root cause
`drawCeilingWebColumn` (`render/object-renderer.js`) built each column as a **bundle of ~26 near-parallel vertical strands** (fanning at the ends) plus horizontal rungs and a few diagonals. Near-parallel verticals read as **stringy white lines** — there was no actual web geometry (no cells, no orb).

## The fix — a woven net with embedded orb webs
`drawCeilingWebColumn` is rebuilt from scratch:

- **Width profile** — broad at the ceiling anchor and the floor, gently pinched mid-hang (like a real hanging sheet-web), scaled by the existing `thick / medium / small` spread.
- **Node grid** — a jittered grid of nodes (`R×C`: **11×8** thick, **8×5** medium, **5×3** small) across that profile.
- **Woven mesh** — the nodes are connected by **vertical radial threads** + **horizontal capture threads with catenary sag** + a **diagonal weave** across ~half the cells. That produces **irregular web cells** — an actual woven mesh, not parallel lines.
- **Embedded orb webs** — **1–2** orb webs are woven into the panel (thick 2 / medium 1 / small 0) via a new reusable **`drawOrbWeb`** helper: radial spokes tied together by a genuine **capture spiral**, a frame ring, and a centre bead — the unmistakable spider-web shape and the thing that was missing.
- **Junction beads** at the net nodes, a **ceiling anchor fan** tying up into the dark, and a **translucent silk sheet** behind the net (thick + medium) so it reads as filled silk.
- **Kept from V0.18.46**: the woven **nest** (`drawWebNest`) bulging in thick columns, clinging **egg sacs**, and **webbed bones** tangled at the base of thick columns.

**Performance**: each thread family (verticals, horizontals, diagonals, fan) is batched into a single `Path2D`/`beginPath`→`stroke`, so the richer look is only a handful of stroke calls per column.
