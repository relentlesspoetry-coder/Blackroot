# V0.16.83 - Necromancer Pet Skeleton Model Revamp Report

## Phase source
- Phase brief: `Pasted text(47).txt`
- Current baseline used: `Blackroot V0.16.82`
- Target produced: `Blackroot V0.16.83`

## Scope
This was a visual/model revamp only. Necromancer spell behavior, pet stats, pet damage, pet threat, pet AI, pet commands, pet pathing, pet duration, cooldowns, mana costs, save behavior, action bar behavior, spellbook behavior, and unrelated systems were preserved.

## Old model ownership
The existing Necromancer pet skeleton visual lived in:

- `render/pet-identity-procedural-model.js`
- Method: `PetIdentityProceduralModel.drawNecroBoneServant(ctx, actor, s)`

The project uses canvas/procedural rendering for this pet model. No true 3D mesh pipeline was present for this pet, so the high-poly brief was translated into a high-fidelity procedural Necromancer pet model using the existing renderer and animation state.

## Replacement details
The old `drawNecroBoneServant` method body was replaced in place. The new implementation remains inside the same owning renderer and keeps the same call path from `PetIdentityProceduralModel.draw()`.

New visual details include:

- cracked asymmetric skull
- one glowing green eye socket and one empty socket
- hanging jaw with uneven teeth and a gold tooth
- broken rusted helmet with a punctured temple silhouette
- exposed cervical vertebrae and hunched spine
- exposed ribcage with protruding broken ribs
- mummified flesh/sinew patches
- rusted breastplate fragment and corroded shoulder/leg armor
- ruined shield prop strapped to the far forearm
- broken rust-pitted cleaver-like longsword
- ragged grave shroud, mail belt, empty scabbard, chain trophy, and cloth decay
- lightweight necromantic rune glow and smoke-wisp strokes

## New helper code
Small drawing helpers were added in `render/pet-identity-procedural-model.js` near the existing pet-render helper functions:

- `drawNecroDecayWisp()`
- `drawNecroRustedPlate()`
- `drawNecroBrokenHelmet()`
- `drawNecroShatteredShield()`
- `drawNecroRuinedCleaver()`
- `drawNecroMummifiedPatch()`

These helpers are local drawing utilities for the existing pet renderer. They do not create a second renderer, loader, model system, or runtime override.

## Directions updated
The renderer continues to use the existing eight-direction pet state:

- north
- northeast
- east
- southeast
- south
- southwest
- west
- northwest

Weapon, shield/forearm gear, head orientation, shroud, and side-facing silhouette are driven by the existing `DIRS` direction metadata and the same actor-facing state used before this phase.

## Animation states updated
The visual revamp uses the existing pet animation state fields and timing:

- idle
- walk / shambling movement
- attack
- hit reaction
- death / collapse fade
- summon/spawn rune effect
- debug-facing display

No animation timing, attack speed, damage timing, pet AI, or command handling was changed.

## Enemy skeleton impact
Enemy skeleton mobs use `render/dark-woods-mob-procedural-model.js`, not `PetIdentityProceduralModel.drawNecroBoneServant()`. They were not changed by this phase.

## 3D/WebGL status
No 3D Prototype renderer, WebGL renderer, mesh engine, external asset dependency, or model-loader path was added or reintroduced. The removed 3D Prototype system remains absent.

## Dead/duplicate code
No shared pet renderer, skeleton helper, enemy skeleton renderer, or old visual branch was deleted. The old Necromancer pet skeleton method body was replaced in its existing owner. No dead/duplicate code removal was required.
