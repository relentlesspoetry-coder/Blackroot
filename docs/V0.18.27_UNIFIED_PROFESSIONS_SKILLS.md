# V0.18.27 — One unified "Professions & Skills" window

User (with screenshots of two overlapping panels):

> "We need to combine both these menus together, as of right now pressing k will
> open them one at a time. Pressing K should only show 1 window with both
> professions and skills, then pressing k again should close the window."

## Background
Two near-duplicate windows existed:

- **Professions** (`systems/profession-system.js`, `#professionWindow`) — Gathering,
  Mining, Fishing, Cooking, Blacksmithing, Tailoring.
- **Skills** (`systems/skills-system.js`, `#skillsPanel`) — the same six **plus
  Meditating**.

Both read the same underlying data (`resourceGatheringSystem.state.skills` +
`craftingSystem.state.skills`), so the only unique thing Skills added was
**Meditating**.

Only the profession system bound **K**. Its `toggle()` opened *both* windows as a
side-by-side "paired view" (`layoutProfessionsAndSkillsWindows`) — but it had **no
`else` branch to close the Skills window**, so after the first toggle the two
drifted out of sync, which read as "opening one at a time".

## Fix — a single window
Made the Professions window the one unified window and retired the standalone
Skills window.

**`systems/profession-system.js`**
- `collectRows()` now appends a **Meditating** row, pulled from
  `game.collectSkillRows()` so the meditation level/XP curve stays owned by the
  skills system (single source of truth).
- Window retitled **"Professions & Skills"** (now 7 rows); summary reads
  "N tracked · N total levels".
- `toggle()` now simply shows/hides the single `#professionWindow`, and defensively
  force-hides the retired standalone skills panel (`game.skillsOpen = false`).
  No more opening a second window or `layoutProfessionsAndSkillsWindows` pairing,
  so **K opens one window and K closes it**.

**`systems/skills-system.js`**
- `toggle()` now **delegates to `game.toggleProfessionPanel(force)`**, so the menu's
  Skills button and the gamepad shortcut open the *same* unified window — never a
  second one. (Legacy standalone-panel path kept only as a fallback if the
  professions UI is somehow absent.)
- `renderSkillsPanel` also calls `professionSystem.update()` so live XP gains
  refresh the visible unified window.

**`systems/ui-system.js`**
- The periodic UI refresh keyed off the now-dead `skillsOpen` flag; it now refreshes
  `professionSystem.update()` while the unified window is open.

## Verification
- `node --check` on all three files — OK.
- Driving the toggles live:
  - `toggleProfessionPanel(true)` **and** `toggleSkillsPanel(true)` → exactly **one**
    visible window (`#professionWindow`, title "Professions & Skills", **7 rows**
    including Meditating); the standalone skills panel stays hidden.
  - `toggleProfessionPanel()` again → window **fully closes**; repeated open/close
    toggles stay clean.
- Boots with **0 console errors**.

## Files touched
- `systems/profession-system.js` — Meditating row; retitled window; single-window toggle.
- `systems/skills-system.js` — toggle delegates to the unified window; render refresh.
- `systems/ui-system.js` — periodic refresh targets the unified window.
