# V0.17.70 — Damage-Type → Combat School Map (Talent spec BUG 2)

**Type:** Pre-existing bug fix, shipped as its own pass (Talent System v1 spec, PART 8 BUG 2 / PART 9)
**Previous version:** V0.17.69
**New version:** V0.17.70

Found while auditing the spell pipeline for the talent system. Must land before the caster classes'
talent trees are authored, because those trees add `when: { damageType: 'fire' }`-style nodes that
would otherwise sit on a broken foundation.

---

## The bug

`data/spells.js` ships ~15 damage types beyond `physical`/`holy`/`true`: `fire`, `frost`, `nature`,
`arcane`, `lightning`, `earth`, `sonic`, `psychic`, `lunar`, `planar`, `spirit`, `primal`, `bone`,
`disease`, `poison` (plus the literal `magic` and `shadow`).

The combat "school" logic checked the damage type **literally** (`damageType === 'magic'`) in three
places, so every one of those types fell through to the **physical** branch:

- `core/stats.js` `outgoingMultiplier` → used `physicalDamageMultiplier` (magicPower / magicDamagePct
  never applied to a Fireball).
- `core/stats.js` `critProfile` → crit rolled off the **physical** crit table (a Wizard's Fireball
  crit off dexterity/`critChance` instead of intelligence/`magicCritChance`).
- `combat-system.js` `damageEntity` → mitigation used armor only (never layered magicResist), and the
  `magicDamageTakenMultiplier` buff check missed them.

## The fix (one table, in the owning module)

`core/stats.js` gains a damage-type → school map and two helpers, exported on `DR.StatSystem`:

```js
const MAGIC_DAMAGE_TYPES = new Set(['magic','fire','frost','arcane','lunar','planar','psychic',
  'nature','spirit','lightning','earth','sonic','primal','bone','disease','poison','shadow']);
damageSchoolFor(damageType) // -> 'magic' | 'holy' | 'physical' | 'true'
isMagicSchool(damageType)   // damageSchoolFor(...) === 'magic'
```

- `outgoingMultiplier` and `critProfile` now branch on `isMagicSchool(damageType)`.
- `combat-system.js` `damageEntity` uses `DR.StatSystem.isMagicSchool(...)` for the magicResist
  mitigation branch and the `magicDamageTakenMultiplier` buff check (holy stays grouped with magic
  for that buff, as before).

### Deliberately unchanged
- **holy** → stays on the physical crit/output path ("existing" per spec). This keeps the V0.17.69
  Paladin talent slice (holy + physical) completely unaffected.
- **physical** / **true** → unchanged (`true` still bypasses all mitigation).
- **unknown / auto-attack default** → `physical` (safe, matches prior behavior).

## Balance note

Mitigation is numerically **inert today** — `magicResist` is 0 on every shipped class/item, so magic
still reduces to armor-only. The **live** change is that caster elemental/shadow/nature/etc. spells
now correctly crit off `magicCritChance` and scale with magic power/`magicDamagePct`. That is a real
DPS shift for casters (the correct one). A numbers-tuning pass should accompany authoring the caster
talent trees.

## Verification (in-browser, port 8899)

- `damageSchoolFor`: all 15+ magic-flavored types → `magic`; `holy`/`physical`/`true` map to
  themselves; unknown → `physical`.
- With a caster source (`magicCritChance 0.30`, `magicDamageMultiplier 1.40`): `fire` crit = 0.30 /
  outgoing = 1.40 (magic path); `holy` = 0.05 / 1.00 and `physical` = 0.05 (unchanged).
- Live `damageEntity`: a 200-base `fire` hit dealt 305 (routed through the Wizard's magic multiplier)
  vs 204 for `holy`/`physical`; no errors thrown.

## Not in this pass

Paged action bar (BUG 1, unlocks levels 10–20), the other 13 talent classes, and the caster balance
tuning that this fix's DPS shift will motivate.
