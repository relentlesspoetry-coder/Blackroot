# V0.17.06 - Mini-Map Window Cleanup + HUD Panel Revamp Report

## Phase
Mini-Map Window Cleanup + HUD Panel Revamp Pass.

The provided phase brief referenced V0.17.03 -> V0.17.04, but the active baseline in this workspace was V0.17.05. This update was applied as V0.17.06 to preserve version continuity.

## Ownership audited

- Mini-Map window ownership lives in the static HUD markup and CSS in `Blackroot V0.17.06.html` under `#minimapWrap`.
- Minimap rendering ownership remains in `render/minimap-renderer.js`; this pass did not change map marker drawing, player marker drawing, or dungeon marker rendering.
- Dungeon info display ownership remains in `systems/dungeon-system.js` through `#dungeonInfoHudPanel`, anchored inside `#minimapWrap`.
- Time display ownership remains in the existing time-cycle runtime using `#timeCycleClock`, `#worldClockTime`, and `#worldClockPhase`.
- Weather display ownership remains in `systems/weather-system.js` using `#weatherPanel`, `#weatherStatus`, `#weatherClimate`, and `#weatherCycleBtn`.
- The broken FPS counter lived in the Mini-Map DOM as `#fpsCounter`, with update wiring in `game.js` through `updateFpsCounter()`.

## Root cause

The Mini-Map HUD had accumulated multiple vertically stacked diagnostic and status widgets in one fixed panel: minimap canvas, dungeon card, static zone line, clock card, weather card, debug Cycle Weather button, and a Mini-Map-local FPS card. The FPS card was visually part of the Mini-Map panel even though real performance diagnostics already belonged to the F3/debug overlay. This made the Mini-Map look like stacked debug widgets instead of a compact HUD panel.

## Changes made

- Removed the `#fpsCounter` DOM node from the Mini-Map panel.
- Removed Mini-Map FPS CSS and responsive/Steam Deck FPS sizing rules.
- Kept FPS measurement alive in `game.js` for F3/debug overlay diagnostics, but no longer requires a Mini-Map FPS element to calculate `currentFps` / `displayFpsEma`.
- Reworked `#minimapWrap` into a compact BlackRoot-style card with a consistent fixed width, reduced padding, compact gaps, rounded border, and dark/gold HUD treatment.
- Kept the minimap canvas itself and preserved marker rendering ownership in `render/minimap-renderer.js`.
- Kept dungeon info under the minimap and compacted its spacing and typography.
- Replaced the inline static zone line style with `#minimapZoneLine.minimapInfoLine`.
- Tightened time and weather card spacing, fonts, and borders.
- Changed Cycle Weather to a debug-only HUD control shown when F3/debug overlay or performance overlay is active.
- Preserved weather-cycle behavior and event binding in `systems/weather-system.js`.

## FPS display handling

The broken FPS display was removed from the Mini-Map window. FPS diagnostics were not removed globally. `game.js` still measures presentation cadence for debug overlay lines such as `FPS:` and performance diagnostics. This preserves useful debug/performance systems while removing the inaccurate/noisy Mini-Map FPS panel.

## 1440p and responsive behavior

The revised panel uses a 204px desktop HUD width with a 184px minimap canvas display, compact section gaps, and smaller typography. Existing fixed HUD anchoring remains `right: 12px; top: 12px`. Smaller-screen and Steam Deck CSS paths were updated to shrink the panel and minimap together without relying on the removed FPS block.
