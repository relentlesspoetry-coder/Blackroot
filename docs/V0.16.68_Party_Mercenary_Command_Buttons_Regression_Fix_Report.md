# V0.16.68 - Party Mercenary Command Buttons Regression Fix

## Summary
Fixed the Party / Companions HUD regression where controllable mercenary/bot command buttons were still missing from the in-player companion panel. The HUD now renders compact per-unit command buttons directly under each controllable companion's HP/mana bars and routes clicks through the existing mercenary/bot command system.

## Root Cause
The previous pass added command controls to the legacy `partyPanel` member-row renderer in `systems/party-system.js`, but the visible screenshot panel is the unified in-player companion HUD owned by `systems/ui-system.js` (`renderUnifiedCompanionHud` / `companionHudFrame`). Bot rows in that unified HUD were rendered as `partyBot` entries without `commands`, and the unified HUD click binding only handled mercenary and pet command attributes. As a result, the active Party / Companions HUD never displayed usable bot/merc command buttons.

## Party / Companions Render Changes
- Added `unifiedCompanionCommandDefs()` for the visible unified HUD command set:
  - Follow
  - Guard
  - Assist
  - Passive
  - Attack
- Added `unifiedCompanionActiveCommand()` to resolve active command state from existing bot/merc fields.
- Added `unifiedCompanionCommandRow()` to render command buttons for controllable `bot` and `merc` entries only.
- Updated `companionHudFrame()` so command buttons render directly after HP/mana/EXP bars and before status icons.
- Updated `buildUnifiedCompanionHudEntries()` so bot/merc roster entries include command metadata and active command state.

## Input / Hitbox Changes
- Updated `bindUnifiedCompanionHud()` to handle `[data-unified-companion-command]` buttons.
- Buttons pass the row’s `data-party-member-id` plus command value to `handlePartyCompanionCommand()`.
- `handlePartyCompanionCommand()` remains the existing party command bridge and calls the existing owned setters:
  - mercenary: `setMercCommand()` through party command handling
  - bot: `setBotCompanionCommand()`

## Command Setter Used
No duplicate command state was added. The visible command buttons route through the existing party command handler and existing mercenary/bot command owners.

## Layout / Scroll Notes
- Buttons are rendered inside each companion row, directly under HP/mana bars.
- `.unifiedCommandRow` was adjusted to a compact five-button grid so all required commands fit in one row at the current HUD width.
- Non-controllable party entries do not receive command buttons.
