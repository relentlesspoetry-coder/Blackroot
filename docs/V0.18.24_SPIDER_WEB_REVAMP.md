# V0.18.24 — Spider-web visual revamp

User (with three in-game screenshots):

> "The spider webs look like complete garbage. Revamp them so that they actually
> look like a spider web, also make them look like they're attached to whatever
> object or surface they're being stuck to."

## Root cause
Every web was hand-rolled the same weak way: **perfect concentric ellipses** plus
**evenly-spaced radial spokes**. Concentric circles + even spokes read as a
radar/target, not a web, and nothing tied the web to the surface it sat on. The
offenders shown:

- `drawCaveWeb` — the big ground "radar" web (3 ellipse rings + 6 even spokes + a floating dot).
- `drawWebbedBush` — 3 straight strands + one partial ellipse.
- `drawDeadTree` `obj.webbed` overlay — the same 3 strands + partial ellipse in the tree crown.

## Fix — one reusable orb-web primitive
New `drawOrbWeb(cx, cy, opts)` in `render/object-renderer.js` builds a believable
orb web:

- **Off-centre hub** (real webs rarely centre it).
- **Irregular radial spokes** — angle and length jittered per spoke.
- **Bowed outer frame** connecting spoke tips.
- **Spiral capture thread** — for each ring, the segment between two consecutive
  spokes is a `quadraticCurveTo` whose control point is pulled **toward the hub**,
  so every segment **sags**. That concave sag is the thing that reads as a web
  instead of a stack of circles.
- **Anchor / bridge threads** — optional `anchors` (surface points in screen
  space) each get a support thread from the nearest frame vertex, tying the web
  to whatever it hangs on.
- **Silky sheen** (a faint wide pass + a crisp thin pass), a small hub, and
  optional **dew** droplets.
- **Deterministic** per `seed` so a static web doesn't shimmer every frame.

```js
drawOrbWeb(cx, cy, opts = {}) {
  // rx/ry, spokes, rings, tilt, alpha, color, sag, anchors, dew, seed ...
  // 1. off-centre hub + irregular spoke endpoints (seeded jitter)
  // 2. traceWeb(): spokes -> bowed frame -> sagging capture spiral
  // 3. anchor bridge threads (behind), then wide sheen + thin crisp pass
  // 4. hub + dew
}
```

Rewired the three shown webs through it:

| Web | Anchors (attachment) |
|---|---|
| `drawCaveWeb` (ground web) | 5 strands out to the surrounding ground/rock; kept the accent as a **silk-wrapped egg sac** tied onto a capture ring (was a floating dot) |
| `drawWebbedBush` | frame tied to the bush's outer **sprig tips** |
| `drawDeadTree` `obj.webbed` | orb strung in the crown, tied to the **branch tips** |

## Also fixed: a V0.18.21 scale bug on webs
V0.18.21's `overworldObjectScale` regex matched `caveWeb` on the substring
`cave` (and other web props on `gate` / `rack`) and was enlarging webs **2.2×**
like built structures. Added an early guard and narrowed `cave`:

```js
if (/web|silk|cocoon|spider/.test(key)) return 1;          // webs are decals
if (/… |caveentrance|cavemouth| …/.test(key)) return 2.2;  // was: |cave|
```
