# V0.18.38 — Silk Web Cavern: infested & dangerous

**Report:** *"The webs on the walls still look super bad… I want thick webs layer on webs on webs, the entire floor should mostly be webs with tiny paths still showing the actual ground floor of dirt. The dungeon still doesn't look like it's infested. Also the baby spiders that spawn from breaking eggs you can't even see them — they need tiny spider models. Eggs breaking should have an animation. I want spiders to web down from the cave ceiling randomly while walking around. We want bigger bodies wrapped in webs that you can break and an animation shows the skeleton falling out. This dungeon needs to feel scary and dangerous."*

Six pieces, each in its owning module. No existing gameplay, art, data, or other systems were removed.

---

## 1 — Floor: MOSTLY webs, thin dirt paths
`render/terrain-renderer.js` → `drawSilkWebCavernSurfaceDetails` (rewritten). Each web tile now layers **webs on webs**: a silk **film base** so the ground reads as covered, a **back web layer** (cooler offset threads), a **dense front lattice** strung between the tile's shared diamond corners, and an **orb-web hub** on most tiles. A coherent low-frequency mask (coarse 3×3 blobs + fine detail) carves **thin winding dirt paths** through it (~17% of tiles show ground). Measured ~9355 web px over a 3×3 patch.

## 2 — Walls: thick layered silk curtains
`render/terrain-renderer.js` → `drawSilkCavernWallWebs` (rewritten). Was a few thin loose strands. Each exposed wall face now gets a **translucent web sheet** + **10 dense vertical drape strands** falling most of the way down + **horizontal binding threads** weaving them into a mesh + **dangling silk beads** + a **corner orb web**. Measured ~5773 px for two faces (was ~735).

## 3 — Baby spiders: tiny, visible hatchlings
Spiderlings hatched from eggs (`makeBabySpider` in `systems/dungeon-system.js`) render at **modelScale 0.46** (~64×40 px vs 139×87 full-size — measured). **Root fix:** the runtime sprite cache keys by `rendererId`, so babies were sharing (and inheriting the size of) the full-grown `spider:skitterer` sprite. `render/entity-renderer.js` now gives spiderlings a distinct **`spider:<role>:baby`** cache key. Eggs hatch a small swarm; ambient ceiling spiders are a touch bigger (0.56).

## 4 — Egg-break animation
New **`silkEggBurst`** VFX (`render/effects-renderer.js`): shell shards fly outward and fall under gravity while a silk puff bursts and fades, plus snapping silk strands. Played by `smashSpiderEgg` when an egg is broken.

## 5 — Ceiling spiders drop on you
`updateSilkWebHazards` now runs an ambient timer (randomised 13–29 s) that calls `maybeDropCeilingSpider`: it picks a walkable tile a few tiles from the player, spawns a small spider there, and plays the new **`silkDrop`** VFX — a silk thread from the ceiling with a tiny spider sliding down it. Capped at 4 active ambient droppers so it stays a scare, not a swarm.

## 6 — Breakable web-wrapped bodies → skeleton
New **`webWrappedBody`** object (`render/object-renderer.js` `drawWebWrappedBody`): a full-size victim cocooned upright in silk, hung into the webs with anchor threads. Walking into it triggers `breakWebWrappedBody` (via the same proximity tick as the eggs), which plays the new **`silkSkeletonFall`** VFX (silk shreds burst; a skeleton falls out and collapses onto the floor), swaps it for a torn **`brokenWrappedBody`** (empty shell + skeleton crumpled on the ground), and sometimes drops loot. Placed per run with consumed-key persistence so it stays broken.

## Important: floors regenerate on entry
Dungeon floors are generated fresh each entry, so the new webbing and objects will **not** appear on a floor you're already standing in — **re-enter the Silk Web Cavern (or change floors)** to see it.
