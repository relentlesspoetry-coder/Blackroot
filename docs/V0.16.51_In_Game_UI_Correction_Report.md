# V0.16.51 - In-Game UI Correction Report

## Summary
Focused correction pass for the currently broken in-game UI areas from the V0.16.50 build sequence. The pass only touched the Character HUD, Bag window, Action Bar, Character Stats window fit/layout, and Character Stats model viewport background.

## Root causes found
- The Character HUD crest was forced to use the sprite-sheet background-size value against individual crest PNGs, causing distorted/cropped class icons.
- The Character HUD inherited the oversized mockup scale from the prior visual revamp rather than gameplay-sized HUD bounds.
- The Bag panel and item cards were still using oversized inventory-card dimensions, making the panel dominate the gameplay screen.
- The Character Stats header was rebuilt from `ui.characterSummary.parentElement`; after the first render that parent changed to the inserted title block, causing class crest markup to recursively accumulate across the header.
- The Character Stats sheet used excessive vertical spacing in equipment/stat cards, causing lower content such as Set Bonuses to be clipped on desktop viewports.
- The paper-doll viewport had no class-themed backdrop despite the class preview systems already owning class background/emblem paths.

## HUD size/icon fixes
- Reduced the HUD grid, medallion, title, meta text, and HP/Mana bars to practical gameplay sizing.
- Corrected HUD crest CSS to render the individual class crest PNG with `background-size: contain` rather than sprite-sheet sizing.
- Updated HUD runtime emblem application from 108px to 72px to match the corrected medallion bounds.

## Bag window scale fixes
- Reduced `#bagPanel` width and top/max-height bounds.
- Reduced the inventory grid minimum card width, card height, icon scale, and direct-stat text scale.
- Preserved existing bag item cards, hover tooltips, and interaction hooks.

## Action bar renderer changes
- Kept the canonical `#hotbar.abilityBar` and `.actionSlot` renderer.
- Strengthened the dark metal/gold frame treatment, integrated EXP bar accents, slot inner frames, purple active/cooldown glow, and centered Meditate button styling.
- Preserved the existing slot order, keybinds, cooldown classes, Auto Attack active class, and spell icon path.

## Character Stats layout fixes
- Repaired header ownership so the direct header identity container is rewritten, preventing recursive class-icon spam.
- Tightened panel padding, section title spacing, equipment slot sizes, stat card sizes, stat-grid gaps, and set-bonus padding so content fits without hiding the bottom section.
- Preserved all equipment slots, stat cards, gear score, attributes, status, and set bonus content.

## Model viewport background changes
- Added a class-themed backdrop layer inside the existing paper-doll viewport using the existing class background and class emblem path helpers.
- Kept draw order behind the existing character canvas and preserved full-body paper-doll rendering.
- Increased paper-doll fit scale/anchor to use the viewport more effectively without clipping weapons, offhands, head, or feet.
