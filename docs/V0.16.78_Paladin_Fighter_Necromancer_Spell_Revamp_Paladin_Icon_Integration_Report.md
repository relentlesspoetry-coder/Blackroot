# V0.16.78 - Paladin + Fighter + Necromancer Spell Revamp and Paladin Icon Integration Pass

## Source files used
- `Paladin Spells(1).txt`
- `Fighter Spells(1).txt`
- `Necromancer Spells(1).txt`
- `Paladin Spell Icons(1).zip`

## Implementation notes
- Current working baseline was V0.16.77, so this patch advances to V0.16.78 even though the uploaded phase text referenced an older V0.16.75 -> V0.16.76 sequence.
- Paladin spell data was updated in `data/spells.js`.
- Necromancer spell data was updated in `data/spells.js`.
- Fighter spell data and behavior were already aligned in the current baseline; this phase preserved the existing Fighter Stamina/Momentum implementation and compatibility.
- Paladin icon assets were added to `assets/spell-icons/paladin/` and registered in `systems/spell-system.js`.

## Behavior owners changed
- `systems/spell-system.js` now owns Paladin revamp spell resolution through `resolvePaladinRevampSpell()`.
- `systems/spell-system.js` now refreshes Paladin and Necromancer spell drafts from canonical spell data.
- `systems/spell-system.js` was updated with Necromancer ID compatibility for the new spell names: Bone Splinter, Raise Skeleton, Bone Armor, Lich Veil, Summon Bone Servant, Command Undead, and Grave Sovereign.
- `systems/status-effect-system.js` now preserves small owner-level status fields required by Paladin protection and Necromancer empowerment effects.
- `systems/combat-system.js` now supports direct-hit charge consumption and Paladin Blessed Intercept damage redirection in the existing damage owner.

## Paladin icon integration
- Used `Paladin Spell Icons(1).zip`.
- The ZIP contained `manifest.txt`; it was preserved in `assets/spell-icons/paladin/`.
- A normalized `manifest.csv` was generated for the project asset structure.
- All 20 Paladin icons mapped exactly by spell name and level.

## Mechanics fully implemented or owner-supported
- Paladin: high-threat holy strikes, taunt, Judged threat mark, cleanse, interrupt, self/group buffs, absorbs, direct-hit charges, anti-undead control/protection, cheat-death, and defensive capstone status.
- Necromancer: undead pet summons, pet command spell routing, pet repair, disease DoTs, curse DoTs/debuffs, drains, Rot Cloud, temporary skeletons, Death Pact, Lich Veil, Soul Leech channel-style data, and Grave Sovereign pet empowerment.
- Fighter: current baseline Stamina/Momentum spell revamp preserved.

## Partial engine limitations documented
- Paladin tank threat is applied by explicit threat bonuses and taunt calls in the spell owner. The generic `damageEntity()` threat multiplier field remains unchanged for unrelated spells.
- Some advanced boss-specific immunity distinctions depend on existing enemy rank/tag data.
- Grave Sovereign uses the current pet/status owner rather than a separate permanent pet transformation pipeline.
