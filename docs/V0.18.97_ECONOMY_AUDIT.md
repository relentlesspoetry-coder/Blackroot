# V0.18.97 ‚Äî Economy Audit: Arbitrage Safety + Value-Ladder Coherence (Roadmap Item 11)

Roadmap Item 11 (Economy and Progression Balance Pass) asks for a coherent level 1‚Äì10 economy with
documented "expected gold value" per category, sane vendor prices and sell values, and no uncontrolled
gold generation. This version makes the economy **measurable**: a read-only audit that documents the
value structure and guards the two properties that actually matter.

## The economy, documented

| | formula | owner |
|---|---|---|
| **Buy** (what you pay a shop) | `sellValue √ó 2` | `systems/event-system.js` `itemPrice` |
| **Sell-back** (what you receive) | `sellValue √ó rarityMult √ó (1 + (level-1) √ó 0.08) √ó 0.35` | `Game.prototype.itemSellValue` |

## The audit ‚Äî `auditEconomy()`

New `Game.prototype.auditEconomy()` (`systems/inventory-system.js`, beside `itemSellValue` /
`validateItemData`). Read-only; never throws.

### 1. Arbitrage safety ‚Äî hard-fail

For every item, buy price must exceed sell-back, or players could farm gold by buying and re-selling.

- **Result: 0 arbitrage across all 396 items.**
- The **tightest** spread in the whole catalogue is still **2.0√ó** (`item_torn_boot`: buy 2 /
  sell-back 1) ‚Äî healthy headroom everywhere.
- Structurally the buy:sell-back ratio is ~**5.7√ó** at level 1 and only narrows as `levelMult` grows;
  it would take level **~59** to invert ‚Äî far outside the level 1‚Äì10 design range. The check encodes
  this so a future change to either formula can't silently open an exploit.

### 2. Value ladder ‚Äî advisory

Median sell value must ascend by rarity tier, so "rare items feel valuable". Rarity keys are
**canonicalised** first, because the game treats `yellow`/`gold`/`orange` as one **Legendary** tier ‚Äî
on raw keys, `gold@260` looked equal to `purple@260`; canonicalised, legendary is n=4 median **600**.

- **Result: a perfectly monotonic ladder, 0 inversions.**

| tier | count | median | range |
|------|-------|--------|-------|
| junk | 3 | 1 | 1‚Äì6 |
| common | 164 | 5 | 1‚Äì40 |
| uncommon | 122 | 52 | 7‚Äì96 |
| rare | 82 | 150 | 15‚Äì260 |
| epic | 20 | 260 | 55‚Äì440 |
| legendary | 4 | 600 | 260‚Äì600 |

(Tier ranges overlap by design ‚Äî a level-1 rare can be worth less than a level-8 uncommon. The ladder
check compares medians, which is the meaningful signal.)

### 3. Zero-value items ‚Äî hard-fail

Items that ought to be sellable but carry no value. **Quest items are exempt**:
`item_glooms_crypt_key` legitimately has no value (it's a key, not tradeable) ‚Äî and because both
formulas floor to `1`, it would otherwise read as a false arbitrage hit.

- **Result: 0.**

## Suite integration

Wired into `runValidationSuite` as an `economy` section: **arbitrage and zero-value are hard
failures** (an exploit or a data bug); **ladder inversions are advisory** balance notes, not release
blockers. The boot log now includes `N economy issue(s)`.
