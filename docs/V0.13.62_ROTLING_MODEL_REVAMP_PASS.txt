Dream Realms V0.13.62 — Rotling Model Revamp Pass

Systems changed
- Added render/rotling-procedural-model.js as the dedicated Rotling-family procedural renderer.
- Routed Dark Woods Rotlings through RotlingProceduralModel from render/dark-woods-mob-procedural-model.js before the generic/rootling fallback path.
- Updated render/entity-renderer.js to preserve dedicated Rotling nameplate anchors.
- Updated Rotling metadata in data/enemies.js, data/npcs.js, systems/editor-system.js, _script_order.json, and HTML script references.

Rotling visual/model changes
- Replaced the simple round rootling/blob presentation with a distinct corrupted woodland creature model.
- Added separated head and torso masses, asymmetrical hunch, visible maw, eye/socket reads, bent root limbs, clawed forelimbs, fungal caps, thorn growths, pustules, bark/root details, and rot-texture breakup.
- Added dedicated palette and variant resolution for base Rotlings, elder/elite Rotlings, and named/spore-crown-style variants.

Animation coverage
- Added renderer support for idle, walk, attack, hit, death, lunge, spit/corruptionCast, claw/swipe, and sporeBurst mappings.
- Idle includes breathing, twitch, hunch motion, and subtle growth motion.
- Walk uses a shambling crooked leg cycle with body bob and limb sway.
- Attack uses maw/claw lunge motion.
- Death uses a grounded corpse pose instead of a standing blob fallback.

Gameplay systems preserved
- No mob stats, aggro, loot, quests, targeting rules, pathing, collision, attack ranges, combat logic, save/load, or elite/rare logic were changed.
- V0.13.60 targeting/threat fixes, V0.13.61 boar corpse fix, wolf/boar revamps, Spirit Healer revamp, terrain/water work, bots, pets, Silk Web Cavern, inventory, paper doll, currency, Purify, resurrection, and Esc menu behavior are preserved.

Tests run
- node --check on every JavaScript file.
- Local HTML script-reference audit.
- Metadata audit for V0.13.62.
- Static routing audit for RotlingProceduralModel and rotlingDedicated path.
- Mock render smoke test for all 8 directions and idle/walk/attack/hit/death/corruptionCast states.
- ZIP integrity verification.

Manual browser-test caveats
- Manual browser testing is still recommended for Dark Woods Rotling herds, click/Tab targeting, health bar/nameplate anchors, corpse/loot readability, save/load, and FPS with multiple Rotlings on screen.
