- `render/paperdoll-equipment-renderer.js`
  - equipment lookup
  - slot aliases
  - main-hand/off-hand item lookup

  - visible paperdoll gear drawing

- `render/humanoid-base-renderer.js`

## Root cause
The reported white-stick visuals came from the paperdoll held-weapon renderer, not from class stats or combat data. The paperdoll renderer had three root problems:

1. It only recognized a small set of weapon text patterns and did not include icon family, item IDs, source IDs, or editor notes in visual classification.
2. It did not draw specialized shapes for bow, crossbow, greatsword, lute/instrument, book/grimoire, skull, totem, symbol, orb/focus, or throwing-knife bundles. Unrecognized items fell into the generic sword branch, which rendered as pale stick-like lines.
3. The off-hand path treated any non-shield/book/orb/focus/skull/totem/symbol item as a dagger, which made Bard lute/offhand, Assassin throwing weapons, and other offhands read as generic white sticks.

A secondary hardening issue was that equipment values can arrive as runtime instances or simple IDs from visual equipment/bot equipment paths. The renderer now rehydrates draft visual metadata from `DR.ITEM_BY_ID` when possible before choosing the shape.

## Changes made

### `render/paperdoll-equipment-renderer.js`
- Added `itemCatalogEntry()` and `hydrateVisualItem()` to resolve string equipment IDs and merge immutable draft visual metadata into runtime visual reads without mutating gameplay data.
- Updated `slotItem()` so weapon/offhand visual resolution receives hydrated item data.
- Expanded `itemText()` to include item IDs, source IDs, icon family, glyph, slot, and editor note.
- Replaced `weaponKind()` with a broader class/item-aware shape classifier.
- Reworked `drawWeaponShape()` to render proper class equipment silhouettes for:
  - staff
  - wand
  - book/grimoire
  - orb/focus
  - symbol/prayer icon
  - totem
  - skull focus
  - lute/instrument
  - bow
  - crossbow
  - throwing knives/fangs
  - dagger/shiv
  - greatsword/two-handed blade
  - axe
  - mace/maul/mallet
  - generic sword fallback
- Reworked `drawOffhand()` to call the same visual classifier instead of falling back to a hardcoded dagger shape for unknown offhands.

## Entity types affected
Because the fix is inside the shared paperdoll equipment renderer, it applies where that owner is used:

- player world models
- character/equipment preview models
- bots using `botEquipment`
- mercenary/companion class models using shared visual equipment
- remote/player-like actors using shared visual equipment

Monster, pet, animal, and independent procedural renderers were not changed.

## Main-hand/off-hand resolution after the fix
- Actual equipped runtime items are used when present.
- String equipment IDs are resolved through `DR.ITEM_BY_ID` when possible.
- Draft visual metadata such as `icon.family`, `icon.color`, class starter item IDs, and item names now participates in visual kind selection.
- Offhand items no longer default to a dagger unless their actual item data resolves to dagger/throwing-knife.
- Two-handed weapons render as a single main-hand class weapon; no offhand is fabricated.
