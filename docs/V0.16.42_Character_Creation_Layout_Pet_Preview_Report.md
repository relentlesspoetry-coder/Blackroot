# V0.16.42 - Character Creation Layout Expansion and Pet Preview Fix Report

## Summary

Updated Blackroot from V0.16.41 to V0.16.42 with a focused Character Creation pass. The active `.cc-*` Character Creation layout now uses more of the available viewport, the center character preview is a larger showcase panel, and Summoner/Necromancer previews render their companion pet models beside the player model.

## Root causes found

- The active Character Creation screen uses the newer `.cc-*` layout, but the modal and grid column sizing were still conservative from the earlier restoration pass, leaving unused viewport space around the creator.
- The center preview was artificially constrained by `#creatorPreview .creatorModelViewport { max-width: 420px; }`, so the model canvas stayed narrow even when the center panel had much more available space.
- The shared class preview renderer drew only the selected player model. Pet-class companion visuals existed in `PetIdentityProceduralModel`, but the Character Creation preview path never invoked that renderer.
- The preview metadata still exposed raw skin/garb hex values, which read like debug/config text rather than useful character identity text.

## Bugs fixed

- Character Creation now occupies more of the available viewport and reads like a full-screen creator instead of a compact modal.
- The center character preview no longer stays capped at a narrow 420px width; it expands with the center stage.
- Player preview scaling now derives from the measured preview rectangle and reserves horizontal room only when a pet-class companion is rendered.
- Necromancer selection now shows the existing Bone Servant pet model beside the player model in the creator preview.
- Summoner selection now shows the existing Azure Shard Familiar pet model beside the player model in the creator preview.
- Companion preview animation is driven by the existing creator idle animation loop and does not change gameplay pet behavior.
- Preview metadata no longer displays raw garb/skin hex values.
