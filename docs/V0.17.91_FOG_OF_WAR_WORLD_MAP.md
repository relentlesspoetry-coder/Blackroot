# V0.17.91 — Fog of War World Map (core)

**Type:** System / UI (map)
**Previous version:** V0.17.90
**New version:** V0.17.91
**Design source:** `Blackroot_Fog_of_War_World_Map_System.txt`

Every zone starts unknown; the player reveals it by walking, and exploration is permanent. Scope
(per the "core now, extras later" decision): tile exploration + feathered reveal + persistence + the map
screen with icons + exploration % + landmark discovery. **Deferred:** the 25/50/75/100% reward milestones,
the Cartographer NPC, and survey points.

---

## Implementation

All in `render/minimap-renderer.js` (Game.prototype fog methods), hooked from `game.js`
(`updateExternalSystems` → `fogUpdate`) and `systems/save-system.js` (persistence).

- **Explored-tile tracking** — `game.fog.zones[key]` holds a packed **1-bit-per-tile** bitmask + a revealed
  count, keyed by `fogZoneKey()` (overworld `dark_woods`, per-cave, per-dungeon-floor — each keeps its own
  map, satisfying "caves and dungeon floors maintain separate fog-of-war maps").
- **Reveal-on-move** — `fogUpdate(dt)` (each frame) reveals a **disc of radius 10** around the player
  (`fogRevealAt`). The fog overlay is a tile-resolution canvas (unexplored = near-opaque dark, explored =
  transparent) drawn scaled with `imageSmoothing` on → **soft feathered edges**, not square blocks.
- **Exploration %** — `ensureFogReachable` flood-fills from camp over non-water land once, giving a
  reachable-landmass denominator so 100% means the whole landmass. Shown on the world-map title
  (`Explored X.X%`).
- **Fog compositing** — on **both** the minimap (`compositeFogMinimap`, under the player/enemy dots) and the
  world map (`compositeFogWorldMap`, under the markers, transformed to the current pan/zoom sub-view).
- **Icons gated on discovery** — `fogMarkerVisible` hides NPC/vendor/landmark/cave icons until their tile is
  revealed; the player marker always shows.
- **Landmark discovery** — `checkLandmarkDiscovery` raises `Landmark Discovered: <name> (Map Updated)` + a
  ring + exploration XP the first time a **landmark-type** marker's tile is revealed (towns/camps/caves/
  dungeons/shrines/spirit-healers/waypoints only — NPCs just appear as icons, no banner spam).
- **Permanent persistence** — `serializeFogState`/`restoreFogState` (bits base64-packed) via the game save
  (`runtime.fog`) **and** a per-character `localStorage` key (`dream-realms.fog.v1:<characterId>`), throttled.
  Survives reload, logout, and death.

## Verification (in-browser, port 8899)

Revealing grows the % as the player walks (0.27% → 1.4% over a short path); tiles read explored where
walked and hidden far away; the town + spirit-healer landmark banners fire on first reveal (not all 25 camp
NPCs); serialize/restore round-trips exactly; localStorage saves; the minimap and world map both draw with no
console errors.

**Visual note:** `preview_screenshot` times out on this WebGL page, so the fog's *look* (feathering,
darkness) is verified structurally only — the user is the visual sign-off. The unexplored-darkness alpha
(currently 236/255) and the reveal radius (10) are easy tunables.
