# Dark Woods 360x360 Expansion — Phase 25: Resource UI Messaging + Feedback

**Phase name:** Resource UI Messaging + Feedback (Phase 25 of 28)
**Previous version:** V0.17.63
**New version:** V0.17.64

---

## Goal

Make resource interactions clear: "Requires Gathering/Fishing Level X", successful gather/catch
feedback, profession EXP-gain feedback, and first-discovery/rare-node bonus feedback — **without
redesigning the HUD**.

## What already existed (audited, kept)

Most feedback was built incrementally across Phases 19–24 and was left in place:
- **Gather/catch success**: `<node> gathered: <summary>.` / `Caught <fish>.` via the status line, the
  harvest cast bar (`completionToast`), and the game log.
- **First-discovery + rare-node bonuses**: `First discovery! +N EXP.` / `Rare node +N EXP.` /
  `First catch! +N EXP.` / `Rare +N EXP.` appended to the success message.
- **Level-up feedback**: `gainSkillXp` logs `<Skill> level N.`; the resource panel meta and the
  skills/professions panels show XP bars.
- **Hard level gate**: `startHarvestNode` blocks and messages when the profession level is too low.

## What this phase changed (surgical, reusing existing surfaces)

1. **Level-gate wording → plan phrasing.** `startHarvestNode`'s block message is now
   `<node>: Requires <Skill> Level X` (was `<node> requires <Skill> X`).
2. **Nearby-node panel shows the requirement when under-level.** The resource panel's nearby line now
   reads `<node> · Requires <Skill> Level X` when the player's profession level is below the node's
   requirement, and `<node> nearby · <Skill> Lv X · G to gather` when they can gather it — so an
   uncollectible node reads clearly.
3. **EXP-gain feedback on gather.** The success message now includes the total profession EXP gained:
   `<node> gathered: <summary>. +N <Skill> EXP.<bonus breakdown>`.
4. **EXP-gain feedback on catch.** The catch message now includes `+N Fishing EXP` alongside the
   existing first-catch/rare breakdown.

**Fishing level gate — intentionally no message.** Fishing is fish-anywhere (no per-fish node to gate);
under-level fish simply don't bite (the Phase 22/23 `eligibleFish` filter). There is no node to attach a
"Requires Fishing Level X" prompt to, so none is shown (adding a fake one would be misleading/spammy).
Documented, not faked.
