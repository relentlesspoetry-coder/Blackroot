# V0.20.0 — The Capstone That Looked Like Level One (Roadmap Item 6)

Auditing the Fighter's 20 spells found the same **dead-data pattern one layer down** — this time in the
VFX **style strings**.

## The find

`systems/spell-system.js:1016`:

```js
this.playAttackAnimation?.(p, t, color, id === 'titan_chop' || id === 'perfect_masterstroke' ? 'slam' : 'slash');
```

**Nothing in the codebase had ever branched on `'slam'.`** Both renderers implemented only `'claw'`;
every other style fell to the same `else` arm and drew an identical horizontal arc.

So the Fighter's two heaviest hits — including **Perfect Masterstroke, the level-20 capstone** — were
**pixel-identical to level-1 Heavy Swing**. And they *sounded* identical too, because the audio line had
the same shape:

```js
this.playSfx?.(style === 'claw' ? 'attack_claw' : 'attack_slash', …)
```

## Two more, found by checking

| style | passed at | consequence |
|---|---|---|
| `'crush'` | `entities/enemy.js:315` | heavy enemy attacks drew a polite swipe |
| `'cast'` | `entities/pet.js:351`, `entities/bot-player.js:546` | **casters were swinging an invisible sword** |

Fixing only `'slam'` would have been the half-fix the golden rules forbid. The root cause is that
`spawnSlash` **did not render every style it was asked for**, so all three are implemented — each a
genuinely distinct silhouette:

- **slam** — heavy near-vertical stroke driven down, plus a ground shock *once it lands*
- **crush** — blunt pound with radial debris, **no blade**
- **cast** — glyph ring with rising motes, **no blade**

Slam's audio reuses `attack_slash` at rate ~0.58 — pitched well down, a heavier weapon landing — rather
than inventing a sample that doesn't exist in `sound-effects/`.

## The check — and two wrong versions of it first

`tools/check-vfx-styles.js`, wired into `validate.sh`.

**Why static, not a suite section.** This class **cannot** be a runtime data validator: the styles are
string literals in **code**, not data, and the renderer's `else` arm means an unimplemented style still
draws *something* — just the wrong thing. Nothing throws. No data is malformed. There is nothing for a
data validator to find.

**Wrong version 1 — a tautology.** I wrote `Game.prototype.validateVfxStyles()` and **deleted it**: its
`requested` list was hardcoded to mirror its `implemented` list, so it could never fail. That is exactly
the dead validation this session has spent its time removing — a check that only pretends to work is
worse than none.

**Wrong version 2 — a grep.** It raised a **false positive** (`'undead_minion'` — a `petType` inside a
ternary *condition*, not a style) and **missed a real one** (`'cast'`, that same ternary's *value*). A
checker that cries wolf is the V0.19.3 mistake again.

**The version that shipped** balances parens to extract the 4th argument, reads only the **value side** of
a top-level ternary (skipping `?.`), and **reports non-literal arguments honestly** instead of guessing:

```
note: non-literal style argument at systems/effects-system.js:329 - not checkable statically
```

## Item 6 status

The Fighter's **cone** and **style** layers are now honest. Still open:

- **The Fighter's 20 spells still share silhouettes at the tier below this.** 8 single-target melee
  spells (Heavy Swing, Crushing Blow, Reckless Strike, Heavy Riposte, Armor Splitter, Bonebreaker, Final
  Swing, Shatter Guard) all draw the same `'slash'`. That is not *dead* data — it's a design gap, and
  unlike the cone it makes no falsifiable claim a checker can catch. It needs a per-spell VFX pass like
  the Assassin's V0.18.29.
- `spell_summoner_mass_dismissal` declares `radius: 4` on a single-target spell.
- 6 classes have had no VFX audit at all.
- The **sprite-cache backend** (`webglVisibleEffectLayerDefault: false`, off by default) is a simplified
  LOD path and does **not** implement slam/crush/cast — consistent with how it already approximates other
  effects, but worth knowing if that layer is ever enabled. Its `slash` bake also keys without `dir`, so
  direction would be baked from whichever effect created the entry first.
