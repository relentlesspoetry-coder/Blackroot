# V0.18.41 — Silk Web Cavern layout + population revamp

**Report:** *"We need to revamp the layout — small narrow tunnels leading into small caverns, with more tunnels into large caverns/rooms. We need more spiders inside: baby spiders, big spiders, little spiders, plus the elites and everything else. The floors need more webs, we need webs stretching from the floor going up to the cave ceiling, we need big piles of webs with junk all webbed up — armor, weapons and bones."*

Implemented while **preserving the authored boss / gate / objective / miniboss structure** — the reachable critical path is untouched.

---

## 1 — Narrow tunnels (`systems/dungeon-system.js` `carveCorridor`)
`carveCorridor` now takes a `radius`. The main room-to-room corridors carve at **radius 2 (~5 tiles wide)** instead of the old fixed ~7–9-wide hall, so the layout reads as tunnels between caverns rather than open halls.

## 2 — Small pocket caverns (`carveSilkPocketCaverns`)
Branches small/medium **dead-end pocket caverns** off the combat rooms via **radius-1 (~3-wide) narrow tunnels**. Each pocket is only carved where its footprint is **>80% dead rock**, so it can never overwrite another room or land on the critical path — **boss/gate/objective reachability is guaranteed intact**. Pockets are pushed into the room list, so they get webbed, dressed, and populated like authored rooms. ~6–9 pockets per floor.

## 3 — More spiders, size-varied (`makeSilkSpiderVariant` + `spawnSilkExtraSpiders`)
On top of the authored elite packs, two minibosses, and boss, every combat room and pocket now gets **filler spiders**:
- **babies** — modelScale 0.46 (more in brood/egg rooms),
- **little skitterers** — modelScale 0.66,
- **big "Broodmother" spiders** — modelScale 1.34 (occasional, in larger caverns; a tanky **non-elite** so it doesn't inflate elite/objective tracking).

Capped per floor (**40 / 48 / 54**) and further limited by available placement space, so it teems without flooding. `render/entity-renderer.js` now buckets the spider **sprite-cache key by size** (`xs`/`s`/`m`/`xl`) — the cache bakes `modelScale` into the bitmap, so without a size bucket a baby would inherit the full-grown skitterer sprite (this generalises the V0.18.38 `:baby` fix).

## 4 — Floor-to-ceiling webs + webbed junk piles
`placeSilkCeilingWebsAndJunk` scatters two new objects (`render/object-renderer.js`):
- **`ceilingWebColumn`** — a thick silk column: a bundle of strands fanning out floor-to-ceiling, rung threads crossing it, a ceiling anchor fan, and a clinging egg sac. Reads as the cavern being spun top to bottom.
- **`webJunkPile`** — a web mound with a victim's gear cocooned in silk: a shield, a sword (hilt + blade + crossguard), ribs and a half-buried skull, with strands wrapping over the whole pile and anchor threads to the ceiling. Placed in combat/den/boneyard/supply rooms and pockets.

The floor keeps the V0.18.40 dense web overlay (`renderSilkWebFloorOverlay`).

## Important: floors regenerate on entry
The layout, pockets, spiders, and web objects are all built during floor generation, which runs on entry — so **re-enter the Silk Web Cavern** (or change floors) to see them; they won't appear on the floor you're already standing on.
