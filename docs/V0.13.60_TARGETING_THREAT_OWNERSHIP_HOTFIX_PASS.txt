Targeting fixes
- Tab targeting now builds a valid hostile target candidate list instead of using a raw nearest-enemy lookup.
- Tab targeting cycles through valid hostile enemies when the current target is already in the candidate list.
- Dead mobs, friendly bots, pets, NPCs, quest givers, temporary loot entities, and invalid actors are ignored.
- Click and right-click targeting now use a shared screen-space target-bounds resolver.
- Large revamped wolf/boar models, elites, named mobs, bosses, and scaled enemy visuals are clickable across their visible body.
- Empty ground clicks no longer select a random nearby enemy through the old world-nearest fallback.

Threat ownership fixes
- Autonomous bots are now valid threat actors in the central threat system.
- Bot-owned pets are now included as valid allied threat candidates.
- Threat target selection now resolves stored threat entries first and only uses proximity aggro when no stored threat exists.
- Mobs no longer use player-biased proximity/nearest fallback while bot or pet threat exists.
- Enemy fallback scoring no longer gives the player an artificial priority bonus.
- Bot attacks, bot spell damage, bot taunts, bot healing threat, and bot-pet attacks credit the bot or pet actor, not the player.
- Player threat is still generated by player attacks, player spells, player heals, player taunts, and player-owned pet actions through existing explicit gameplay paths.

Player threat rules
- Player receives threat only from player-owned threat-generating actions or explicit proximity aggro when no stored threat exists.
- Player is not inserted into threat tables as a default target when bots engage enemies.
- Empty or invalid threat tables return neutral/no target rather than silently falling back to player.

Bot/pet threat rules
- Bots and bot-owned pets can own threat table entries.
- Fighter bot taunts credit the Fighter bot.
- Bot healing threat credits the healer bot through addHealingThreat.
- Bot-owned pet attacks credit the pet actor, and that pet is now targetable by the threat resolver.

Gameplay preserved
- No mob stats, loot tables, quest data, class damage formulas, bot behavior, pet behavior, pathfinding, collision, terrain rendering, inventory, paper doll, currency, resurrection, Purify/Cure Poison, save/load, or Esc menu behavior was redesigned.
