# V0.18.32 — Walk-through portals

User:

> "We need to fix the Portal system, when you press T and open a portal you are
> instantly teleported. I want the player to have to walk through the portal to
> teleport back to camp. the port is a one time use, so you can teleport back to
> camp and then return to your original location then the portal will close."

## Root cause
The walk-through machinery already existed in `systems/portal-system.js`:
- `tryCastPortalSpell` (T) only **opens** a portal link — it never teleports.
- `updatePortalRuntime` triggers `useActivePortal` when the player comes within
  `PORTAL_TRIGGER_RADIUS` of the active portal.
- `useActivePortal` walks the link through `origin → (teleport to camp) → return →
  (teleport back) → collapse`.

But it played as an **instant** teleport because the portal spawned **on top of the
player**: `findPortalSpawnPoint` tried steps `[PORTAL_SPAWN_FORWARD=0, 0.35, 0.65,
0.95]` — the first is the player's exact tile, and all are inside the
`PORTAL_TRIGGER_RADIUS` (0.96). So after the 0.42 s spawn grace the stationary
player was already inside the trigger zone and it fired immediately. The **return
portal** had the same bug — it spawned at `capturePortalLocation(player.x,
player.y)`, i.e. the camp arrival tile, so the player got bounced straight back.

## Fix
Open both portals a couple of tiles away, clearly beyond the trigger radius, so the
player must **walk into** them.

- `PORTAL_TRIGGER_RADIUS` 0.96 → **1.1**
- new `PORTAL_SPAWN_DISTANCE = 2.5`
- new `findPortalPlacement(ax, ay, dirX, dirY)` — scans for a walkable spot
  ~2.5 tiles out, preferring the facing direction then fanning to other
  angles/distances, and always at least `PORTAL_TRIGGER_RADIUS + 0.6` from the
  anchor (falls back to a straight forward nudge if nothing is walkable).
- `findPortalSpawnPoint` (origin portal) and the **return-portal** spawn in
  `useActivePortal` both use it.

## New flow
1. Press **T** → a portal opens ~2.5 tiles in front of you (no teleport).
2. **Walk into** it → travel to Dead Lantern Camp.
3. A **return portal** opens ~2.5 tiles from where you land — restock/shop freely.
4. **Walk into** the return portal → return to exactly where you cast it; the link
   collapses. (One round trip, then closed.)

Casting from camp is symmetric (opens a portal to your last wilderness anchor).

## Verification
- `node --check systems/portal-system.js` — OK.
- `findPortalPlacement` returns points **exactly 2.5 tiles** from the caster for
  east / north / diagonal / no-facing inputs, all beyond the 1.1 trigger radius
  (so a stationary caster can never auto-trigger). `findPortalSpawnPoint` → 2.5.
- Boots with **0 console errors**.
- Full in-world confirmation is the user's (press T and walk through).
