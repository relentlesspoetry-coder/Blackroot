# V0.16.47 - Character Stats Window Mockup Revamp Report

## Summary

Revamped the Character Stats window toward the supplied ornate Blackroot mockup while keeping the existing character-window owner, equipment data, stat calculations, gear score, status text, set-bonus text, and close behavior.

## Root cause found

The existing Character Stats window still used compact shared sheet sizing and conservative paper-doll fit constants. That made the frame feel flatter than the mockup, left the header without a strong class crest anchor, kept equipment slots visually compressed, and prevented the model stage from reading as a premium full-body showcase.

## Model viewport implementation details

The existing paper-doll canvas renderer remains the single renderer. The patch increases the preview-stage dimensions in the canonical Character window CSS and adjusts `renderPaperDollClassModel()` scale/origin math to use the measured viewport size with higher class-aware scale targets, width/height clamps, and a pedestal-aligned foot anchor.

## Equipment slot layout changes

Equipment slots remain split into the same left/right slot groups: head, shoulders, chest, legs, hands, feet, cape on the left; amulet, earrings, rings, weapon, offhand, charm on the right. The cards now use larger framed styling, slot glyphs for empty slots, item icons when equipped, item names, and the existing summarized item stat lines. Slot data attributes and tooltip binding remain unchanged.

## Stat panel layout changes

The right panel keeps the same stat data but presents it with a larger character summary card and icon-backed two-column stat cards. Gear Score, Damage, HP, Mana, Armor, CDR, Phys Crit, Magic Crit, Phys Output, Magic Output, Healing, Speed, Attributes, Status, and Set Bonuses are preserved.
