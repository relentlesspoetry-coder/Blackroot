# V0.20.78 ‚Äî The Tail Draws Over the Rider, and Both Seats Match

Reported: *"the tail should be rendered on top of the player. Their legs should have the same position as
the front view."*

## The tail

The rear band was a shallow **14px** strip anchored at the back line and stopping at the rider's knee ‚Äî
deliberately shallow, back in V0.20.73, because the report at the time said the legs already looked right.
The consequence is that the tail, which hangs *below* the rump, stayed behind the rider entirely.

It now runs down past the legs exactly as the front band does:

```js
const riderFootY = Math.round(oy - this.mountSaddleHeight(def, modelScale) + 28);
const bandBottom = Math.min(bottom, Math.max(backY + 14, riderFootY + 4));
```

Box goes from **‚àí53..‚àí39** to **‚àí53..‚àí20**.

This makes the two views symmetric, which is the underlying idea the code had been missing: **from the
front the rider's legs straddle the animal's head; from behind they straddle its tail.**

## The seat

`kneeOut`/`footOut` were `27/31` in front and `17/19` behind. Both are now **27/31**.

The narrow rear seat was never independently motivated ‚Äî it existed because the old rear band was a thin
strip that a wide seat would have poked out of. Now that the band runs past the legs, the opposite holds:
a *narrow* seat sits inside the band and gets swallowed by it. The two changes in this version are
therefore not independent; the seat change is what makes the tail change survivable.

`frontOn` became unused as a result and was deleted rather than left dangling.

## Width is bounded by the rider, not the animal

This is the one place the two views legitimately differ, and it took a failed attempt to see why.

The front band takes its width from the head-and-withers rows, because a front silhouette *has* a narrow
head band to measure. **A rear silhouette does not** ‚Äî the rump is inherently wide, 131‚Äì139px across on
the black wolf. Measuring it the same way produced a **147px** band that buried the rider's left leg:

| zone | before clamp | |
|---|---|---|
| left leg, non-animal pixels | 92.6% ‚Üí **63.2%** | buried ‚ùå |

So the rear band is bounded by the **rider** instead:

```js
const legClear = 27;                 // tracks the ¬±31 foot splay in buildRidingRig
minX = Math.max(minX, ox - legClear);
maxX = Math.min(maxX, ox + legClear);
```

The tail lands between the legs; the legs stay visible either side of it. **These two constants are
coupled ‚Äî if the splay changes, this must change with it.**
