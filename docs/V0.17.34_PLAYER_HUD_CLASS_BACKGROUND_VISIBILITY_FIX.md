## Root Cause
The Player HUD was resolving the correct class background, but V0.17.33 restored the panel base by stacking a very dark top overlay above the resolved class image. That made the class art read as a plain black panel even though the resolver path was populated. The Party HUD was not affected because its companion cards use a separate card-surface backdrop without the extra heavy Player HUD wash.

## Player HUD Background Compositing Changes
- Kept the opaque panel base so the HUD is not see-through.
- Reduced the top dark wash so the class art remains visible.
- Reordered the Player HUD background layers so class art is the dominant visual surface under a readability veil.
- Increased brightness/saturation slightly on the Player HUD backdrop only.
- Left the Player HUD layout and bar placement unchanged.

## Class Background Resolver Usage
The existing canonical resolver remains the source of the class background path and class color variables. This phase changed the compositing that was hiding the resolved art, not class lookup or class data.

## Party HUD Regression Verification
The Party HUD card renderer and layout were not changed. The companion card class-background treatment remains the visual reference.
