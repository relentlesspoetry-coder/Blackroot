# V0.21.4 — Reference Grade

Measured against the user's supplied reference art, not the migration plan. Where they disagreed, the
reference won.

## Fog was the biggest culprit

Found by finally taking a screenshot instead of trusting statistics.

| | hex | luma | saturation |
|---|---|---|---|
| with fog | `#adadaa` | 172.8 | **0.017** |
| fog cleared | `#997d5b` | 128.5 | 0.405 |
| reference | `#5a4632` | 72.6 | 0.444 |

The terrain material was already close in hue and richness — a 0.039 saturation gap. Fog was draining
it to grey. **A weather effect that takes saturation to 0.017 is a bug, not an atmosphere.** Flagged.

## Brightness, fixed in two stages

Stage one grades `material.base` in the chunk build. It **stalled at luma 93**: cutting
`ambient.value` 0.565 → 0.40 (−29%) bought only 10%, because the cached detail passes, depth wash and
boundary work paint on top ungraded.

Stage two grades the **finished layer** after every pass — the same "find where it is actually drawn"
lesson V0.21.3 learned about `drawGroundDetail`. That let `value` ease back to 0.52 so materials keep
their identity instead of being crushed to mud.

**Result: `#5f4f3b`, luma 81.0** against the reference's 72.6 — within 12%, hue matching.

### The safety claim, verified after a bad probe

`source-atop` tints only pixels the layer already painted, so padding cannot become a dark slab.

| | grade ON | grade OFF |
|---|---|---|
| alpha sum | 64,886,172 | 64,886,172 |
| clear px | 23,929 | 23,929 |
| opaque / semi | 254,016 / 839 | 254,016 / 839 |

Zero alpha added. A first probe sampled (1,1) and assumed empty padding — it is painted terrain,
because padding exists precisely so neighbouring tiles get drawn.

## Saturation above 1 — contradicting the plan

The plan specifies `ambient.saturation` 0.72 (desaturate). Ours measured 0.405 against the reference's
0.444 — **already below**. Cutting took it to 0.339, away from target. The reference is desaturated
relative to a bright fantasy palette, not relative to this build, which was already muddy.

## The bug behind three failed attempts

`DR.utils.smoothNoise` returns a **spread of 0.0001** across 16,000 points — a constant 0.5135. Through
`|n − 0.5|` that gave a flat **0.0151 alpha everywhere** (min = median = max): a uniform 1.5% tint over
the entire world. The report that the ground "looks the same" was exactly right, and two of three
attempts went into raising a multiplier that could never have helped.

Fixed with `seededNoise` (0.98 spread) on an integer lattice with smoothstep bilinear interpolation —
both range and smoothness, which neither helper provides alone. Alpha now varies 0 → 0.189.

## Shrine size correction

V0.21.2 shipped it several times too large. I reported "1.93 tiles wide", which measured the **platform
only**, ignoring backplate pylons 150px above it and pillars another 74px — the silhouette stood 3–4×
character height.

Frame 320→208, platform radius 132→76 (1.12 tiles), pylons 150→84, pillars 74→44, emissive alpha
0.55→0.30 (its translucent look), occlusionHeight 118→70, contactShadow rescaled.

**Measuring one part of an asset and reporting it as the asset's size** is the mistake not to repeat.

## Remaining gap to the reference

- saturation 0.379 vs 0.444
- detail density — the reference has dense pebbles, dirt clumps and grass tufts; the `drawCached*`
  passes do run, so this is likely density tuning rather than new code
- fog desaturation, which needs its own fix
