# V0.18.31 — Cleric spell-animation revamp

Third class of the per-class VFX pass (after Assassin V0.18.29, Bard V0.18.30).
User: "Now do the Cleric."

## Problem
The whole Cleric kit ran on generic VFX in `resolveClericRevampSpell`
(`systems/spell-system.js`):
- `heal` / `cleanse` / `buff` / `groupBuff` / `aoeHeal` → `spawnRing`
- holy damage (Smite / Judgment / Exorcise) → `spawnBolt`
- `revive` → nothing

Unlike the Bard, the Cleric had **no** bespoke draw functions at all — this was a
from-scratch build (like the Assassin).

## What was built

### `render/effects-renderer.js` — a holy VFX set
- `drawClericBlessEffect(e)` — one function, keyed by `e.style`:
  - `heal` — radiant bloom, expanding rings, sunburst, rising sparks + a golden cross
  - `hot` — a gentle sustained glow + slow motes (Renewing Prayer)
  - `aoeheal` — expanding radiant ripple over the party with per-ally glows/crosses
  - `ward` — a golden dome + facets + shield sparkle (scales up for group wards)
  - `cleanse` — an upward purifying flush that sheds dark specks
  - `nova` — a radiant repelling burst (Turn Undead)
  - `revive` — a rising column of light + ascending motes + a cross forming above
  - `bless` (default-ish) — the heal bloom, for prayer/mercy buffs
- `drawClericSmiteEffect(e)` — a **pillar of light slams down** onto the target then
  blooms; styles `smite` / `judgment` (wider) / `exorcise` (extra white flare + rising
  specks, anti-undead).
- `drawHolyCross` + `drawSunburst` motif helpers. Everything draws under `screen`
  composite so holy light reads additively.

### `systems/effects-system.js` — spawners
- `spawnClericVfxEffect(type, payload)` — generic spawner (mirrors the Assassin one).
- `spawnClericSpellEffect(spell, source, target, options)` — dispatches every
  `clericSpellId` via a `CLERIC_VFX` config (fx type + style + theme colours: green
  heals/regen, gold wards, blue/white cleanse, white exorcise, gold-white
  revive/nova). Single-target effects `followId` the ally; AoE/nova anchor at the
  group centre.

### `systems/spell-system.js` — wire up, delete the generic VFX
`resolveClericRevampSpell` now calls `spawnClericSpellEffect` at each branch (heal,
cleanse, buff, groupBuff shields **with allies**, aoeHeal **with allies**,
revive-on-success, and the holy-damage path). The `spawnRing` / `spawnBolt` calls
are removed.

## Spell → animation

| Spell | Effect |
|---|---|
| Minor Heal / Greater Heal / Radiant Touch | radiant heal bloom |
| Renewing Prayer | sustained regen glow |
| Divine Light / Hymn of Renewal | party-wide radiant ripple |
| Holy Ward / Sanctuary / Blessed Barrier / Guardian Prayer / Divine Intervention | golden ward dome |
| Cleanse / Purify Soul | upward purifying flush |
| Prayer of Mending / Avatar of Mercy | blessing bloom |
| Turn Undead | repelling radiant nova |
| Radiant Revival | rising column of light |
| Smite | pillar of light + bloom |
| Judgment Light | wider judgment pillar |
| Exorcise Evil | white anti-undead pillar |

## Verification
- `node --check` on all three files — OK.
- Live: all 8 `bless` styles + 3 `smite` styles render with **0 throws** and the
  canvas transform returns to identity.
- `spawnClericSpellEffect` dispatches every `clericSpellId` to the correct themed
  effect (spot-checked 12 spells).
- `grep` confirms **0** generic `spawnBolt` / `spawnRing` remain in
  `resolveClericRevampSpell`. Boots with **0 console errors**.
- In-world visual is the user's sign-off.

## Note for the Druid (later in the pass)
The Cleric `heal` branch shares its code shape with the Druid resolver
(`spawnRing?.(ally.x, ally.y, color, 14); return; } if (spell.kind === 'cleanse')`),
so a naive edit matched both — disambiguate Druid edits by the `woundedThresholdPct`
line / log text.
