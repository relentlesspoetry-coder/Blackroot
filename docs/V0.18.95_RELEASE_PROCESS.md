Rounds out the initial Item 22 pass: two final regression scenarios, and the validation process

## Two more regression checks (stat-pipeline self-test → 14/14)

| # | check | detail (passing) |
|---|-------|------------------|
| 12 | higher level yields higher max HP | `L15 329 > L5 263` |
| 13 | current-HP ratio is preserved across a gear change | `ratio 0.49, max 263 → 295` |

- **#12 level growth** — confirms character level is a live stat source (`computePlayerDerived` uses
  level); higher level → higher derived stats.
- **#13 HP-ratio preservation** — `recalculatePlayerStats` snapshots `hp/maxHp` at the top and restores
  that ratio at the end. So equipping a +maxHp item while at ~50% HP keeps you at ~50% of the *new*
  (higher) max — not a free heal, not a stale absolute value. This guards a subtle, easily-broken
  gear-swap behaviour.

## Release process codified

### `validate.sh` (new) — CLI checks

Runs the automatable half and prints `RESULT: PASS`/`FAIL`:

- `node --check` on every JS file (excluding `blender/` and `backups/`)
- `JSON.parse` on every JSON file
- **version consistency** across `core/config.js` (const + top `// <VER>:` comment), the
  `Blackroot <VER>.html` file, `VERSION.txt`, `PATCH_NOTES.md`, and `README.txt`, plus a check that
  exactly one `Blackroot *.html` exists.

### `RELEASE_CHECKLIST.md` (new) — full process

Documents the per-version flow: back up the previous version first → bump every surface → run
`validate.sh` → confirm the **in-browser** boot validation suite is PASS
(`DarkWoodsGame.runValidationSuite()`) → report and stop. Includes a table of which checks block a
release vs are advisory (the 7 descriptor warnings are advisory — Roadmap Item 1).
