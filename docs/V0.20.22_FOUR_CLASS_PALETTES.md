# V0.20.22 — Four Classes With No Face (Roadmap Item 6, caster half closed)

## The claim this version exists to correct

V0.20.21 made the caster palettes reachable, and I reported that **eight classes gained an identity**.

**False, and I never checked it before writing it.** `shaman`, `summoner`, `enchanter` and `ranger`
match **no regex in `vfxStyle`** and still resolved to the generic default. The wiring worked
perfectly; **there was nothing on the other end to reach.**

## Authored — the four missing palettes

The user delegated the design call.

| class | primary / secondary | motion | reasoning |
|---|---|---|---|
| **Shaman** | storm-teal `#5fd8e0` / totem-amber `#ffd98a` | `wisp` | storms and ancestors — spirit smoke, not arcane runes |
| **Summoner** | planar violet `#9d7dff` | `rune` | conjuring circles; violet against the Wizard's blue |
| **Enchanter** | mesmeric rose `#ff8fd0` | `wave` | illusion bends the air into sigils |
| **Ranger** | moss `#a8d86a` / leather `#d9b378` | `precise` | a hunter doesn't announce themselves |

**Motions reuse existing draw branches — nothing new was invented.** Only `rune`, `wisp` and `wave`
have ring branches; `bloom`/`wisp`/`rune` have pulse branches. **`precise` has neither, so the Ranger
gets colours only.** Stated plainly rather than dressed up: giving a hunter a flourish means writing a
new renderer, which is a build, not a palette.

## Placement was the entire fix — and only the real spell names revealed it

`vfxStyle` tests **in order**, and these four classes cast spells named after *other* schools:

| class | its own spells | would match |
|---|---|---|
| Shaman | Lightning Spark · Stormcall · Chain Storm · Ghostfire | **the Wizard line** |
| Summoner | Arcane Spark · Soul Link · **Elemental Rotation** | Wizard / **Necro — `"Rotation"` starts with `rot`, so even V0.20.21's `\brot` claims it** |
| Enchanter | Arcane Binding · Glyphstorm | the Wizard line |

**Placed low, the Shaman's entire storm identity would render wizard-blue and the new class terms
would never fire — the exact `warden` shadow bug of V0.20.21, authored by me this time.**

So the four lines go **high**, above necro/wizard/cleric — but deliberately **below the poison and
blade lines**, which name an **effect** rather than a caster and should still win. `\b`-anchored so
`summoner` isn't claimed by *Summon Familiar*, nor `ranger` by a stray *stranger*.

**I would have got this wrong by reasoning.** Reading the spell list is what decided it.

## My first audit was vacuous — again

It checked **280 spell names** through `spawnRing` and reported **0 collisions**.

**Meaningless.** Nothing sets `effect.label` on a ring — **only `spawnStatusPulse` does** — so the
spell name never entered the key. I was auditing a field that wasn't there.

**Caught only by a negative control:** a necromancer casting *"Phantom Army"*, which **must** be
stolen by the enchanter line, reported `canDetect: false`. Without that planted failing case,
*"280 spells, 0 collisions"* would have shipped as proof.

> A test that cannot fail is not evidence. This is the third vacuous probe of the session — the
> `.every()` on empty arrays, the `r.summary` key mistake, and now this. The only one caught *by
> design* rather than by luck is this one.

It also sharpened the model: **rings key off `sourceClass` alone**, so spell names can only collide on
**status pulses**.

## Re-run properly: 9 real collisions, none of them mine

| collision | verdict |
|---|---|
| Assassin *Poison Dart* · *Venom Bolt* · Rogue *Venom Edge* → poison green | **by design** — the poison line is deliberately first; a poison dart *should* read as poison |
| Fighter **Bonebreaker** → necromancy (`bone`) | questionable |
| Cleric **Purify Soul** → necromancy (`soul`) | questionable |
| Druid **Barkskin Blessing** → holy (`bless`) | questionable |
| Druid **Moonfire** → arcane (`fire`) | questionable |

**All pre-existing. All left alone.** The four questionable ones are judgment calls about authorial
intent, not obvious bugs — and "fixing" intent on a guess is exactly how the last several bugs in this
session got written. Reported, not touched.

## Still open — unchanged, and not quietly dropped

- **`combatImpact`/`combatSpark` remain generic.** They come from the **damage** path, and
  `drawCombatImpactEffect` only branches on `motion === 'impact'` — class context would hand a Wizard
  an undrawable `rune` and alter melee.
- **`decorateEffectVisualPayload` pooling**: `kind`/`style`/`school` still default from the recycled
  effect. Latent; belongs to an effect-pooling audit.

## The lesson

V0.20.21 said: *don't ask whether the renderer exists, ask whether anything can reach it.*

**This version adds the third form:** *and then ask whether there was anything there to reach.*
The wiring was perfect and four classes still looked like nothing, because the thing being wired to
had never been written. **I confirmed all three of this session's blind spots in one version** — I
claimed without checking, I audited a field that didn't exist, and I nearly authored the same
shadow bug I had just finished fixing.
