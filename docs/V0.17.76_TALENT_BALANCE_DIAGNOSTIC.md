# V0.17.76 — Talent Balance-Diagnostic Pass

**Type:** Balance (the measurable slice of Talent System spec V0.17.64)
**Previous version:** V0.17.75
**New version:** V0.17.76

The spec's balance phase asks for "1000 fights/spec" tuning to a **+22..30% throughput** band. A full
fight-sim (rotations, cooldowns, procs, boss damage, pets, active mitigation) is a large project of its
own. This pass builds the *measurement* half of it and acts only where the measurement is trustworthy.

---

## The tool

`tools/talent-balance-sim.js` — a **dev-only** simulator. It is **not** in `_script_order.json` and is
never loaded by the game (zero runtime cost). Paste it into the browser console and call
`runTalentBalanceReport({verbose:1})`.

It measures the throughput uplift of a **fully-invested** spec (16 points, capstone taken) over an
untalented level-20 character of the same class, using the **real pipeline**: talent-patched spells
(`getClassSpells`), `talentOutgoingMultiplier`, `StatSystem` crit/mitigation, and the runtime stat/mod
effects. Improvements over a naive first cut:

- **Per-spec metric** (not class role): each spec is classified DPS / EHP / HPS from its own nodes, so a
  damage spec inside a healer/tank class (Cleric Exorcist, Druid Mooncaller, Paladin Judicator) is
  measured as DPS.
- **Spec-biased rotation, unlimited mana**: the rotation favours the spec's empowered spells and matching
  damage types, so specialised trees are measured playing to their strengths (fixes casters reading
  near-0%).

## Documented model blind spots (why we do NOT tune to every number)

- **Pet specs** (Summoner Beastcaller/Swarmweaver, Necro Bonecaller): the damage is the *pet's*; a
  player-cast rotation doesn't simulate it, so `petDamage` trees read far too **low**.
- **Tank specs**: EHP here is passive (armor + `damageTaken`). Active mitigation — Shield of Faith
  charges, Stonehide, Oathbound Stand — is a cooldown the static model can't value, so tanks read too
  **low**.
- **Healer specs**: the HPS proxy uses one best heal spell and can over-count `healingDone`, so healers
  read too **high**.
- **Capstone procs** don't fire in a static model, so their value isn't counted.

The model is **reliable for direct-damage burst specs**; everything else is directional only.

## Tunes applied (only where the model is reliable)

All are reductions, so the PART 7 rails still hold.

| Spec | Change | Uplift |
|---|---|---|
| Ranger **Beaststalker** | Marked Prey 4→3%/rank; Apex Predator +4→+2 Attack | 47% → **38%** |
| Assassin **Nightblade** | Marked 5→3.5%/rank | 38% → **31%** |
| Druid **Mooncaller** | Moonlight 6→5%/rank | 42% → **39%** |

Beaststalker/Mooncaller remain high-30s: they are aggressive/glass-cannon DPS specs that this model
*over*-credits (weapon-attack scaling; a caster trading healing for damage), so pulling them from the
mid-40s to high-30s is the correct conservative step — forcing them to ≤30% would over-nerf against an
inflated measurement.

Pet, tank, healer, and DoT specs were **left alone** on purpose — fitting them to a static model that
can't see their real value would make the live game worse, not better.

## Verification

`node --check` on the tool and `data/talents.js`. Re-ran the three tuned specs after the edits to confirm
they dropped as intended (above). Reductions only ⇒ no PART 7 rail regressions.

## Still deferred (the genuinely-remaining work)

A **real fight-sim** — spawn a spec'd character vs the Silk Web boss set, run rotations with cooldowns,
procs, pets, and active mitigation live, over N fights — plus playtesting. That is what can finally
close the ±22..30% band for pet/tank/healer specs, which no static model can judge.
