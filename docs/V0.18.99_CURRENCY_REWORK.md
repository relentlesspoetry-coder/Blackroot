# V0.18.99 — Full Currency Rework (Roadmap Item 11)

**Directive:** *"everything in the game should use the current currency not just gold."*
**Currency ladder (confirmed):** 100 copper = 1 silver, 100 silver = 1 gold, 100 gold = 1 platinum.

The constants already matched that ladder (`COPPER_PER_SILVER=100`, `COPPER_PER_GOLD=10000`,
`COPPER_PER_PLATINUM=1000000`) — the problem was that the money **paths** didn't use it.

## Root cause

The authored money values — item `sellValue`, quest rewards, chest gold, class-training and mercenary
costs — are on a **silver** scale. But every money path spent/credited them via `addGold`/`spendGold`
(**×10,000**) instead of silver (**×100**): a **100× overcharge**. That's why a basic hood cost ~950
level-8 kills, and why V0.18.98's audit found 395/396 items with a 20,000×–100,000× buy:sell spread.

## The scale was evidenced, not guessed

buy = `sellValue × 2`; a level-8 kill drops ~232 copper:

| reading | Homespun Hood (11) | Iron Helm (76) | Greathelm (192) | verdict |
|---|---|---|---|---|
| as **gold** *(old behaviour)* | 948 kills | 6,552 | 16,552 | absurd |
| as **copper** | 0.1 kills | 0.7 | 1.7 | everything free |
| as **silver** ✅ | **9 kills** | **66** | **166** | sensible ladder |

Silver is also the only reading under which quest rewards (12–140), training (25–85) and the
deliberate rare-gold-coin mob drop (`+10000` copper at 2–9% on elites+) are all coherent.

## The rework — copper is the internal unit, `formatCopper` is the display

| file | change |
|---|---|
| `inventory-system.js` | New `spendCopper` / `addSilver` / `spendSilver` / `silverToCopper` (`spendGold` now delegates to `spendCopper`). **`itemSellValue` returns COPPER** — the silver→copper conversion lives at that one formula site, which automatically makes every existing copper consumer correct: `vendorSellInventoryIndex`'s `addCopper`, the tooltip's `formatCopper`, the bags-full overflow, and `auditEconomy`. |
| `event-system.js` | `itemPrice` returns **copper** (silver ×2 ×100); buy check compares copper directly (was `price * 10000`); payment via `spendCopper`; price + purchase log render via `formatCopper` (was `${price}g`); chest gold credits `addSilver`. |
| `quest-system.js` | Quest money rewards credit `addSilver`. |
| `npc-system.js` | Class training pays copper — **and fixes a latent bug**: the affordability check read the legacy `player.gold` field while the spend went through `spendGold`/`coinCopper`, so the two could disagree. |
| `mercenary-system.js` | Contracts pay copper. |
| `inventory-system.js` (`auditEconomy`) | Compares copper vs copper. |

## Result (verified in-browser)

**Out-of-scale items: 395 → 0.** Buy:sell spread now a healthy **3×–6×** catalogue-wide.

| | buy | sell-back | spread | L8 kills |
|---|---|---|---|---|
| Homespun Hood | 22S | 3S 84C | 5.7× | 9 |
| Iron Helm | 1G 52S | 35S 11C | 4.3× | 66 |
| Hollowsteel Greathelm | 3G 84S | 1G 4S 83C | 3.7× | 166 |

Mob drop (L8) **2S 32C** · Quest **50S** · Training **25S** · Rare boss coin **1G**.

Money flow end-to-end: `addSilver(50)` → 5,000 copper; `spendCopper(25 silver)` → 2,500 left;
overspending is **rejected** with no negative balance.
