## Performance Presets

The preset buttons now write the active preset, update runtime performance mode, mark UI dirty, request resize, clear stale sprite-cache entries, and persist the selection.

Preset intent:

- `Quality`: higher DPR/cull padding/effect budgets with sharp model rendering enabled.
- `Balanced`: default moderate DPR, culling, and effect budgets with sharp model rendering enabled.
- `Performance`: reduced DPR, tighter culling, slower nonessential UI refresh, lower effect budgets, sharp model rendering enabled.
- `Rescue`: lowest safe visual cost, aggressive culling and reduced effect budgets, sharp model rendering enabled.

Low-Spec Mode remains compatible with the preset system.

## Model Blur Root Cause

The blur came from the normal gameplay runtime sprite cache path, not from the procedural model art. F3/debug mode bypassed the runtime cache and drew procedural models directly, which made models suddenly look sharper while debug was active. Normal mode baked procedural actors into an offscreen canvas at unscaled base size, then scaled that bitmap to the final draw size each frame. That introduced soft model edges.

## Sharp Rendering Fix

- Runtime sprite-cache keys now include the quantized final draw scale.
- Cached procedural actors are baked at the final draw scale instead of base scale.
- Cached sprites are drawn one-to-one to the target context instead of scaled during draw.
- Cache bake and target draw both force `imageSmoothingEnabled = false` and clear canvas filters.
- Direct procedural fallback drawing also forces no smoothing/no filter.

This makes normal gameplay match the sharpness of the old F3/debug direct-render path without requiring debug mode.
