# V0.18.6 — High-Poly Spider Webs (Silk Web Cavern graphical upgrade pass)

Detailed high-poly source geometry for the Silk Web Cavern's web assets, for
normal-map baking / detail projection onto the original low-poly web geometry.

## What this is (and the important reality)
The Silk Web Cavern high-poly bake mesh
(`assets/models/highpoly/silk_web_cavern_environment_highpoly_quadmesh_*.obj`) is
an **offline bake/reference asset** — no runtime code loads it (the game renders
in Canvas2D). So this pass touches **nothing** in pathing, collision, or the live
dungeon; it upgrades the source geometry that a baker projects onto the low-poly
webs.

The original generator that produced `v0.15.05` was not in the repo, so this pass
adds a reproducible one: **`tools/generate-silk-web-highpoly.js`**.

## Preservation contract (the CRITICAL constraint)
The generator is an **up-res-in-place** pass:

- The **six structural groups** (`floor_{1,2,3}_structural_floor_quads` and
  `_structural_wall_faces`) — 96% of the mesh — are copied through
  **byte-identical**: same vertices, same faces, same indices. This locks room
  centers, dimensions, corridor links, overall coverage, and boss-arena bounds.
  *Independently verified: all 6 groups identical, per-floor floor-plane bounding
  boxes identical, and the entire original 622,224-vertex block is preserved as an
  exact prefix.*
- Every **web-feature group** (anchor cables, hanging-silk curtains, egg sacs,
  bone/debris strands, gouge rings, shed exoskeletons, egg clutches) is
  regenerated as real high-poly geometry anchored to its **existing bounding
  box / endpoints**, so nothing moves.
- **Non-web** groups (venom pool lips) pass through unchanged.
- New brief detail (prey cocoons, floor/crevice web-fill mats, web balls,
  adhesive droplet beads, bound foreign objects) is added **only within** each
  floor's existing coverage footprint.

## Detail passes (all real geometry — no displacement/normal-map tricks)
- **Anchor cables** → twisted rope of helical sub-strands with sag, an anchor
  fan-splay gripping the stone, adhering broken threads, and dried-adhesive beads;
  thicker/tackier near the brood, greyer/saggier in disused passages (age
  heuristic from floor + room).
- **Web curtains** → subdivided sagging/billowing sheets with a felt-thick center,
  ragged tears (curled snap-back strands), a bottom fringe of dangling threads +
  dew beads (round when fresh, shrivelled when old), and trapped debris
  (insects, bone slivers, husks) through the layers.
- **Prey cocoons** → mummiform capsules with criss-cross wrapping bands, a
  victim-form limb/hand pressing out, and torn-open feeding variants with a hollow
  interior and a few bones; hung from curtains in cocoon/storage/victim rooms.
- **Egg sacs** → teardrop lat-long bodies with papery surface noise, radiating
  support strands, spiderlings (body + legs), and hatched variants (exit hole +
  shed exoskeleton flecks).
- **Gouge tears** → three-clawed Broodwarden slashes with curled torn strands and
  a hasty thin cross-hatch repair patch.
- **Crevice fill** → dusty tangled floor mats with billowy edges and discarded
  web balls, in each floor's corners.
- **Global** → adhesive droplets, and inextricably bound foreign objects (gold
  coin, glass vial, arrow, ripped page) on some curtains.

## Topology
- Quads on all primary surfaces (cables, sheets, sacs, cocoons); triangles only on
  fine threads/strands. **No n-gons.**
- Deterministic (seeded per feature name), so re-runs are stable.

## Output (verified)
- `assets/models/highpoly/silk_web_cavern_environment_highpoly_quadmesh_v0.18.6.obj`
  (+ `.json` metadata), matching the original coordinate space and per-floor Z
  bake-separation.
- **887,310 vertices** (622,224 original preserved + 265,086 new).
- **281,782 quads + 98,824 triangles, 0 n-gons** (independent scan).
- Face indices all in range (1..887,310); 23 MB → 36 MB.
- Feature groups upgraded: 102 curtains, 71 cocoons, 62 egg sacs, 60 debris
  strands, 27 foreign objects, 16 anchor cables, 12 crevice-fill mats, 2 gouge
  tears, 2 shed exoskeletons.

## Regenerating / tuning
`node tools/generate-silk-web-highpoly.js`. Density is a single tunable (`DENSITY`
at the top of the script; 1.0 = shipped baseline) — raise it for an even denser
bake without changing placement. Bake the result onto the original low-poly webs.
