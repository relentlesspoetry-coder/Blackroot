# V0.17.90 — Mob Distribution Rework

**Type:** Gameplay / world (mob spawns)
**Previous version:** V0.17.89
**New version:** V0.17.90

Makes the Dark Woods feel alive and not empty, with distance-based leveling and explicit group-size tiers.

---

## Root causes fixed

1. The old solo-roamer generator (`darkWoodsSoloPoint`) only scattered within **x:14–192, y:18–188** — so
   the 360×360 expansion's outer content (Bandit's Fall out to ~280,110; Stone Hedge to ~250,300; the deep
   Gloamroot reaches) was empty except a handful of named dens.
2. Group sizes were halved by `DARK_WOODS_HOSTILE_DENSITY_SCALE = 0.5`.

## What changed (`systems/mob-spawn-system.js`)

A deterministic **full-coverage tiered generator** (`darkWoodsTieredEncounters`) replaces the old confined
solo-roamer + roaming-group lists. It hash-lattice scatters well-spaced **roaming** encounters across the
entire content area (dist 22–205 from camp) with:

- **Exact group-size tiers** — solo=1, small=2–3, medium=3–5, large=5–7 — flagged `exactCount` so both
  `scaleDarkWoodsCountPair` (the 0.5 halving) **and** `desiredCountCap` (which otherwise caps roaming groups
  at 3) leave them at full size.
- **Family by distance band** — near: wolves/rotlings/boars → mid: boars/spiders/wisps/stags → deep:
  wisps/stags/deep_mixed → outer: deep_mixed/ashroot.
- **Tier weighting by distance** — near camp is mostly solo/small; the deep/outer reaches carry more
  medium/large.
- **Level by distance** — derived from the existing `overworldProgress` / `chooseLevel`. Verified gradient:
  near ~3.1, mid ~6.4, deep/outer ~9.1.
- **Distance-ramped acceptance** (`0.12 + progress·0.26`) — sparse near the "safe ground" camp (where the
  base anchors already sit), denser out in the previously-empty regions — keeping the total population near
  the original (~344 vs the old ~300) while filling the outer regions.

Untouched: the themed camps (`DARK_WOODS_MOB_CAMPS`), the named rares (Old Tusk / Lumen-Wisp / Ashroot
Elder / Rurik), and the quest/landmark spawns. `DARK_WOODS_POPULATION_VERSION` 4→5 re-seeds cleanly over
existing saves (the surplus-removal pass drops old auto-seeded anchors).

## Performance

The engine already has an AI **awake/throttled/sleeping** banding system (+ adaptive downshift), so only
mobs near the player run full AI; distant ones sleep. Total count was kept close to the original that
already ran acceptably, so this is a redistribution (fills the empty outer zone) rather than a raw density
spike. A synthetic all-awake stress test showed ~0.4 ms/mob, but that path is never hit in real play. Final
density is a tunable (`step`, the acceptance ramp, and the tier weights) and can be adjusted from real-device
playtesting.

## Verification (in-browser, port 8899)

Rebuilt the population and inspected it: full-map coverage (Bandit's Fall 92, Stone Hedge 25, Gloamroot 120
mobs where there were near-zero); a clean distance-level gradient (near 3.1 → mid 6.4 → deep/outer 9.1); all
four tiers present (solo 75, small 62, medium 19, large 5 encounters); 58% roaming with every mob carrying a
roam radius; named rares intact; no console errors.
