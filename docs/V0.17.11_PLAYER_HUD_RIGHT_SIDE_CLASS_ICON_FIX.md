## Root Cause Found

The class-art node had a grid-column declaration but appeared before player vitals in DOM order, while both the player row and expandable Party / Companions HUD were children of the same outer grid. If the outer grid display was disrupted or superseded, normal source flow placed the icon before the vitals, producing the observed left-side placement. The player row also lacked a dedicated layout boundary separate from companion content.

## Layout / Render Changes Made

- Added a dedicated `playerHudCore` grid inside the existing player HUD.
- Kept the existing panel width, padding, and 84px art reservation; the HUD was not expanded.
- Player vitals occupy the flexible first column and class art occupies the fixed second column.
- Party / Companions remains a separate full-width sibling below the player core.

## Left-Side Icon Removal Details

- Moved the class-art element after the vitals in source order.
- Kept exactly one `hudPortraitIcon` element and no left-side placeholder or duplicate.
- The removed left-first ordering can no longer become the fallback placement.

## Right-Side Icon Placement Details

- The class-art element explicitly owns grid column 2 and uses `justify-self: end`.
- The existing 84px transparent emblem remains large and unframed.
- All class selection continues through the existing `applyClassEmblem` and shared class asset resolver.
