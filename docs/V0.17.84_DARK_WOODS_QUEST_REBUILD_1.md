# V0.17.84 — Dark Woods Quest Rebuild I

**Type:** Content + engine (quests)
**Previous version:** V0.17.83
**New version:** V0.17.84
**Design source:** `Blackroot Dark Woods Quests.txt` (25-quest set)

Per the user's direction ("Remove all quests from Dark Woods. Add the new quests from this txt.",
**Remove everything** + **Full faithful build, phased**), this begins a multi-version rebuild:
all 11 legacy quests are removed and replaced with the authored 25-quest set. This first version
delivers the reusable objective-verb engine plus the first two regions (9 quests).

- The entire 5-quest **Silk Web Cavern** dungeon chain (Clear the Webbed Mouth → … → The Silk Matriarch).

## New objective verbs (`systems/quest-system.js`)

- **`discover`** — `DR.DISCOVERY_POINTS` (in `data/quests.js`): a zone-keyed registry of named landmarks
  with `{ x, y, radius }`. `checkDiscoveryObjectives()` runs on the quest tick and fires
  `advanceObjective('discover', poiId)` the first time the player enters a point's radius; the POI is
  recorded in `state.discoveredPois`. Coordinates align with the real Phase-15 micro-landmark/story-scene
  props and the named-rare spawn anchors already in the world.
- **overworld `interact`** — `DR.INTERACT_POINTS`: quest-scoped world objects activated with **E** via
  `tryQuestWorldInteract()`. A point is only actionable while an active quest has an incomplete `interact`
  task matching its `objId`, so points never pre-consume. Behaviours:
  - `relightLantern` — relights the nearest dark `lanternPost` prop (visible change; the renderer draws a
    flame for `intact`/lit lanterns and dark for `broken`). **[01]** additionally darkens its 5 target
    lanterns while active (`ensureQuestLanternDarkStates`) so relighting each is a dark→light moment.
  - `grantItem` — grants 1 of an item on activation (drives the well-coin and journal pickups).
  - Multiple points may share an `objId` to express "interact with N instances".
- **`deliver`** — deliver-to-an-NPC accrues on item pickup (like `collect`) and is **consumed on turn-in**;
  deliver-to-a-POI is handed over at the destination via E (consumed there). `isPoiTarget(to)` distinguishes.
- **rotating daily** — a task with `rotateOptions: [...]` picks today's target by a daily index
  (`resolveRotatingTasks`), so a repeatable's ask changes each day.
- **auto-offer / auto-complete** — giver-less HIDDEN quests (`autoOffer` / `autoComplete`) are accepted the
  moment their prerequisites + `requiredConditions` are met and resolve the moment their objectives are met.

Robustness: `syncWorldObjectivesOnAccept` seeds `discover` (already-visited POIs) and `collect`/deliver-to-NPC
(current bag contents) at accept time, so the verbs are order-independent. `state` now persists
`discoveredPois` + `consumedInteractPoints`.

and retrofit to [08].
