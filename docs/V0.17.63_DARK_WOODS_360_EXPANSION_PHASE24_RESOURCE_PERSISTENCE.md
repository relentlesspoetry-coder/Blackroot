# Dark Woods 360x360 Expansion — Phase 24: Resource Persistence + Anti-Farming

**Phase name:** Resource Persistence + Anti-Farming (Phase 24 of 28)
**Previous version:** V0.17.62
**New version:** V0.17.63

---

## Goal

Prevent resource abuse through reloads, zone/floor transitions, and off-screen respawns. Per the plan,
rare nodes must not instantly respawn by leaving/re-entering Silk Web Cavern, reloading, opening/closing
menus, or moving off-screen and back. **No duplicate save system.**

## What was already in place (verified, not rebuilt)

The core anti-farm mechanism already existed and only needed verification:

- **Wall-clock depletion.** `completeHarvest` records
  `state.depletedNodes[zoneKey:x,y:type] = { depletedAt, respawnAt: nowMs() + respawnSeconds*1000, … }`
  — an **absolute** timestamp. `isAvailable`/`remainingSeconds` compare `respawnAt` to now; the timer
  never resets on re-entry, it only counts down in real time.
- **Persistence.** The resource runtime state (skills, `depletedNodes`, `discovered`) is serialized to
  **both** localStorage (`dream-realms.resource-runtime.v2`) and the game save (`resourceRuntime`, via
  `world-serializer`/`save-system`), and restored via `importState`/`normalizeState` on load.
- **Deterministic node positions → stable keys.** Overworld herbs are placed by a deterministic map
  scan; Silk Web herbs and the Phase-23 pools sit at fixed `SILK_WEB_FLOOR_CONFIG` room offsets. So a
  depleted node re-seeds at the **same** coordinates on reload/re-entry → the same `zoneKey:x,y:type`
  key → the persisted depletion still matches. This is the property that makes leave/re-enter farming
  impossible.
- **No reset on visibility/zone change.** Nothing clears `depletedNodes` on off-screen culling, zone
  transitions, or menu toggles; re-seeding (`ensure*`) is idempotent and never touches depletion.

## What this phase added

- **Respawn-range randomization** (the plan's `respawnSecondsMin/Max`). Added those fields to all 8 herb
  defs (`data/resources.js`) using the exact master-plan ranges, and changed `completeHarvest` to roll
  `randInt(min, max)` when a def provides a range (falling back to the single `respawnSeconds`). A
  randomized respawn is less predictable/farmable than a fixed interval, and matches the plan's tables:

  | resource | range (s) | | resource | range (s) |
  |---|---|---|---|---|
  | Lantern Moss | 60–90 | | Webcap Fungus | 180–240 |
  | Thornberry | 90–150 | | Widow's Veil | 300–480 |
  | Gloomcap Mushroom | 180–240 | | Queen's Silkroot | 720–1200 |
  | Wispbloom | 300–420 | | Blackroot Herb | 600–900 |

- **Fishing reconciliation (documentation).** Fishing is fish-anywhere with no depletable nodes, so the
  plan's per-fish respawn timers do not map; the per-cast timer (≈3.5–7 s) is the natural throttle.
  Documented; no depletion/duplicate system added (rare fish are already gated by level + deep-water +
  weight).

### Results
- **JS/JSON:** all pass. Clean boot at V0.17.63 (31 resource types), **zero console errors**.
- **In-browser anti-farm verification (no-cache server):**
  - **Overworld (Blackroot, rare):** harvest → `depletedNodes` gets a future `respawnAt` (e.g. 737 s,
    inside 600–900); `isAvailable=false`; a `serializeState`→`importState` round-trip (= reload) keeps it
    depleted with the **exact** remaining time (737 s); re-running `ensureDarkWoodsOverworldHerbs` (0
    duplicate nodes) does **not** reset it.
  - **Respawn randomization:** 15 harvests produced 14 distinct respawn durations, all within 600–900 s.
  - **Dungeon (Queen's Silkroot, rare):** harvest on floor 3 → depleted (1027 s, inside 720–1200); after
    **leaving to the overworld and re-entering** the dungeon (fresh run, herbs re-seeded), the node
    re-seeds at the **exact same position** and stays **depleted with the exact remaining time preserved**
    (1027 s); also survives a reload while in the dungeon.
  - Def ranges load correctly: Blackroot [600,900], Queen's Silkroot [720,1200].

## Known limitations
- Silk Web dungeon resource grids (`editorResources['silk_web_cavern:F<n>']`) remain in-memory
  (re-seeded per floor entry); only the depletion state is persisted. This is sufficient for anti-farming
  because the re-seed is deterministic (same key), as verified.
- Fishing has no depletion model (fish-anywhere), by design.
