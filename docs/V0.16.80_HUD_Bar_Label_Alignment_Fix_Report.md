# V0.16.80 - HUD Bar Label Alignment Fix Report

## Scope
Implemented the HUD Bar Label Alignment Fix pass against the current V0.16.79 baseline, advancing the project to V0.16.80 because the uploaded project baseline was newer than the phase brief's older version labels.

## Shared bar rendering owner
The player HUD, target/mercenary panels, action-bar EXP meter, and unified Party / Companions HUD are rendered from `systems/ui-system.js`. The legacy Party panel row renderer is owned by `systems/party-system.js`.

## Bars updated
- Player frame HP bar: `HP:` now renders left, numeric value centered.
- Player frame resource bar: `Mana:`, `Stamina:`, `Energy:`, or `Focus:` now renders left, numeric value centered.
- Target HP bar now uses the same split label/value markup.
- Mercenary panel HP/resource/EXP bars now use the same split label/value markup.
- Unified Party / Companions HP/resource bars now use the same split label/value markup.
- Legacy Party panel HP/resource bars now use the same split label/value markup.
- Action bar EXP bar now renders `Exp:` left and current/max centered.

## Implementation notes
- Added intentional split label/value helpers in `systems/ui-system.js` instead of overlaying text after bar rendering.
- Updated unified companion bar markup and fast-update hitboxes/text updates to modify the existing bar text node.
- Updated `systems/party-system.js` legacy party row markup for consistency when that panel is used.
- Updated CSS so `.barText`, `.partyValueText`, and `.partyHealth span` position labels left and values centered.

## Protected behavior
- Resource colors were preserved: HP red, Mana blue, Stamina amber/orange, Energy yellow, Focus green, EXP purple.
- Fill percentages, current/max values, resource spending/regeneration, EXP progression, action-bar layout, party row commands, and spell behavior were not changed.
- No duplicate HUD/bar renderer, cascade patch, wrapper hotfix, or timeout-based fix was added.

- Exact full JSON validation command was attempted and timed out in the sandbox due repeated Python startup cost.
- Equivalent `python3 -S -m json.tool` JSON validation passed.
