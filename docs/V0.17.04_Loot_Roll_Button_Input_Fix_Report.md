## Loot Roll UI Ownership
Loot roll prompt ownership lives in `systems/loot-system.js`:
- `ensureLootRollPanel()` creates the DOM prompt owner.
- `renderLootRollPanel()` renders the active player-facing prompt.
- `submitLootRollChoice()` records Need / Greed / Pass against the active roll session.
- `updateLootRollSessions()` advances timers, bot/mercenary responses, auto-pass, and resolution.
- `maybeResolveLootRollSession()` and `resolveLootRollSession()` resolve the existing session.

## Button Hitboxes and Click Input
The roll prompt is DOM-based, not canvas-hitbox based. The button hitboxes are native `<button>` elements inside the fixed-position `#lootRollPanel`.

Changes made:
- Repositioned the prompt owner to a centered fixed HUD position using `left:50%`, `bottom:128px`, and `transform:translateX(-50%)`.
- Raised the prompt above normal HUD/gameplay layers with a high z-index while keeping it non-modal and combat-safe.
- Added a single delegated panel submit handler used by both `click` and `pointerup`.
- The handler now calls `preventDefault()` and `stopPropagation()` only when a loot-roll button is clicked, preventing stale UI/global handlers from consuming the same click.
- Buttons now use `type="button"` so they cannot accidentally trigger form-like behavior if the DOM context changes later.

## Confirmed Root Cause
The root regression was in `renderLootRollPanel()` session selection and prompt lifetime ownership:

- The old renderer displayed every `rolling` session with `playerEligible`, regardless of whether the local player had already responded.
- `submitLootRollChoice()` could record the choice, but `renderLootRollPanel()` immediately re-rendered the same still-rolling session while bots/mercenaries were pending.
- That made Need, Greed, and Pass appear to do nothing because the same card stayed visible after the click.
- The old prompt renderer also showed up to three roll sessions at once, which made stale prompt/hitbox behavior easier to reproduce when multiple roll items were queued.

This pass fixes the owner by making the player-facing prompt render only sessions where the local participant has not responded and has not been dismissed.

## Decision Recording
`submitLootRollChoice(sessionId, choice)` now:
- Normalizes and validates the session ID.
- Gets the local participant through `localLootRollParticipant(session)`.
- Rejects already-responded or already-dismissed local participants.
- Records exactly one choice.
- Generates a roll only for Need/Greed.
- Stores Pass with no roll value.
- Marks the local participant as responded and prompt-dismissed.
- Re-renders the prompt before resolution so the current card closes immediately.
- Calls the existing resolution path without duplicating roll-session ownership.

## Need / Greed / Pass Behavior
- Need still validates the player can equip the item before recording.
- Disabled Need remains disabled and does not submit or dismiss the prompt.
- Greed records a Greed roll and dismisses the prompt.
- Pass records a pass, no roll value, and dismisses the prompt.
- Late/double clicks are ignored because the local participant is already responded/prompt-dismissed.

## Prompt Dismissal and Queue Advancement
`renderLootRollPanel()` now filters through `playerLootRollPromptPending(session)`.

Behavior after a valid click:
- The current prompt disappears immediately.
- The roll session remains alive internally until all participants respond or the timer resolves.
- Bot/mercenary delays and decisions continue unchanged.
- If another roll is queued and the player has not responded to it, that next prompt is shown.
- If no player prompts are pending, the panel hides and clears its HTML so stale buttons cannot consume clicks.

## Scaling / Fullscreen Handling
The prompt is fixed-position DOM UI anchored to the viewport center-bottom. Because the buttons are native DOM elements, browser hitboxes track visual button geometry automatically under 1440p scaling, fullscreen, and responsive resizing. No canvas coordinate conversion or duplicate hitbox system was added.
