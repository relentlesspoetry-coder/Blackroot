Dream Realms V0.15.76 — Bogling Sprite Rebake and Validation Pass

Reviewed and refined the Bogling models with sharper layered poly-mesh detail, corrected amphibian hand and foot anatomy, clearer webbed fingers and toes, improved grounding and direction-specific layering, and updated equipment/tool alignment while preserving male/female Bogling support, all color variants, character creation, runtime rendering, and save/load compatibility.

This pass audited the Bogling hand/foot geometry introduced in V0.15.69-V0.15.75, replaced the flat kite-shaped hand and the single-blob foot polygon with a shared digit-and-webbing helper used by both hands and feet (individual tapered fingers/toes with true membrane webbing in the gaps, not one solid fan), and confirmed the V0.15.74/V0.15.75 arm/leg attachment fixes hold across all 8 directions, both genders, and every color variant. All 192 Bogling sprite sheets (2 genders x 8 palettes x 12 animation states) were rebaked from the corrected procedural renderer via the existing Canvas2D export pipeline so runtime and character-creation rendering (which prefer the baked sprite sheets over the live procedural fallback when available) match the fixed anatomy exactly. Full validation was run: node --check on every project JavaScript file, python3 -m json.tool on every project JSON file, and a check that every <script src> in the main HTML resolves to a real file.
