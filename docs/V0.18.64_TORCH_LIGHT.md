# V0.18.64 — Torch Light

> "the torch isn't emitting any light, i want the caves the feel dark and light sources should project light through the darkness."

Showcase: [V0.18.64_torch_light.png](V0.18.64_torch_light.png) — two torches casting warm pools out of the dark cave, with a faint player light in the middle and drifting fog.

## Root cause
`drawTorch` already drew a small warm glow, but it's painted **inside the world pass**. The Silk Web Cavern's darkness — `renderSilkCavernAtmosphere` — is a **full-screen overlay drawn after the world**, so it painted straight over the torch's little glow. The torch read as completely unlit.

## Fix
1. **Register torches as lights.** `drawTorch` (`render/object-renderer.js`) now pushes each visible torch into `game._silkFrameLights` (its unzoomed screen position + colour). `game.js render()` resets that list each frame just before the world draw, so it holds exactly the torches currently on screen.
2. **Darken + light pass** in `renderSilkCavernAtmosphere`:
   - The cave is **darker** (tint `0.4`→`0.52`, vignette `0.55`→`0.66`) so it reads pitch-black away from any light — which is what makes the torch pools read as *light*.
   - After the darkening, a **light pass** runs: each registered torch projects a warm, gently **flickering** pool — an additive (`lighter`) radial gradient, bright amber core falling off to orange — **projected from unzoomed screen space through the live camera zoom**, so the pools track pan/zoom exactly. The **player** also gets a much dimmer warm light so your party is never in total black with no torch nearby (torches clearly dominate).

Net effect: a dark, enclosed cave with torches punching warm pools of light out of the black. Gated to the Silk Web Cavern (where the darkness overlay lives).
