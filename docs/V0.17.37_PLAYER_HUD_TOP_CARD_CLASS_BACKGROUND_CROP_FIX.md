## Root Cause
The class-background resolver was resolving the correct image, but the Player HUD backdrop was attached to the entire `#hud` container. That container includes both the top player card and the Party / Companions stack. Because the class art used `background-size: cover` across the full tall HUD, the top player card displayed the dark upper crop of the image and read visually as a black panel.

The Party cards looked correct because each companion card owns its own wide, short class-background surface. The Player HUD needed the same crop behavior on only the top player card, not a full-HUD background.

## Fix Details
- Scoped `.hudClassBackdrop` to the top Player HUD card area using the same wide-card crop philosophy as companion cards.
- Kept the backdrop as a single owned surface. Legacy optional image/veil children remain disabled.
- Updated the runtime helper so inline class-background styles also position the backdrop as a top-card surface.
- Preserved the existing Player HUD layout and Party / Companions card layout.

## Party HUD Regression Verification
Party cards were not redesigned. Their existing class-background card renderer, status text, bars, slot count, and command buttons were preserved.
