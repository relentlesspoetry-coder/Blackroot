# V0.16.58 - Character Select and Class Layout Polish Report

## Summary

This phase updates only the Character Select screen, the Main Menu floating class-icon layout, and the Character Creation class-card order. Gameplay, combat, spells, inventory, bags, character stats, action bar, and the in-game character HUD were left untouched.

- `Blackroot V0.16.58.html`

## Systems Touched

- Character Select presentation/layout
- Main Menu floating class icon placement/sizing
- Character Creation class-card ordering
- Version metadata and release documentation

## Character Select Layout Changes

- Expanded `#characterSlotModal` from the capped centered panel to a viewport-responsive full-screen framed panel using up to `98vw` by `95vh`.
- Increased the selected character showcase area by converting the Character Select body to a two-row layout: large featured showcase plus a clear four-slot switcher row.
- Enlarged the featured preview stage and increased the featured character render scale from `2.6` to `3.15`, keeping the existing animated preview renderer and class-background path.
- Enlarged slot cards, slot emblems, labels, and spacing while preserving the existing `data-slot-select` and `data-slot-action` hitboxes.
- Kept Play/New/Delete/Create, Load Game, Link Save Folder, Account Menu, and Close controls in the same owning screen and interaction path.
- Reduced the footer help text footprint so it no longer wastes vertical layout space.

## Main Menu Floating Icon Layout Changes

- Rebuilt the `MAIN_MENU_FLOATING_CLASS_SIGILS` placement data into explicit left/right lane assignments.
- Increased floating class icon sizes and opacity for stronger readability.
- Added `floatingClassIconBoundsForSpec()` so movement remains bounded to each icon's assigned side instead of drifting into the central logo/menu zones.
- Preserved existing safe-zone resolution against the logo, menu buttons, and open modals.
- Preserved the existing class-emblem rendering and animation/update path; no duplicate floating icon system was added.

## Character Creation Class Sorting Changes

- Sorted the real Character Creation class gallery order alphabetically by display name:
  `Assassin`, `Bard`, `Cleric`, `Druid`, `Enchanter`, `Fighter`, `Necromancer`, `Paladin`, `Ranger`, `Rogue`, `Shaman`, `Summoner`, `Warden`, `Wizard`.
- The sort uses the existing `CLASSES` lookup and class names as gameplay IDs, so cards remain connected to their real icons, roles, backgrounds, colors, and selection behavior.
- No duplicate class arrays or fake cards were added.

## Class Data / Order Safety Notes

- Character Creation still assigns `this.selectedClass = name` from the canonical `CLASSES` object.
- Class icons still come from `classEmblemImagePath(name)`.
- Class roles still come from `CLASSES[name].role`.
- Class backgrounds continue to resolve through the existing preview/background renderer using the selected class name.
- Randomize remains appearance/gender only and does not randomize class.
