# V0.18.17 — Talent Tree UI revamp (spec backgrounds, no locks, route lines)

Three changes to the talent panel (`systems/talent-system.js`), from user-supplied
spec background art.

## 1. Per-spec painted backgrounds
The user provided 14 `<Class> Spec Backgrounds.png` images, each containing the
**three spec panels side by side** (ornate portrait frames). These were cut into 42
per-spec PNGs and dropped into `assets/talents/<specId>.png` — the exact filenames
the panel already auto-discovers (see `assets/talents/README.md`), so no wiring was
needed; each spec column now shows its own art under the existing dark readability
veil + spec-colour glow.

- Cutter: `tools/cut-spec-backgrounds.py` (gutter-detected split near the thirds,
  downscaled to ~480px wide). Reproducible if the source art is re-exported.
- Naming/order came from `data/talents.js` (`DR.TALENT_TREES[class].specs[].id`),
  so left/middle/right panel → spec 0/1/2.

## 2. Lock icons removed
The `🔒` glyphs on locked tiles and locked tier gates were removed (and the unused
`.talTileLock` CSS). Locked tiers are still clearly indicated by the existing greyed
/ desaturated styling (`.talLocked` — dim border, `grayscale(.7) brightness(.55)`),
which the user preferred.

## 3. Route connection lines
Talent nodes are tier-gated (spend N points in a tree to unlock the next tier), so
the "route" is the T1→T5 ladder. The weak per-tier stub was replaced with clear
connectors drawn behind the tiles:
- **Vertical link** (`.talTierLink`) from each tier up to the one above — the tier
  ladder.
- **Horizontal rung** (`.talTierRung`) joining the sibling nodes within a tier,
  sized `(nodeCount-1)×78px` so it spans exactly icon-center to icon-center.
- Both are dim while locked and **glow in the spec colour once the tier is reached**.
- Tiles are layered above the connectors (`z-index`), so lines read as connections
  between the node icons.

## Verified (in-browser, 0 console errors)
- 3 spec columns each load their background image; **0 lock icons** in the panel.
- 12 vertical links + 6 horizontal rungs render; spending points lights the reached
  ones (4 active).
- Connector geometry is exact: a rung's centre sits at the icon centre height and
  spans precisely between the two node icon centres (411→489px, width 78).
- All 42 background PNGs serve over http.
