# V0.20.39 — High-Contrast Target Ring (Roadmap Item 20)

The **4th of 12** accessibility features — plus an honest finding on the one I *didn't* build.

## The finding first — UI/font scaling is a refactor, not a toggle

UI/font scaling is the highest-value remaining accessibility item, so I checked its tractability
before anything else. It is **not a clean toggle** in this build:

- **165 hardcoded `px` font-sizes** in the HUD stylesheet (only 1 `em`).
- **105 independently `position: fixed/absolute` panels**, each anchored to a screen edge.
- **No `--ui-scale` variable** and **no single container** wrapping the HUD.

So scaling means either a **px → rem refactor** across 165 sites or a **per-panel zoom** across ~105
panels — a real multi-version refactor with layout-overflow risk, not a pref. **Reported for a
dedicated pass rather than half-built.**

## What I did build — high-contrast target ring

`drawActiveTargetIndicator` (one clean function) now reads a `highContrastTarget` pref (off by default,
its own Accessibility row, persisted). When ON, the current-target reticle draws:

| | normal | high-contrast |
|---|---|---|
| main ring colour | amber `#ffe56d` | **white `#ffffff`** |
| line width | 3 | **5.4** (×1.8) |
| opacity | 0.96 | **1.0** |
| backing | translucent brown | **solid black `#000000`** |
| ticks | pale amber | **white, thicker** |

So a low-vision player can always tell which enemy is selected. **Only the indicator's look changes —
targeting itself is untouched.**

## Verified

Captured the real stroke parameters by instrumenting the canvas context:

| | main ring |
|---|---|
| OFF | amber `#ffe56d`, lw **3**, alpha **0.96** |
| ON | white `#ffffff`, lw **5.4**, alpha **1.0**, over black backing |

Pref in defaults ✅. **Suite PASS:** `ok: true`, items 396/396, **0 console errors**. `validate.sh` PASS.

## The lesson

**Check tractability before committing to the marquee item.** Font scaling *sounds* like the obvious
next accessibility win, but a five-minute look at the stylesheet showed it's a refactor, not a toggle —
so I built the clean, high-value thing that *was* a toggle (the target ring) and documented the refactor
honestly instead of forcing a fragile zoom hack and calling it done.
