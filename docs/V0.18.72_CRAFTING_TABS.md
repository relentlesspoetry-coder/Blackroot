# V0.18.72 — Crafting Window Category Tabs

> "Crafting UI needs cleaned up better. We need buttons for each category — Food, potions, Armor, Weapons etc. You click the button and then it shows the recipes for that category."

## What changed (`systems/crafting-system.js`)
The crafting window (`refreshWindow`) now has a **category tab bar** across the top instead of one long grouped list:
- an **"All"** tab (the full grouped list), plus
- **one button per category the current station actually makes** — Armor / Weapons / Cooked Meat / Cooked Fish / Meals / Health Potions / Mana Potions / Buff Potions / Materials / Other — each showing its **recipe count**.

Clicking a tab **filters the list to that category**. The active tab is highlighted; the choice (`runtime.selectedCraftCategory`) **persists** across re-renders and while crafting; and the window **opens already filtered** to the first category rather than as one long scroll. The per-category headers from V0.18.70 now only appear inside the "All" view.

Category clicks are handled in the window's click listener alongside the existing craft button — a `button[data-craft-cat]` sets `selectedCraftCategory` and re-renders.
