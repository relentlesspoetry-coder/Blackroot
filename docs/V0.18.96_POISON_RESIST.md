# V0.18.96 — Poison Resistance Implemented (dead-stat follow-up)

The `poisonResist` item stat — flagged by V0.18.93's data-integrity check as recognised-but-
unimplemented and kept on a whitelist — is now a real, working stat. **Implemented** rather than
retired: poison resistance is thematically core to the Silk Web Cavern venom content.

## The problem

These items carried `poisonResist`, but it was in no stat pipeline and read by nothing — a dead stat:

| item | poisonResist |
|------|-------------|
| item_venom_stained_boots | 3 |
| item_old_venomsacs_gland | 4 |
| item_silk_web_antivenom_charm | 6 |
| (antivenom trinket) | 8 |
| Antivenom potion (buff) | 20 |

## The implementation (mirrors the existing `magicResist` rating → reduction pattern)

**`core/stats.js`**
- `'poisonResist'` added to `STAT_KEYS`.
- `STAT_RULES.poisonResist` — a **niche** resistance (only matters vs poison/venom), so it stacks more
  generously and caps higher than magic resist: `cap 0.75, denominatorBase 24, denominatorPerLevel 3.0`.
  It never fully negates poison.
- `computePlayerDerived` derives `poisonResistRating` + `poisonResistReduction` (same additive-rating →
  diminishing-return shape as magic resist; opt-in via the stat; `0`/inert otherwise).
- New `poisonResistReductionFor(entity)` helper mirrors `magicResistReductionFor`; both exported on
  `DR.StatSystem`.

**`systems/inventory-system.js`**
- `recalculatePlayerStats` assigns `player.poisonResistRating` / `player.poisonResistReduction` from
  the derived stats.
- `validateItemData` no longer whitelists `poisonResist` — it is a real `STAT_KEY` now, so a genuine
  typo still stands out.
- Regression guard added to `runStatPipelineSelfTest` (#14): a `poisonResist` item yields a positive,
  capped reduction.

**`systems/status-effect-system.js`**
- In the DoT tick, poison/venom effects (`isPoisonLikeEffect`) now reduce their tick damage by the
  victim's `poisonResistReductionFor` before it is dealt.

## Curve behaviour

`reduction = (rating / (rating + (24 + level*3))) * 0.75`. For a level-6–7 player:

- 3 venom pieces → rating ~17 → **~21%** poison reduction.
- Adding the Antivenom potion (+20) pushes it toward the cap for the duration.
- Capped at **75%** — poison always does something.

## Verification

- `node --check` clean; JSON valid; boots with **0 console errors**.
- `poisonResist` is a `STAT_KEY`; **no gear → 0 reduction** (inert).
- 3 venom pieces → rating **17**, reduction **0.206** (within the 0.75 cap); `poisonResistReductionFor`
  matches the derived value.
- A **100-damage poison DoT → 79** (the reduction applies to real poison damage).

  0 compiler errors.
