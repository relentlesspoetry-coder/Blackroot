## The core architectural decision

Talents do **not** grant new castable abilities. The hotbar is already full (20 compiled slots, 20
spells per class). Instead, talents **empower the 20 class spells that already exist** in
`data/spells.js`, plus add flat stat ranks, runtime combat multipliers, and passive capstone procs.
Every Paladin signature ability the spec wanted to touch already exists by exact name â€” verified
against the live spellbook â€” and the compiler passes through arbitrary spell fields
(`spell-compiler-system.js:270-273`), so `charges` / `damageTakenMultiplier` / `power` /
`threatMultiplier` are all patchable on the compiled object.

### Performance

`talentModifier()` is called from `damageEntity` (hottest path). Unconditional multiplicative mods are
pre-folded into a single scalar per bucket (`talentModsFlat`); only the handful of conditional mods are
iterated â€” never a 294-node scan. `applyTalentSpellMods()` returns an index-preserving shallow-clone
copy, memoized on `(sourceArrayRef, talentRevision)`, so it recomputes only on rebuild.

## Deviations from the spec (real fields vs prose)

- Consecrated Ground uses `tickDamage` (it is a ground DoT), not `power`.
- Radiant Bulwark uses `absorbFlat`/`absorbMaxHpPct`; Sunlit Aegis has no absorb field, so Blessed
  Bulwark instead strengthens its `magicDamageTakenMultiplier`.
- Oathbound Aegis's "take less damage while Oathbound Stand is active" is implemented by patching that
  buff's own `damageTakenMultiplier` (no new runtime `when` needed).
- Capstones lean on existing status-buff fields honored by combat (`damageTakenMultiplier`,
  `lethalSave`, `holyDamageMultiplier`) plus a timed "dynamic mod" for threat. Unbreakable Oath's
  lethal-save currently spans the full 8s (spec: first 3s); Reckoning is a holy-damage buff rather
  than a next-cast-free consumption. Both are noted for the faithful pass.
