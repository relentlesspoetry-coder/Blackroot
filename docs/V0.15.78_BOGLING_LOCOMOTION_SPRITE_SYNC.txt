Dream Realms V0.15.78 — Bogling Locomotion Sprite Sync

Verified the new Bogling hop-waddle and frog-kick locomotion frame-by-frame across genders, directions, and color variants, rebaked the Bogling walk and swim sprite sheets that had gone stale against the corrected procedural model, and ran a full validation pass.

Audited the V0.15.77 Bogling Amphibian Locomotion pass by rendering exact frames of the 8-step hop-waddle and frog-kick cycles (via the `animationPhase` override already supported by HumanoidAnimationSystem.buildPose) rather than trusting the source alone. Confirmed swim correctly overrides walk in the animation priority order, confirmed the frog-kick power stroke shows clearly spread webbed feet with the forelimbs staying subordinate for stabilization, and confirmed no limb detachment across south/east/north/southeast directions for both genders. Found that the on-disk Bogling walk and swim sprite sheets (32 files: 2 genders x 8 palettes) predated the hop-waddle/frog-kick rewrite and still contained the previous generic locomotion baked in; regenerated all 32 through the existing Canvas2D export pipeline so runtime and character-creation rendering, which prefer baked sprite sheets over the live procedural fallback when available, now match the corrected locomotion. All other Bogling sprite sheets (idle/attack/cast/hit/death/meditate/dance/harvest/mine/fish) were unaffected by this pass and left untouched. Full validation was run: node --check on every project JavaScript file, python3 -m json.tool on every project JSON file, and a check that every <script src> in the main HTML resolves to a real file.
