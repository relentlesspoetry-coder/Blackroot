# V0.18.1 — Patch Notes source fix (stop the freeze)

The in-game Patch Notes panels were frozen on **V0.17.25** — every release since then
updated `PATCH_NOTES.md` (per the golden rules) but the panel never showed it.

## Root cause: dual-source drift

The panels (`#splashPatchNotesPanel`, `#menuPatchNotesPanel`) only ever rendered a
**hand-embedded copy** of the notes baked into the HTML as `.patchNoteEntry` sections.
`refreshPatchNotesPanels` just **cloned that embedded copy** between the two panels;
**nothing read `PATCH_NOTES.md`**. So the markdown file (canonical) and the embedded copy
were two independent sources, and the embedded one stopped being maintained after
V0.17.25. Tellingly, the container already carried `data-patch-notes-source="PATCH_NOTES.md"`
— the intended source that was never wired up.

## Fix (`systems/ui-system.js`)

- **`renderPatchNotesMarkdown(md)`** — parses `PATCH_NOTES.md` into the exact
  `patchNoteEntry` markup: `##` → entry title `<h4>`, `###` → sub-header `<h5>` (kept
  *inside* the entry, not a new one), `-` → bullet, 2-space-indented `-` → nested bullet,
  and inline `**bold**` / `` `code` `` / `*em*`.
- **`ensurePatchNotesFromSource()`** — fetches `PATCH_NOTES.md` **once** (only over
  `http[s]`; `file://` can't fetch) and stamps the rendered HTML into every
  `.patchNotesEntries` container. `refreshPatchNotesPanels` calls it, so on a served build
  the panel is **always the live `PATCH_NOTES.md`**.
- Graceful fallback: if the fetch can't run (`file://`) or fails, the embedded copy stays.

## Fix (`tools/sync-patch-notes.py`, new)

A reusable release script that regenerates the **embedded fallback** from `PATCH_NOTES.md`
(same parsing rules), replacing the inner content of the populated `.patchNotesEntries`
container in the current `Blackroot V*.html`. This keeps `file://` builds current and
avoids a stale flash before the runtime fetch lands.

**Release-process addition:** after editing `PATCH_NOTES.md`, run
`python3 tools/sync-patch-notes.py`. (The runtime fetch handles served builds
automatically; the script keeps the standalone-file fallback in sync.)

## Verification (in-browser)
- Panel first entry was `V0.17.25 - Rotling Root…`; now `V0.18.1 - Patch Notes Actually
  Update Now`, then `V0.18.0`, `V0.17.99`, … (screenshot).
- 240→241 entries + 158 `<h5>` sub-headers render; embedded and fetched output are
  identical; `**bold**` renders; tag balance intact (div 220/220, section/ul balanced);
  0 console errors.
