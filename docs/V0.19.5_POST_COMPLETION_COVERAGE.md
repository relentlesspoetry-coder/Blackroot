# V0.19.5 — Post-Completion Coverage (Roadmap Item 1)

## The key finding: the two "gaps" weren't gaps

Filling them would have created **dead data** — the exact anti-pattern this session has spent its time
removing.

### (a) "5 quests missing `beforeOfferText`" — correct as-is

`beforeOfferText` is resolved inside an **NPC dialogue** function (`systems/npc-system.js`):

```
npc.dialogueVariants → npc.dialogueText → npc.dialogueId → quest.beforeOfferText
```

It can **only display when the player talks to a real NPC**. All 5 quests lacking it have **no NPC
giver**:

| quest | why it has no NPC |
|---|---|
| `quest_follow_the_blood`, `quest_lookouts_last_watch` | hidden / autoOffer, `giver: null` |
| `quest_ruriks_bargain`, `quest_fall_of_rurik` | given by "Rurik the Fallen" — a **mob** |
| `quest_fourth_member` | auto-offers at a POI |

**Verified: of the 21 quests with a real NPC giver, 21/21 already have `beforeOfferText`.** The state
is complete; those 5 were correctly omitted.

### (b) Repeatables never reach the post-completion branch

A completed **repeatable** is re-offerable, so the picker returns `mode: 'accept'` rather than falling
through to the post-completion line (`systems/quest-system.js` modes: complete / accept / progress /
locked, then the fallthrough). So `postCompletionText` only ever displays for a **non-repeatable**
quest reachable through a **real NPC**.
