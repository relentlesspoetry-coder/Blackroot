# V0.20.25 — A Poison You Can See While It Kills You (Roadmap Item 6 fidelity)

## The spec's other named case

> *"A damage-over-time effect must persist while the debuff is active."*

Same root as V0.20.24: the entity renderer had no per-frame pass over active buffs, so a DoT ticked
with no lasting mark — a target could be burning, poisoned and bleeding at once and look untouched
between ticks. V0.20.24 built the pass for shields; this extends it to DoTs.

The correctness property carries over: the plume is drawn straight off `actor.buffs`, so its visual
lifetime **is** the DoT's gameplay lifetime — appears the frame the DoT lands, gone the frame it ends.

## The finding that shaped the whole design

**`buff.color` does not encode the element.** Ground-truthed against real casts:

| DoT | element | `buff.color` ships | correct tint |
|---|---|---|---|
| **Ignite** | fire | **`#6fa8ff` — the Wizard's blue** | orange |
| Bleeding Vein | bleed | `#cfd4d9` grey | red |
| Grave Rot | disease | green | green |

A fire DoT tinted by `buff.color` would read as **frost**. The spec requires the visual to communicate
**damage type** — the one field `buff.color` gets wrong. So `dotElementColor` derives from `damageType`
first, then `tags`, and **never** from `buff.color`. Verified: Ignite renders `#ff7a3d` orange, not
blue.

`isDotDebuff` keys on the **definition** — `hostile && periodicDamage > 0 && remaining > 0` — not the
`'dot'` type string, which some periodic debuffs omit.

## The deferral concern, measured and dissolved

DoTs are on nearly every enemy in a fight, so this was deferred in V0.20.24 pending a clutter/perf
pass. Both halves are now addressed with numbers, not hope:

**Clutter** — every DoT on an actor **merges into one plume** tinted by the strongest, motes capped at
4. A 5-DoT enemy shows one affliction, not five.

**Cost** — measured:

| scenario | cost |
|---|---|
| unafflicted actor (early-exit) | **0.01 µs** |
| fully-afflicted actor (5 DoTs) | **3.83 µs** |
| **30 fully-afflicted enemies** | **0.11 ms/frame** — 0.7% of a 60fps budget |

The plume itself is cheap: one element-coloured ground taint ellipse + up to 4 small rising motes, no
gradients. Mote count scales with total DoT dps — the spec's **"Power"** cue. No fade-**in** (DoTs are
short and want instant combat feedback); fades **out** only in the final 0.6s so expiry reads.

## The lesson

V0.20.24's was *"ask what draws while this buff is alive."* This one adds: **when you finally draw it,
don't trust the colour the data hands you — check what it actually means.** `buff.color` was present,
non-null, and authored, and it said a fire spell was blue. The element lived in `damageType`, one field
over. Same shape as V0.20.23's radius that meant tiles to one function and pixels to the next: **a
value being present is not the value being right.**
