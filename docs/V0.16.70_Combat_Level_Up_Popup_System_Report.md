# V0.16.70 - Combat Level-Up Popup System Report

## Summary
Implemented a centered, translucent combat level-up popup that appears when the player gains one or more combat levels. The popup shows the final current combat level, remains visible for a five-second lifetime, drains a countdown bar, fades during the final 700ms, and then removes itself from active UI state.

## Combat Level-Up Detection
Combat level-up detection remains owned by `Game.prototype.checkLevelUp` in `systems/combat-system.js`. The method now records the old level before resolving XP rollover, processes all level-ups normally, and triggers the popup once with the final new level only when `finalLevel > oldLevel`.

This avoids polling from UI and avoids triggering from save/load, stat recalculation, character-panel open, or non-combat profession/skill level systems.

## Popup State Location
Popup state lives on the game instance as `this.combatLevelUpPopup` and uses this canonical shape:

```js
{
  active: true,
  level,
  startedAt,
  durationMs: 5000,
  fadeOutMs: 700
}
```

## Popup Rendering Location
Rendering is owned by `systems/ui-system.js` through:

- `showCombatLevelUpPopup(level)`
- `updateCombatLevelUpPopup(deltaMs)`
- `renderCombatLevelUpPopup()`
- `hideCombatLevelUpPopup()`

The popup is a DOM overlay with `pointer-events: none`, so it renders above normal HUD without consuming movement, combat, hotbar, menu, mouse, touch, or keyboard input.

## Chat/System Log Hook
Added the required log line through the existing log owner:

```text
You have reached Combat Level X!
```

It is emitted once per resolved combat level-up batch, using the final level after all XP rollover is complete.

## Sound / Particle Hooks
The existing combat level-up code already plays the `level_up` SFX and spawns a gold ring effect at the player during each level-up. No duplicate sound system, particle system, or heavy popup particle system was added.

## Multiple-Level Handling
If the player gains multiple combat levels from one XP resolution, the normal stat/level loop still resolves each level, but the popup is shown once with the final level.

## Non-Combat Level Protection
Profession, crafting, gathering, fishing, and meditation level systems were not modified. They do not call the combat-level popup owner.

## Input Safety
The popup uses `pointer-events: none` and does not add any click, key, pointer, or focus handlers. It cannot create dead input zones or block action-bar input.

## Performance Notes
The popup avoids `setTimeout`, avoids new canvases, avoids per-frame heavy particles, and updates only opacity, transform scale, and countdown-bar transform while active. Sparkles are CSS-only and limited to ten small elements inside the panel.
