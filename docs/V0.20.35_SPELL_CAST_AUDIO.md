# V0.20.35 ‚Äî Spells That Make A Sound (Roadmap Item 18, first pass)

## The gap

The spells were given a distinct **look** over V0.20.21‚Äì29, but they cast in near-silence. Measured:
across ~280 spells, **only 3 `playSfx` sites existed in `spell-system.js` ‚Äî all pet summons.** So:

| spell | before |
|---|---|
| Fireball (aoe) | **silent** |
| Cleave (melee) | **silent** |
| Mesmerize (debuff) | **silent** |
| Arcane Bolt (bolt) | only `hit_thud` on *impact* ‚Äî no cast |
| Lightning Spark | only `hit_thud` ‚Äî no cast, no thunder |
| Minor Heal | `heal_chime` on the target only |

There was no **cast-moment** sound. The audio parallel to the VFX gap, exactly.

## The fix ‚Äî the same funnel as the VFX identity

`resolveClassSpell` ‚Äî the dispatcher every class spell passes through ‚Äî now plays a **cast sound**
chosen by element/kind from the **existing** SFX manifest (asset files verified present):

| spell trait | cast sound |
|---|---|
| lightning / storm / shock / thunder | `thunder_roll` |
| melee / physical / slash / cleave | `attack_slash` |
| everything else (fire, frost, arcane, heal, debuff‚Ä¶) | `magic_cast` |

- **Impact/effect sounds are untouched** ‚Äî `hit_thud` on bolts and `heal_chime` on heals still fire;
  the cast sound layers *on top*. No doubling.
- Slight random pitch (`rate 0.94‚Äì1.06`) + `0.08s` cooldown so rapid casting doesn't sound robotic or
  spam.

## Honest limit

The manifest has **no per-element cast assets** ‚Äî there's no `fire.wav`/`frost.wav`/`arcane.wav`, so
those elements share `magic_cast`. Making fire *sound* different from frost (the way they now *look*
different) needs **new sound files, which I can't author.** This pass differentiates as far as the
shipped assets allow ‚Äî silent ‚Üí audible, with thunder and blade where matching sounds exist ‚Äî and the
selector (`spellCastSfxId`) is one function that a future audio drop can extend the moment new assets
land.
