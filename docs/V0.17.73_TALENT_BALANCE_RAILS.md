# V0.17.73 — Talent Balance-Rails Enforcement

**Type:** Balance (the enforceable slice of Talent System spec V0.17.64 / PART 7)
**Previous version:** V0.17.72
**New version:** V0.17.73

The spec says the PART 7 rails are "hard invariants — enforce them in the compiler, not in review."
This pass does exactly that. The **full** balance pass (sim 1000 fights/spec, re-tune every per-rank
number, make the approximated capstones mechanically faithful) still needs a fight-simulation harness
and stays deferred — this is the deterministic, verifiable part.

---

## What changed

### 1. Cost / cooldown / cast-time 25% floor (rail 2)

`applyTalentSpellMods` now floors `cost` / `cooldown` / `castTime` at **25% of the spell's base value**
for **all** operations. Previously additive reductions (`op:'add'`, e.g. cooldown `-0.5/rank`) only
floored at 0, so several stacked reductions could drive a value toward zero. Multiplicative fields keep
their existing `[0.25×, 4×]` base clamp.

### 2. Automated rails validator + 6 retunes

An in-browser validator checks all **294 nodes** against the PART 7 caps:

| Rail | Check |
|---|---|
| damageDone | per node ≤ ×1.20 unconditional / ≤ ×1.35 conditional |
| damageTaken | per node ≥ ×0.85 unconditional / ≥ ×0.70 conditional |
| critChance | ≤ +6 rating at max rank |
| capstone | `icd` ≥ 5s, has effects |
| rail 5 | no node references gold / xp / loot / profession |
| rail 1 proxy | spec-wide product of **unconditional** damageDone ≤ ×1.20 |

All per-node caps already passed. The rail-1 proxy flagged **6 specs** that stacked 2–3 filler
generic-damage nodes to ×1.23–×1.38 (Warden Thornwarden, Fighter Breaker, Rogue Shadowblade, Ranger
Beaststalker / Trapper / Deadeye). Each had one node retuned so the spec's unconditional damageDone is
now ≤ ×1.15:

- Thornwarden **Bramble Field**, Breaker **Shatter Guard**, Beaststalker **Beastbane** → gated to
  `targetHpBelow 0.5`.
- Shadowblade **Ambush** → gated to `targetHpAbove 0.8` (opener theme).
- Beaststalker **Apex Predator** → +Attack; Trapper **Area Denial** → +Damage Power; Deadeye
  **Armor Piercing** → +Crit Damage.

### Already enforced (unchanged)

- **Rail 4** — sum of `damageDone` mods clamped to ×2.0, `damageTaken` floored at ×0.40, in
  `talentModifier` (`MOD_CLAMP`).
- **Rail 3** — every capstone has `icd ≥ 5`, a visible tell (`spawnRing` + `logCombat`), and the
  `fireTalentProc` reentrancy guard prevents proc→proc chains.
- **Rail 5** — talents only patch their own class's spells (by construction) and never grant castables.

## Verification (in-browser, port 8899)

Validator reports **0 violations** across all 14 classes after the retunes. No console errors.

## Still deferred (spec V0.17.64 proper)

Fight-sim tuning of the provisional per-rank numbers to the +22..30% throughput band, and rewriting the
approximated capstones (DoT detonation, chain lightning, summon-one-of-each) into their full mechanics.
