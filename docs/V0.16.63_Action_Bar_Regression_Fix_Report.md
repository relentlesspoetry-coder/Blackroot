# V0.16.63 - Action Bar Regression Fix Pass

## Summary
Fixed the V0.16.61/V0.16.62 action bar regression without redesigning the full HUD. The pass keeps the current lightweight action bar visual direction, corrects the mounted Meditate button clipping, adds real button interaction states, and reduces cooldown overlay layout work during combat.

## Root cause
The action bar used `contain: layout paint` while the Meditate button was mounted outside the panel bounds with `overflow: visible`. Paint containment clipped the mounted button despite the visible overflow setting. Cooldown overlays were also updated by changing `height` during combat, which forces layout/paint work on every cooldown tick. The Meditate button was bound as a click target but lacked owned visual state updates for hover, pressed, active meditation, and disabled states.

## Action bar performance changes
- Removed paint containment from `#hotbar.abilityBar` so mounted children can render without clipping.
- Kept containment to layout/style only.
- Reduced heavy action bar and Meditate button shadow blur.
- Changed cooldown overlays from per-update `height` writes to transform-based `scaleY()` via `--cooldown-pct`, avoiding layout work while cooldowns tick.
- Preserved one canonical action bar renderer and did not add a duplicate UI path.

## Meditate button clipping fix
- Kept the button centered inside the action bar owner.
- Adjusted the mounted offset to keep the full button visible and visually connected to the frame.
- Ensured the action bar panel no longer clips the button.

## Meditate button interaction states
- Added normal, hover/focus, pressed, active-meditating, and disabled styles.
- Reused the existing `#steamDeckMeditateBtn` element and existing `toggleMeditate()` behavior.
- Updated runtime state in `renderHotbarCombatState()` so the button reflects meditation state triggered by either mouse/touch or the keyboard keybind.
- Updated the input binding to use native `click` activation with pointer/key pressed-state classes instead of duplicate pointerup/click/touch activation.

## Preserved behavior
- Auto Attack remains key 1.
- Spell slots remain keys 2-0.
- Spell click/hover behavior remains delegated through the existing hotbar input owner.
- Existing meditation behavior and keybind remain unchanged.
- EXP text and formatting remain unchanged.
