Implemented the Assassin, Bard, and Cleric level 1-20 spellbook revamp using the uploaded balance documents as the design source. ## Spell Data Changes
- Assassin: replaced the existing level 1-20 spellbook with Throwing Knife through Perfect Ambush, including Focus costs, ranged/crossbow identity, poison stack metadata, trap metadata, marked-target bonuses, and execute bonuses.
- Bard: replaced the existing level 1-20 spellbook with Quick Note through Dreamsong Crescendo, including major song metadata, target caps, aura radii, regen/mana ticks, debuffs, movement support, and crescendo support.
- Cleric: replaced the existing level 1-20 spellbook with Minor Heal through Avatar of Mercy, including direct heals, HoTs, cleanse metadata, wards/barriers, group heals, emergency save, revive, and anti-undead holy damage metadata.

## Behavior Changes Implemented
- Assassin now uses Focus as its player-facing resource label while preserving the existing internal mana field for compatibility.
- Marked for Death applies one active Assassin mark and replaces prior Assassin marks from the same player.
- Poison Payload stacks apply as a shared poison DoT up to the existing status stack cap.
- Assassin trap spells resolve through the spell owner with trap metadata, target validation, poison/slow/bleed effects, and Shadow Fuse interaction.
- Bard major songs apply through group statuses and enforce a two-major-song limit per Bard/source on affected allies.
- Bard debuffs and sonic attacks apply actual damage/debuff gameplay instead of tooltip-only behavior.
- Cleric HoTs, cleanses, wards, group heals, emergency save, revive, and holy damage resolve through the existing spell/combat/status owners.
- Combat now honors status absorb pools and lethal-save effects.

## Compatibility Notes
Existing saves continue to load. The spell-system owner now synchronizes Assassin, Bard, and Cleric editor spell entries to the canonical revamp definitions so older saved editor spell data cannot keep obsolete spell costs or tooltips. No icon asset paths were renamed or replaced.

## Unsupported / Simplified Mechanics
- Ground-targeted trap placement is implemented through the current target/path-based spell owner rather than a full terrain placement cursor. The spell data includes trap duration/radius metadata for future ground-placement expansion.
- Charge consumption for some long-duration Bard/Cleric buffs is represented in status metadata where possible; only mechanics already supported by current combat/status hooks consume automatically.
