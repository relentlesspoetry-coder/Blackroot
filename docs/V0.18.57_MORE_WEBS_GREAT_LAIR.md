# V0.18.57 — Silk Web Caverns: more + thicker webs, and the Great Web Lair

Your request: way more (and super-thick) floor-to-ceiling webs (~30% more), plus one extremely large web with everything inside it — multiple nests, tons of tiny spiders, lots of bones, lots of eggs, stalagmite columns — pretty massive.

Showcase: [V0.18.57_great_web_lair.png](V0.18.57_great_web_lair.png) — two super-thick columns (left) and the massive lair (right).

## 1. More + thicker floor-to-ceiling columns
- **~30% more columns**: `placeSilkCeilingWebsAndJunk` `columnCount` raised — minimal 4→5, big 9→12, pocket 3→4, else 6→8.
- **Denser tunnels**: the corridor column pass lattice `%8 → %6`, so the tunnels carry more webs.
- **Super-thick**: the thick bias is raised so about **half** the columns are the thick variant (`c%2`, with a lower room-size gate), and `drawCeilingWebColumn`'s thick `spread` is **2.7 → 3.2**, so thick columns are ~3.5 tiles wide.

## 2. The Great Web Lair (new `giantWebLair`)
New `drawGiantWebLair` (`render/object-renderer.js`) — a single massive floor-to-ceiling web structure (~300 px tall, ~4–5 tiles wide), placed **once per floor** in the biggest non-safe/non-boss/non-treasure cavern. It contains **everything**:
- a huge translucent silk sheet + a sagging web scaffold and catenary capture threads;
- **7** embedded orb webs;
- **3** web nests (`drawWebNest`);
- **~24** egg sacs;
- **6** bone clusters + **5** cocooned bodies (`drawBonesCluster` / `drawWebCocoon`);
- **3** webbed stalagmite columns inside it (`drawCaveStalagmite`);
- **~26** animated tiny spiders crawling all over it (`drawTinySpider`, with the leg gait);
- a **big matriarch spider** lurking in the centre, eyes aglow.

Placement: `placeSilkCeilingWebsAndJunk` finds the biggest eligible room and drops one `giantWebLair` at its centre (passable / visual-only).

No gameplay changes.
