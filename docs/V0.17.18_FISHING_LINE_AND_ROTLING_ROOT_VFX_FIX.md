## Fishing Line Root Cause
The fishing line origin was being derived from `getFishingPoleTipWorld()`, a world-space approximation near the player. The actual pole is drawn later by the character renderer as a screen-space procedural overlay anchored to the actor hand. Because the line did not consume that rendered rod-tip anchor, it could visually detach from the pole.

## Fishing Anchor Fix Details
- `drawFishingActionOverlay()` now publishes `player.fishingRodTipScreen` at the exact rendered rod tip each frame.
- `systems/fishing-system.js` now resolves the line origin through that fresh rod-tip screen anchor before falling back to the old world projection.
- The anchor validity check includes frame age and camera yaw/zoom/position so stale anchors are not reused after camera movement.
- Bobber position and fishing mechanics remain unchanged.

## Rotling Root VFX Implementation Details
- Added `rotlingRootEntangle` to the existing effect renderer.
- The effect draws dark root tendrils, green sickly highlights, soil compression, and lower-body wraps around the rooted actor.
- The visual grows in quickly, pulses while active, and fades/retracts when released.

## Status-Effect Integration Details
- The status-effect owner now synchronizes the root VFX from active effects with the `root` tag.
- Existing root movement blocking remains driven by `statusSpeedScalar()`.
- VFX instances follow the rooted actor and are retired when the root status expires, is cleansed, or is otherwise removed.

## Known Limitations
- The live rod-tip anchor depends on the Canvas character renderer publishing the fishing overlay. If an alternate hybrid renderer bypasses the player draw path for one frame, the fishing renderer uses the existing world-space fallback for that frame.
- Browser visual QA is still required to approve root readability at all camera zoom levels.
