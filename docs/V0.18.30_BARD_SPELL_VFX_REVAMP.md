# V0.18.30 — Bard spell-animation revamp

Second class of the per-class VFX pass (after the Assassin, V0.18.29). User: "Now
do the Bard spell animations."

## Problem
The **entire** Bard kit ran on generic VFX in `resolveBardRevampSpell`
(`systems/spell-system.js`):
- `bolt` / `debuff` → `spawnBolt`
- `aoeDebuff` / `buff` / `groupBuff` → `spawnRing`

A rich bespoke Bard toolkit already existed in `render/effects-renderer.js` (song
waves with ally buff-flashes, a sonic crescent blade, a drowsy lullaby, a
discordant corona, a healing ribbon, note glyphs) — but it was **orphaned dead
code**:
1. its dispatcher `spawnBardSpellEffect` keyed off **stale spell names**
   (`warhymn`, `soniccut`, `valorchorus`, `discordantnote`, `refrain`) that match
   no current spell — only `lullaby` matched;
2. it was never called anyway: `spell-system.js:3323` dispatches to
   `resolveBardRevampSpell` and **returns** before the old `triggerBardSpellEffect`
   call sites (~L3446+).

## What changed

### `render/effects-renderer.js` — theme by color
`drawBardSongWaveEffect` and `drawBardSonicCutEffect` were hardcoded gold; they now
read `e.color` / `e.color2` (gold as fallback), so each song/attack reads by role.

### `systems/effects-system.js` — dispatch by `bardSpellId`
Rewrote `spawnBardSpellEffect` to dispatch off `options.spell.bardSpellId` via a
`BARD_VFX` config (fx family + theme colors + `grand` flag) covering all 20 spells,
and to gather allies via `bardAlliesForSpell` for the ally buff-flashes.

### `systems/spell-system.js` — wire it up, delete the generic VFX
`resolveBardRevampSpell` now calls `spawnBardSpellEffect` at each branch (sonic
bolt/debuff, AoE discordant, self verse, group song); the `spawnBolt` / `spawnRing`
calls are **removed**.
