# V0.17.80 — Spec-Name-Themed Talent Backgrounds

**Type:** UI / UX (talent panel)
**Previous version:** V0.17.79
**New version:** V0.17.80

V0.17.78/79 put a faint blurred **spell icon** behind each talent tree. Per request, that is replaced
with a **themed silhouette derived from the spec's name** — so the background actually reads as the
spec's fantasy (bones, skull, flame, moon, blade…), not a random ability icon.

---

## How it works (`systems/talent-system.js`)

- **`THEME_SVG`** — a library of **19 hand-drawn, single-colour vector motifs**: `skull, bone, flame,
  ice, bolt, mountain, thorn, leaf, moon, blade, drop, claw, arrow, gear, rune, eye, note, shield,
  radiant`. Each is drawn in the spec's own colour (`%C%` is replaced with `spec.color`).
- **`specTheme(spec)`** — keyword-maps the spec **name** to a motif. It matches the **name only**: the
  spec `id` embeds the class name (`warden` → `ward`) and the `role` embeds flavour words (`stagger` →
  `stag`, `anti-undead`), both of which mis-themed specs when included.
- The column's emblem now renders that motif (large, faint at ~13% opacity, bottom-centred behind the
  tiles). The V0.17.78 colour glow + tinted gradient + noise grain + vignette + spec-tinted border all
  remain.

## Drop-in art still wins

If a spec has painted art (an `assets/talents/<specId>.png` file or an explicit `bg:` field, per
V0.17.79), that image is used **and the procedural silhouette is hidden** — the art is the theme.

## Verification (in-browser, port 8899)

All 42 specs audited to a sensible name-based motif (the earlier `id`/`role` collisions — Wildheart,
Breaker, Wildmender, Exorcist, Sanctifier — are fixed by matching name-only). Every column renders a
themed `<svg>` emblem with no spell icons; gradients, borders and colours intact; no console errors.
(Full-page screenshots time out on the live canvas loop, so this was verified via DOM/computed-style
inspection — eyeball the silhouettes in-game and any that read poorly can be refined in `THEME_SVG`.)
