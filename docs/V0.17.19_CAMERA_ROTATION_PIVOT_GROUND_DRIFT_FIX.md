## Root Cause Found
The camera projection path did not have an explicit single world-space pivot owner for every render path. Player/entity rendering was effectively anchored to the player-derived projection, while some terrain paths, especially independently composited Hybrid/WebGL terrain, could compose terrain through a separate screen-space layer during yaw changes. That made the player appear pinned while the ground layer could visually drift under the player during camera rotation.

A secondary issue was that camera shake was sampled independently in each `worldToScreen()` call. When shake was active, different terrain corners/chunks/entities could receive different offsets in the same frame, which could amplify apparent drift.

## Camera Pivot Changes
- Added `resolveCameraTargetWorld()` as the canonical camera pivot resolver.
- The active pivot is the player world coordinate when a player exists.
- `camera.x`, `camera.y`, `camera.targetWorldX`, and `camera.targetWorldY` are synchronized to that world-space pivot instead of acting as an independent screen-space pan.
- Added render-frame yaw delta tracking so the renderer can detect active camera rotation without guessing from UI state.

## World-to-Screen Transform Changes
- Rebuilt `getCameraProjectionCache()` around the canonical pivot.
- Added explicit pivot fields: `pivotWorldX`, `pivotWorldY`, and `pivotElev`.
- Added `projectWorldToScreenUnzoomed()` as the shared projection primitive behind `worldToScreen()`.
- Stabilized camera shake to one offset per render frame instead of random offsets per projected point.

## Screen-to-World Transform Changes
- Updated `screenToWorld()` to invert through the same cached projection.
- Added optional `{ z }` plane support.
- Updated Hybrid camera round-trip audit to pass the same z-plane it projected with.

## Tile Rendering Alignment Changes
- Terrain chunk axis vectors now use the cached projection cosine/sine instead of recomputing from camera yaw independently.
- WebGL terrain candidate axis vectors now use the same cached projection values.
- Independent WebGL terrain promotion is disabled while yaw is actively moving so terrain and entities use the same Canvas projection path during rotation.

## Player/Entity Alignment Verification
- Player, NPCs, mobs, props, portals, cave entrances, VFX, and tile surfaces continue to call the same `worldToScreen()` projection path.
- The player is no longer visually pinned through a separate transform from the ground tile projection.

## Targeting/Input Regression Notes
- Movement math was left unchanged to avoid reintroducing inverted-controls regressions.
- Click targeting continues to use `screenToWorld()` through the updated inverse transform.
- Tab targeting is not changed because it is world/entity based.
- Collision remains world-coordinate based and was not modified.
