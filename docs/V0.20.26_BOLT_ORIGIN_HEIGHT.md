# V0.20.26 — Bolts Leave The Hand, Not The Belly (Roadmap Item 6 fidelity)

## This opens by correcting my own estimate

I told the user, twice, that projectile-origin was a **14-renderer, all-or-nothing build** and the
**lowest-value** of the remaining options. **Measuring killed that claim.**

Every humanoid — across **both** the class-identity path (Fighter/Rogue/Cleric/Enchanter/Summoner/
Necromancer) **and** the humanoid-base path (Wizard/Necromancer/Druid/Bard/Merc/generic) — publishes
its feet (`_lastGroundAnchor`) and head (`_nameplateAnchor`) **identically** every frame:

| class | feet Y | head Y | body px |
|---|---|---|---|
| Wizard / Necromancer / Cleric / Druid / Ranger | ~633 | ~535 | ~97 |

Dead uniform. And the bolt origin sat at **~37–55% up the body — the belly** — lifted a **fixed 24px**
that ignored body height entirely.

So the cast-**height** fix is **one uniform change in the bolt path**, not fourteen. My "14 renderers"
was about the exact per-class *hand-side*; the height was a shared, measurable, one-place fix all along.
I should have measured before quoting scope — the same lesson this session keeps teaching.

## The fix

- **`spawnBolt`** derives `originLift` from the caster's own feet/head span — the cast point sits at
  **~62% up** (chest/hand height) — instead of a hardcoded 24.
- **`drawHighQualityBoltEffect`** lifts the **source** by `originLift` (falling back to 24 for any
  caller that passes no anchors).

| property | how |
|---|---|
| **scale-correct** | uses the real on-screen body height, so a 1.5× enemy's bolt scales with it |
| **camera-safe** | a px lift on the re-projected source each frame — exactly what the fixed 24 did |
| **uniform** | both anchors exist for every humanoid, every class |
| **aim unchanged** | only the SOURCE moves; target/tip stay on the enemy's body |
| **safe fallback** | no anchors → lift 24 (verified) |

## Verified

| class | originLift | drawn origin (% up from feet) | was before |
|---|---|---|---|
| Wizard | 34 | **62%** | 54% |
| Necromancer | 35 | **62%** | 54% |
| Shaman | 34 | **62%** | 55% |
| Summoner | 33 | **62%** | 55% |

The "before" column is the tell: the old origin was an **inconsistent 37–55%** depending on each bolt's
`height` option. The fix doesn't just raise it — it makes it **consistent** at the anchor-derived cast
height. No-anchor fallback keeps 24. A live screenshot shows a blue bolt visibly leaving the wizard's
upper body.

## Honest scope — what this does NOT do

This fixes origin **height** and makes it uniform. The exact per-class **hand-side** — the cast hand
sits ~16px to one side in the rig, and which side depends on facing — remains a larger per-renderer
refinement and is **not** claimed here. The height was the wrong-*by-design* part (a fixed nudge that
ignored body height); it's right now. The horizontal is a polish pass on top.

Assassin projectile untouched — its own spawner sets no `originLift`, so it keeps the fixed 24.

## The lesson

The recurring one, turned on myself: **I quoted a scope without measuring it.** "14 renderers,
all-or-nothing, lowest value" was three claims, and the measurement — every humanoid publishes the same
two anchors — dissolved all three. The exact-hand build is real; the height fix that mattered most was
one function. *Measure the thing before you price it,* including when the thing is your own estimate.
