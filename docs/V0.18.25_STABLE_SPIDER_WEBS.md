# V0.18.25 — Spider webs hold their shape

Follow-up to the V0.18.24 web revamp. User:

> "the spider webs look really good but they move / change shape as you walk around."

## Root cause
`drawOrbWeb` (V0.18.24) is deterministic **per seed** — its spoke jitter, hub
offset and dew all come from a seeded PRNG, specifically so a static web doesn't
shimmer. But the three callers seeded it from the **screen** point:

```js
// drawCaveWeb        seed: s.x * 7.1  + s.y * 13.3
// drawWebbedBush     seed: s.x * 5.7  + s.y * 9.1
// drawDeadTree web   seed: s.x * 6.3  + s.y * 11.7
```

`s = this.worldToScreen(x + 0.5, y + 0.5, elev)` moves every frame as the camera
pans, so the seed changed continuously → the web re-rolled its random geometry
while you walked. The shape was deterministic per *frame*, but not per *web*.

## Fix (`render/object-renderer.js`)
Seed on the **world tile** instead, which is fixed for a given prop. `drawObject`
already receives the world tile `(x, y)`, so it stamps a stable seed onto the
object once, before any dispatch:

```js
drawObject(x, y, obj, elev) {
  const s = this.worldToScreen(x + 0.5, y + 0.5, elev);
  if (this.tryDrawObjectSpriteSheetFrame?.(obj, s)) return;
  // Stable per-object seed from the WORLD tile (not the moving screen point).
  if (obj && obj._propSeed == null) obj._propSeed = Math.floor(x) * 73856 + Math.floor(y) * 19349;
  ...
}
```

The three web callers now pass `seed: obj._propSeed`. World objects persist in
the world grid, so the stamp is computed once and reused every frame.

Only the **seed** changed. The web's geometry is still built relative to its
centre `(cx, cy)`, which tracks the camera correctly — so the web stays glued to
its tile *and* now keeps one fixed shape.

## Verification
- `node --check render/object-renderer.js` — OK.
- `obj._propSeed` for tile (47, 136) = `47*73856 + 136*19349` = **6102696**, and
  is unchanged across repeated `drawObject` calls.
- Drew the **same** web (same `_propSeed`) at two different screen positions and
  compared equal-size pixel boxes around each: **0 differing pixels** — the shape
  is now translation-invariant (it was reshaping before).
- Boots with **0 console errors**.

## Files touched
- `render/object-renderer.js` — `drawObject` stamps `obj._propSeed` from the world
  tile; `drawCaveWeb`, `drawWebbedBush`, and the `drawDeadTree` webbed overlay seed
  `drawOrbWeb` from it instead of the screen point.
