# V0.16.54 - Main Menu Weather Cleanup and Fog Polish Report

## Summary
Implemented a focused main-menu weather cleanup pass from the active V0.16.53 build. The pass removes the visible black diagonal weather stripe artifacts, locks the main logo in place, and adds subtle fog ambience under the logo plus a low fog layer near the bottom of the menu.

## Root Cause Found for Black Stripe Artifacts
Two visual owners contributed to the artifact:

1. The main-menu rain integration was using the shared rain curtain pass, which paints broad diagonal/transformed translucent bands. That is useful for heavy world rain, but it read as black/striped bands over the dark menu composition.
2. The menu backdrop pseudo-element also contained two static diagonal black `linear-gradient` bands from an older gothic vignette treatment.

Both were removed from the main-menu path rather than hidden with a darker overlay.

## Root Cause Found for Logo Movement
The hero logo pseudo-element still used the `blackrootHeroBreath` animation, which applied a subtle translate/scale animation to the logo artwork. The animation was removed from the logo owner so the logo remains fully static while rain, fog, and class sigils continue moving.

## Rain / Weather Layering Changes
- The main-menu rain canvas remains non-interactive with `pointer-events: none`.
- Rain still renders through the existing weather drawing primitive.
- The menu-specific call now disables the broad rain curtain pass to avoid diagonal bands.
- Rain density/opacity remains reduced while modal windows are open.

## Fog Implementation Details
- Added `DR.Weather.drawFogLayer()` as a small shared helper inside `weather-system.js`.
- The helper reuses the existing weather-system seeded particle/gradient style without mutating gameplay weather state.
- Main menu fog draws two targeted regions:
  - a centered fog bank beneath the logo / welcome area
  - a wider low fog bank near the bottom / Exit button area
- Fog is drawn on the existing rain canvas before rain streaks so it remains behind class sigils, buttons, and modal windows.

## Canvas State Safety Notes
- The weather pass remains wrapped in `ctx.save()` / `ctx.restore()`.
- After drawing, the menu renderer resets transform, alpha, filter, and composite mode.
- The fog helper uses local canvas state and restores after drawing.
