# V0.20.33 — Stun It And The Fire Never Falls (Roadmap Item 19)

## The join of two systems

This is where the crowd-control work (V0.20.27/30) and the telegraph work (V0.20.31/32) reward each
other. A telegraphed enemy ground-AoE already gave the player a 0.5s **dodge** window. Now that same
window is an **interrupt** window: hard-CC the caster inside it and the strike never lands.

The spec's boss requirement *"interrupt opportunities"* and enemy-review item *"interruptible abilities"*
— delivered by connecting mechanics that already existed, not a new system.

## Implementation

At the single point every telegraphed strike resolves (the `enemy_aoe` case in
`updatePendingSpellImpacts`): when the strike comes due, if the caster has an active control buff
tagged from `INTERRUPT_CC_TAGS` — `stun` / `silence` / `freeze` / `frozen` / `sleep` / `knockback` /
`knockdown` / `interrupt` — the strike is **cancelled** and logged *"X's attack is interrupted!"*

**Deliberate exclusions:**
- **`root` and `slow` do NOT interrupt** — a rooted caster can still finish its spell.
- **Dying still commits** — a caster killed in the window still drops its already-loosed ground effect
  (V0.20.31's choice, unchanged). Only a *deliberate interrupt* cancels.

## Why this is balanced, not a giveaway

- Interrupting requires landing a hard-CC inside a **0.5s window** — a skill check.
- **Bosses resist CC** (V0.20.30: stun duration cut 40–55%), so an interrupt on a boss is a tight,
  earned play — but landing one still stops the attack. The CC-resistance tiers and the interrupt
  window now define each other.
- Every player stun/silence/freeze becomes a **real interrupt tool** — abilities that previously just
  shortened enemy uptime now have a decisive use.

## Verified

| caster state when strike is due | result |
|---|---|
| no CC | **lands** ✅ |
| stunned / silenced / frozen | **interrupted** (0 damage) ✅ |
| **rooted / slowed** | **lands** — root/slow don't interrupt ✅ |
| expired stun (remaining 0) | lands ✅ |

**End-to-end:** enemy telegraphs an AoE → player stuns it inside the window → **AoE interrupted**; the
same cast with no stun **lands**.

## The lesson

The best fixes this session came from *connecting* things that were each half-built: the caster-VFX
funnel wired identity to renderers, the CC gate wired a dead field to the damage path, and this wires
the CC system to the telegraph system. Neither the stun nor the telegraph was new here — the value was
in the seam between them.
