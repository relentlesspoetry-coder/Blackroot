## Root Cause of Green Character Preview Background
The class background asset was resolving, but the Character Model preview still depended on DOM/CSS background layering. The tall portrait viewport cropped the landscape class-background asset into a narrow dark center slice, and remaining fallback/vignette layers made Warden/Druid nature backgrounds read as a flat green panel. The player model canvas was transparent, so any fallback panel below it could visually dominate the result.

## Root Cause of Plain Player HUD Background
The HUD used the canonical resolver variables, but the actual visible HUD surface was still dominated by the compact HUD panel background and heavy dark overlay. The class backdrop was present but too visually suppressed to read as class art.

## Class Background Resolver Changes
- Kept `classBackgroundVisualSpec()` as the canonical lookup path.
- Routed HUD and Character preview surfaces through `applyClassHudVisualStyle()`.
- Added direct inline background-image application to the actual image-bearing surfaces while still deriving paths from the canonical resolver.
- Preserved class key normalization through the existing emblem/class metadata path.

## Asset Loading / Key Fixes
- Added `drawClassBackgroundPanelBackdrop()` to use the same resolved `assets/ui/class-backgrounds/<class>.png` asset at canvas-render time.
- Added a load hook so the paper-doll/class preview can redraw after a class background image finishes loading.
- Missing assets still fall back cleanly through the existing dark fallback and debug warning path.

## Character Preview Render-Order Changes
- The class background is now drawn directly into the paper-doll preview canvas before the player model.
- The drawn background uses a cover layer plus a visible hero/contain layer so the class art is readable in the tall Character Model panel.
- The character model still renders after the background and remains readable.

## Player HUD Background Render Changes
- The HUD class backdrop now receives the resolved class background as the visible surface layer.
- The compact HUD panel background was reduced so it no longer visually hides the class background.
- The HUD overlay remains dark enough to preserve name, level/class text, HP bar, and resource bar readability.

## Fallback Behavior
- If a class background image is missing or still loading, the preview/HUD uses a clean dark fallback.
- Missing class-background warnings remain gated behind the existing debug/dev path and are not emitted every frame.
