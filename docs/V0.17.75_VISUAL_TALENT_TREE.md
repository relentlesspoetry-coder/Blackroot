# V0.17.75 — Visual Talent-Tree Revamp (Spell-Icon Tiles)

**Type:** UI / UX (talent panel)
**Previous version:** V0.17.74
**New version:** V0.17.75

The talent panel was a list of text rows. It is now a proper **tree of real spell-icon tiles**, and you
**invest points by clicking the icon**.

---

## What changed (`systems/talent-system.js`, presentation only)

`renderTalentPanel` was rebuilt; the engine (spend/refund/gates/effects) is untouched.

### Icons
Each node borrows a real class spell icon via `this.spellIconHtml(...)` — the same renderer the action
bar and spellbook use, so the art matches the rest of the game:

1. explicit `node.icon` (if ever set),
2. the first `k:'spell'` effect's matched spell (the ability the node empowers),
3. otherwise a spec spell chosen by a stable hash of the node id (so tiles vary rather than repeat),
4. last resort: a synthetic spec-colored glyph.

### Layout & states
- Three spec columns; each column is a vertical stack of **tier rows**. The gate shows at the row's
  left as `T{n}` + `{gate}★` (met) or `{gate}🔒` (locked). Tiles are centered per tier.
- Tile visual states:
  - **available to spend** — gold border + glow,
  - **invested** — spec-color border + rank badge,
  - **maxed** — green border + glow,
  - **locked** — dimmed, grayscaled, lock overlay,
  - **capstone** — round tile, spec-color halo, ★.
- A bottom-right badge shows `rank/max`. The hover tooltip keeps the current / next-rank description
  and the tier-gate requirement.

### Interaction
Click a tile → `spendTalentPoint`; right-click → `refundTalentPoint`. The delegated handlers are
unchanged (still keyed on `[data-tnode]` / `.talLocked`), so clicking the icon just bubbles up to them.

### Tree feel
- **Tier connectors** — a vertical link sits above each tier (T2–T5); it lights up in the spec colour
  once that tier is unlocked (`treePoints ≥ gate`), so the "spine" of the tree fills as you descend.
- **Per-spec progress bar** — each column header has a spec-coloured fill bar (`treePoints / 18`) with
  a tick marking the capstone threshold (12 points).

## Verification (in-browser, port 8899)

Level-20 Paladin: 21 tiles render, every one with real spell art (image icons). State classes, border
colors and sizes are correct (maxed = green glow; capstone = 54px round, spec-colored border). Clicking
an icon invests `0/3 → 1/3`; right-click refunds `1/3 → 0/3`. No console errors. (Full-page screenshots
time out against the live canvas loop, so this was verified via DOM inspection of computed styles and
click dispatch.)
