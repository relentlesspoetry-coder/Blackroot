## Layout Changes Made

- Reduced the player HUD width range from 360–430px to 320–356px.
- Reduced the minimum height from 112px to 88px.
- Reduced outer padding, column gap, and the dedicated class-art column.
- Kept the text and bar column flexible so name, metadata, HP, resource, and status content determine practical height.
- Preserved the full-width Party / Companions child below the player HUD row.

## Class Art Positioning Changes

- Kept one class emblem on the right through the existing `applyClassEmblem` and shared class resolver paths.
- Sized the transparent class emblem to 84px in an 84px right-side column.
- Added only transparent drop-shadow separation so the art reads directly against the class-themed backdrop.
- No Bard-specific or per-class layout branch was added.

## Old Square Portrait / Frame Handling

- Replaced the `hudPortrait` wrapper presentation with the unframed `hudClassArt` layout role.
- Removed its border, filled background, rounded box, inset shadow, clipping, and obsolete wrapper ID.
- Preserved `hudPortraitIcon` because it is the canonical element updated by the existing UI renderer.
- No left-side placeholder or duplicate class image remains.
