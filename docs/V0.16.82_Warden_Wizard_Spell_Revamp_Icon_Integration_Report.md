# V0.16.82 - Warden + Wizard Spell Revamp and Icon Integration Report

## Uploaded sources used

- Spell document: `Warden Spells.txt`
- Spell document: `Wizard Spells(1).txt`
- Icon package: `Warden Spell Icons(1).zip`
- Icon package: `Wizard Spell Icons(1).zip`

## Spell data updated

- Warden spell data was replaced in `data/spells.js` under `DR.CLASS_SPELL_BOOK.Warden`.
- Wizard spell data was replaced in `data/spells.js` under `DR.CLASS_SPELL_BOOK.Wizard`.
- Warden class metadata in `data/classes.js` now uses the requested level-1 mana target and reinforces the nature/stone sustain-control tank role.
- Wizard class metadata in `data/classes.js` now uses the requested level-1 mana target and reinforces the fragile scholarly ranged DPS role.

## Behavior systems updated

- Warden behavior was added inside the owning spell system in `systems/spell-system.js` as `resolveWardenRevampSpell`.
- Wizard behavior was added inside the owning spell system in `systems/spell-system.js` as `resolveWizardRevampSpell`.
- Existing dispatch in `resolveClassSpell` now routes Warden and Wizard revamped spells through those owning handlers.
- Existing editor-spell refresh logic now treats Warden and Wizard as revamp classes so runtime spellbook data updates cleanly from source definitions.

## Warden mechanics represented

Fully represented through current engine-compatible status fields and spell resolver logic:

- High tank threat multipliers and flat threat.
- Earthen Roar taunt / forced-focus support through the existing taunt owner.
- Barkskin and Stonehide charge-based mitigation.
- Grove Mend and Ancient Pulse regeneration with healing-cap metadata.
- Thorn Guard retaliation metadata.
- Root Grasp, Briar Lash, Thornsnare, Quaking Challenge, and Ancient Warden Form control metadata.
- Living Wall, Grove Shelter, and Primal Guard group-protection/redirect metadata.
- Ironroot Stance, Titanroot, Heart of the Grove, and Ancient Warden Form status buffs.

Partially represented due to current engine limits:

- Living Wall is represented as a protective zone-style group buff and visual ring, not full collision terrain.
- Mountain's Patience delayed damage conversion is represented as status metadata for the combat owner; exact per-hit conversion depends on combat damage handling support.
- Thorn Guard multi-attacker retaliation uses status metadata; exact multi-attacker retaliation depends on damage event consumption support.

## Wizard mechanics represented

Fully represented through current engine-compatible status fields and spell resolver logic:

- Arcane/fire/frost school metadata.
- Mana Shield and Arcane Mirror absorb/protection metadata.
- Ignite, Flame Wave, Meteorfall, and Cataclysm burn/DoT behavior.
- Frost Lance, Ice Prison, Frost Nova, and Shatter setup status behavior.
- Shatter bonus damage against chilled/frozen/rooted targets.
- Arcane Barrage multi-hit behavior.
- Overchannel, Time Slip, and Prismatic Surge charge-limited buff behavior.
- Meteorfall and Archwizard's Cataclysm delayed AoE impacts through the existing pending spell impact update owner.

Partially represented due to current engine limits:

- Prismatic Surge uses charge-limited school metadata and broad school bonuses; exact visible school-rotation UI is a recommended follow-up.
- Mana Shield mana-conversion drain is represented as metadata; exact mana drain on absorb depends on combat damage owner support.

## Icon integration

- Warden icons were added to `assets/spell-icons/warden/`.
- Wizard icons were added to `assets/spell-icons/wizard/`.
- `manifest.txt` from each ZIP was preserved in the class icon folder.
- `manifest.csv` was generated for each class documenting spell-to-asset mapping.
- Warden and Wizard icon registries were added to `systems/spell-system.js` through the existing `CLASS_SPELL_ICON_PATHS` owner.

## Icon mapping

- Warden icons mapped directly by unlock level and spell name. The uploaded ZIP encodes apostrophes as `#U2019` for `Mountain’s Patience`; it was mapped to the canonical spell key `mountain_s_patience`.
- Wizard icons mapped directly by unlock level and spell name. The uploaded ZIP encodes apostrophes as `#U2019` for `Archwizard’s Cataclysm`; it was mapped to canonical spell key `archwizard_s_cataclysm`.

## Save compatibility

- Spell names were preserved from the uploaded documents.
- Generated spell IDs remain deterministic as `spell_warden_<spell_slug>` and `spell_wizard_<spell_slug>`.
- No save wipe or action-bar wipe was added.
- No separate save migration was required because the current generated spell ID pattern remains stable for the implemented names.
