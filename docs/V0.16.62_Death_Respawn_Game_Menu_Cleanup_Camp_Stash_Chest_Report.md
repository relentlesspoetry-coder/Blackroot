## Death Respawn Full-Health Fix

Root cause: the player respawn owner restored the player at a partial HP value, and supporting friendly recovery paths did not consistently resolve actor max HP before restoring health. That allowed respawned/recovered friendly units to return below full HP.

Changes:

- `Player.respawnAt()` now restores HP to the resolved `maxHp` value instead of a partial percentage.
- `restoreActorAfterRespawn()` now recalculates the player or scalable companion actor first, then sets HP to the resolved max HP.
- Mercenary revive now recalculates mercenary level/equipment stats before restoring HP to `maxHp`.
- Player respawn now restores active mercenary and pet/summon HP to max HP after moving them to the respawn anchor.
- Bot respawn now restores its companion pet/summon HP to max HP after moving it to the bot recovery anchor.
- Existing respawn location rules and mana restoration behavior were preserved.

## Camp Stash Chest

Added a permanent `stashChest` world object inside the main Dead Lantern Town camp service area.

Stash details:

- Type: `stashChest`
- Label: `Camp Stash`
- Interaction prompt: `Open Stash`
- Placement: near the central camp service area, reachable without combat.
- Collision: non-blocking, so it does not block critical camp paths.
- Persistence: reuses the existing Bank/Stash `this.bank` data model and save/load behavior.

## Bank/Stash Persistence Notes

The patch does not create a second storage inventory or a new save format.

The in-world Stash chest calls the existing Bank/Stash UI and storage functions:

- `ensureBankSystem()`
- `toggleBank(true)`
- existing deposit/withdraw handlers
- existing save/load bank serialization

Older saved worlds are protected by ensuring the camp Stash chest is reinserted during world runtime rebuild after save/load.

## Rendering / Interaction Notes

- Added a lightweight procedural Stash chest renderer to the existing object renderer.
- Extended runtime chest interaction lookup so nearby Stash chests are prioritized before resource chests.
- Interacting with the Stash chest opens the existing Stash storage UI.
- Resource chest behavior remains unchanged.
- The Stash chest is not lootable and does not use random chest loot-table logic.
