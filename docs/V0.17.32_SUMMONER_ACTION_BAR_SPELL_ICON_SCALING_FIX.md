## Root Cause of Small Action Bar Spell Icons
The shared spell action slot renderer already inserted the canonical spell icon markup into each hotbar slot, but the active action-bar CSS treated the icon as a contained thumbnail. Bound spell icon PNGs are landscape-framed images, while the hotbar slot is closer to a compact vertical button. `object-fit: contain` preserved the full source frame but left large unused vertical space inside each slot, so Summoner spell art rendered like small thumbnails and the label/cost text visually competed with the icon area.

## Action Bar Slot Scaling Changes
- Kept the existing action bar slot size and grid.
- Changed filled spell slots so the icon container is inset only slightly from the slot border.
- Switched bound hotbar spell images to slot-filling `object-fit: cover` within the existing icon container.
- Preserved aspect ratio while making the icon art the dominant slot visual.
- Enlarged generated/fallback hotbar glyphs so unmapped spells still read as proper buttons.

## Spell Icon Resolver Changes
No spell data or resolver mapping was changed. The fix intentionally reuses the existing canonical `spellIconHtml()` / `spellIconDescriptor()` path.

## Text/Cost Overlay Changes
- Keybinds remain in the upper-left corner.
- Spell labels remain as compact truncated overlays near the bottom.
- Mana/resource cost remains as a compact bottom-right overlay.
- Cooldown overlay remains driven by the existing runtime `--cooldown-pct` path.
