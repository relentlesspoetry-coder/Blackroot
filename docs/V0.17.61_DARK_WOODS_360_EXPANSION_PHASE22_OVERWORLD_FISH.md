## Implementation

Fishing is a **fish-anywhere-near-water** system (`systems/fishing-system.js`) using a hardcoded
`FISH_TABLES` object keyed by zone — **not** placed nodes (Phase 18 finding #3). So this phase is
unlike the herb phases:

1. **Fish entries** added to `FISH_TABLES.dark_woods` (kept the existing generic fish; added the 5 new
   named ones). Each new entry carries `levelRequired`, and Gloamroot Catfish additionally
   `deepWaterOnly:true` + `rareNodeXpBonus:20`.
2. **Fish items** — 5 `item_*` materials in `data/items.js` (`type:'resource'`, `icon.family:'fish'`,
   `sellValue` = the plan's vendor values). Needed so vendor value is correct (the fishing
   `addMaterialItem` fallback would otherwise flat-value every fish at 4).
3. **Per-fish level gate (the new mechanic)** — new `eligibleFish(table, fishingLevel, deepWater)`
   filters the weighted pick to fish whose `levelRequired <= fishingLevel`, and drops `deepWaterOnly`
   fish unless the cast is in deep water. Existing fish (no `levelRequired`) count as level 1. Falls
   back to level-1, non-deep fish so a cast always has a valid pool. `resolveCatch` now does
   `weightedPick(eligibleFish(zoneFishTable(zone), fishingLevel, waterSpot.rare))`.
4. **Authoritative Fishing level** — the gate reads
   `resourceGatheringSystem.skill('Fishing').level` (the single source of truth since Phase 19), with
   a fallback to the fishing system's mirrored `state.level`.
5. **Rare fish in deeper water** — reused the existing `spot.rare = waterDepth >= 9` +
   `deepWaterBonus` hook: `deepWaterOnly` keeps Gloamroot Catfish to deep-water casts.
6. **First-discovery (+15/30/50) and rare-fish bonus (+20)** already worked via the Phase 19
   `resolveCatch` wiring (reads `fish.levelRequired` and `fish.rareNodeXpBonus`); this phase just
   populates those fields.

**Zone exclusivity is automatic:** `zoneFishTable()` returns `FISH_TABLES.dark_woods` only for the
overworld, and fishing `render`/`update` early-return in dungeons — so these fish never appear in
caves/dungeons or other zones.

**Scope notes / deliberate deviations:** existing generic fish (Small Pondfish, etc.) keep their
level-1 behavior (not retro-gated) — the plan targets the 5 new fish. Fishing yields 1 fish per catch
(the plan's per-fish yield ranges apply to node gathering, not fish-anywhere). Region-specificity
(e.g. Brambleback only in Bramblefen water) is treated as flavor, not enforced — the validation
requires level + depth gating, which is implemented; region gating could be a later refinement.

## Validation

### Commands
```bash
find . -name "*.js" -print0 | xargs -0 -n1 node --check      # ALL JS OK
find . -name "*.json" -print0 | xargs -0 -n1 python3 -m json.tool > /dev/null   # ALL JSON OK
```

### Results
- **JS/JSON:** all pass. Clean boot at V0.17.61, all 5 fish items present, **zero console errors**.
- **In-browser (no-cache server), drove `resolveCatch`:**
  - Fish item vendor values: 2 / 4 / 8 / 15 / 35 (per plan).
  - **Level gate** (200-sample batches, resetting level each cast): at **Fishing 1 shallow** only
    Duskmud Minnow appears among the new fish; at **Fishing 10 shallow** Duskmud/Lanternfin/Brambleback/
    Ghostscale appear but **not** Gloamroot Catfish; at **Fishing 10 deep** all 5 including Gloamroot
    Catfish.
  - **First-discovery + rare:** Duskmud Minnow → "Caught Duskmud Minnow. First catch! +15 EXP." (27 XP);
    Gloamroot Catfish (deep) → "Caught Gloamroot Catfish. First catch! +50 EXP. Rare +20 EXP." (132 XP).

## Known limitations
- Region-specific spawn flavor (Brambleback only in Bramblefen water, etc.) is not enforced — fish are
  gated by level + water depth only.
- Fishing has no depletion/respawn model (fish-anywhere); the plan's per-fish respawn timers are a
  Phase 24 reconciliation, not applicable here.
- Yield is 1 fish per catch (the plan's yield ranges apply to node gathering).
