# V0.20.31 — Step Out Of The Fire (Roadmap Item 19)

## The defect

Regular enemy ground-AoE abilities (`executeDataAbility` kinds `aoe` / `venom_rain` / `silk_lines` /
`poison_cloud` / `snare` — ~20 abilities across 35 enemies) spawned their ring and dealt the damage on
the **same frame**:

```js
game.spawnRing?.(target.x, target.y, color, ringPx);           // ring appears
game.enemyAoeDamage?.(target.x, target.y, radius, hit(), this, color);  // damage — same frame
```

**Zero reaction window** — the ring *was* the hit. That's the spec's *"clear telegraphs"* and
*"dangerous without relying on unavoidable one-shot attacks"* requirements, unmet. Silk Web bosses
already telegraph (V0.18.42), but the data-driven ability path didn't.

The user chose a **0.5s** window (dodgeable but demanding).

## The fix — reuse, not rebuild

Two existing systems did the whole job:

1. **`bossGroundTelegraph`** — its draw is generic (not silk-coupled). A danger marker appears at the
   target, sized `ringPx / 5.8` so its drawn circle matches the ability's real blast, with `life`
   spanning the window.
2. **`queuePendingSpellImpact`** — the delayed-impact queue the player's Meteorfall already uses,
   already ticked each frame (`game.js:2766`). A new `enemy_aoe` kind lands the strike 0.5s later via
   `enemyAoeDamage` / `enemyAoeStatus`.

**Damage is snapshotted at cast**; the strike is **committed** (lands even if the caster dies in the
window — the ground effect was already loosed). **Cleave stays instant** — it's a swing at the enemy's
own body, not a ground effect.

## Verified through the real path

## Scope / follow-up

Scoped to the **data-driven object abilities** (`executeDataAbility`) — the main system, ~20 ground-AoE
abilities. The older **string-format abilities** (`ash_cloud`, `death_bloom`, `blightwood_pulse` on
specific minibosses, in `enemy.js`'s string rotation) are a separate code path and **still land
instantly** — a bounded follow-up that would apply the same telegraph pattern.

The 0.5s window is a single constant (`AOE_TELEGRAPH_DELAY` in `executeDataAbility`) — trivially
tunable if it plays too tight or too loose.

## The lesson

The mirror of the whole VFX arc. There, the job was making the *player's* abilities communicate what
they do. This is the same question turned on the enemies: *does the attack tell you it's coming?* And
the fix reused the exact delayed-impact machinery the player's Meteorfall already ran on — the engine
had the telegraph pattern; the data-driven enemies just never used it.
