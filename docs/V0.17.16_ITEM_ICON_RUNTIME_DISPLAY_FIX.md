## Root Cause Found
The previous item icon pass attached icon keys to canonical item definitions, but `itemIconDescriptor()` resolved the runtime item's `id` before `itemId` / `sourceItemId`. Runtime inventory and equipment items use a numeric serial `id`, so the canonical item definition and `DR.ITEM_ICON_KEYS_BY_ID` mapping were skipped. The renderer then fell through to generated category glyphs.

## Asset Loading Fixes
- Added explicit runtime PNGs for the starter profession tools shown in QA: Basic Fishing Rod, Crude Pickaxe, Herbalist Knife, and Worn Hatchet.
- Added `DR.ITEM_ICON_ASSET_KEYS` so the runtime resolver can distinguish real item icon assets from missing icon keys.
- Added image error fallback CSS/markup so a bad asset path falls back to generated glyph art instead of showing a broken image.

## Item Resolver Fixes
- Added `resolveCanonicalItemDef()` and `resolveItemIconKey()` in the owning inventory/item UI system.
- Canonical stable IDs now resolve in this priority: `itemId`, `sourceItemId`, canonical/template IDs, then string `id` only when it is an actual item definition key.
- Existing saves no longer require icon fields on item instances; they resolve from canonical item definitions.
- Missing mappings/assets/unresolved saved items log once behind debug/dev mode.

## Inventory Renderer Fixes
- Bag/inventory slots now render `itemImageIcon` when a mapped asset exists.
- Stack badges, rarity borders, drag/drop attributes, and equip/use handlers were not changed.

## Equipment Renderer Fixes
- Filled Character window equipment slots continue calling the shared `itemIconHtml()` path.
- Empty slot glyphs remain only for empty equipment slots.
- Filled slots now resolve actual mapped item icons before generated/category glyphs.

## Tooltip Renderer Fixes
- Tooltip header icons use the same `itemIconHtml()` resolver as inventory and equipment.

## Saved Item Hydration Behavior
No save migration or wipe is required. Existing item instances resolve icon art from `itemId` / `sourceItemId` and canonical item definitions at render time.
