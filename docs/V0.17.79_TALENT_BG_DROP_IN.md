# V0.17.79 — Drop-in Painted Art for Talent Spec Backgrounds

**Type:** UI / content pipeline (talent panel)
**Previous version:** V0.17.78
**New version:** V0.17.79

V0.17.78 gave every spec a procedural themed background. This adds the bridge to the fully-**painted**
mockup look: hand-drawn per-spec scene art with **zero code edits**.

---

## How it works (`systems/talent-system.js` → `renderTalentPanel`)

- **Auto-discovery.** For each spec, the panel probes `assets/talents/<specId>.png` once per session via
  a cached `new Image()` load (result kept in `this._talentBgCache`). If it loads, it becomes that
  column's background and the panel re-renders; if it 404s, the spec keeps its procedural colour+emblem
  theme and the miss is cached so it isn't re-probed.
- **Explicit override.** A `bg:` field on a spec in `data/talents.js` always wins over the convention.
- **Fallback-safe.** `specBackground(spec, bgUrl)` layers the painted image under a dark tinted veil
  (for text readability) **with the full procedural theme beneath it** — so a missing / slow / broken
  image never causes a visual regression.

## Using it

Drop a PNG named after the spec id into `assets/talents/` and reopen the talent panel:

```
assets/talents/assassin_boltweaver.png
assets/talents/wizard_pyromancer.png
assets/talents/paladin_oathkeeper.png
...
```

`assets/talents/README.md` documents the convention and lists all 42 spec ids. Recommended art is a tall
portrait (~300×540); it's shown with `background-size: cover`, so exact dimensions aren't critical.

## Verification (in-browser, port 8899)

A test PNG placed at `assets/talents/assassin_boltweaver.png` was auto-picked-up as that column's
background (`this._talentBgCache.assassin_boltweaver` = the path), while the other two Assassin specs
stayed on the procedural theme (cache `false`). All columns kept their fallback gradients; no console
errors. The test image was removed after verification — only `README.md` ships in `assets/talents/`.
