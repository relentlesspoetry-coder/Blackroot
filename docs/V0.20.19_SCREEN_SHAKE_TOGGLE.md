# V0.20.19 — You Can Turn The Shaking Off (Roadmap Item 20)

**Item 20's first real accessibility feature — 1 of 12** — and the one with the strongest case:
**vestibular**. Camera shake is one of the more reliable ways to make somebody feel genuinely unwell,
and there has never been a way to turn it off.

## Where shake comes from

| source | |
|---|---|
| melee swing | `combat-system.js:1471` |
| critical strike | `combat-system.js:1667` |
| portal | `portal-system.js:278` |
| decay | `effects-system.js:276` |

## Gated at the consumer, not the producers

Shake is **written from four places** and **read in exactly one** — `getCameraProjectionCache`:

```js
const shake = this.uiPrefs?.reduceScreenShake ? 0 : clamp(Number(this.camera?.shake || 0), 0, 8);
```

**One line covers every source — plus any future one, for free.** Gating each producer would be four
edits, four chances to miss a fifth added later: the cascade-patch the golden rules forbid, avoided by
finding the funnel.

**It zeroes the READ, not `camera.shake` itself.** The producers keep incrementing and `effects-system`
keeps decaying exactly as before, so **no other system can notice the setting exists**, and toggling it
back mid-session is instant with no state to reconcile.

## Design decisions

| decision | why |
|---|---|
| **off by default** (shake stays on) | a default must never silently change how the game looks for a player who didn't ask |
| **its own Accessibility section**, not under HUD | it isn't a HUD preference, and someone looking for it will look for the word |
| **changes nothing else** | the hits still land, crits still crit. An accessibility option that quietly makes you worse at the game isn't an option, it's a penalty |
| reuses the existing `settingKey` path | persists through V0.20.18's `uiPrefs` save **for free** |

**This is the first feature that needed V0.20.18.** Without it the toggle would be forgotten on every
reload — and someone who turns off screen shake because it makes them ill should not have to turn it
off again tomorrow.

### A probe that measured nothing

My first test called `resolveCameraProjection` / `updateCameraProjection` — **neither exists**. It
returned `undefined` for *both* states, and `undefined !== 4` would have looked like a passing gate if
I'd only checked "did it change". The real function is `getCameraProjectionCache`.

That's the same failure as V0.20.15's `setFacingFromDelta?.()`: **assuming a method name and reading
the silence as success.** Caught this time by asserting the *expected values* (`4` and `0`) rather than
just a difference.

### A stale button that isn't mine

The toggle's button text stays `Off` after a synthetic `.click()`, though the pref flips and persists
correctly underneath.

**The shipped `Show Minimap` toggle behaves identically under the same test** — and a manual re-render
shows the correct state for both. So it's either a pre-existing quirk or an artifact of dispatching
`.click()` rather than a real user click. **My row is indistinguishable from ones that shipped long
before me**, and chasing it would be chasing a test artifact.
