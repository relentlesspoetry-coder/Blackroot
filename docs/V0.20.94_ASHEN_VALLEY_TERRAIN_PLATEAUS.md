# V0.20.94 — Ashen Valley Terrain Forms Plateaus, Not a Staircase

Reported: *"They look like they're floating."*

**It was my terrain generator, not the renderer.**

## The two screenshots settled it

One showed a hairline zigzag with isolated blocks at its corners running across nearly flat ground. The
other showed a real cliff falling into water, rendering perfectly — proper tall faces, visible depth.

The renderer was never at fault. I had spent the previous version and four failed attempts assuming it
was.

## Measured

| | Dark Woods | Ashen Valley |
|---|---|---|
| elevation edges per 1,000 tiles | **2** | **107** |
| 1-unit steps in sample | 8 | **4,602** |
| pixels per elevation unit | \(28\) | \(28\) |
| pixels per tile | \(59.5\) | \(59.5\) |

One elevation unit is **28px — nearly half a tile.** These were never collapsing hairlines; they were
full-height walls, packed **53× more densely** than Dark Woods. That is the endless staircase, and it is
exactly why it read as floating slabs: there is no coherent ground, just an unbroken field of small
plateaus each 28px above the last.

## Root cause

```js
elev = Math.floor(basin * 2.6 + n2 * 1.7 + n3 * 0.7 + ridge - 0.95);
//                              ^ noise scale 14  ^ noise scale 7
```

High-frequency terms inside a `floor()`. The result flips by one every couple of tiles.

## Fixed in two parts — the first was not enough

**1. Low-frequency relief, blurred before flooring.** Elevation now comes from broad regional noise only,
with **three box-blur passes over the continuous height before it is floored**. Blurring *before*
flooring is the whole trick: it produces broad flat ground with occasional real cliffs, instead of a step
wherever the noise crosses an integer boundary.

**107 → 67.** Better, and the measurement said not good enough.

**2. The ridge term keyed off tile type.**

```js
const ridge = (type === TILE.STONE || type === TILE.RUIN) ? 1.35 : 0;
```

Tile type is chosen from the same `n1`/`n2`/`n3` noise I had just removed from elevation — so this
smuggled per-tile churn straight back in through the back door. Highland mass is now its own smooth
field.

**67 → 52.**

`n2`/`n3` still drive tile **type**, where per-tile variation is exactly what is wanted.

## On the remaining number, honestly

52 edges per 1,000 is still well above Dark Woods' 2 — and I set "<25" as a pass threshold and missed it.

**That threshold was worthless.** Dark Woods scores 2 because it is **99.5% flat**: 31,533 of 31,684
sampled tiles sit at elevation zero. It has no relief at all, so it cannot be the target for a zone whose
whole northern third is supposed to be highlands.

The meaningful measure is how much unbroken ground lies between steps:

| | value |
|---|---|
| average flat run | **18.3 tiles** (~1,000px) |
| 1-unit steps | **4,602 → 2,038** |
| distinct elevations retained | 5 |

18 tiles of flat ground between height changes is a plateau. Two or three is a staircase.

## Not fixed — stated plainly

**Nothing casts a shadow onto the ground at the foot of a cliff.** That is a real missing feature and it
contributes to how grounded the terrain looks. Four attempts failed, each for a different reason, and
each was reverted:

1. Cast outward from the wall foot, in the wall pass — **invisible**. Proven by painting the same quad
   opaque magenta: **0 magenta pixels** on screen.
2. Receiver-side, immediately after the tile's own ground — **invisible**, same test.
3. Receiver-side in the occlusion-cap pass — ran 56×/frame and drew **0 quads**, because that pass only
   visits tiles that *have* tall faces, never the lower ground receiving the shadow.
4. Caster-side in the occlusion-cap pass — **82 quads/frame, no errors, and no measurable darkening**
   (0.1 luma, identical to drift).

Along the way I also captured `ctx` once instead of resolving it at call time, invalidating one magenta
test until redone — the same mistake as the object-renderer probe in V0.20.86.

**What that sequence actually proved:** I do not understand this renderer's compositing order well
enough to place a ground-level effect in it. The passes involved — `depth-underlay`, cached ground
blits, `occlusion-cap`, `shore-cap`, `structural-walls` — need to be read end to end before trying
again, not probed.

## The wider lesson

Every audit I wrote for the previous two versions measured **geometry**, because the first genuine bug
(missing corner joins, V0.20.92) was geometric. The second was **shading** (V0.20.93). The third was
**source data** — the terrain itself. Three different layers, and I kept reaching for the previous
version's instrument.

The screenshots did more than any of my probes, twice. Asking for them earlier would have saved a
version.
