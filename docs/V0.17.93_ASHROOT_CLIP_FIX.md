# V0.17.93 — Ashroot Horror model clip fix

**Type:** Bug fix (rendering)
**Previous version:** V0.17.92
**New version:** V0.17.93

The Ashroot Horror's model rendered with its top (canopy/head) cut off by a flat edge.

---

## Cause

Dark-woods mobs are sprite-cached with the shared `BEAST_BOUNDS` canvas (300×240, originY **170** — i.e.
170px of headroom above the creature's feet). But the ashroot/deadroot family is much taller than a wolf or
boar:

- Ashroot Horror — `anchorY 122` × config scale 1.22 × the mob's `visualScale`.
- Ashroot Elder — `anchorY 142` × config scale 1.45 × `visualScale`.

That's ~180–250px of body above the feet, overrunning the 170px canvas headroom, so the cached sprite
clipped the top (the flat cut visible in the report, with background above it — confirming a canvas clip,
not a screen edge).

## Fix (`systems/runtime-sprite-cache-system.js`)

- Added a `TALL_MOB_BOUNDS` preset (340×350, originX 170, originY **290** → 290px of headroom).
- Routed **only** the ashroot/deadroot actors to it in `boundsFor`, matched on
  `actor.mobVisualKey` / `family` / `rendererId` (`/ashroot|deadroot/`), checked **before** the generic
  `'mob'` branch (because `'tallmob'.includes('mob')`).
- Every other mob (wolves, rotlings, boars, spiders, …) keeps the normal `BEAST_BOUNDS` — no wasted memory.

## Verification (in-browser, port 8899)

`boundsFor` returns the tall bounds (originY 290) for `ashrootHorror` and `ashrootElder`, and the unchanged
`BEAST_BOUNDS` (originY 170) for `gloomWolf` / `rotling`. An Ashroot Horror spawns and renders many frames
with no console errors.

**Visual note:** `preview_screenshot` times out on this WebGL page, so the canopy no longer being clipped is
the user's visual sign-off. If any headroom is still short (e.g. the Elder's crown at capstone scale),
`TALL_MOB_BOUNDS.originY`/`height` are the single knob to raise.
