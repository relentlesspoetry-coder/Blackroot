## Model Preview Background Fix Details
The Character model preview no longer relies on the flat fallback stage as its visible background. The preview viewport now uses `--pd-class-bg`, which is populated through `DR.creatorClassBackgroundImagePath(className)`, and the existing `paperDollClassBackdrop` layer was strengthened so the class background is visibly present behind the model while preserving the existing frame and model canvas draw order.

## Class Background Resolver Usage
The implementation reuses the existing class background resolver exported by `systems/ui-system.js`. No class-specific hardcoded asset path was added.

## Belt Slot Integration Details
`belt` was added to `DR.EQUIP_SLOTS` and `DR.SLOT_LABELS`, the item compiler and runtime slot normalizer now canonicalize `belt`, `waist`, and `sash` to `belt`, and the Character window renders Belt as a normal equipment slot.

## Equipment / Save / Load Changes
No new save schema was introduced. The existing equipment object persists any `belt` key the same way it persists every other equipment slot. Save-load now calls `ensureEquipmentSlots()` immediately after restoring equipment, so older saves get a null `belt` key without a wipe or schema migration.

## Nameplate Layout Changes
World nameplates now use a shared two-line composition:
- Top line: name plus class icon when the entity has a playable class.
- Bottom line: `Lvl:X` plus current action/status text.

Class emblem resolution was corrected for the full playable roster so Assassin, Paladin, Warden, Ranger, Wizard, and Shaman nameplates no longer collapse into broader fallback class icons.
