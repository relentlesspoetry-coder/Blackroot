# V0.18.74 — Loot Audit Triage: Every Item Obtainable (Item 12 follow-up)

The V0.18.71 obtainability audit flagged **10 items** with no data-level source: 2 starter bags, 5 gathering tools/rods, and 3 quest-script-given items. This closes all of them, and the audit now reports **0 unobtainable**.

## 1. Vendor stock is shared data + the camp store stocks the basics (`systems/event-system.js`)
- The per-shop stock lists are lifted out of `shopItemIds()` into a single shared const **`SHOP_ITEM_LISTS`**, exposed as **`DR.SHOP_ITEM_LISTS`** (one source of truth; `shopItemIds` now just reads it).
- The **camp general store** (`shop_camp_basic_goods`) now also stocks the basic gathering tools a new character needs — `item_crude_pickaxe`, `item_worn_fishing_rod`, `item_basic_fishing_rod`, `item_herbalist_knife`, `item_worn_hatchet` — plus the starter bags `item_linen_pouch` and `item_travelers_pack`. The pickaxe is also at the **blacksmith**; the herbalist knife also at the **herbalist**.

## 2. The audit sees vendor + quest-grant sources (`core/registry.js` `auditItemObtainability`)
It now also cross-references:
- **`DR.SHOP_ITEM_LISTS`** → source label `vendor:<shop>`.
- **`DR.INTERACT_POINTS` / `DR.DISCOVERY_POINTS`** `grantItem` → source label `quest-grant:<poi>` (quest-script pickups like the well coin, Aldric's journal, the torn ledger).

## Result (verified live)
- **262 / 262 items sourced, 0 unsourced.** Every item now has a reachable, recognized source.
- Spot checks: `item_crude_pickaxe` → `vendor:shop_camp_basic_goods` + `vendor:blacksmith`; `item_linen_pouch` → `vendor:shop_camp_basic_goods`; `item_hunters_journal` → `quest-grant:poi_hunter_journal`; `item_tarnished_coin` → `quest-grant:poi_well_coin`. Unsourced list is **empty**.
- `node --check` clean; JSON clean; boots with **0 console errors** (the boot obtainability `console.info` no longer fires, since nothing is unsourced).
