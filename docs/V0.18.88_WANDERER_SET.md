# V0.18.88 — The Dark Woods Wanderer Set (Roadmap Item 4 D, capstone)

The capstone of the itemization pass: the game's **first true armour set**, with a real full-set
bonus wired into the existing meditation and XP systems. Completing this means **every equip slot now
has a full item pool**.

## The set (`data/items.js`)

Four blue, level-8, leather pieces (all classes) with strong solo stats:

| id | name | slot | armour | stats |
|----|------|------|--------|-------|
| item_wanderer_hood     | Wanderer's Hood     | head  | 4 | hp 14, mana 10, speed, magicResist |
| item_wanderer_garb     | Wanderer's Garb     | chest | 6 | hp 22, mana 14, defense 3, magicResist |
| item_wanderer_leggings | Wanderer's Leggings | legs  | 5 | hp 18, mana 10, defense 2, speed |
| item_wanderer_boots    | Wanderer's Boots    | feet  | 4 | hp 14, speed 0.05, defense 2 |

Registered as `DR.ARMOR_SETS.dark_woods_wanderer` with `pieceIds`, `bonus: { xpPercent: 0.03,
meditationSpeedPercent: 0.15 }`, and `bonusText`.

## Full-set bonus

Wear all four → **+3% experience** and **15% faster meditation**.

## How it works (surgical — reuses existing systems, no separate tick loop)

**Set detection is by item id.** The item compiler carries only a fixed field set (`compileOne`), so
a custom `setId` on the draft would be dropped. Runtime instances store the source id in `itemId`
(their `id` is a numeric instance id), so the helper checks `itemId || sourceItemId || id`.

- **`game.js`** — new `equippedArmorSetCount(setId)` / `hasFullArmorSet(setId)` read the player's
  `this.equipment` map. `meditationTickRate()` shortens the **existing** meditation tick interval by
  `meditationSpeedPercent` for the player only when the full set is worn (15s → 12.75s). The tick
  cadence is reused — nothing new loops.
- **`systems/combat-system.js`** — `awardPlayerXp()` is the single canonical XP award path (the party
  bonus is already applied there). It now adds `xpPercent` to the same multiplier when the full set
  is worn, so the bonus applies uniformly to every XP source, and appends `(Wanderer Set: +3%)` to
  the log.
- **`systems/ui-system.js`** — the character sheet's Set Bonuses box (previously a hardcoded "No
  active set bonuses.") is now dynamic: it lists each set the player has ≥1 piece of, shows
  `X/total`, and marks the full-set bonus active/inactive.

## Sourcing (audit-guarded)

All four are rare set pieces and drop only from the tougher Dark Woods loot tables
(`data/loot-tables.js`) at low chances — named rares (Hood, Garb), ashroot horrors (Leggings), common
chest (Boots). No vendor/craft (a rare set shouldn't sit in the camp store).
