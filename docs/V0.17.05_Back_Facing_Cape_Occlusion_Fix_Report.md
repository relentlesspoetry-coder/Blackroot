# V0.17.05 - Back-Facing Cape / Cloak Occlusion Fix Report

## Phase
Back-Facing Cape / Cloak Occlusion Fix Pass.

## Ownership
- Cape/cloak paperdoll ownership lives in `render/paperdoll-equipment-renderer.js`, specifically `drawBack(ctx, rig, pass)` and `drawBodyLayer(ctx, rig, 'back', arg)`.
- Shared humanoid draw-order ownership lives in `render/humanoid-base-renderer.js` inside `Base.draw(ctx, actor, options, nowMs)`.
- Direct race-rendered humanoid paths in `render/race-identity-procedural-model.js` and `render/race-sprite-sheet-renderer.js` also call the same paperdoll `drawBodyLayer` owner, so they were updated to pass the new cape layer phase explicitly.

## Root cause
The paperdoll cape/back-slot renderer was always invoked in the pre-body back layer. That worked for front and side silhouettes, where capes should sit mostly behind the torso with side drape visible. It was wrong for north/northeast/northwest back-facing rows: the back-facing torso and torso paperdoll then rendered after the cape and hid the cloak, so the actor looked almost like a front-facing body instead of showing a back garment draped over the shoulders/back.

## Implementation
- Added `capeDepthLayer(rig)` inside the existing paperdoll equipment renderer.
- `capeDepthLayer(rig)` returns `front` for `rig.dir.backVisible` rows and `back` for front/side rows.
- Changed `drawBack(ctx, rig, pass)` to render only during the matching `back` or `front` cape pass.
- Updated `drawBodyLayer(ctx, rig, 'back', arg)` to forward the pass argument to `drawBack`.
- Updated `render/humanoid-base-renderer.js` so the old pre-body back pass remains for front/side cape behavior and a second call point after torso/torso-paperdoll draws only the same cape for back-facing rows. This is not a duplicate renderer; it is the same owner gated to one pass per frame.
- Updated direct Bogling/Ratkin/race sprite paperdoll call sites so shared race-rendered humanoids use the same back/front cape pass contract.

## Layering rules after fix
- Front-facing: cape remains in the original behind-body pass.
- Side-facing: cape remains mostly behind the torso with silhouette drape.
- Back-facing: torso base and torso paperdoll draw first, then the cape/back-slot garment draws over the back torso so it visibly covers the model's back. Head, arms, weapons, offhands, and later hooks retain their existing order.

## Entity coverage
This fix applies through the shared humanoid/class/paperdoll render path used by player characters, bots, mercenaries, and companion/class-rendered humanoids. The direct race-rendered Bogling/Ratkin paths were also updated to avoid leaving a separate no-arg cape path behind.

## Protected systems
No item stats, equipment resolution, inventory behavior, loot behavior, combat behavior, movement, bot AI, mercenary AI, save/load behavior, weapon identity, white-stick fixes, global-crossbow fixes, auto-attack arm-anchor fixes, or weapon occlusion rules were changed.
