# V0.19.8 — Cleave Draws Its Swing (Roadmap Item 6)

## The violation

Item 6's own first example: **"a cone attack must visibly use the real cone dimensions."**

V0.19.6 and V0.19.7 made the cone true in **targeting**. The picture was still wrong — in the
**opposite** direction. The fighter branch ended in:

```js
this.spawnRing?.(p.x, p.y, color, id === 'cleave' ? 16 : 22);
```

So Cleave **tested a 100° wedge in front** and **drew a full 360° ring around** the player. Not an
approximation of the wedge — its inverse. A player learning the ability by watching it was being taught
something that was never true.

**And the ring never showed real dimensions at all.** `spawnRing`'s radius is **screen pixels** — a
hardcoded `16` — with no relation to the spell's real **2.05-tile** reach. It looked like information
and wasn't.

## The fix — a `spellCone` effect type

`spawnSpellConeVfx` (`systems/effects-system.js`) + `drawSpellConeEffect` (`render/effects-renderer.js`).

Three deliberate design points:

### (a) `radius` is WORLD units

The renderer samples the arc in **world space** and projects each point through `vfxScreen` — the same
transform the world uses — rather than approximating with a canvas ellipse. An ellipse's **parametric
angle is not the world angle** under an iso projection, so an arc drawn that way would not line up with
the cone that kills. Sampling and projecting is exact regardless of the transform.

### (b) It deliberately bypasses the sprite cache

`render-backend-system.js`'s `effectSpriteCacheKey` keys on `type:color:style:radius:progress` — **facing
is not in the key**. A baked cone would point the wrong way for every direction but one. This is the same
trap as the **spider `modelScale` cache bug**, and sidestepping it is *why* a new type was added rather
than extending `'ring'`.

Verified the main loop (`game.js:4152`) falls through to `this.drawEffect` when the WebGL queue rejects a
renderable — so a non-eligible type still draws.

### (c) The sweep never overshoots

Uses `1 - (1-p)³` and lands **exactly** on the real radius. The ring's `easeOutBack` **overshoots past 1**,
which would paint the swing landing further out than it actually hits.

### Pool safety

Checked before adding custom fields: `recycleEffectEntry` **deletes every key except `__pooled`** before
pooling, so a recycled entry cannot carry a stale `coneDegrees` into a plain ring.

## What stays a ring

`groundbreaker` and `whirlwind_cleave` declare **no cone** and genuinely **are** omnidirectional. They keep
the ring — the honest picture for a ground slam and a spin.

## Verification — by driving the real renderer

Handed `drawSpellConeEffect` a **recording ctx** and an **identity projection**, reading the world geometry
back out of the path it traced:

| check | result |
|---|---|
| wedge centred on the actor | **yes** |
| arc span (facing due **north** — the axis case V0.19.7 fixed) | **40°–140°** = exactly 100° centred on 90° |
| outer edge | exactly **2.05** — Cleave's real radius |
| reach across 7 frames | **0.92 → 2.05** monotonic, **zero overshoot** |
| end-to-end `resolveFighterRevampSpell` | cleave → **CONE(2.05/100°)**; groundbreaker & whirlwind_cleave → **RING(22)** |
| Berserker's Roar | drawn cone **2.05 → 2.65**, angle unchanged — matches the real buffed radius |

## Item 6 status

**The cone thread is now closed end to end** — declared in data, honoured in targeting, aimed by a correct
facing, and drawn at its true dimensions. Across V0.19.6–V0.19.8 that thread produced: an inert
`coneDegrees` field (279/280 spells), a 360°-hitting "cone", a 72° visual/behaviour gap, a 45° facing bug
in six abilities including shipping Cleave and Backstab, and a ring that never showed real dimensions.

**Item 6 remains open.** Still outstanding:
- `spell_summoner_mass_dismissal` declares `radius: 4` on a single-target spell — may be legitimate for a
  `tempMinions` effect.
- The remaining **279 spells' VFX have not been audited** against their descriptions. The cone was found
  because the data made a falsifiable claim (`category: "AoE Ranged / Cone"`); most spells' visual claims
  live only in prose, so this needs reading, not a validator.
- The facing `||` idiom still exists in `render/` and `systems/fishing-system.js` (see the V0.19.7 report).
