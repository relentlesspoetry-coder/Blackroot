# V0.20.9 — Press I And Look At Something (Roadmap Item 17)

Item 17's list — *inspect supply crates, use water sources, sit on benches, use training dummies,
inspect tents, examine landmarks* — describes objects **V0.20.8 had just placed** in Dead Lantern Camp,
every one of which was scenery you walked past.

## What was built

`systems/world-examine-system.js` — wired the three ways this codebase requires (`_script_order.json`,
an HTML script tag before `game.js`, an `install(Game)` call), plus a per-instance
`installWorldExamineSystem()` at boot beside `installDebugTools` — the pattern already in use.

**22 camp object types examine**, keyed by the type the renderer already draws, each with multiple
authored lines.

| rule | how |

## The key is the story

**The first version bound E** and deferred explicitly: before examining, it asked every other system's
own finder (`npc` / `chest` / `stash` / `quest` / `waypoint` / `station` / `resource` / `event` /
`entrance` — **all nine verified to exist** before being called) whether it had a target, and stayed
silent if any did. It reused their proximity rules rather than duplicating them. It looked right.

**Then it was measured.** Standing at a lone path torch, nearest NPC **2.83 tiles** away:

```
claimants: { npc: "Selene Glasswhisper" @-5,-8, distance 2.83 }
```

`findNearbyNpc` reaches **~3.25 tiles**, and this camp holds **27 NPCs**. An NPC is in range nearly
everywhere — so examine would have been **silent in the exact place its objects live.**

A feature that looks finished and never fires is precisely the pattern this session has spent its time
deleting. **Reasoning about the design did not catch it. Measuring it did.**

So examining moved to **its own key** (`examine: 'i'` in `defaultKeyBindings`; `i` was free, bags are
on `b`). It's always available, and it can never take a press from talking, looting, gathering or
crafting — Item 17's "must not interfere" met **by construction rather than by arbitration**.

The whole `interactKeyIsClaimed()` deferral was then **deleted**, not left behind: a function that no
longer decides anything is new dead code.

### A guard that never guarded

That draft called `this.findLootableCorpseNear?.(4.5)` — **a method that exists nowhere in the
codebase**. With `?.` it would have evaluated to `undefined` forever: a check that never checks. Caught
before shipping. Corpse looting needs no check anyway — its handler (`game.js` ~984) is registered on
the **capture phase** (`}, true`), so it already runs before every bubble listener.

## A line that lied about the forest

`flower` was written for the camp's Medicinal Flowers:

> *"planted deliberately in rows" / "someone weeds these"*

But the table is keyed by object **type**, and the Dark Woods is full of **wild** flowers nobody
planted. Every one of them would have claimed a gardener. **True of one object and false of hundreds is
the same failure as a cone that draws a circle.** Cut. Camp-specific flavour for that one object needs
keying by *name*, not type.

**It was found by a test I got wrong.** I asserted "pressing I in empty woods is silent"; it spoke,
which looked like a bug. It wasn't — there was a flower there. Chasing my own false failure is what
exposed the real one.

## Item 17 status

**Delivered:** inspect crates · use water sources · benches · training dummies · tents · bedrolls ·
lanterns and torches · supply stacks · racks · tables · landmarks (standing stones) · clutter — with
feedback, no rewards, and no message spam.

**Still open:**
- **Sit on benches / rest at campfires / light and extinguish fires** — these are *actions* with player
  state and animation, not text. The log seat currently says "you could sit a while" and can't be sat
  on; that line is honest about what it is, but the verb needs an animation and a pose.
- **Environmental storytelling** (recent attacks, missing people) — needs authored *scenes* placed in
  the world, not a text table.
- **Hidden caches / abandoned journals / shrines** — content that doesn't exist yet.
- **Interaction prompts** — Item 17 wants a readable prompt across input methods; examining is
  currently discoverable only via the key. A "look" prompt when standing beside something examinable
  would want the same treatment `interactLabel` was supposed to give — a field that is, incidentally,
  written on 4 objects and **read by nothing**.
