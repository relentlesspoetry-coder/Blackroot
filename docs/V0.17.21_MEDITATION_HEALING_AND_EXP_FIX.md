## Root Cause: Missing Healing
Meditation recovery was calculating per-second recovery values, then calling `healEntity()` every frame with `hpPerSec * dt`. `healEntity()` floors incoming healing before applying it, so low-level Meditation values such as 2 HP/sec became `Math.floor(2 * 0.016) == 0` on normal frames. The visual Meditation state appeared, but HP did not recover.

## Root Cause: Rapid EXP Gain
`calculateMeditationRegen()` returned `xpPerTick`, and `updateMeditationForEntity()` consumed it every active update. Separately, the world nameplate rendered this as an `Exp:` bar while meditating, making Meditation function as an EXP-progress state instead of a recovery state.

## Recovery Tick Changes
- Added `applyMeditationRecoveryTick()` in `game.js`.
- Active meditation now accumulates delta time and applies recovery only on controlled Meditation ticks.
- Recovery clamps HP and the class resource field (`mana`, which the existing class UI labels as Mana/Focus/Stamina/Energy) to their max values.
- Tick catch-up is capped to avoid a large recovery burst after a long paused frame.

## EXP / Progression Safeguards Added
- Active Meditation no longer grants Meditation XP or player combat XP.
- `calculateMeditationRegen()` now returns `xpPerTick = 0` for active recovery.
- `updateMeditationForEntity()` snapshots player combat XP during the Meditation recovery update and restores it if any recovery path mutates it unexpectedly, logging once in dev console.
- Kill, quest, and party EXP paths were not changed.

## Interruption Behavior Verified
Existing interruption ownership was preserved:
- movement calls `cancelMeditation(..., 'movement')`
- attacks/combat call Meditation cancellation through combat ownership
- spell casts and portal casts already cancel Meditation
- death/combat checks still stop Meditation during the update loop
