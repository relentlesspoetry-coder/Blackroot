# V0.18.42 — Silk Web Cavern boss: entrance web-seal + attack telegraphs

**Report:** *"The first floor boss, upon entering the boss room, we need 3 spiders to walk to the entrance of the boss room and web up the entire entrance so the player can't leave until the boss is dead. This needs to be fully animated. We also need to make sure the bosses and mini bosses attacks show up as indicators with red visual markings on the ground where it will strike."*

---

## 1 — Boss-room web seal (floor 1)
**`systems/dungeon-system.js`** — `dungeonSystem.updateBossSeal(dt)` (a state machine) + `startSilkBossRoomSeal` / `placeSilkSealBarrier` / `unsealSilkBossRoom`, wired into the dungeon update loop.

- **Trigger:** when the **floor-1 boss** first engages (`silkWebBossEncounterEngaged`), once per boss (`_bossSealDone`).
- **Entrance:** computed from the boss arena toward the approach room (`skirr_approach`) — the walkable chokepoint tiles where the corridor meets the boss room. Only that entrance is sealed (not the treasure exit).
- **Walk phase (~2.6s):** 3 spiders scuttle from the boss to the entrance, spread across the doorway, dragging silk lines — animated via `silkSealCrawler` effects (`render/effects-renderer.js` `drawSilkSealCrawlerEffect`, a spider with a walk-bob + trailing thread).
- **Web phase (~1.4s):** blocking **`bossWebSeal`** barrier objects are placed across the entrance tiles (`map.blocked = true` + `solid` objects), which **grow in over ~1.2s** into a floor-to-ceiling silk curtain with an orb web (`render/object-renderer.js` `drawBossWebSeal`). The spiders scurry off to the sides.
- **Sealed:** the player is trapped in the boss room.
- **On boss death:** `unsealSilkBossRoom` tears the barrier apart (a silk burst per tile), restores the tiles' original object/blocked state, and clears the seal.
- **Robustness:** `unsealSilkBossRoom` only restores tiles that **still hold our `bossWebSeal`**, and the seal auto-clears if the boss leaves the floor list — so re-entering or a regenerated floor can never be corrupted by a stale seal.

## 2 — Boss / miniboss attack telegraphs
**`systems/dungeon-system.js`** — `castSilkWebBossAbility`'s damaging ground branch no longer applies damage instantly. Instead:

- It **queues a delayed strike** (`queueSilkBossStrike`, delay ~0.8–0.95s) and spawns a **red ground danger marker** (`render/effects-renderer.js` `drawBossGroundTelegraphEffect`): a filling red ellipse (fills as the strike nears) + a pulsing boundary ring showing the full AoE + radial hazard ticks.
- `updateSilkBossStrikes` (run every frame in `updateBossAbilities`) lands the strike when its delay elapses: damage + poison/web (where applicable) to everyone in the radius, with a bright red impact flash (`drawBossStrikeHitEffect`). If the caster dies first, the strike fizzles.
- Applies to **both bosses and minibosses** (both route through `castSilkWebBossAbility`). The non-ground abilities (summon adds, cocoon prison, shield, heal, retreat, frenzy) are unchanged.
