# V0.18.39 — Silk Web Cavern entrance: trigger fix + graphical overhaul

**Report:** *"We also need to fix the Entrance — once you walk into the entrance you should teleport inside. Right now you walk in and then you have to walk back before it triggers. Also the entire cave entrance needs a graphical overhaul, it looks pretty generic, it doesn't have any webs, there's no mountain or hill attached to it. We want it to look real."*

Two fixes, both scoped to the Silk Web Cavern entrance. The Hidden Tree Cave entrance and all interior cave transitions are untouched.

---

## 1 — Walk-in trigger fires late
**Root cause (`systems/cave-system.js` `checkZoneTransition`):** the overworld cave object is **passable** (`placeObject(..., false)`), so walking into the mouth carries the player *north* onto/through the opening tiles. The front trigger was an ellipse centred at **`ey + 3.05`** with radii `2.05` (x) / `1.35` (y) — it only covered `py ∈ [ey+1.7, ey+4.4]`, i.e. the ground **below** the mouth. So walking in overshot the band to the north, and it only fired when the player walked back **out** into it.

**Fix:** the trigger now covers the whole mouth — centre **`ey + 1.4`**, radii `2.5` (x) / `2.7` (y), covering `py ∈ [ey−1.3, ey+4.1]` and `px ∈ [ex−2, ex+3]`, including the opening at/above the object origin. Walking **into** the mouth now teleports immediately, while the south extent (`ey+4.1`) stays close to the old approach so you don't trigger from far away.

## 2 — Generic entrance → real, webbed spider lair
**`render/object-renderer.js` `drawCaveEntrance`**, gated on the Silk Web Cavern's `variant === 'web'` (so Hidden Tree Cave's `moss` variant is unaffected):

- **Hill / mound:** the mouth is now seated in a real earthen hill drawn **behind** the rock — a dark mossy-earth body, a grassy crown highlight, an exposed dirt bank at the base, and grass tufts fringing the crown — so the cave reads as dug into a hillside instead of a floating rock arch.
- **Webbing:** the opening is choked with silk — a **giant orb web** (12 radial spokes + 5 concentric rings) filling the mouth, **10 heavy drape curtains** hanging from the top, **3 hanging egg-sac cocoons** on silk lines, and **corner webs** tucked into the top corners. The old sparse 4-strand look is gone.
- The generic flat purple glow orb is **dimmed into a webbed hollow** (a dark fill over the opening), leaving only a faint sickly purple halo behind the webs.

## Validation
- `node --check` — clean on all changed files and the full non-`blender/` JS set.
- JSON — all non-`blender/` JSON parses clean.
- Runtime (served `Blackroot V0.18.39.html`): boots with **0 console errors**; `drawCaveEntrance` runs without faults.
- **Caveat:** isolated pixel-probing of `drawCaveEntrance` is unreliable in this harness (object-renderer captures its canvas context in a module closure at install, so reassigning the scratch context doesn't redirect it). The new art uses the **identical** primitives (`this.fillPoly`, the `P()` scaler, `ctx.stroke`) as the existing entrance that already renders correctly in-game, and is gated to the web variant. Both the new entrance look and the walk-in trigger want an in-game eyeball pass.
