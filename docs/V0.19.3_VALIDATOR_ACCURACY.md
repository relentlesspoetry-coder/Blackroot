# V0.19.3 — Validator Accuracy: Quest Giver/TurnIn False Positives

Five of the seven long-standing descriptor warnings were **false alarms about content that works**.
This fixes the check (not the data), and triages the two that are real.

## The problem

`core/registry.js` validated quest `giver`/`turnIn` with `checkRefByName('npc', ...)` — the **NPC table
only**. But a giver/turnIn is a *display name*, and it legitimately may not be a townsfolk NPC:

| flagged | reality | warnings |
|---|---|---|
| **"Rurik the Fallen"** | a **mob** (`mob_bandit_leader`). `quest_fall_of_rurik` is a bounty — its task is literally `{type:'kill', target:'Rurik the Fallen'}` — and `quest_ruriks_bargain` is his branch. **Both `autoComplete`**, and the data's own note says *"Rurik is not a turn-in NPC"*. | ×4 |
| **"The Survivor"** | `quest_fourth_member` **`autoOffer`s** at `foundAtPoi` and turns in to Iren; its escort task self-describes the actor. No NPC giver exists or should. | ×1 |

**A validator that cries wolf is worse than no validator** — noise trains you to ignore it. So the
check was fixed, not the content.

## The fix

New `checkQuestPersonName(quest, field)`:

- a giver/turnIn name is valid if it resolves to an **npc, mob, or boss**;
- the reference is **optional** when the quest cannot have that NPC:
  - `giver` → `autoOffer` / `foundAtPoi` (a *place* hands it over)
  - `turnIn` → `autoComplete` (nobody takes it back)

**Not merely weakened** — verified that injecting a bogus giver (`"Nobody McNotreal"`) is **still
caught**.

## Result: 7 → 2 warnings, both real

Both were investigated and **deliberately not auto-fixed** — each is unfinished content needing a
design call, not a reference typo:

### 1. `event "quest_hook"` → `questHook: "quest_darkwood_first_steps"`

That quest exists **only** in `systems/bot-player-system.js`'s separate **bot** quest catalog and in an
editor default draft — never in `DR.QUEST_BY_ID`. So the event silently does nothing today.

The only real level-1 starter is `quest_light_the_way`, which **already has Road Warden Cael as its
giver**, so simply repointing the hook risks double-offering it.
**Decision needed:** write the quest, repoint the hook, or remove the event.

### 2. `dungeon_glooms_crypt` → `blackroot_catacombs`

The dungeon has a valid `zoneId` (`dark_woods`) but dangling `caveZoneId: 'blackroot_catacombs'` /
`entranceZoneId: 'blackroot_catacombs_f4'`. That cave doesn't exist in `DR.CAVE_BY_ID` (only
`mossfang_cave` and `silk_web_cavern`), and `data/default-zones.js` lists it beside `ashroot_hollow` /
`crystal_grotto` / `forgotten_mine` as **planned** cave zones.
**Building the catacombs is content work**, not a reference fix.

- Descriptor warnings **7 → 2**, both remaining ones real.
- A bogus giver is **still caught** — the check wasn't just switched off.
