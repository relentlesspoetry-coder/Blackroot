# V0.18.67 ‚Äî Food Buffs Actually Apply (Roadmap Item 3 begins)

First player-facing slice of **Roadmap Item 3 (Campfire Cooking / Cooking / Alchemy)**, built directly on the V0.18.65/66 effect-policy foundation.

## Root cause
Cooked-food items in `data/items.js` already declared a temporary buff via `useEffect.buff`:

| Food | Buff | Effect |
|---|---|---|
| Grilled Minnow | Well Fed | +1 defense, 120s |
| Cooked Eel Skewer | Eel Vitality | +8 hp, 180s |

‚Ä¶but the consumable-use path (`Game.prototype.useInventoryItem`, `systems/item-compiler-system.js`) only ever applied `useEffect.hp` / `.mana` / `.castsSpellId`. It **never applied `useEffect.buff`** ‚Äî so every food buff was dead data. Eating the food healed you, but the buff never happened.

## Fix
1. **Apply `useEffect.buff`** ‚Äî `useInventoryItem` now applies the buff as a status effect via `this.applyStatusEffect` (stable per-item buff id, the buff's own `effectCategory`, name/duration/mods/color), counts it toward "did anything happen", and includes it in the no-usable-effect guard. Because the buff carries `effectCategory: 'food'`, the **central effect policy governs it** ‚Äî `food` is `maxConcurrent: 1` / `replaceStronger`, so eating a second food **replaces** the first food buff instead of stacking.
2. **Categorize + cooldown the foods** (`data/items.js`) ‚Äî the two buff-foods now set `effectCategory: 'food'` on their buff and a per-item `cooldownMs` (10s / 12s), so a food heal+buff can't be spammed (Item 3 requires an item cooldown). The cooldown key defaults to the `itemId`, so different foods don't share one timer.

## Verification (end-to-end, live, on a mock player)
- Compiled `item_grilled_minnow` carries `useEffect.buff { name:'Well Fed', effectCategory:'food', mods:{defense:1} }` and `cooldownMs: 10000`.
- **Eat Grilled Minnow** ‚Üí hp **50‚Üí68** (+18), mana **20‚Üí28** (+8), buff **"Well Fed"** applied (`effectCategory: food`, +1 defense), cooldown set on key `item_grilled_minnow`.
- **Eat Cooked Eel Skewer** next ‚Üí the food buff is **replaced**: only **"Eel Vitality"** remains (1 food buff cap enforced by the policy).
- `node --check` clean; JSON clean; boots with **0 console errors**.

## Why it matters
This is the plumbing every future consumable buff uses. The 20+ cooked foods, fish dishes, and herbal potions (rest of Item 3) each become fully governed ‚Äî heal + categorized buff + cooldown + one-at-a-time ‚Äî simply by declaring `useEffect.buff` with an `effectCategory`. It also generalizes to **potions** (`potion_buff` category) with no extra code.
