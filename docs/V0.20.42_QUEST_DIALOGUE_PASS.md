# V0.20.42 — Quest Dialogue & Narrative Pass (Roadmap Item 1, COMPLETE)

Roadmap Item 1 asks for a complete authored dialogue pass over all 26 Dark Woods quests, covering eight
dialogue states, in character, with no fourth-wall / UI / developer language.

## The method mattered more than the volume

The existing quest set was already strong — 26 quests, all with offer / in-progress / completion text,
a fully authored Rurik branch, clean objective summaries, and **0 validator violations** going in. So
the work was *finding the true gaps* without inventing fake ones.

A standalone harness (`scratchpad/quest_audit.js`) flagged every quest missing any of the eight states.
Then — the session's recurring lesson applied to the audit itself — I **read the real render paths**
before authoring, because a flag is only a gap if the game would actually speak it:

| harness flag | verdict after reading the render code |
|---|---|
| 6 × missing `lockedText` | **false positive** — all prereq-only quests. `unavailableQuestReason` returns a name hint for prereqs; `lockedText` is consumed only when `requiredConditions` fail (quest-system.js:429), and every quest with `requiredConditions` already has it. |
| 5 × missing `beforeOfferText` | **false positive** — giver-less breadcrumbs (`giver:null`), or branch/auto-offer quests where the branch modal / auto-offer replaces the approach-and-greet moment (npc-system.js:1413 needs a giver). |
| 10 × missing `postCompletionText` | **mostly N/A** — repeatable dailies re-offer (mode `accept`) rather than hitting the post-completion fallback (quest-system.js:1249); giver-less/dead-giver quests have no speaker. |
| 3 × failable missing `failureText` | **REAL** — see below. |

Padding the false positives would be dead data — the exact anti-pattern this project forbids.

## The three real fixes (each root-caused in its owning module)

### 1. Failure dialogue for defense events (state 5)
`failureText` was read in **only one place**: the escort permanent-failure path (quest-system.js:788).
The three **defend** quests failed with a single fixed system-narrator line — the same for every quest,
in no one's voice. Roadmap state 5 explicitly requires failure dialogue for *defense events*.

Fix: a new `logDefendFailure(questId, reason)` routes all three defend-failure branches (overrun /
left-gate / hold-timeout) through the quest's own in-character `failureText` first, then the mechanical
reset note — because a defend is **retryable**, unlike the permanent escort loss. Quests without a
`failureText` fall through to exactly their old line, so nothing else changes. Authored `failureText`
for `quest_guides_warning`, `quest_what_the_gap_lets_in`, and `quest_the_long_root`, each written in
the retryable-loss register (the hold cracked; make it again).

### 2. In-character in-progress lines (state 3)
State 3 requires in-progress dialogue to "remain in character rather than repeating raw objective
text." Eleven quests violated this — most bluntly `quest_the_long_root`, which literally **spoke**
`"(300s)"` and `"6 waves"`, and three night quests that shouted `"NIGHT ONLY"`. These are timer/UI
notations in an NPC's mouth, and the validator's word-list missed them.

All 11 were rewritten into each speaker's voice (Corven, Cael, Branna, Selene, Thalen, the
Quartermaster, the Cook). Nothing is lost: the mechanical detail already lives in the `tasks[].label`
objective summaries and the live defend HUD (wave counter + hold timer).

### 3. Post-completion for the Rurik ally path (state 8)
`quest_ruriks_bargain` — the fork where you kill Cael's patrol and join Rurik — had no
`postCompletionText`, so Rurik (who *lives* on this path) replayed his turn-in line forever. Added a
post-completion line acknowledging the changed world (you're now one of his).

## Validator hardened (Roadmap Item 22 discipline)

`validateQuestDialogue` now also flags parenthetical seconds-notation (`/\(\s*\d+\s*s\s*\)/`) in spoken
fields — the class of dev-token that let `"(300s)"` ship. **Proven non-vacuous:** it matches the
pre-fix line, and passes clean prose and ordinary numbers ("three coins and six berries").
