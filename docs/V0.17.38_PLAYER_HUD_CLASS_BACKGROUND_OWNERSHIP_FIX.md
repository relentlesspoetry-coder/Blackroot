## Root cause
The Player HUD backdrop was still using a darker bespoke background stack than the Party HUD cards. The class art resolver was working, but the top-card surface applied a heavier black treatment that suppressed the class art and made the panel read as black.

## Fix details
- Reused the Party HUD class-card backdrop treatment for the Player HUD top card.
- Kept the backdrop scoped to the top player card only.
- Preserved the Party HUD cards unchanged.
