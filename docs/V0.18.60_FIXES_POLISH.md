# V0.18.60 — Fixes & Polish (part 1 of the request)

Your latest message was a big list. This version ships the quick fixes/polish; the larger visual items are queued (see bottom).

## 1. Darkness overlay glitch (fixed)
`game.renderSilkCavernAtmosphere` (`render/object-renderer.js`, added V0.18.59) sized its full-screen dark tint + vignette + fog with `canvas.width/height` — **device pixels**. The canvas is `cssWidth * dpr` with a matching dpr transform on the ctx, so at a zoom where `dpr ≠ 1` the fill didn't line up with the visible area and the darkness only covered part of the screen (your screenshot). It now sizes everything with `window.innerWidth/innerHeight` (**CSS pixels**) — exactly like `drawVignette`, which already worked full-screen. Verified: a corner pixel now darkens (200 → 83) where it previously stayed light.

## 2. Torch — smaller + smoother flame
`drawTorch` is now wrapped in `translate(s) + scale(0.62)` (≈40% smaller). The flame animation was a fast two-sine flicker; it's now a **lower-amplitude blend of three slow sines** at a slower base rate, with a gentler sway and slower sparks — a smoother, calmer flame.

## 3. "Silk Web Caverns" title card
`loadDungeonFloor` (`systems/dungeon-system.js`) sets `game.dungeonTitleCard = { text: 'Silk Web Caverns', at }` when the loaded dungeon is `silk_web_cavern`. `renderSilkCavernAtmosphere` draws it: a big centered serif title with a soft glow and a decorative rule, that **scales in** and **fades out ~2 s** later, then clears itself.

## 4. Click-to-move while rooted (bug fixed)
Keyboard movement already checked `hasStatusTag(player, 'root')`, but **click-to-move didn't**, so you could still walk while rooted. Now:
- `setClickMoveTarget` rejects the move (and logs "You are rooted.", rate-limited) when rooted;
- `updateClickToWalk` stops an **in-progress** click-move the instant root is applied.
