# V0.17.92 — Animated Quest Markers over NPCs

**Type:** UI / world (quest markers)
**Previous version:** V0.17.91
**New version:** V0.17.92

Replaces the old small static purple badge above quest-giving NPCs with a **large, bobbing,
softly-glowing** marker so quest sources read at a glance.

---

## What changed (`systems/npc-system.js`)

- **`questMarkerMode(game, npc)`** — resolves the NPC's best quest state via the existing
  `questSystem.chooseQuestFromSource` (which already prioritises complete > accept > locked > progress):
  - `accept` → **gold `!`** — a quest is available to take.
  - `complete` → **gold `?`** — a quest is ready to turn in.
  - `progress` → **silver `?`** — a quest is in progress here (come back later).
  - locked / done → **no marker**.
- **`drawAnimatedQuestMarker`** — draws the marker at the NPC's `_nameplateAnchor` with a sine **bob**, a
  pulsing **screen-blended halo**, and an **outlined glyph** (dark outline + drop shade + bright face). It
  scales up when the player is within interaction range.
- The marker is drawn in the NPC render loop (`render()`), replacing the old ~8px purple `!`/`✓` badge.
  Only NPCs with `questIds` ever show a marker, and it updates live as quests are taken/completed.

## Verification (in-browser, port 8899)

- Road Warden Cael → `accept` (available `!`); the Cleric trainer → `locked` (prerequisite unmet → no
  marker); accepting a quest flips the giver's active quest to `progress` (silver `?`) and completing its
  objective flips it to `complete` (gold `?`).
- The NPC render pass ran many frames with no console errors.

**Visual note:** `preview_screenshot` times out on this WebGL page, so the marker's exact look (bob height,
colours, glow) is the user's visual sign-off. Glyph size/colours/bob are easy tunables.
