## Root Cause of Green Character Preview Background
The Character Model preview had been routed through competing background layers across `.dollCenter`, `.paperDollPreview`, and `.paperDollClassBackdrop`. A prior cleanup removed one duplicated class-art layer, but left the visible preview surface dominated by dark/green fallback gradients. For nature-toned classes such as Druid and Warden, the class image was either visually buried or not applied to the winning surface, so the panel still read as a flat green placeholder instead of class art.

## Root Cause of Plain Player HUD Background
The Player HUD depended on the normal full UI refresh to apply `--party-class-bg` to `#hud`. Fast HUD refreshes updated HP/Mana/text but did not guarantee that the HUD class-background variables and backdrop surface were applied. The HUD also used a dark background treatment that visually overpowered the class art when the backdrop was present.

## Class Background Resolver Changes
- Added `classBackgroundVisualSpec(className)` as the canonical class background lookup path.
- Normalizes class names through the existing class emblem key resolver.
- Resolves a single background path, emblem path, and class accent color for all UI surfaces.
- Exposes the resolver through `DR.classBackgroundVisualSpec`.
- Keeps missing background warnings behind the existing debug/dev flag and logs each missing key once.

## Asset Loading / Key Fixes
- Uses `assets/ui/class-backgrounds/<normalized-class-key>.png` for all class background surfaces.
- Uses `assets/ui/class-emblems/<normalized-class-key>.png` for the paired emblem.
- Preserves the existing image load path and adds an error hook for missing class background assets.
- No new assets were added and no class-specific hardcoding was introduced.

## Character Preview Render-Order Changes
- `paperDollMarkup()` now pulls the background/emblem/color from `DR.classBackgroundVisualSpec()`.
- `.paperDollPreview.classModelPaperDoll` is now the single class-background surface for the Character Model preview.
- `.paperDollClassBackdrop` now provides only vignette/emblem readability treatment instead of owning another independent class-art layer.
- The model canvas still renders above the background and keeps the existing character presentation renderer.

## Player HUD Background Render Changes
- Added `ensurePlayerHudClassVisuals()` to keep the Player HUD class visual variables applied from both full and fast HUD update paths.
- The HUD backdrop receives class background variables directly from the canonical resolver.
- HUD dark overlay opacity was reduced so class art is visible while preserving readable name, level/class text, HP bar, Mana/resource bar, and class icon.
- The HUD layout, size, and class icon placement were preserved.

## Fallback Behavior
- If a class background cannot resolve, the preview/HUD falls back to a clean dark background instead of a green panel.
- Missing asset warnings are debug-gated and emitted once per class key.
