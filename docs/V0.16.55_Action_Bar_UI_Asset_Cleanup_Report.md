# V0.16.55 - Action Bar UI Asset Cleanup and Overlay Pass Report

## Summary
Applied a focused action-bar-only cleanup pass. The canonical `Action Bar UI` PNG remains the real action bar shell, transparent pixels were normalized, the in-game scale was reduced, and the live overlays were recalibrated to the asset frame.

## Asset cleanup performed
- Rewrote `assets/ui/Action Bar UI.png` from the transparent canonical source.
- Normalized fully transparent pixels to RGBA `(0, 0, 0, 0)` so no hidden checkerboard/matte color remains under alpha.
- Preserved the action bar artwork, slot frames, EXP track, and Meditate frame.
- Kept the game-facing asset name as `Action Bar UI`.

## Transparency/background-removal notes
The asset used by the game has real alpha transparency around the action-bar shell. No DOM/CSS overlay or fake background mask was added.

- Reduced the desktop asset-backed bar from the prior oversized clamp to `clamp(660px, 54vw, 980px)`.
- Added asset-specific responsive overrides so older generic hotbar media rules do not reintroduce padding or oversized layout.
- Preserved bottom-center anchoring and the PNG aspect ratio.

## Overlay alignment fixes
- Tightened EXP fill bounds to stay inside the top track.
- Recalibrated slot group bounds and gap spacing for all ten slots.
- Repositioned icon, hotkey, label/cost, cooldown, and active-outline overlays inside the PNG slot frames.
- Recentered and resized the Meditate overlay/hitbox to the PNG Meditate frame.

## Input/hitbox verification notes
The existing action slots and Meditate button remain the live DOM hitboxes. Their layout now follows the same asset-scaled owner, so pointer hitboxes track the visible frame rather than a detached overlay. Keyboard bindings `1-0` remain owned by the existing spell/action bar system.
