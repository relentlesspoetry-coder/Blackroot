## Root Cause
The Player HUD was still visually owned by the legacy black HUD background stack. V0.17.35 added optional `.playerHudClassBackdropImage` / `.playerHudClassBackdropVeil` child nodes, but the stable HUD surface `.hudClassBackdrop` still had an important `background-image: none` override and the root HUD retained a black important background. When the optional image layer did not become the visible paint owner, the HUD stayed black even though the class resolver returned the correct asset path.

## Fix Details
- Promoted `.hudClassBackdrop` itself to the owning Player HUD class-art surface.
- Removed runtime reliance on optional image/veil child nodes.
- Set the Player HUD backdrop background stack to: dark readability overlay, class-color glow, resolved class background art, and light bottom veil.
- Kept the Player HUD root as a semi-opaque panel base instead of a fully black image-obscuring surface.
- Preserved the already-correct Party HUD card renderer and companion command layout.
