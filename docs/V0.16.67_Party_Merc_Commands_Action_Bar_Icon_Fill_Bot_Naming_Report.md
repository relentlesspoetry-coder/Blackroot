## Mercenary / Bot Command Controls
Restored compact command chips directly inside controllable companion rows in the Party / Companions panel. The controls expose the existing expected command set:

- Follow
- Guard
- Assist
- Passive
- Attack

Mercenary commands call the existing `setMercCommand()` owner. Bot commands use a small owned `setBotCompanionCommand()` helper in `systems/bot-player-system.js` so bot command state remains on the bot runtime object instead of introducing a separate party-side command store.

The active command is visually highlighted with `aria-pressed` state. Party HP/mana bars, class labels, level labels, slot count, and status text are preserved.

## Action Bar Icon Sizing Fix
The action-bar CSS now treats spell art as the full-slot background layer for spell slots. The spell icon element is positioned across the full slot interior, while key labels, spell names, mana readouts, cooldown overlays, and disabled/resource overlays remain layered above it.

This fixes the previous inset/tiny icon presentation without replacing icon assets, adding class-specific icon hacks, or creating a second icon renderer.

## Bot Naming Assignment and Migration
Added bot display identity normalization in the existing bot player owner. Placeholder names matching patterns such as `Fighter Bot`, `Ranger Bot`, or plain `Bot` are replaced once with deterministic fantasy names from the BlackRoot bot name pool. The bot type label remains separate as `<Class> Bot`.

Examples of resulting display format:

- World nameplate: `<Name> · Fighter Bot`
- Party row: `<Name> - Fighter Bot - Level 1`

Existing saves with placeholder bot names are normalized through `normalizeBotState()` / `ensureBotDisplayIdentity()` and then persist through the existing bot save snapshot. New bot snapshots now store `botTypeLabel` and `botPartyCommand` alongside the existing command state.
