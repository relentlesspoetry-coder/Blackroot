# V0.17.08 Player HUD Class Background and Icon Pass

## Scope

This phase updates only the player HUD class presentation. It does not redesign the Party / Companions HUD or change gameplay systems.

## Implementation

- Generalized the existing party class-card style resolver into the shared `classHudVisualStyle` path while retaining the prior public alias for compatibility.
- Applied the same canonical class key, class background asset, emblem asset, and class color variables to the player HUD during its normal full UI update.
- Added an owned player-HUD backdrop layer that uses the shared class variables with readability gradients matching the party card design language.
- Changed the player HUD grid to reserve a right-side 104px class-art column.
- Enlarged the player class emblem to 94px inside a 100px right-side frame.
- Preserved the existing player name/meta, HP/resource bars, status tray, and unified companion HUD attachment.

## Files and Systems

- `systems/ui-system.js`: shared class visual resolution and normal player/party HUD rendering paths.
- `Blackroot V0.17.08.html`: player HUD markup and owning layout/style rules.
- `core/config.js`, `VERSION.txt`, `README.txt`: V0.17.08 metadata.
- `PATCH_NOTES.md`: V0.17.08 release notes.

## Compatibility

- All class names continue through `classEmblemKey` and `classEmblemMeta`; no class-specific Assassin-only branch was added.
- Party frames still use the same renderer, variables, assets, and layout.
- The unified companion HUD remains a full-width grid child below the player frame.
- No gameplay or balance logic changed.

## Validation

- Full JavaScript syntax sweep: passed with zero errors.
- Full JSON parse sweep: passed with zero errors.
- The HTML contains no inline JavaScript; all referenced external scripts were included in the JavaScript syntax sweep. HUD markup/style structure was reviewed separately.
- All 14 mapped class background and class emblem image files were confirmed present and non-empty.

## Manual Regression Tests

1. Enter the world with each playable class and confirm the correct class backdrop and right-side emblem.
2. Verify player name, level/gender/class line, HP/resource labels and values, and buff/debuff icons remain unobstructed.
3. Change or load a different class and confirm the HUD art refreshes on the normal UI update.
4. Damage/heal and spend/regenerate the class resource to verify bar updates.
5. Add and remove buffs/debuffs to verify status tray layout and tooltips.
6. Form a party and summon commandable companions; verify party frames and command buttons.
7. Check normal and scaled HUD layouts at the supported 2560x1440 target and a smaller window.
8. Open menus and overlapping UI windows to verify layering remains unchanged.
