All new class starter items use `item_starter_v01689_*` ids and are white, level 1, stack size 1, class-restricted, weak starter equipment. Runtime item creation marks granted items as `starter: true` and `bound: true`.

## Removed/replaced old active class starter gear
Old active class starter gear object definitions using `item_starter_<class>_*` ids were removed from `data/items.js`. Old class starter ids were also removed from the active `classStarterGearIds()` map.

Profession starter tools, starter bags, gathering tools, fishing tools, loot drops, and unrelated starter/non-class items were not removed.

## Inventory-system changes
`systems/inventory-system.js` now maps every supported class to exactly five V0.16.89 starter item ids.

`grantClassStartingGear()` now resolves the real equipment slot before auto-equipping:

```js
const slot = this.resolveEquipmentSlotForItem?.(item, true) || this.canonicalEquipmentSlot?.(item.slot) || item.slot;
```

It then equips only if the resolved slot exists and is empty. This allows offhand/charm-style starter items to auto-equip correctly while still preserving overflow behavior if a slot is occupied.

## Save/load duplication fix
`systems/save-system.js` now persists starter gear grant state through character saves:

- `starterGearGranted: Boolean(this.classStartingGearGranted)`

On load, the runtime restores:

- `this.classStartingGearGranted = Boolean(data.starterGearGranted ?? c.starterGearGranted)`

`systems/inventory-system.js` also adds `hasAnyClassStarterGear(className)` and uses it before granting starter gear so a character that already has V0.16.89 class starter gear cannot receive a duplicate kit.

## Two-handed handling
Fighter and Ranger receive two-handed starter weapons and identity charms instead of forced offhand shields/foci:

- Fighter: `item_starter_v01689_fighter_rustcleaver` + `item_starter_v01689_fighter_bruisers_token`
- Ranger: `item_starter_v01689_ranger_greenwood_bow` + `item_starter_v01689_ranger_frayed_quiver`

The existing equipment system's two-handed behavior was preserved; the full equipment system was not rewritten.
