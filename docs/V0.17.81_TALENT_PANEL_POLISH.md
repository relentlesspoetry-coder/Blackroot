# V0.17.81 — Talent Panel Polish

**Type:** UI / UX (talent panel)
**Previous version:** V0.17.80
**New version:** V0.17.81

Polish pass on the talent panel.

---

## Changes (`systems/talent-system.js`)

- **Styled hover tooltip.** The plain native `title=""` tooltip on talent tiles is replaced by a custom
  `#talentTooltip` element built by `Game.prototype.talentTileTooltipHtml(specId, nodeId)`. It shows:
  - the node name (★ prefix for capstones), spec-coloured,
  - current rank `x / max`,
  - the current-rank description,
  - the **next-rank** description in the spec colour (when there is a next rank),
  - the tier-gate requirement on locked tiles.

  It has a spec-coloured left border, follows the cursor, clamps to the viewport, and hides on
  `mouseout` and when the panel closes. The redundant native `title` (and the per-tile tooltip-string
  build) were removed so there is no double tooltip.
- **Tile hover feedback.** Hovering a talent lifts and scales its icon (`translateY(-2px) scale(1.06)`).

## Verification (in-browser, port 8899)

Hovering a tile shows the styled tooltip with the correct name / rank / current + next-rank text and a
spec-coloured border; a locked tile shows the "Requires N points in <spec>" line; no native `title`
remains; no console errors.

## Known limitation

The preview screenshot tool times out on this WebGL/canvas page (even with the render loop frozen), so
the V0.17.80 themed SVG silhouettes and this panel's overall look are verified **structurally** (correct
elements, computed styles, mapping) but not by a captured image. Any silhouette that reads poorly in-game
is a one-line fix in the `THEME_SVG` map.
