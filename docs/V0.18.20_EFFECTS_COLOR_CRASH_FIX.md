# V0.18.20 — Fix: combat effect render-fault crash (NaN color)

Found during a full-game bug/error review.

## The bug
During combat, casting spells triggered **repeated render faults** every frame while
certain impact effects were on screen:

```
render fault in world-effect SyntaxError: Failed to execute 'addColorStop' on
'CanvasGradient': The value provided ('rgba(NaN,11,236,…)') could not be parsed as a color.
  at Game.drawSoftGlow (render/effects-renderer.js:62)
  at Game.drawImpactBloom (render/effects-renderer.js:225)
  at Game.drawCombatImpactEffect (render/effects-renderer.js:492)
```

The game's render-fault wrapper caught it (no hard crash), but each faulting effect:
- spammed the console with an error every frame,
- was skipped (not drawn), and
- burned CPU on the throw/catch.

Notably this was **invisible to `window.onerror`** (the render wrapper swallows it),
so it only showed up when checking `console.error` directly.

## Root cause
`hexToRgb(color)` (render/effects-renderer.js:30) parsed channels with bare
`parseInt(hex.slice(...), 16)` and **no NaN guard**. A malformed color string (some
effect passed one whose first two chars weren't valid hex) produced `{ r: NaN, … }`,
and `rgba()` interpolated that straight into `rgba(NaN,11,236,…)` — which
`CanvasGradient.addColorStop` rejects, throwing on every draw.

## The fix
Make the color parser degrade gracefully — a color parser should never emit `NaN`.
Each channel is now clamped to a valid 0–255 int, falling back to 255 if it can't be
parsed:

```js
const ch = v => { const n = parseInt(v, 16); return Number.isFinite(n) ? Math.max(0, Math.min(255, n)) : 255; };
```

This fixes every caller of `rgba`/`hexToRgb` in the effects renderer, so no malformed
color can ever crash a gradient again. The offending effect now renders in a valid
(slightly-off) colour instead of faulting.

## Verified (in-browser)
- Parser: `rgba('#zz0bec', .5)` → `rgba(255,11,236,0.5)` (was `rgba(NaN,…)`); all
  malformed inputs (`''`, `null`, `rgb(...)`, junk) yield valid rgba; valid hex
  unchanged (`#6fa8ff` → `rgba(111,168,255,…)`).
- Reproduced the exact combat scenario (Wizard, all 20 spells cast repeatedly in
  melee combat, effects spawning): **0 new render faults** (was continuous).
- Broad sweep across Cleric/Necromancer/Bard/Druid casting all spells + panels:
  **0 new console errors**.
