## New engine

### `defend` verb (`systems/quest-system.js`)
A `defend` task holds a fixed point through sequential waves:
```
{ type:'defend', target:'camp_south_gate', x, y, radius, spawnDx, spawnDy, mobLevel,
  waves: [ [{mob, count}], ... ], ally: { name, color, hp, attackMin, attackMax },
  required: <waves.length>, progress: 0 }
```
- `updateDefendEncounters()` (per-frame) finds the active defend task; when the player is within `radius`
  it starts the encounter (`startDefendEncounter`) — progress 0, wave 0 spawned, optional ally spawned.
- Progress = **waves cleared**. When every mob of the current wave is dead the next wave spawns; when the
  last wave is cleared the task completes and the encounter tears down.
- **Reset conditions**: if the player dies or wanders more than `radius + 8` from the point, the remaining
  mobs + ally are despawned and progress resets to 0 (retry the hold).
- **Ally** (`spawnQuestAlly`): a temporary friendly combatant built from the `Pet` actor
  (`temporaryPet:true`, `command:'assist'`) — Road Warden Cael for [05]. Despawned when the encounter ends.

Wave mobs use two new `mob-spawn-system.js` helpers:
- `Game.prototype.spawnQuestWaveEnemy(mobId, x, y, level, opts)` — builds a `DR.Enemy` from a mob draft
  (`makeEnemyType`), tagged `questWaveMob` with **no spawn slot**, so the normal respawn/population scan
  ignores it (never respawns, never counted against zone density). Finds a valid tile via
  `findValidMobSpawnNear`.
- `Game.prototype.despawnQuestEntity(entity)` — removes an entity from `enemies`/`overworldEnemies`/
  `entities`. The defend loop despawns each wave mob the instant it dies, so no corpses linger.

### `timed` modifier
A quest (or task) carrying `deadlineSeconds` gets a wall-clock `deadlineAt` set at accept
(`questDeadlineSeconds` + `acceptQuest`). `updateTimedQuests()` fails and removes the quest if the deadline
passes before completion; the quest HUD panel shows a `⏱ Ns` countdown while a timed quest is active.
(Not used by [05] — wave-clear — but ready for the [11]/[20] race quests.)

## Content

**[05] The Guide's Warning** (`quest_guides_warning`, L3, giver Road Warden Cael) — defend
`camp_south_gate` at (100,113) through 3 waves: **3× rotling → 4× rotling → 5× gloom wolf**, with Cael
fighting alongside. Reward: 300 xp / 35 g / `item_ironbark_shield`. Bound to `npc_road_warden`
(now offers Light the Way, The Guide's Warning, Thin the Pack).

## Verification (in-browser, port 8899)

Started a session at the gate: the encounter begins on arrival; the three waves spawn and advance with the
correct mob types (rotling → rotling → gloom wolf) and progress 0→1→2→3; the Cael ally spawns as a friendly
combatant and is cleaned up at the end; a full clear completes the quest, grants the Ironbark Shield, and
removes **all** wave mobs from the world; walking away mid-encounter tears it down and resets progress to 0
with mobs despawned; and an expired `deadlineAt` fails its quest. No console errors.
