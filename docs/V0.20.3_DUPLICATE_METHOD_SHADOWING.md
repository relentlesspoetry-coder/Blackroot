# V0.20.3 — The Method That Ate Its Twin

Found while verifying V0.20.2. **Confirmed pre-existing**: `render/object-renderer.js` was byte-identical
to the shipped `Blackroot V0.20.1.zip` backup — untouched by this session.

## The bug

`object-renderer.js` defined **`drawOrbWeb` twice**:

| line | signature |
|---|---|
| 608 | `drawOrbWeb(cx, cy, opts = {})` — options form, **own internal `rnd`** |
| 2269 | `drawOrbWeb(cx, cy, r, silkP, alpha, rnd, ky = 0.88)` — positional, `rnd` **passed in** |

JavaScript accepts a duplicate method name **silently**, and the **last definition wins**. So the
options-form implementation was **unreachable dead code**, and its three callers were really hitting the
positional one:

```js
this.drawOrbWeb(s.x - 2, s.y - 8, { rx: 16, ry: 12, … });   // object where `r` was expected
                                                            // → rnd === undefined → rnd() throws
```

**`TypeError: rnd is not a function`, every frame.** Webbed bushes and web-strung trees had not been
drawing **at all**.

The reason nobody noticed: `drawObject`'s per-object `try/catch` swallowed each throw as a
`"render fault in world-object"` and moved on. The safety net worked *perfectly* — it caught the same
error thousands of times and never let it become a visible crash. The scenery just quietly wasn't there.

## The second, silent victim

`drawCaveStairs` was **also** defined twice:

| line | signature | used by |
|---|---|---|
| 572 | `drawCaveStairs(s, obj, label = 'DOWN')` | the `caveStairsDown` / `caveStairsUp` shape cases |
| 1498 | `drawCaveStairs(s, obj)` — V0.18.61's silk-web cave mouth | `drawDungeonStairs`, for `silk_web_cavern` only |

The later one won, so **every** cave staircase rendered the silk-web steps and **lost its UP/DOWN
label** — the exact opposite of what V0.18.61's own comment promised:

> *"natural rough-hewn STONE cave stairs … for the Silk Web Cavern (other dungeons keep the old
> stylised steps)"*

That comment is also what made this fix **unambiguous rather than a guess**: the intent was written
down. Without it, "which twin was meant to win" would have been a design question to escalate, not a bug
to fix.

## The fix — name each method for what it draws

- `drawOrbWeb` (options form) → **`drawAnchoredOrbWeb`** — ties a web to explicit branch-tip `anchors`,
  which the positional one cannot do. Its 3 callers repointed.
- `drawCaveStairs` (silk-web form) → **`drawNaturalCaveStairs`** — the rough stone cave mouth. Its 1
  caller repointed.

**The two working positional `drawOrbWeb` callers are untouched** — zero regression risk on the paths
that were already fine.

## The check — `tools/check-duplicate-methods.js`

Wired into `validate.sh`. This class is invisible to **`node --check`** *and* to **reading the file**,
because the code that never runs still looks finished. Nothing else in the toolchain looks for it.

| | |
|---|---|
| found | **both** bugs |
| false positives across 193 files | **0** |
| negative-tested | re-introducing the `drawCaveStairs` collision → **exit 1**, names both line numbers; restoring → **exit 0** |

It's deliberately heuristic (ES method shorthand at a consistent indent, keyword-filtered) and reports
what it finds with line numbers rather than guessing — a checker that cries wolf gets ignored, which is
worse than no checker.

### A note on the console tool

It returns a **stale buffer across reloads**. After the fix it still showed faults at lines **2273/4298** —
but the edits had shifted those to **2277/4306**, which is how the staleness was *spotted* rather than
trusted. The fix was then proven by calling the crashing function directly and by counting faults
through an interceptor on a fresh page.

## Why this kept happening

This is the fourth distinct shape of the same underlying failure this session:

| version | dead thing | why it looked fine |
|---|---|---|
| V0.19.6 | `coneDegrees` inert for 279/280 spells | field read for exactly one spell id |
| V0.19.9 | authored `'east'`/`'west'` poses unreachable | `0 \|\| 1` made the branch impossible |
| V0.20.0 | `'slam'`/`'crush'`/`'cast'` never drawn | renderer's `else` arm drew *something* |
| **V0.20.3** | **an entire method** unreachable | the duplicate silently won |

Each one was code or data that **claimed** to do something and didn't, with nothing failing loudly
enough to notice. Three of the four now have a checker.
