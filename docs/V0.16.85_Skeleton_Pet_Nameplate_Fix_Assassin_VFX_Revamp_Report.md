## Skeleton pet nameplate ownership
The skeleton pet visual anchor is owned by:

The shared nameplate layout is owned by:

## Root cause of overlap
The V0.16.83 high-detail skeleton model extended the visible skull/helmet silhouette upward, but the skeleton pet still used the older approximate anchor:

```js
actor._nameplateAnchor = { x, y: y - 118 - s.lift * 0.25 };
```

The shared nameplate renderer correctly placed the nameplate stack above the anchor line, but the anchor itself was too low for the enlarged skeleton silhouette. That made the HP/nameplate stack touch or overlap the pet head.

## Corrected anchor calculation
The skeleton renderer now derives its nameplate anchor from the actual skull position calculated by the model render state:

```js
const skeletonTopY = skullY - 28;
actor._nameplateAnchor = { x, y: Math.round(skeletonTopY - 16 - s.lift * 0.15) };
```

This keeps the anchor tied to the real procedural skull/helmet top while preserving the existing shared nameplate stacker and HP bar layout.
