# V0.18.22 — Seamless Dark Woods ground depth-wash

Follow-up to V0.18.21. User loaded V0.18.21 and reported (with a screenshot):

> "you replaced the circles with square tiles, the ground needs to look seamless."

## Root cause
V0.18.21 fixed the *overlapping-circle* depth wash by replacing the per-tile
filled circle with a per-tile `fillRect`. But the terrain is baked in a **square
grid space** (`px = (x - worldMinX) * scale`, one axis-aligned rect per tile) and
that baked chunk is later drawn to **isometric screen space** — so:

1. Each flat tile rect became a hard-edged **diamond** on screen.
2. The depth-darkening alpha is `depthFactor * 0.22`, and `depthFactor` varies
   smoothly per tile — but a *flat* per-tile fill makes that smooth variation
   **step** at every tile boundary. Those hard steps read as a diamond
   checkerboard across the forest floor.

The base terrain doesn't show a grid because it deliberately breaks tile/chunk
boundaries with broad overlapping patches (terrain-renderer.js ~L490-501); the
depth wash had no such smoothing.

## Fix (`render/terrain-renderer.js`)
Compute the wash once per chunk into a **low-resolution buffer** (1 pixel per
tile) and draw it back **bilinearly interpolated**, so the exact same intensity
becomes a continuous gradient instead of hard per-tile steps:

```js
// Precomputed once per chunk, before the per-layer composite loop:
let depthWash = null;
{
  const dx0 = worldMinX - 1, dy0 = worldMinY - 1;          // 1-tile border
  const dbw = (startX + size + padding) - dx0 + 1;
  const dbh = (startY + size + padding) - dy0 + 1;
  if (dbw >= 2 && dbh >= 2) {
    const buf = this._terrainDepthWashBuf ||
      (this._terrainDepthWashBuf = document.createElement('canvas'));
    if (buf.width  < dbw) buf.width  = dbw;
    if (buf.height < dbh) buf.height = dbh;
    const bctx = buf.getContext('2d');
    const img = bctx.createImageData(dbw, dbh);
    const data = img.data;
    let anyDepth = false;
    for (let j = 0; j < dbh; j++) for (let i = 0; i < dbw; i++) {
      const df = this.darkWoodsDepthFactor?.(dx0 + i, dy0 + j) || 0;
      if (df <= 0.02) continue;
      anyDepth = true;
      const o = (j * dbw + i) * 4;
      data[o] = 5; data[o+1] = 8; data[o+2] = 7;            // #050807
      data[o+3] = Math.min(255, (df * 0.22 * 255) | 0);     // same intensity as before
    }
    if (anyDepth) {
      bctx.putImageData(img, 0, 0);
      depthWash = { canvas: buf, sw: dbw, sh: dbh,
        dx: (dx0 - worldMinX) * scale, dy: (dy0 - worldMinY) * scale,
        dw: dbw * scale, dh: dbh * scale };
    }
  }
}

// Inside each elevation layer's composite pass (still under source-atop):
if (depthWash) {
  const prevSmoothing = c.imageSmoothingEnabled;
  c.imageSmoothingEnabled = true;                          // bilinear
  c.globalAlpha = 1;
  c.drawImage(depthWash.canvas, 0, 0, depthWash.sw, depthWash.sh,
              depthWash.dx, depthWash.dy, depthWash.dw, depthWash.dh);
  c.imageSmoothingEnabled = prevSmoothing;
}
```

Key properties:
- **Seamless within a chunk** — bilinear upscale interpolates the per-tile alpha
  into a smooth gradient; no diamond grid.
- **Seamless across chunks** — the buffer samples a **1-tile border** beyond the
  chunk, and `darkWoodsDepthFactor` is a pure world-position function, so a
  chunk's edge values equal its neighbour's → matching seams.
- **Same look, no rings** — a smooth field has neither hard tile steps nor the
  overlapping-circle rings of the pre-V0.18.21 version.
- **Correctly clipped** — drawn under the existing `source-atop` composite, so it
  only darkens each layer's own ground tiles (water/gaps untouched), exactly as
  the old per-tile wash did.
- **Cheap** — one small buffer (`this._terrainDepthWashBuf`, grown as needed and
  reused), built at chunk-bake time only (not per frame).

## Verification
- `node --check render/terrain-renderer.js` — OK.
- Game boots with **0 console errors**.
- Replicated the exact op sequence (`createImageData` → `putImageData` →
  `drawImage` with `imageSmoothingEnabled`) against a synthetic `depthFactor`
  ramp: ran with **0 throws**; the upscaled output scanline steps by at most
  **1/255** between adjacent pixels, versus **~6/255 in a single pixel** at each
  tile edge for the old flat fill (the visible grid) — i.e. the same intensity
  change is now spread across the whole tile width. Per-tile intensity preserved.
- Full in-world visual is the user's sign-off (this WebGL/canvas page can't be
  screenshotted by the preview tooling, and no saved character exists to auto-load).
