# V0.20.11 ‚Äî A Map Of What Nobody Reads

## Why

This session found **six** dead fields **by accident**, one at a time:

| version | authored | read by |
|---|---|---|
| V0.19.6 | `coneDegrees` on a spell calling itself a cone | one hardcoded spell id |
| V0.19.9 | the `'east'`/`'west'` poses | nothing ‚Äî `0 \|\| 1` made the branch unreachable |
| V0.20.0 | `'slam'`, `'crush'`, `'cast'` styles | nothing ‚Äî the `else` drew a slash |
| V0.20.6 | `portrait` on **all 33 NPCs** | **nothing at all** |
| V0.20.10 | `interactLabel` on 4 objects | nothing ‚Äî a hardcoded twin, a missing panel |
| V0.20.9 | `findLootableCorpseNear` ‚Äî **mine** | nothing ‚Äî the method never existed |

Each was somebody's care that never reached a player. **Tripping over them is not a strategy.**

## The tool

`tools/check-dead-fields.js` ‚Äî collects every field name authored in `data/*.js` and cross-checks for
**any** read outside `data/`: `.field`, `['field']`, destructuring, or the name used as a lookup string.

```
node tools/check-dead-fields.js          # fields authored 3+ times
node tools/check-dead-fields.js --all    # every candidate
```

**29 candidates** at the 3+ threshold.

## Confirmed findings

Both verified by targeted grep across `systems/ render/ core/ entities/ game.js`:

### `requiresHeavyWeapon` ‚Äî 13 spells, 0 reads

**Thirteen abilities declare they need a heavy weapon.** The only heavy-weapon logic in the entire
codebase is `inventory-system.js`'s equip check (`twoHanded || hands === 2 || /greatsword|greataxe‚Ä¶/`),
which is about *equipping*, not casting.

**Every one of them can be cast holding a dagger.** This is the `coneDegrees` pattern exactly: an
authored gameplay gate that never gates. Implementing it is a **balance change** ‚Äî 13 spells become
conditional ‚Äî and deserves its own version and its own testing, which is why it is reported here rather
than done in passing.

### `classMechanic` ‚Äî 40 spells, 0 reads

`"classMechanic": "primal_schools"` and friends, authored on 40 spells, read by nothing. Needs a design
call: what was it meant to drive?

## Triaged ‚Äî dead, but *not* a broken feature

The tiles' **`footstepSound` / `footstepEffect` / `footstepRateModifier` / `terrainName` /
`decalEffectOptional`** are all unread ‚Äî but footsteps **work**.

`systems/terrain-footstep-system.js` reads **`footstepProfile`**, which all 11 tiles *also* author, and
which carries the same information more completely (`particleEffect`, `terrainName`,
`stepIntervalModifier`, colours). Those five are **superseded legacy duplicates**.

**The honest fix is retirement, not implementation** ‚Äî the V0.18.96 `poisonResist` decision, on the
retire branch. Knowing which branch a finding belongs on is the whole point of looking.

## Why it's advisory, not a build failure

It's a heuristic, and **a checker that cries wolf gets ignored** ‚Äî which would be worse than no checker:

- **dynamic access** (`obj[key]` in a loop) reads a field without ever naming it
- a field may exist purely to **round-trip through save/load**
- a name can be **written** in code (a default profile literal) with no data read at all

## Both tools were wrong, in opposite directions

This is the part worth keeping.

**The tool was wrong first:** it missed **quoted** keys ‚Äî `"classMechanic":` is not `classMechanic:` ‚Äî
so an early triage said the field was authored 0 times.

**Then my triage grep was wrong the other way:** it counted `terrainName: 'Grass'` ‚Äî the game *defining*
a default profile in code ‚Äî as a **read** of tile data. It reported 10 confident reads that weren't
reads at all.

Two tools, two confident answers, opposite directions, both wrong. **The two confirmed findings above
are the two I then checked by hand.** The tool's own output says it reports *candidates, not verdicts*,
because that is honestly what it is.

Capitalised names are skipped: `loot-tables.js` is a map keyed `'Woods'`/`'Rare'`/`'Boss'`, and
reporting catalogue keys as dead fields is noise (34 ‚Üí 29).
