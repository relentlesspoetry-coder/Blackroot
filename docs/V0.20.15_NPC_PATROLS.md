# V0.20.15 — Two People Who Asked To Walk (Roadmap Item 16)

Item 16's goal: *"make camps feel inhabited rather than static."* **The data had already asked for it.**

## The find

Every one of the 33 NPCs authors:

```js
patrol: { mode: 'short_loop', radius: 3, speed: 0.8 }
```

**Nothing has ever read it for behaviour.** The two apparent reads are both false:

| apparent read | reality |
|---|---|
| `editor-system.js:1758` | merely **copies** the field |
| `dungeon-system.js:2656` | reads `encounter.patrol` — **a different object** |

**This is why the dead-field scanner never flagged it.** The scan is **name-based, not type-aware**: a
field name read on *any* object counts as read everywhere. The same false negative as `maxRank` in
V0.20.14 (read on talent nodes, never on professions). **The candidate list is a floor, not a ceiling.**

## Two kinds at once — that's new

| mode | count | verdict |
|---|---|---|
| `stationary` | **31** | **EMPTY** — describes exactly what they already do |
| `short_loop` | **2** | **BROKEN** — claims a walk, has never taken a step |

Only the two that asked get movement:

- **Mossfang Scout** — lore, 0 quests, radius 3, speed 0.8
- **Road Warden Cael** — quest_giver, **6 quests**, radius 2, speed 0.65

## Item 16's safety rules — each honoured, each verified

| rule | how | verified |
|---|---|---|
| *critical quest NPCs must never become inaccessible* | Cael's authored radius is 2 — he orbits his own post and **cannot leave it** | **0 strayed**; still found by `findNearbyNpc` from his home tile, all 6 quests intact |
| *NPCs must not overlap* | a step onto an occupied tile is **refused**, not forced | **0 overlaps** |
| *must respect collision* | `isWalkable` gates every step | — |
| *must not block portals, exits, narrow paths* | bounded to the authored radius around **HOME**, captured once so the loop can't drift off-post each lap | 0 strayed beyond radius |
| *dialogue should pause movement safely* | the active NPC holds still while their window is open | Cael moved **0.0000** across 120 ticks of conversation, resumed after |
| *must recover if an NPC becomes stuck* | **there is no stuck state to recover from** — the angle is *derived*, so a refused step is skipped and the next angle tried | — |
| *deterministic and save-compatible* | a pure function of the clock + a hash of the NPC's own id (so two patrollers never march in step). **No patrol state is stored** | `noStoredPatrolState: true` — nothing to save, migrate, or desync; a reload resumes mid-stride |

## A first attempt moved nobody — V0.20.4's lesson, repeating

`patrol` is authored on the **draft**. `allNpcs()` returns raw placed **nodes** — only
`interactWithNpc` merges the draft in. **I read the runtime copy instead of the descriptor**, exactly as
the quest journal did in V0.20.4, where the fix was the same sentence: *read the descriptor, not the
runtime copy.*

I'd written that lesson down and didn't recognise it wearing different clothes. Now resolved from
`DR.NPC_DRAFT_BY_ID` and cached per node so 28 NPCs don't re-resolve a catalogue lookup every frame.

## Item 16 status

**Delivered:** patrol routes (the two authored) · role-appropriate positioning (V0.20.8 put the
supplies, herbs and practice gear beside the right people) · collision · no overlap · dialogue pause ·
quest-NPC accessibility · deterministic and save-compatible.

**Still open — and all of it needs authoring or animation, not plumbing:**
- **Work / rest / meal periods, guard shifts, night behaviour** — need a *time-of-day schedule* field
  on NPCs. `getWorldLightState()` and `setWorldTime` already exist, so the clock is there; the
  schedules are not.
- **Sleeping locations, campfire gatherings** — need authored destinations. The camp now *has* bedrolls
  and a fire to gather at (V0.20.8), so the targets exist; nothing says who goes where.
- **Ambient animations** — idle, eating, sitting, warming hands, work animations. These are animation
  work, the same class as Item 17's "sit on benches".
- **Weather / emergency reactions, quest-state positioning** — need design.
- **The other 31 NPCs** don't need patrols. `stationary` is a real answer for a shopkeeper behind a
  stall. What they need is *schedules*, which is a different field.
