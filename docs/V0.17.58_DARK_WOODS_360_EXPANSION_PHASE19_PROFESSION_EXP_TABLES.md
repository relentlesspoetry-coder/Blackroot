## Implementation notes (root cause / decisions from the Phase 18 audit)

### 1. EXP table (Gathering / Mining / Fishing share one curve)
`resource-gathering-system.js` owned a single power curve `max(50, floor(100┬Ělevel^1.45))` used by
all three skills. Replaced it with the plan's exact per-level table:

```
PROFESSION_XP_TABLE = [100, 150, 225, 325, 475, 675, 950, 1300, 1750]  // index 0 = level 1Ôćĺ2
```

Cumulative totals to reach levels 2..10 are now **exactly** the plan's "Total EXP Required" column
(100 / 250 / 475 / 800 / 1275 / 1950 / 2900 / 4200 / 5950) ÔÇö verified in Node.

**Tier ceiling handling:** the plan calls level 10 the "Max for Dark Woods tier." Rather than a hard
cap (which would return `Infinity`/`null` and break the four EXP-display consumers, and block future
higher-tier content), `nextLevelXpFor` continues at the final finite step (1750) for level Ôëą 10.
This keeps every display finite and lets a future tier extend the table later. The plan's intended
ceiling is still honored in spirit: Dark Woods resources cap the player around level 10, and 5,950
total XP reaches it.

**Sharing:** the curve + first-discovery helper are exposed once as `DR.Professions`
(`{ XP_TABLE, maxLevel, nextLevelXp, firstDiscoveryBonusForLevel }`) so the separate fishing system
reuses the exact same math instead of duplicating a table. Load order note: `fishing-system.js`
loads before `resource-gathering-system.js`, so fishing reads `DR.Professions` at call time (runtime)
with a fallback ÔÇö verified working.

### 2. Dual Fishing level reconciled (single source of truth)
The audit flagged that `fishing-system.js` tracked Fishing level in **two** places with **different
curves** (its own `state` on `max(65,80┬Ělevel^1.32)` AND the resource-gathering `Fishing` skill on
the shared curve) ÔÇö they diverged. Fix: the resource-gathering **`Fishing` skill is now the single
source of truth**.
- `fishing-system.gainXp` grants to `resourceGatheringSystem.gainSkillXp('Fishing', ÔÇŽ)` (authoritative
  leveling + the single level-up log), then **mirrors** the resulting level/xp back into its own
  `state` via new `syncLevelFromResource()`.
- `fishing-system.refreshPanel` re-syncs each frame, so the fishing panel, professions panel, and
  skills panel all show one Fishing level.
- `nextLevelXp()` now uses the shared table too.
- The old self-incrementing loop + duplicate "Fishing level N" log remain only as a fallback for the
  case where the resource system is unavailable.

### 3. First-time discovery bonus (+15/30/50/75 by tier)
New persisted `state.discovered` map in `resource-gathering-system.js` (added to `defaultState`/
`normalizeState`, so it serializes via the existing `resourceRuntime` save + localStorage ÔÇö no new
save system). On the first successful gather of each resource id (`completeHarvest`) and the first
catch of each fish species (`fishing-system.resolveCatch`, using the same shared `discovered` map),
a tier bonus is granted once: L1ÔÇô2 +15, L3ÔÇô5 +30, L6ÔÇô7 +50, L8+ +75
(`DR.Professions.firstDiscoveryBonusForLevel`). Short inline feedback is appended to the existing
status line (fuller UI feedback is Phase 25's scope).

### 4. Rare-node bonus (data-driven, ready for Phases 20ÔÇô23)
Added a data-driven `rareNodeXpBonus` field read from the node/def (gathering) and the fish table
entry (fishing). Zero for everything today; Phases 20ÔÇô23 will set +20/+30 for Blackroot Herb /
Queen's Silkroot / Gloamroot Catfish / Broodpool Angler per the plan. No hard-coded ID list.

### 5. Left untouched (already correct / not in scope)
- **Hard level-gate + "requires" message**: already exists in `startHarvestNode`
  (`"<node> requires <skill> <level>."`) and matches the plan's primary Required-Level Rule. Phase 25
  will align the exact wording ("Requires Gathering Level X").
- **Soft-failure**: already exists (success chance scaling with skill advantage). The plan's failure
  table is explicitly optional; not rebuilt (no duplicate failure system).
- **Mining**: incidentally adopts the same shared tier-1 table (the three skills always shared one
  curve). Harmless and more consistent; Mining is not a Dark Woods resource.
