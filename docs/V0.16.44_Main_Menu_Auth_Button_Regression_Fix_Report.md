# V0.16.44 - Main Menu Auth Button Regression Fix Report

## Summary

Fixed the main-menu account modal regression reported after V0.16.43 where Login, Cancel, and Load Game JSON could become unresponsive. The fix stays inside the existing UI ownership path and preserves the main menu, account, and save-picker routing.

## Root cause found

The V0.16.43 Character Creation full-layout conversion left `renderCreatorSummary()` referencing appearance-label locals (`faceLabel`, `hairLabel`, `eyeLabel`, and `skinLabel`) that were no longer defined in that function. During `initUI()`, the top-level main-menu delegate was bound first, then Character Creation summary rendering ran and threw before the later account-modal button bindings and save-picker bindings could be installed. That left the menu partially interactive: the Login modal could appear through delegated main-menu action routing, but the modal's own Login/Cancel/Load Game JSON controls were not reliably bound.
