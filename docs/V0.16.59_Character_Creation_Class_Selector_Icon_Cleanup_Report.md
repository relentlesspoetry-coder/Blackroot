# V0.16.59 - Character Creation Class Selector Icon Cleanup Report

## Summary

This phase updates only the Character Creation class selector visuals and hitbox footprint. The selector now presents each class as a large standalone class emblem with the class name and role label directly underneath. Main menu, Character Select, character stats, in-game HUD/action bar, combat, bags, inventory, spells, and gameplay systems were not changed.

## Class Selector Visual Changes

- Replaced the visible boxed class-option presentation with transparent standalone icon/name/role options.
- Removed the visible rectangular card border and card fill from `.classIconOption`.
- Removed the circular badge/ring element from the active renderer markup.
- Kept the current two-column alphabetical class layout and scroll behavior.

## Icon Size Changes

- Increased `.cc-class-gallery-emblem` to use the former circular badge footprint.
- Increased the icon hit footprint through `.cc-class-icon-halo` while keeping the visual presentation icon-first.
- Adjusted text spacing below the larger icon to prevent overlap between class options.

## Removed Circle/Card Rendering Details

- Removed the active `.cc-class-icon-ring` DOM element from the class option renderer.
- Removed visible card frame styling from `.classIconOption` by clearing border, background, radius, and card shadow.
- Selected state no longer uses the old square card highlight or circular ring; it now uses icon glow plus a small diamond marker under the emblem.

## Hitbox / Selection Behavior Notes

- The active class option remains the real `button[data-class-option]` renderer, but its footprint is reduced to the icon/name/role content rather than a large visible card.
- Clicking the class icon area selects the correct class through the existing `selectClass()` handler.
- Double-click creation behavior remains wired through the same existing button handler.
- Class ID mapping still uses `button.dataset.classOption = name`, preserving class creation behavior.
