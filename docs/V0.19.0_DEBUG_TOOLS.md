# V0.19.0 — Developer Debug & Content Inspection Tools (Roadmap Item 23)

Roadmap Item 23 asks for development-only tools so the expanded game can be tested without slow manual
setup. This ships them — and completes the foundation track.

## The toolset — `DarkWoodsGame.debug.*` (`systems/debug-tools-system.js`)

| group | commands |
|---|---|
| mode | `on()` · `off()` · `enabled` · `help()` |
| items | `grantItem(id, qty)` · `removeItem(idOrIndex)` · `money(silver)` |
| character | `applyBuff(idOrEffect, secs)` · `removeBuff(id)` · `setSkill(name, level)` · `statBreakdown()` |
| world | `setTime(hour)` · `setWeather(kind)` |
| inspection | `lootTable(id)` · `lootTables()` · `item(id)` |
| validation | `validate()` · `economy()` · `selfTest()` |

## Safety — every roadmap requirement, met and verified

| requirement | how |
|---|---|
| **Development-only** | Debug mode is **off by default**; every command refuses with a warning until explicitly enabled (`debug.on()`, `?debug=1`, or `DR.CONFIG.DEBUG`). |
| **Not accessible in player builds** | Attaching is harmless — the commands are gated, not merely hidden. |
| **Debug state visibly indicated** | A fixed **DEBUG** badge appears on `on()` and is removed on `off()`. |
| **Debug actions logged** | A single `run()` funnel gates, logs (console **and** in-game log), and `try/catch`es every command. |
| **Use authoritative systems** | `grantEditorItem`, `applyStatusEffect`/`removeStatusEffect`, `resourceGatheringSystem.skill()`, `setWorldTime`, `setWeather`, `runValidationSuite`/`auditEconomy`/`getPlayerStatBreakdown` — no poking unrelated state. |
| **Avoid impossible save states** | Unknown item ids and unknown loot tables are rejected. |

### A subtlety worth noting

`resourceGatheringSystem.skill(name)` **silently falls back to `'Gathering'`** for an unknown name. So
`setSkill` validates the name against `state.skills` *first* — otherwise a typo would quietly set the
wrong skill, exactly the kind of impossible-state the roadmap warns about.

### Deliberately not stubbed

Tools whose systems don't exist yet — **mounts/taming** (Item 7), **spell-VFX trigger** (Item 6),
**NPC schedules** (Item 16) — are omitted rather than faked. They land with their systems.

## Wiring

- `systems/debug-tools-system.js` (new) — `DR.DebugToolsSystem` with the standard `install(Game)` pattern.
- Added to `_script_order.json` and the HTML script tags (before `game.js`).
- `DR.DebugToolsSystem.install(Game)` sits with the other installers in `game.js`.
- `Game.prototype.installDebugTools()` runs once at boot, wrapped so it can never break boot.

## Verification (in-browser)

- Attached with `enabled: false` by default; a command while disabled returns `false` and **no badge exists**.
- After `on()`: badge appears, commands run. After `off()`: badge removed, commands re-gated.
- `lootTable()` resolves **46 tables** with item names + chances; `lootTables()` lists ids.
- `item()` returns the draft + compiled instance and the sell-back in real currency (`60C 26S 0G 0P`).
- `validate()` and `economy()` both return ok; `help()` prints 24 lines.
- Unknown item id → rejected; unknown loot table → `null`.
- `node --check` clean; JSON clean; **0 console errors**; validation suite **PASS**.
