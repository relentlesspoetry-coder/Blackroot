# V0.18.53 — Silk Web Caverns: squish crash fix + more webs & spiders

Your screenshot showed the world blacked out with `CanvasRenderingContext2D.ellipse: Negative radius` / "render recovery failed" after stepping on a ground spider, plus: more ground spiders, and web columns in every room and tunnel.

## 1. CRASH FIX — stepping on a ground spider faulted the renderer
**Symptom:** squishing a floor critter threw `CanvasRenderingContext2D.ellipse: Negative radius`, which the render-fault recovery couldn't paper over → black screen.

**Root cause (a clock mismatch):**
- `dungeonSystem.updateSilkCritters` stamps a squished critter's `c.deadAt` with `nowMs()`, and `nowMs = () => Date.now()` — a ~`1.7e12` epoch-millisecond value.
- `game.renderSilkCritters` (`render/object-renderer.js`) computed the squish-splat fade `k = min(1, (now - deadAt)/420)` with `now = performance.now()` — **milliseconds since page load**, a small number.
- So `performance.now() - Date.now()` is hugely **negative** → `k` negative → `(0.7 + k*0.5)` negative → the splat `ctx.ellipse(..., c.size*1.7*(neg), c.size*0.8*(neg), ...)` gets a **negative radius** → throw.

**Fix:** `renderSilkCritters` now uses `Date.now()` (matching `c.deadAt`), and `k` is clamped with `Math.max(0, Math.min(1, …))` as a defensive guard so a negative radius can never occur again. Verified with a render test that reproduces the exact path (a dead critter with `deadAt = Date.now()`) — it no longer throws.

So the passages between rooms are webbed too — no bare tunnel.
