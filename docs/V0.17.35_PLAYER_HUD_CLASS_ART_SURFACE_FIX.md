## Root Cause
The Player HUD was resolving the correct class-background asset, but the final HUD CSS/background stack still owned the visible surface. Its layered dark fallback/overlay rules could visually collapse the Player HUD into a plain black panel even while the Party HUD cards correctly showed class art.

## Fix Details
- Added a concrete `.playerHudClassBackdropImage` element inside `.hudClassBackdrop` and synced its `src` directly from the canonical class background resolver.
- Forced the old `.hudClassBackdrop` CSS background-image stack off for the Player HUD only.
- Added a controlled readability veil over the image instead of using multiple opaque CSS background layers.
- Updated the fast HUD class-visual sync guard so the image surface is rebuilt if missing or stale, even when the dataset key already matches.
- Left Party HUD card rendering unchanged.
