## Ownership audited

### Idle/fighting held weapon path
- File: `render/paperdoll-equipment-renderer.js`
- Owner: `PaperdollEquipmentRenderer.drawBodyLayer(ctx, rig, 'weapon')`
- Responsibility: resolves equipped/default paperdoll weapon/offhand visuals and draws class-appropriate items such as dagger, crossbow, lute, mace, shield, wand, staff, book, focus, skull, totem, bow, and greatsword.

### Auto-attack swing path
- File: `render/humanoid-base-renderer.js`
- Owner: `buildRig()` attack-pose arm displacement and `drawSharedAutoAttackOverlay()` / `drawMeleeAutoAttackOverlay()`
- Responsibility before this pass: displace attack arms and draw a generic attack overlay.

## Root cause

The white-stick bug was attack-state specific. Idle/fighting weapons used the paperdoll/class held-weapon renderer correctly, but the auto-attack overlay still drew its own generic weapon primitive during melee swing rendering.

The stretched-arm bug came from the same attack path treating the animated hand endpoint like the weapon tip. V0.16.87/V0.16.88 increased swing amplitude, which made the main-hand anchor travel too far from the shoulder/elbow. Because the paperdoll weapon attaches to that hand, and because the generic overlay also drew a long line from that point, the result looked like a long white stick attached to an unnaturally stretched arm.

Ranged weapon classes were also affected because the shared attack-pose displacement and melee overlay could still run for attack-state actors where the correct bow/crossbow presentation should have remained in the class/paperdoll weapon owner.

## Changes made

### `render/humanoid-base-renderer.js`
- Added `autoAttackRenderClassId()` to preserve Assassin and Ranger identity instead of collapsing Assassin into the Rogue meditation style for attack overlay decisions.
- Added `isRangedAutoAttackPose()` so ranged weapon classes skip the melee arm displacement and melee overlay.
- Added arm segment clamping helpers:
  - `limitArmSegment()`
  - `clampAttackArmProportions()`
- Reduced attack-pose hand displacement so the hand remains a grip anchor, not the weapon tip.
- Preserved the full wind-up pose shape while keeping elbow/hand within stylized anatomical limits.
- Converted `drawMeleeAutoAttackOverlay()` into a trail/impact-only overlay.
- Removed live normal-gameplay drawing of the generic attack weapon/rod primitive from the overlay path.
- Kept lightweight slash arcs and impact glints as cosmetic cues only.

## Placeholder/debug draw handling

No bottom-of-file patch or duplicate renderer was added. The old live overlay weapon primitive was not kept in the normal render path. The paperdoll/class held-weapon renderer remains the source of actual weapon visuals.

## Entity types affected

The fix applies anywhere the shared humanoid renderer and shared auto-attack overlay are used:

- player characters
- bots using the class/humanoid renderer
- mercenaries using the class/humanoid renderer
- party companions using the class/humanoid renderer
- class-preview/world models that share the same renderer path

Monster and pet renderers were not changed.

## Classes covered

The fix is class-agnostic, but the corrected routing explicitly preserves:
