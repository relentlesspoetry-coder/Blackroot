# V0.18.3 — World/UI fixes (mob spread, map fog, map UI, mob swim)

Four fixes from user screenshots.

## 1. Mob distribution thinned out far from camp (`systems/mob-spawn-system.js`)

The Dark Woods map is **360×360** (`activeMapSize()` = 360; the HUD's "200×200" is a stale
`CONFIG.MAP_SIZE` label — a separate cosmetic bug, left as-is). Camp is at **(100,100)** —
the top-left — so the east/south/deep-corner half is a long way out. The tiered generator
capped spawns at **distance 205** from camp and its density ramp **saturated at distance
106** (acceptance maxed at 0.38), so density fell with distance and the far reaches were
nearly empty.

Fixes in `darkWoodsTieredEncounters`:
- distance cap `205 → 300` (cover the whole content radius),
- `progress = clamp((dist-10)/96)` → `clamp((dist-22)/208)` (ramp over the full distance),
- acceptance `0.12 + 0.26·progress` → `0.17 + 0.41·progress` (far reaches denser).

Plus **13 new well-spaced camps** in `DARK_WOODS_MOB_CAMPS` covering the east, south, and
deep SE corner (deeper = tougher families). `DARK_WOODS_POPULATION_VERSION 5 → 6` re-seeds
over existing saves.

**Verified:** total enemies **345 → 414**; mobs beyond distance 120 went **109 → 203**; 79
camp-tagged mobs now spread across the whole map.

## 2. World map "black until discovered" (`render/minimap-renderer.js`)

`buildFogOverlay` unexplored alpha `236 → 255` (was ~92% opaque, so terrain faintly showed
through the world map). To keep the small minimap's softer look, `compositeFogMinimap` now
draws the overlay at `globalAlpha 0.92`. **Verified:** overlay unexplored pixel alpha = 255,
rgb ≈ [3,5,4]; explored = 0.

## 3. Map UI overlap (`systems/ui-system.js` + HTML CSS)

`toggleMap` now toggles `document.body.classList` `worldMapOpen`, and new CSS hides all
gameplay chrome while it's set — `#hud`, `#hotbar`, `#minimapWrap`, `#menuDock`,
`#mercPanel`, `#partyHud`, `#externalSystemsHud`, `.unifiedCompanionHud` — so nothing draws
over the map (its own Close/Center controls live inside `#mapOverlay`). Cleared on close.
**Verified:** those elements compute `display: none` while open; class removed on close.

## 4. Mob swim submersion (`systems/swimming-system.js` + `render/entity-renderer.js`)

Enemies could swim (V0.17.97) but rendered walking on top of the water. Now:
- `updateAuxiliarySwimStates` also applies `applySwimmingRenderState` to nearby (≤44 tiles)
  enemies, giving them the full swim state (`swimming` / `swimSubmerge` / `swimAnim` /
  `swimState`).
- `EntityRenderer.drawEntity` wraps the procedural enemy model draw in a **waterline clip**
  (hides the submerged lower body) and calls the new `drawMobSwimSurface` to draw an
  animated water ellipse behind + a bright surface rim in front.

**Verified:** a mob placed in water gets `swimState 'swimming'`, `swimSubmerge 0.52`,
`isEntitySwimming true`, and `render()` runs with **0 errors**. The exact look is the user's
sign-off — the preview tab was backgrounded, so a screenshot wasn't capturable this pass;
the submerge is a modest ~13px waterline + ripple (won't hide the mob), easy to tune via
the `6 + 13·blend` factor.
