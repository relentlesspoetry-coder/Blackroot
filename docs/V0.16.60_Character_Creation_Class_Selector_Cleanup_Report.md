# V0.16.60 - Character Creation Class Selector Cleanup Report

## Summary

This phase is limited to the Character Creation class selector visuals. The selector now renders as icon/name/role only, with the remaining visible square boxes removed from the active class option styling. Class icons were slightly enlarged and the class name/role stack was moved closer to the icon without touching or overlapping it.

## Class Selector Visual Changes

- Kept the existing real `.classIconOption` selector elements instead of creating a second selector system.
- Suppressed the generic global `button::after` pseudo-frame for `.classIconOption`, which was the remaining source of the visible square boxes.
- Kept class options visually transparent so they read as standalone icon/name/role groups.
- Tightened the gap between icon and label stack.
- Allowed role labels to wrap when needed instead of forcing hard ellipsis on long role text.

## Icon Size Changes

- Increased class icon container size from the V0.16.59 range of roughly `82px-108px` to `96px-126px`.
- Increased emblem image size from roughly `78px-102px` to `94px-122px`.
- Preserved hover and selected-state icon glow/brightness without restoring a circle ring or card border.

## Removed Circle/Card Rendering Details

- No class data or render path was duplicated.
- No circular ring renderer was restored.
- No old square card highlight was restored.
- The remaining square boxes were caused by shared button chrome, not separate class-card markup; the fix was applied directly to `.classIconOption` so the active owner suppresses that chrome itself.

## Hitbox / Selection Behavior Notes

- Existing class selection behavior remains on the active class option button.
- Clicking the icon itself continues to select the class because the icon is contained in the real class option control.
- Label text remains part of the same accessible control; no fake overlay hitbox was added.
- Selected state remains visible through icon glow and gold class-name emphasis.
