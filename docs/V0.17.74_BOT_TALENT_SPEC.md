# V0.17.74 — Bot Talent-Spec Selection

**Type:** Talent System v1 spec, final phase (V0.17.65)
**Previous version:** V0.17.73
**New version:** V0.17.74

The last item in the spec's build order: give the simulated bot population **visible builds**.

---

## What changed

Bot players now pick a talent **spec** once they reach level 5, surfaced as their type label
(e.g. "Judicator Paladin", "Exorcist Cleric", "Duelist Rogue", "Planebinder Summoner").

**Rail 5 compliant — label only.** Bots get no talent points, stat ranks, or effects; talents remain
strictly player-only. `bot.talentSpec` is a display/role identity string that mirrors the player's
`talentSpec` field, so AI role hints and any future faction/dialogue gates can read either uniformly.

### Implementation (`systems/bot-player-system.js`)

- **`botTalentSpecFor(bot)`** — returns `null` below level 5; otherwise picks one of
  `DR.TALENT_TREES[class].specs` via the existing `stableStringIndex` FNV hash of the bot's id. Stable
  per bot, varied across the roster.
- **`Game.prototype.updateBotTalentSpec(bot)`** — sets `bot.talentSpec`, `bot.talentSpecName`, and a
  spec-aware `bot.botTypeLabel` (`"<Spec> <Class>"`); clears them below level 5.
- Called at **`spawnBotPlayer`** (spawn), in **`awardBotXp`** on any level change (so a bot that
  dings 5 gets its spec), and across the roster in **`ensureBotPlayers`** (covers loaded bots).

## Verification (in-browser, port 8899)

All 14 roster bots receive a valid, class-appropriate spec at level 6; deterministic across repeated
calls; `null` under level 5; every assigned spec id belongs to that bot's class; 14 distinct specs
across the roster. No console errors.
