# V0.18.68 — Herbal Potions (Roadmap Item 3 D)

First batch of Dark Woods herbal potions, built directly on the V0.18.67 consumable-buff plumbing and the V0.18.65/66 effect policy.

## Recon
- The **campfire** is a world object (`type: 'fire'`) that maps to `station_campfire` via `stationForObject`, so **cooking is functional** at the camp.
- The **Herbalist Table** (`objectType: 'herbalistTable'`) is **removed** from the camp by `ensureStartingCampRevamp` and never re-placed, so a dedicated alchemy station isn't usable yet.
- Item 3's premise is that **campfires become the functional early-game crafting hub for cooking *and* alchemy** — so these potions are **brewed at the campfire** (craftable now, no world-placement work).

## Added
### `data/items.js` — six functional potions
| Potion | Effect | Category | Cooldown |
|---|---|---|---|
| Minor Healing Draught | instant **+45 HP** | `potion_heal` | 15s (shared potion-heal) |
| Minor Mana Draught | instant **+35 MP** | `potion_mana` | 15s (shared potion-mana) |
| Regeneration Tonic | +10 HP + **HoT** (6 HP / 3s / 30s) | `potion_buff` | 20s |
| Clarity Tonic | +8 MP + **mana-regen** (5 MP / 3s / 30s) | `potion_buff` | 20s |
| Bramblehide Elixir | **+4 defense / +3 armor**, 180s | `potion_buff` | 30s |
| Antivenom Draught | **+20 poison resist**, 180s | `potion_buff` | 30s |

Instant heal/mana route their cooldown to a **shared potion group**; the buff potions apply a categorized `potion_buff`, so the effect policy keeps **only one potion buff active at a time** (a new one replaces the old).

### `data/crafting.js` — six campfire recipes
`profession: cooking`, `stationId: station_campfire`, turning gathered Dark Woods herbs (gloomleaf, wispbloom, lantern moss, thornberry, blackroot, gloomcap mushroom, webcap fungus) into the potions. Level 1–5, tiered.
