## Mechanics Fully Implemented
- Druid: HoT application/refresh through existing status effects, group HoTs, direct and smart heals, cleanse/heal rain, root/slow control, Druid-themed buffs, group sustain buffs, lunar/nature damage and DoTs.
- Enchanter: damage, control statuses, control diminishing returns, mesmerize/charm limits, rune buffs, illusion/decoy buffs, threat blur status, soft-control fields, and short teleport/decoy behavior for Illusory Step.

## Mechanics Partially Supported Due to Current Engine Limits
- Enchanter charm marks and suppresses hostile aggression, but the current enemy AI has no full temporary allied-minion ownership model. A future AI-owner pass should make charmed enemies actively fight for the player.
- Enchanter decoys/runes use existing absorb/charge/status state; the engine does not yet route every incoming attack through a redirect hook.
- Druid healing caps are stored on statuses where the status owner supports them, but the current status tick loop does not yet decrement all pulse/trigger caps generically. The effects expire by duration and status replacement rules today.
- Boss/elite reduced behavior is represented through data fields and conservative status values; a future encounter-rank hook can apply exact per-rank modifiers globally.

## Save Compatibility
- Existing saves continue using the existing spell compiler and default class slot system.
- Druid and Enchanter were added to the canonical revamp refresh set so stale local editor spell data is refreshed from `DR.SPELL_BY_ID`.
- Spell IDs remain generated from class + spell name; unchanged spell names preserve most existing slot compatibility.
