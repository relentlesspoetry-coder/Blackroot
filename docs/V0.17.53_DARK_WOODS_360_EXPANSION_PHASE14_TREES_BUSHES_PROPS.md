# Phase 14 — Dark Woods 360x360 Expansion: Trees, Bushes, Terrain Props

**Phase:** 14 of 28 (Dark Woods Zone Expansion + Resource Identity master plan)
**Previous version:** V0.17.52
**New version:** V0.17.53

---

## 1. Goal

Per the master plan: add tree variety (standard/tall ancient/thin dead/moss-hung/rooted barrier), bush
variety (small/tall/thorn/dead/berry/mossy/flowering/web-covered), and forest floor + larger natural props
(fallen logs, stumps, moss rocks, root clusters, etc.), with only large props blocking movement.

## 2. Audit First

Per Phase 13's own recommendation and this project's established discipline, I audited
`render/object-renderer.js`'s existing tree/bush prop variety before writing anything. Findings:

- **Tree**: `drawTree` already has real procedural variety — 9 shapes (`variant % 9`), broad/tall/sparse
  crown silhouettes, 5 hue palettes, and an `obj.mossy` overlay flag (added Phase 8, for moss-hung trees).
  No ancient/large variant existed.
- **Dead tree**: `drawDeadTree` already exists with a `variant` (color) and `obj.webbed` overlay (added
  Phase 6). It had no size (`scale`) parameter — always one fixed size.
- **Bushes**: `thornBush` and `webbedBush` (both Phase 6) already cover thorn/web-covered. Generic `brush`
  (small bush clump) existed but had no variant support at all — no dead/berry/mossy/flowering options.
- **Rock**: `drawRock` took no `obj` parameter at all (the call site didn't even pass it) — no moss variant.
- **Stump**: `drawAshStump` exists but is deliberately burnt/ash-palette (used at Bandit's Fall and the
  Gloomheart Tree). No natural forest-stump palette existed.
- **Ground props**: no fallen-log or root-cluster prop existed. `rootArch` (Phase 8) is a walk-through arch
  and `hangingRoots` (Phase 8) is a vertical cave-wall feature — neither is a flat forest-floor prop. No
  "rooted barrier" (blocking root obstacle) existed anywhere.

Consistent with the project's "reuse over invention" convention, everything below extends the existing
functions with new `obj` flags/params rather than duplicating geometry, except for the three genuinely
missing prop types (fallen log, root cluster, root barrier).

## 4. Implementation Notes

- **Ancient trees** reuse `drawTree`'s existing trunk-palette variables to draw one extra gnarled burl/knot
  partway up the trunk, and raise the scale cap from 1.34 to 1.85 (default 1.5 when `ancient` and no explicit
  `scale` given). No new function.
- **Dead tree scale** wraps the function body in a `translate(s.x,s.y) → scale → translate(-s.x,-s.y)` around
  the anchor point, since the existing geometry is written entirely in absolute `s.x`/`s.y` terms rather than
  relative coordinates — the smallest possible diff to add size variety without touching every point.
- **Bush variety** is a single `drawBrush(s, obj)` with `obj.kind` selecting palette (`dead`: browns/greys,
  `mossy`: darker greens, default: existing green) plus optional berry dots / flower dots / moss speckle
  overlay drawn on top of the same clump silhouette — one function instead of four near-duplicate ones.
- **Root barrier** ("rooted barrier" in the plan) is the one genuinely new large prop: a blocking tangle of
  surface roots, visually related to (but distinct from) the existing `rootCluster`/`rootArch`/`hangingRoots`
  — reuses the same gnarled-line drawing style at a larger scale.
- **World-gen wiring**: the sparse forest dressing pass (`placeWorldLandmarks`, the only whole-zone tree/bush
  scatter pass — distinct from the region-scoped dressing passes built in Phases 6 and 8) now pulls
  `darkWoodsDepthFactor(x,y)` (Phase 13's region-derived 0→1 gradient) and a second noise channel to bias
  variety by depth: ancient/dead/mossy trees, dead/mossy bushes, natural stumps, and root barriers all scale
  up with depth; berry/flowering bushes scale up in shallow (safe) regions instead. The existing density
  thresholds (which `r` value places an object at all) were left untouched — only the *sub-variety* selection
  is new, via the independent `r2` channel, to avoid disturbing already-tuned density balance.
- Existing region-scoped dressing (`ensureSilkWebApproachDressing`, `ensureHiddenTreeCaveConcealment`) was
  deliberately left untouched — those own their own thorn/webbed/mossy flavor already and are out of this
  phase's scope.

## 5. A Real Bug Found and Fixed

Functional verification (Section 6) caught a real logic bug before it shipped: the tree branch originally
checked the dead-tree condition (`depth > 0.55 && r2 > 0.93`) *before* the ancient-tree condition
(`depth > 0.6 && r2 > 0.985`). Since the ancient range (`r2 > 0.985`) is a strict subset of the dead-tree
range (`r2 > 0.93`), and depth > 0.6 implies depth > 0.55, every coordinate that could ever satisfy the
ancient condition was already caught by the dead-tree check first — **`tree(ancient)` was unreachable dead
code**. Fixed by reordering the checks so the narrower, rarer condition is evaluated first (same fix pattern
as Phase 9's overlap bug and Phase 12's stray `#` — caught by execution, not by reading the code).

## 6. Validation

- `find . -name "*.js" -print0 | xargs -0 -n1 node --check` — clean, all files.
- All `*.json` files parse cleanly via `JSON.parse`.
- **Functional simulation, not just code review**: reimplemented the exact region lookup, depth-factor
  formula, and the new selection decision tree standalone in Node, using the real `seededNoise` function and
  the real region box/ring definitions from `data/default-zones.js`, then swept the full 360x360 coordinate
  space (step 3) bucketed by region. Confirmed by execution:
  - `dead_lantern_outskirts` (the starter-safe region, depthFactor 0): **zero** dead trees, ancient trees,
    mossy trees/rocks, natural stumps, or root barriers — the starter area stays clean, matching the rule
    established in Phase 13.
  - `gloamroot_depths` (depthFactor 1.0, the deepest region): all new variety present, including root
    barriers (rare, as designed) — confirmed only after the Section 5 fix; before the fix, `tree(ancient)`
    had a zero count everywhere in the sweep.
  - Berry/flowering bush kinds only appeared in shallow-depth buckets; dead/mossy bush kinds only in
    deep-depth buckets, as designed.

**Not validated (no browser available, same limitation as Phases 2–13):** actual visual appearance in-game,
and no frame-rate measurement was possible. The "no performance regression" claim rests on the same design
argument as Phase 13 (a few extra `obj` flag checks and conditional draws inside an already-sparse,
already-gated scatter pass) rather than a measurement.

## 8. Known Limitations

- Tall bush is implemented as a `scale` param on the existing `brush`, not a visually distinct silhouette —
  it's larger, not structurally different.
- "Small bush" from the plan's list is just the existing default `brush` — no separate smaller variant was
  added, since the existing clump is already the smallest bush in the game.
- Root barrier placement is coordinate-random (noise-gated), not hand-placed for interesting chokepoint
  effect — it may occasionally land somewhere its blocking behavior feels arbitrary rather than deliberate.
  A future pass could hand-place a few for effect instead of relying purely on procedural scatter.
- No new prop types were added to the level editor's placement palette (consistent with Phase 6 precedent).
