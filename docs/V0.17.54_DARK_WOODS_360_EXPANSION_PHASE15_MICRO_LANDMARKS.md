## 1. Goal

Per the master plan: add 4 authored micro-landmarks (Abandoned Hunter Camp, Old Forest Well, Collapsed Watch
Post, Root Shrine) and 4 authored environmental-storytelling scenes (Bandit Ambush Remains, Failed Spider
Hunt, Lost Pilgrim Shrine, Old Road Collapse). Validation criteria: scenes are readable; scenes do not block
roads; micro-landmarks do not crowd major landmarks; no new cave/major landmark bloat.

Checked `render/object-renderer.js`'s existing prop set (the same switch-case list audited in Phase 14) for

items: `fire` (firepit), `bedroll` (already takes `obj.color`), `brokenCart`, `crate`, `sackPile`,

Also checked `systems/mob-spawn-system.js`'s `DR.DEFAULT_WORLD`-adjacent spawn-anchor list (~70
`{zone:'dark_woods', x, y, spawnId}` entries left over from the original, denser pre-Phase-2 map) and the
existing hand-placed landmark coordinates from Phases 1-14. This zone is spatially crowded in several
quadrants; two of my first-draft scene coordinates landed almost exactly on top of existing mob spawn
anchors (see Section 5) before a standalone distance sweep caught it.

## 5. Implementation Notes

- **Scene structure**: each of the 8 scenes is a small hand-authored cluster (3-6 props, 4-6 tile radius),
  built with a local `scene(cx, cy, radius, props)` helper inside `ensureDarkWoodsMicroLandmarks` that first
  clears the footprint via the existing `flattenArea` (same helper used for the Gloomheart Tree/Bandit's Fall
  ground prep in Phase 9) and then places each prop at an explicit `(dx, dy)` offset from the scene center.
  This mirrors the existing landmark-building style rather than the sparse-noise-scatter style used for
  Phase 6/13/14's zone-wide dressing - these are one-off authored scenes, not a repeatable density pass.
- **Coordinate selection, verified by execution, not eyeballed**: wrote a standalone Node script that parsed
  the real `{zone:'dark_woods', x, y}` mob-spawn-anchor list out of `systems/mob-spawn-system.js` (~70
  entries) plus the known landmark coordinates from Phases 1-14, then computed each candidate scene's
  distance to every anchor/landmark. The first-draft coordinates for "Bandit Ambush Remains" and "Lost
  Pilgrim Shrine" came back at 2.2 tiles from the nearest mob spawn anchor - functionally on top of it. Both
  were replaced with a small brute-force search (grid-scan maximizing minimum distance within the correct
  named region) before implementation. Final placement: all 8 scenes sit ≥10.8 tiles from every mob spawn
  anchor and every prior landmark, and ≥39 tiles from each other.
- **Reused-prop styling choices**: "small stone marker" (Lost Pilgrim Shrine) and "stone idol" (Root Shrine)
  both reuse `standingStone` (a full megalith asset from Phase 10) rather than a new smaller marker prop -
  visually identical asset to the Stone Hedge Ruins ring stones, but placed 100+ tiles from that landmark in
  each case, so it reads as "a similar-style solitary stone," not a duplicate of the ruins. Documented as a
  known limitation (Section 8) rather than building a dedicated smaller marker.
- **Deliberately out of scope, per the plan's own optional/hook language**: the Old Forest Well's "optional
  night wisp appearance" and "possible quest object location," and the Root Shrine's "rare herb spawn" and
  "druid/nature quest hook" all depend on systems that don't exist yet in this plan (day/night state -
  Phase 17; resource nodes - Phase 20; quest hooks - no dedicated phase, out of this plan's explicit scope).
  Only the visual landmark and (for Root Shrine) a documented deferred spirit/wisp encounter slot were built.
  I did **not** spawn an actual wisp mob at the Root Shrine, since Phase 11's wisp spawn-draft system is
  keyed to the Stone Hedge waypoint specifically and adding a second wisp spawn point was judged out of this
  phase's "scenes, not new mob camps" scope - noted as a gap in Section 8, not silently dropped.

## 6. Validation

- `find . -name "*.js" -print0 | xargs -0 -n1 node --check` — clean, all files.
- `find . -name "*.json" -print0 | xargs -0 -n1 python3 -m json.tool > /dev/null` — clean, all files.
- **Functional simulation, not just code review**: re-implemented the exact `flattenArea`/`placeObject`
  calls from `ensureDarkWoodsMicroLandmarks` standalone in Node against a full 360x360 mock map/objects
  grid (same `map[y][x] = {type, elev, blocked}` / `objects[y][x]` shape as the real `Game` instance), with
  `placeObject` throwing on out-of-bounds writes or same-tile collisions. Result: all 36 props across the 8
  scenes placed with **zero exceptions and zero same-tile collisions**, and all 36 objects landed within the
  4..356 safe-interior bounds used by the existing sparse-dressing loop (none in the boundary-wall strip).
- **Coordinate-safety simulation** (Section 5): confirmed by execution, not assertion, that every scene sits
  ≥10.8 tiles from every real mob spawn anchor in `systems/mob-spawn-system.js` and every prior landmark.

**Not validated (no browser available, same limitation as Phases 2–14):** actual visual appearance in-game,
console-error-free boot, and no frame-rate impact (8 small one-time prop clusters added to an already-large
object grid; expected negligible, not measured).

## 8. Known Limitations

- "Small stone marker" (Lost Pilgrim Shrine) and "stone idol" (Root Shrine) reuse the full `standingStone`
  asset rather than a dedicated smaller marker prop - see Section 5.
- Old Forest Well: no night-only wisp appearance, no quest-object wiring (both explicitly optional/possible
  in the plan; the systems they'd depend on - day/night, quests - don't exist yet in this project).
- Root Shrine: no rare herb spawn (depends on Phase 20's resource system, not built yet), no druid/nature
  quest hook, and no actual spirit/wisp mob spawned (Phase 11's wisp spawn-draft system is scoped to the
  Stone Hedge waypoint; adding a second wisp spawn point was judged out of this phase's scope) - the visual
  landmark is built, the encounter is not.
- "Optional journal note" for the Abandoned Hunter Camp was skipped (explicitly marked optional in the plan;
  no lightweight lore-object/note system exists to hang it on without building new UI).
- Lost Pilgrim Shrine's "faint wisp glow" is represented by a static `magicResidue` prop, not a new animated
  aura - building a second full VFX aura (beyond the one Phase 12 built for the Stone Hedge waypoint) was
  judged out of this phase's surgical scope.
