# Dark Woods 360x360 Expansion — Phase 17: Atmosphere / Time-of-Day Identity

**Phase name:** Atmosphere / Time-of-Day Identity (Phase 17 of 28)
**Previous version:** V0.17.55
**New version:** V0.17.56

---

## Goal

Strengthen the haunted-forest mood of Dark Woods using lightweight, time-of-day-driven
atmosphere, per the master plan:

- Morning fog in low areas
- Fireflies at night
- Stronger waypoint glow at night
- Deeper shadows under ancient trees
- Wisps more visible at night
- Faint lantern glow along roads

Hard constraint from the plan: **do not build a heavy weather framework if one does not
already exist.** One already exists, so this phase adds **no** new weather system, clock,
render loop, or timer — it only hooks the existing world-light state into systems that were
already drawing live every frame.

---

## Root cause / implementation notes

### Owning systems (audited before writing, per project convention)

- **`systems/time-cycle-system.js`** owns the in-game clock and `getWorldLightState()`, which
  returns the authoritative per-frame light object (`phaseKey`, `nightStrength`, `moonStrength`,
  `fogAlpha`, `fireflyBoost`, …), already merged with weather via `applyWeatherToLight`. This is
  the single source of day/night truth. **Not modified** — only read from.
- **`systems/atmosphere-system.js`** already renders Dark Woods fog pockets, fireflies, and
  ambient creatures live each frame, and already reads the light state.
- Waypoint aura, lanterns, and trees are all drawn **live** (they animate off `performance.now()`),
  so they are uncached and safe to enhance in place.
- **Mob bodies are sprite-cached** (`entity-renderer.js drawCachedModelOrFallback`), so a
  night-reactive glow could not be baked into the wisp's procedural model — it had to be a live,
  uncached overlay drawn around the cached sprite. This is the key design decision of the phase.

### The six items

1. **Morning fog in low areas** — `systems/atmosphere-system.js` `renderDarkWoodsAtmosphere`.
   Added a `morningFog` factor (1 at `phaseKey==='dawn'`, 0.7 at `'morning'`, 0 otherwise) and a
   per-tile `lowness` from `tile.elev` (elevation is an integer 0–4; elev 0 = valley floor pools
   fog, tapering to ~0 by elev 2). These combine into `morningPool`, which lowers the fog spawn
   threshold and raises fog strength in low ground during dawn/morning only. Night/dusk/day fog is
   provably unchanged (`morningPool` is 0 at every non-morning phase). This reweights the existing
   deterministic fog-pocket seeding — no new fog pass.
2. **Fireflies at night** — **already satisfied** by the pre-existing `fireflyBoost = light.fireflyBoost`
   gating (confirmed in the Phase 16 audit; `fireflyBoost` is ~0.9 at night, ~0.1 by day). **Left
   as-is** — no change, documented here for completeness.
3. **Stronger waypoint glow at night** — `render/effects-renderer.js` `drawWaypointAuraEffect`.
   Derived `nightBoost = 1 + nightStrength*0.85` from the world light and scaled the ground-glow,
   aura, radial-ring, rune-ring, and mote alphas by it (each capped so it never blows out), plus a
   small night-scaled radius bump on the aura/ground glow. Reuses the existing effect draw call
   (no new effect, timer, or loop).
4. **Deeper shadows under ancient trees** — `render/object-renderer.js` `drawTree`. When
   `obj.ancient`, the ground shadow becomes a wider soft outer pool plus a darker inner core
   (drawn inside the already-scaled context, so it grows with the ancient tree). Static identity
   (not time-driven) — matches the plan's framing.
5. **Wisps more visible at night** — `render/entity-renderer.js` `drawDarkWoodsMobVisualModel`.
   Because the wisp body is sprite-cached, added a live, uncached, screen-blended radial halo
   behind wisp-family mobs, alpha-scaled by `nightStrength`. Only living wisp-family mobs get it;
   it is inherently camera-culled because the method only runs for on-screen entities.
6. **Faint lantern glow along roads** — `render/object-renderer.js` `drawLanternPost`. Lit
   lanterns (intact/hanging/wisp/bandit) now draw a faint warm halo (pale green for wisp-lit)
   behind the lamp that strengthens with `nightStrength`. Broken/fallen lanterns stay dark.

All effects are subtle, gated so they vanish by day, and camera-culled via the existing passes.

## Known limitations

- The "deeper shadows under ancient trees" effect is static identity (not time-of-day driven),
  matching the plan's phrasing; it does not deepen further at night.
- The wisp night-halo is a fixed cyan-white for all wisp palettes rather than per-palette-tinted,
  chosen for simplicity; it reads correctly against all four wisp palettes.
- Deer "flee" behavior remains the Phase 16 stateless per-frame offset (unchanged this phase).
- In-browser automated verification requires the local static server (`.claude/launch.json`,
  `python3 -m http.server`); it is not part of the shipped game.
