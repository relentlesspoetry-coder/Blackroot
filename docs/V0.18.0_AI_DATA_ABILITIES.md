# V0.18.0 — AI overhaul IV: mobs use their data ability kits

Final piece of the AI arc, and a minor-version rollover marking the whole
performance + AI overhaul complete. Makes the per-mob ability data actually fire.

## Root issue

Minibosses, bosses, and some mobs carry **object-format `abilities`** in
`data/enemies.js` / `data/npcs.js`, e.g.:

```js
abilities: [{ id:'venom_pool', name:'Venom Pool', kind:'poison_cloud', damageScale:.8, radius:4.5, cooldown:5.5 }]
```

…but **nothing consumed them**. The combat AI only ran the bespoke
`tryAshrootHorrorAbility` plus three hardcoded family checks (spider venom / wolf
rend / root snare). So every boss's designed kit — AoE, cleave, poison clouds,
snares, shields, frenzies, heals, summons — was dead data; bosses just auto-attacked.

## Fix

### Generic executor (`entities/enemy.js`)
- **`tryDataAbility(game, target, d)`** — hooked into `tryTacticalAbility` (after the
  ashroot check, before the family fallbacks). Iterates the mob's object-format
  `abilities`, skips any still on **per-ability cooldown** (`this._abilityCd`, keyed by
  id), skips out-of-range non-self abilities, and runs the first eligible one via
  `executeDataAbility`. On success it sets that ability's cooldown and a short
  `tacticalCooldown` gap so abilities don't spam.
- **`executeDataAbility(game, target, ab, d, kind)`** — dispatch by `kind`, damage =
  `getEnemyAutoAttackRawDamage(this,target) × ab.damageScale` (balanced, not arbitrary):

  | kind | effect |
  |---|---|
  | `aoe` / `venom_rain` / `silk_lines` | AoE burst at the target |
  | `cleave` | AoE around the caster + swing anim |
  | `poison_cloud` | AoE burst + `venom_bite` DoT on those hit |
  | `snare` | root (`root_snare` control) to everyone in radius |
  | `debuff` | applies `ab.status`; movement-type statuses also slow (`moveSpeedMultiplier`) |
  | `shield` | self buff, `damageTakenMultiplier -0.5` for `ab.duration` |
  | `buff` / `frenzy` | self move-speed buff + a faster next swing |
  | `boss_heal` / `heal` | self heal +12% max HP |
  | summon/charge/blink/lunge/retreat/cocoon_prison | **skipped** (return false → next ability / fallback) — future work |

### New game primitives (`systems/combat-system.js`)
The player's `aoeDamage` only hits `this.enemies`, so enemies needed a mirror:
- **`enemyAoeDamage(x, y, radius, amount, source, color, options)`** — damages the
  player + merc + pet + bots (+ bot pets) inside `radius`, with an optional per-target
  `options.status`.
- **`enemyAoeStatus(x, y, radius, effect, source)`** — status-only wrapper (snare/debuff).

## Verification (in-browser)
- A test mob given `aoe` / `poison_cloud` / `snare` / `shield` fired them: AoE dealt 44
  damage to the player; `poison_cloud` applied a `venom_bite` DoT; `shield` applied
  `mob_shield` to the caster; per-ability cooldowns set.
- 0 console errors; camp render unchanged (screenshot).
- **Perf holds**: `performanceSettings` memo still stable, `aiUpdateBand` ~0.001 ms/call,
  steady-state `update` ~11 ms. The executor only runs on attack (via `tryTacticalAbility`),
  never per-frame for idle mobs — a transient 40 ms spike during testing was a disturbed
  scene right after a mass-aggro test, not a regression (settled back to ~11 ms).
