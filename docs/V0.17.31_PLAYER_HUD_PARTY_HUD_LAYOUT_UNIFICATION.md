## Player HUD Background Treatment Changes
The Player HUD now applies the same class-card visual treatment used by the Party / Companions cards: the canonical class background art, a dark readability veil, class-color radial accenting, and a right-side emblem watermark layer. The player name, level/class line, HP bar, resource bar, status tray, and right-side class icon remain in the existing Player HUD structure.

## Party HUD Layout Changes
The companion card renderer now emits each party/companion entry as a compact Player HUD-style frame. Identity and bars live on the left, the class emblem anchors the right side, and command buttons remain available below the bars. This preserves the existing party data and command behavior while making party cards visually match the Player HUD language.

## Shared HUD Styling / Helper Changes
Both Player HUD and Party HUD continue to resolve class backgrounds through the canonical class visual resolver. The Player HUD no longer owns a visually separate dark-panel treatment; it uses the shared class-card layering style. Party cards reuse the same framing, bar, typography, and icon placement philosophy without creating a duplicate HUD system.

## Class Background Resolver Usage
No class-specific hardcoding was added. The existing class normalization and background resolver continue to drive both HUD surfaces.

## Command Button Layout Verification
The command row still renders the existing Follow, Guard, Assist, Passive, and Attack commands for bot/merc companion entries. Active command styling and click handlers remain attached through the existing `data-unified-companion-command` path.
