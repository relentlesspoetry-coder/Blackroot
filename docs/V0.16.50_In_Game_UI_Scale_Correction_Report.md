# V0.16.50 - In-Game UI Scale Correction Report

## Summary
Implemented a focused in-game UI scale correction pass for the Bag window and Action Bar only. The pass reduces the oversized post-revamp Bag panel, bag item cards, action bar frame, action slots, XP meter, and Meditate button while preserving the new Blackroot dark/gold/purple visual direction and all gameplay behavior.

## Root Cause Found
The previous visual revamp increased the owning CSS/layout dimensions for `#bagPanel`, `.bagItemGrid`, `.bagSlot`, `#hotbar.abilityBar`, `.actionSlot`, `.actionXpMeter`, hotbar icon containers, and `#steamDeckMeditateBtn` beyond practical gameplay HUD scale. The issue was not a gameplay or interaction bug; it was layout constants and component dimensions being too large in the canonical UI stylesheet.

## Bag Window Scale/Layout Changes
- Reduced `#bagPanel` width, top anchor, max-height, and padding so it occupies less gameplay space.
- Reduced bag item grid minimum column width and grid gaps.
- Reduced `.bagSlot` height, padding, grid row sizing, text sizes, and stack badge sizing.
- Added bag-slot-specific icon sizing so inventory icons stay readable without forcing oversized cards.
- Reduced direct card stat summary output from three lines to two lines to keep compact cards readable while preserving hover tooltip detail.

## Action Bar Scale/Layout Changes
- Reduced `#hotbar.abilityBar` width, min-width, padding, and border radius while preserving bottom-center placement.
- Reduced XP meter height and text size.
- Reduced action slot height, minimum width, icon container size, label font size, hotkey size, and readout size.
- Reduced the Meditate button width, padding, font size, and vertical offset.
- Preserved cooldown and Auto Attack active highlight selectors in the canonical action bar CSS.
