PASS 52 - UI THEME CONVERSION (Character Select redesign, Character/Stats window
conversion, in-game UI theme unification)
Version: V0.16.39 -> V0.16.40
Date: 2026-07-05

SCOPE OF THIS PHASE
-------------------
1. Character Slots screen: complete visual redesign (featured-slot showcase).
2. Character / Stats window: full theme conversion + model preview enlargement.
3. In-game UI: theme conversion pass across HUD/action bar/bag bar/chat/minimap/
   panels/buttons, applied at the root theme layer rather than per-window.

No other systems were touched. No gameplay, save data, or account logic changed.

ROOT CAUSE FOUND
-----------------
The stylesheet already contained a single, comprehensive "Pass 51: Full UI revamp"
theme block (~830 lines, intentionally placed late in <style> so it wins the CSS
cascade) that already themes nearly every shared in-game UI class: .panel, button,
.windowHeader, .bar/.fill, #hud, #hotbar, .slot, .equipSlot/.itemSlot/.bagSlot,
#characterPanel/#bagPanel/#spellPanel/#settingsPanel/#menuPanel/etc, .logTab,
#menuToggleBtn, .patchNotesButton, and more - largely gold-trimmed and dark already.

The actual defect was narrower than "the whole UI is untouched": this canonical
theme's --accent token and a handful of hardcoded colors inside that same block
were still teal/green (#6fd7c8 and various rgba(127,223,205,...) values), and
--panel/--panel2/the body background/a few component backgrounds still carried a
faint green-black tint left over from the pre-Pass-51 palette. A separate,
unrelated --br-* token set had also been added in a prior phase (V0.16.39, for
Character Creation only) that duplicated part of this same intent without
referencing Pass 51's tokens - two parallel "gothic theme" efforts existed instead
of one.

Root-cause fix: edited the Pass 51 block itself (the canonical, already-late,
already-comprehensive theme owner) rather than adding a third parallel system or a
bottom-of-file override. Shifted --accent and every hardcoded teal/green
occurrence inside that block to violet, and shifted the green-black tint in
--panel/--panel2/body background/.panel/.slot/#hotbar/the shared equip-slot
background/the shared #characterPanel-and-friends background to navy-violet. Kept
--line/--gold/the serif heading font exactly as Pass 51 already had them (already
correct gold/serif). This single, targeted edit cascades to essentially every
system in the game because those systems already read from these tokens/rules.

A second, unrelated discovery: assets/ui/class-backgrounds/*.png (14 fully painted,
per-class illustrated backdrops - radiant beams, wings, class emblem motifs) and
their full canvas-drawing helper (drawCreatorClassEmblemBackdrop, with load-then-
redraw handling, vignette, and animated ambient flare) already existed in
systems/ui-system.js and are used by the character creator when it draws with
creatorBackdrop:true. This is the canonical "class-themed background motif"
renderer the task asked for - it was reused as-is for the new Character Select
featured-slot showcase (see below), not rebuilt.
