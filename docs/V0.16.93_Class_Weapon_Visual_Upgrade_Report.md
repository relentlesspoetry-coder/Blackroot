## Ownership

Class/default held-weapon visuals are owned by `render/paperdoll-equipment-renderer.js`.

The canonical path preserved by this pass is:

1. `slotItem(actor, slot)` resolves equipped visuals first.
2. `resolveVisualSlotItem(actor, slot)` falls back to `classDefaultVisualItem(actor, slot)` only when equipped visuals are missing/invalid for the actor class.
3. `weaponKind(item, actor)` classifies the item into a shared weapon category.
4. `drawWeaponShape(ctx, kind, hand, offHand, side, item, actor)` renders the category using the resolved item and actor class theme.

This pass upgraded the renderer in that existing owner. No duplicate paperdoll renderer, weapon resolver, or overlay pass was added.

## Class-specific themes

`weaponTheme(item, actor, kind)` applies class-aware palettes and material accents without changing equipment data:

- Assassin: black steel, violet metal, red-black execution accents, venom glints.
- Ranger: natural wood, green leaf/feather accents.
- Bard: polished wood, gold trim, purple/rose stage tones.
- Cleric: pale gold, cream, blue-white holy gem accents.
- Paladin: gold/steel holy tank trims.
- Fighter: dark iron, chipped edge, heavy leather wraps.
- Rogue: blackened steel, bright cutting edge, poison glint.
- Warden: bark, moss, vine, and stone colors.
- Wizard/Enchanter/Summoner: arcane/planar gem and focus colors.
- Shaman: carved wood, feather/bone/ritual colors.
- Necromancer: bone, blackened bands, grave-green accents.
- Druid: living branch, leaves, vines, and warm wood.

## Equipped-item override preservation

Equipped item visuals still resolve before class defaults. This pass changes only how resolved visual categories are drawn. It does not alter inventory equipment, item stats, class restrictions, save data, or starter gear mapping.

## Previous bug-fix preservation

- White-stick placeholders were not restored.
- Assassin/crossbow fallback was not made global.
- Auto-attack arm length and grip anchoring were not changed.
- Auto-attack still uses the same resolved held-weapon path from the prior fixes.
- No weapon stat, item stat, damage, attack speed, hit chance, threat, class mechanic, or save/load behavior was changed.

## Verified by static inspection

- Weapon visual ownership remains in `render/paperdoll-equipment-renderer.js`.
- `CLASS_DEFAULT_VISUAL_ITEM_IDS` remains per class and has not been replaced by a global crossbow fallback.

- `drawOffhand()` now uses `drawShieldShape()` for shields/bucklers/barkguards and otherwise calls the upgraded `drawWeaponShape()`.
