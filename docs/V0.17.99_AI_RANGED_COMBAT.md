# V0.17.99 — AI overhaul III: smarter combat (ranged casters + target priority)

Final slice of the requested AI arc (pathfinding+swimming V0.17.97, AI perf V0.17.98,
combat V0.17.99). Makes ranged enemies actually fight at range, and sharpens target
selection — without touching per-hit damage or melee behavior.

## Root issues

- **Every enemy was hardcoded `range: 1.35`** (melee) in the `Enemy` constructor. So the
  data's ranged casters — Grave Wisp / Duskwisp (`aiProfile: 'ranged_caster'`, floating
  spirits) and the silk-web casters — walked into melee and swung. The `aiProfile` tag and
  the rich per-mob weighted `abilities` data were ignored by the combat loop.
- **Target selection** (`pickEnemyTarget` → `actorScore`) was essentially nearest-actor
  with tiny nudges.

## Fixes

### 1. Ranged caster AI (`entities/enemy.js`)
- **Constructor**: detects ranged mobs from data tags
  (`aiProfile`/`animationProfile`/`rendererId` matching `/ranged|floatingcaster/`) and
  gives them `range: 6.5` + `isRangedAttacker` (+ `rangedKiteRange: 3.2`,
  `rangedAttackColor`). Melee mobs keep `range: 1.35`, unchanged.
- **`performRangedAttack(game, target, d)`**: spawns a travelling bolt (`game.spawnBolt`
  with a projectile speed) and applies the **same** damage as the melee auto
  (`getEnemyAutoAttackRawDamage`) — balance-neutral, just from distance — then sets the
  usual `combatCooldown`/`attackTimer`.
- **`kiteFromTarget(game, target, dt)`**: steps directly away from the target (staying
  inside leash + on walkable ground) while continuing to face it, so it fires as it
  retreats.
- **Combat block**: gained a kite branch (`isRangedAttacker && d < rangedKiteRange` →
  retreat + fire if ready) and the attack branch is now ranged-aware — fire **only with
  line of sight**, otherwise reposition to clear the shot. The existing chase branch
  (`moveEnemyToward`, from V0.17.97) already keeps ranged mobs at `range * 0.86` ≈ 5.6
  tiles. The **entire melee path is byte-for-byte unchanged** (only reached when
  `!isRangedAttacker`).

72 / 350 mobs in the test zone are now proper ranged casters.

### 2. Smarter target priority (`systems/combat-system.js` `actorScore`)
Lower score = more attractive. Added:
- `score -= (1 - hpFrac) * 1.1` — favor **wounded** targets (finish near-dead ones).
- `score += min(0.9, maxHp / 700)` — favor **squishier** targets (low max HP =
  caster/healer) over tanks.

Distance still dominates (so a mob won't ignore whoever is meleeing it), and the **threat
table still takes precedence** (`getThreatTarget` before scoring), so taunts/threat are
unaffected — this mainly sharpens opening target acquisition.

## Verification (in-browser)
- Classification: Duskwisp `isRangedAttacker` / range 6.5; Ashroot Horror melee / 1.35.
- `performRangedAttack` from 5 tiles: `fired`, dealt 25 dmg, spawned a bolt effect, set a
  cooldown.
- Kiting: a ranged mob placed 2.0 tiles from the player backed away to 3.2 (`lastTacticalLabel
  = 'kiting'`) over ~25 frames.
- Mixed 6-mob aggro for 60 frames: **0 console errors**, `update` still ~5 ms (V0.17.98
  perf intact). Player died — combat is lethal/working, not a bug (AFK level-1 vs 6 mobs).

## Notes / future polish
- Ranged encounters are now harder to simply walk away from (same per-hit damage, but the
  mobs use their range) — intended "smarter".
- Broader use of the weighted per-mob `abilities` data (AoE / summon_adds / shield / cleave
  kinds already in `data/enemies.js`) is not yet wired into the generic AI; melee ability
  variety still runs through the existing `tryTacticalAbility` family checks (spider venom /
  wolf rend / root snare). That's the natural next step if more ability variety is wanted.
