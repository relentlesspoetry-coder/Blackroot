# V0.17.82 — Talent Balance Pass 2 (enhanced measurement + conservative trims)

**Type:** Balance
**Previous version:** V0.17.81
**New version:** V0.17.82

A second balance pass, and — importantly — an honest one about the limits of measuring balance without
playtesting.

---

## What was tried

The static throughput model (V0.17.76) can't judge three things: **pet damage**, **capstone procs**, and
**tank active mitigation**. I extended the measurement with analytical terms for each:

- pet: product of `petDamage` mods × an assumed pet-DPS share,
- proc: capstone effect magnitude × estimated trigger frequency ÷ icd,
- mitigation: `damageTakenMultiplier` / absorb cooldowns folded into EHP,

and re-measured all 42 specs.

## What it showed (the honest finding)

The added terms introduce **as much noise as signal**:

- the proc estimates **over-credit** onCrit/onCast DPS specs (Judicator, Weaponmaster jumped to +43–45%),
- healers still read **inflated** (a `healingDone` double-application; Lightbearer +65%),
- support / ally-shield specs still can't be measured at all (Minstrel +0%, Sanctifier +1%).

Tuning the pet / tank / healer / support specs to these numbers would risk making the game **worse**, not
better. So I didn't.

## What was actually changed

Only the two direct-DPS specs that are reliably highest in the **clean** model (where the sim is
trustworthy) and were still top after the V0.17.76 pass — both aggressive glass-cannon archetypes,
trimmed conservatively:

| Spec | Change | Clean-model uplift |
|---|---|---|
| Ranger **Beaststalker** | Predator Shot power patch 6% → 5% / rank | 38% → **34%** |
| Druid **Mooncaller** | Crescent Surge power patch 6% → 5% / rank | 39% → **37%** |

Everything else is left as-is. Reductions only ⇒ the PART 7 rails still hold.

## The honest conclusion

Talent balance is now at the limit of what static analysis reliably supports:

- **Rails** are enforced in code (no broken combos) — V0.17.73.
- **Direct-DPS** outliers are measured and trimmed — V0.17.76 + this pass.
- **Pet / tank / healer / support** specs cannot be balanced by simulation — the models are approximations
  and the numbers are noisy. These need **human playtesting**: play a spec, report whether it feels
  weak/strong, adjust. That loop is reliable where simulation is not, and it's the recommended next step
  for finishing balance.

## Verification (in-browser, port 8899)

Both tuned specs dropped as intended (clean-model DPS uplift above); no console errors.
