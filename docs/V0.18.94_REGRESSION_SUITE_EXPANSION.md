# V0.18.94 — Regression Suite Expansion: Death/Respawn, Aggregation, Alias Guard (Roadmap Item 22)

Item 14/22 call for regression tests across equip / unequip / death / respawn / save-load. This
version grows the stat-pipeline self-test from **8 to 12 checks**, adding a real death/respawn test, an
aggregation test, and a regression guard for the V0.18.93 dead-stat fix — all on the isolated
scratch-player harness.

### #9 — alias regression guard

Equips an item written with the display-name attribute `intellect` and asserts the real
`intelligence` attribute rises. This **locks in the V0.18.93 fix** (`ATTRIBUTE_ALIASES`) so the
dead-stat bug can never silently return — if someone removes the alias, this check fails at boot.

### #11 — real death/respawn path

Runs the game's **actual** respawn stat routine, `Game.prototype.restoreActorAfterRespawn`, on the
scratch player. That function only mutates the actor it is passed (recalc → `hp = maxHp`, plus
actor-scoped state resets — no camera/UI/global side effects), so it is safe to exercise here. It
asserts:

- **no modifier duplication**: `maxHp` after respawn equals the live `maxHp` (the roadmap's "death and
  respawn cannot duplicate modifiers"), and
- **full-HP restore**: `hp === maxHp`.

Because it calls the real function, a future regression in respawn is caught immediately.

## Safety

All checks run on a scratch player + equipment; the real `this.player` / `this.equipment` are saved
and restored in a `finally` (confirmed restored even after the real `restoreActorAfterRespawn` call).
The whole body is `try/catch`-wrapped; advisory-only.
