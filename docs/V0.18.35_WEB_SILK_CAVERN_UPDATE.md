# V0.18.35 — Web Silk Cavern Update

Four requested "Bug Fixes & Updates – Web Silk Caverns" features, implemented surgically inside the owning modules. The goal was to make the Silk Web Cavern clearly communicate **"spiders live here and people die here."** No existing gameplay, art, UI, data, or systems were removed.

All four features reuse machinery that already shipped for **venom sacks** — the per-frame player-proximity trigger in `updateSilkWebHazards`, the `interactedObjects` consumed-key persistence, and the environmental-poison allow-list — so there is **no new hazard subsystem, no timers, no monkey-patches, and no duplicate systems.**

---

## 1 — Environmental update: web density (spiders live here)
**Request:** *"Cover walls and floors with far more spider webs, making caverns feel thoroughly infested."*

**Implementation — `systems/dungeon-system.js`, new `placeSilkInfestationDressing(...)`**, wired into floor generation right after `placeSilkVenomSacks`:
- For **every room on every floor** it scatters, in addition to the existing dressing passes:
  - **Floor / hanging / wall webs:** `caveWeb` (orb webs, egg-accent suppressed), `silkStrands` (hanging strands), `webCurtain` (layered wall webbing) — counts scale with room area and floor (royal floor 3 gets more and shifts to the pink `#ff9de8` royal silk palette).
- Placement uses the existing `placeSilkDressingObject` → `placeSilkObject` → `findExistingDungeonFloorTile`, which **only fills empty floor tiles** (`isExistingDungeonFloorTile` rejects any tile that already holds an object), so the density can never overwrite web gates, anchors, venom sacks, stairs, or existing decor — it fills the gaps between them.
- All scattered objects are `passable: true, visualOnly: true, nonDamaging: true`, so they change appearance only, never collision.

## 3 — Atmospheric update (people die here)
**Request:** *"Revamp the dungeon's atmosphere to clearly communicate 'spiders live here and people die here.'"*

Handled in the same `placeSilkInfestationDressing` pass, using existing **visual-only high-fidelity props** (they route to `drawSilkCavernHighFidelityDetail`):
- `webBones` — Web-Wrapped Remains
- `boneMidden` — Bone Midden
- `preyCocoonRack` — Cocooned Victims (larger rooms)

Combined with the denser webs, every room now shows both the infestation and its victims. `silkCocoon` was deliberately **not** used for scatter because it is an *interactable* object (rescue/loot logic at `dungeon-system.js:2664`); scattering it would have created unintended interactables. The V0.18.34 web-wrapped-corpse art (`drawSilkCocoon`) is untouched and still used where the generator already places corpses.

## 2 — New interactable: Spider Eggs
**Request:** *"Add smashable spider eggs. Breaking one either drops items OR spawns baby spiders."*

- **Object + art:** new `spiderEgg` type — `render/object-renderer.js` `drawSpiderEgg` (a pale, bulbous, silk-veiled clutch with a faint embryo shadow and a slow live pulse) and `drawSpiderEggBurst` (cracked, emptied shells). Royal floor-3 clutches use a rose palette.
- **Placement:** `placeSilkNestEggs(...)` — denser in egg/brood/nursery/warren rooms, sparse in combat/boss rooms; capped per floor; skipped if already consumed this run.
- **Smash (walk-over):** the eggs are `passable`, so stepping into one triggers `smashSpiderEgg(...)`:
  - **~45%** → hatches **2–3 baby spiders** ("Spiderlings" = weak, non-elite `Webling Skitterer`s built with `createSilkDungeonEnemy` and pushed into both `dungeonEnemies` and `enemies`, exactly like the boss "summon adds" path).
  - **otherwise** → **drops loot** via `game.rollEditorLootTable('loot_silk_web_cavern', …, { fallbackTableId: 'loot_common_chest' })`.
  - If there is nowhere to place spiders, it falls back to loot so the smash always rewards.
  - Leaves a `spiderEggBurst` shell and records a consumed-key so it does not respawn.

## 4 — New hazard: Venom Eggs
**Request:** *"Small, green-tinted eggs that are dead and filled with venom. Walking over OR smashing them poisons the player."*

- **Object + art:** new `venomEgg` type — `render/object-renderer.js` `drawVenomEgg` (small, **dead** matte-green eggs — no living pulse — cracked and leaking venom, with a sickly ground haze) and `drawVenomEggBurst` (venom splatter + broken green shell).
- **Placement:** `placeSilkNestEggs(...)` — denser in venom/pool rooms, some in brood/combat rooms; capped per floor; consumed-key aware.
- **Trigger (walk over / smash):** the shared `updateSilkWebHazards` proximity tick now handles `venomEgg` — it applies `venomEggPoisonEffect(obj)` (id `venom_egg_poison`, 12 dmg/s for 12 s, stacking) and bursts the egg via `markVenomEggBurst`.
- **Scrubber compatibility:** `isAllowedExistingSilkPoison` now allows `venom_egg_poison` (and the "Venom Egg" source), so this poison survives `removeSilkEnvironmentalPoisonFromFriendlyTargets` — the same allow-listing the venom sacks already rely on.
- Reported as a hazard tile (like `venomSack`/`webHazard`); spider eggs are a reward, so they are **not** hazard tiles.

## Files changed
- **`render/object-renderer.js`** — 4 new draw routines (`drawSpiderEgg`, `drawSpiderEggBurst`, `drawVenomEgg`, `drawVenomEggBurst`) + 4 switch cases.
- **`systems/dungeon-system.js`** — new helpers (`spiderEggConsumedKey`, `venomEggConsumedKey`, `venomEggPoisonEffect`, `markVenomEggBurst`, `spawnBabySpidersFromEgg`, `smashSpiderEgg`), new placement passes (`placeSilkInfestationDressing`, `placeSilkNestEggs`) wired into floor generation, extended `isAllowedExistingSilkPoison`, extended the `updateSilkWebHazards` walk-over trigger, and added `venomEgg` to the hazard-tiles list.
- **Version / docs:** `core/config.js` (version const + build name + top changelog comment), `Blackroot V0.18.34.html` → **`Blackroot V0.18.35.html`** (title + prepended patch-note entry), `PATCH_NOTES.md`, `VERSION.txt`, `README.txt`, this report.

*No collision, movement, or renderer code paths were changed — eggs are non-blocking because `hasBlockingObjectAt` treats `passable: true` objects as walkable (`systems/collision-system.js:56`), which is what makes the walk-over trigger fire.*

**Backup:** the previous version was snapshotted to `backups/Blackroot V0.18.34.zip` before any edits (existing backups were not overwritten).

## Manual tests
1. Enter the **Silk Web Cavern**. Every room should look heavily webbed (floor + walls + hanging strands) and littered with corpses/bones/cocooned victims — clearly "spiders live here and people die here."
2. Find a pale **Spider Egg** clutch and walk into it. It should smash and either (a) hatch 2–3 spiderlings that attack, or (b) drop loot — then leave a cracked shell.
3. Find a small green **Venom Egg** and walk over it (or smash it). You should take stacking venom poison ("Venom Egg Toxin") and see it burst into a green splatter.
4. Re-enter a floor you already cleared — smashed eggs should **stay** smashed (they don't respawn within the run).
5. Confirm the level's web gates, anchors, venom sacks, stairs and treasure are all still present and reachable (the density pass only fills empty tiles).
