# V0.18.18 — Talent tree UI cleanup (fit + alignment + cleaner route)

Follow-up polish to the V0.18.17 revamp (`systems/talent-system.js`), addressing
"make everything fit in the window and look clean."

## The real "text doesn't fit" cause
Measured at the user's ~963×669 window: the 5-tier panel was **~91 px too tall for
the window and scrolled**, hiding the bottom tier. (`max-height` reserved 96 px and
the panel sat at `top:70px`, so at 669 px only ~573 px was usable.) It also overran
the **left** edge horizontally, because it was pinned `right:120px` while being
~940 px wide.

## Fixes
- **Fits vertically (no scroll).** Compacted the per-tier footprint — tier padding
  `10/6→4/2`, icons `50→48` (capstone `54→52`), tile gap `4→3`, name reserve
  `24→21 px`, tighter column header/bar — and let the panel use the window
  (`max-height: calc(100vh - 60px)`, `top:44px`). Now the whole tree (all 5 tiers ×
  3 specs) fits with **0 px cut off** at 963×669.
- **Fits horizontally.** The panel is now **centred and viewport-clamped**
  (`right: max(13px, calc(50vw - 470px))`), so the near-full-width panel always
  lands fully on screen instead of running off the left edge.
- **Aligned grid.** Node names get a **uniform reserved height**, so 1-line and
  2-line names no longer stagger the rows — every tier lines up across all three
  columns.
- **Cleaner route lines.** Replaced the per-tier connector stubs (which floated and
  overlapped the name text) with **one continuous "route spine" per column**,
  JS-positioned through every tier's node centre, glowing in the spec colour from
  the top down to the last reached tier and dim below — plus the horizontal sibling
  rungs. It reads as a real branching route.

## Verified (in-browser at 963×669, 0 console errors)
- Panel fits both ways: left 13 / right 950 (within 963), bottom 619 (within 669),
  **no scroll, nothing clipped**.
- Spine runs exactly first-node-centre → last-node-centre and auto-adapts to the
  compact layout; sibling rungs align to icon centres.
- All node names uniform height; tiers aligned across columns; no lock icons; each
  spec shows its background.
