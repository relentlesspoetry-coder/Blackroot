# Dark Woods 360x360 Expansion — Phase 20: Dark Woods Overworld Herbs

**Phase name:** Dark Woods Overworld Herbs (Phase 20 of 28)
**Previous version:** V0.17.58
**New version:** V0.17.59

---

## Goal

Add the 5 Dark Woods-only overworld herbs (Lantern Moss, Thornberry, Gloomcap Mushroom, Wispbloom,
Blackroot Herb) — resource defs, item defs, and region-based placement — reusing the existing
resource-node and item systems. **No new resource system.**

## Implementation

### Resource node defs (`data/resources.js`, `DR.RESOURCE_TYPES`)
5 new `herb_*` entries using the existing schema (from the Phase 18 audit):

| id | Lvl | xpReward | respawnSeconds | yield (drops min–max) | rarity | region | vendor |
|---|---:|---:|---:|---:|---|---|---:|
| `herb_lantern_moss` | 1 | 10 | 75 | 1–3 | common | Dead Lantern Outskirts | 2c |
| `herb_thornberry` | 2 | 14 | 120 | 1–3 | common | Bramblefen Thicket | 3c |
| `herb_gloomcap_mushroom` | 4 | 24 | 210 | 1–2 | uncommon | shaded mid/deep forest | 7c |
| `herb_wispbloom` | 5 | 34 | 360 | 1 | uncommon | Stone Hedge Clearing | 14c |
| `herb_blackroot` | 7 | 55 | 750 | 1 | rare (`rareNodeXpBonus:20`) | Gloamroot Depths | 32c |

- `tool: 'Hands'` (no tool gate). `respawnSeconds` = midpoint of the plan's range (Phase 24 can add
  min/max randomization). Declarative `allowedZone/allowedRegion/allowedBiome` scope metadata added
  (placement already enforces exclusivity physically; a later phase can hard-guard on these).

### Item defs (`data/items.js`)
5 matching `item_*` materials (`type:'resource'`, `sellValue` = plan vendor value, `icon.family:'herb'`,
rarity white/green/blue). The node `drops` reference the item by display name (resolved by
`resolveDropItemId`).

### Placement — `Game.prototype.ensureDarkWoodsOverworldHerbs` (`systems/world-system.js`)
A **deterministic terrain scan**. The overworld is fully deterministic (`generateMap` uses only
`seededNoise`, no `Math.random`), so the scan picks identical, terrain-valid coordinates every run.
Per herb spec: scan a region bounding box (stride 2), and for each tile require a natural-ground tile
type (grass/dark-grass/forest-floor/dirt/underbrush), the correct Phase-3 region
(`getDarkWoodsRegionAt`), distance constraints (near town for low-level, ≥116 from town for rare
Blackroot, ≥9 off the Stone Hedge waypoint centre for Wispbloom), and `isWalkable(x+0.5,y+0.5)`.
Candidates are hash-ordered for deterministic scatter and spaced apart; the first N are placed.
**Idempotent**: a herb type is skipped entirely if any node of it already exists in the grid.

Counts: Lantern Moss ×5, Thornberry ×4, Gloomcap ×4, Wispbloom ×3, Blackroot ×2 (18 total).

### Call sites (save-safe on new games AND existing saves)
- **New games:** called from the `generateMap` sequence, right after
  `flattenOverworldWalkableTerrain` (so the scan sees the finalized walkable map).
- **Existing saves:** called from `rebuildWorldRuntimeAfterWorldLoad` (`systems/save-system.js`)
  **after** the saved `editorResources` grid is applied (world load *replaces* the grid, so
  pre-Phase-20 saves would otherwise never get the herbs). Idempotent, so double-calling is safe.

### Loot-table correctness fix (`resource-gathering-system.js` `tableForResource`)
Found and fixed a real bug during verification: a herb node with no explicit `lootTableId` fell back
to the generic `loot_gathering_herbs` table, which just drops `item_gloomleaf` regardless of the node
— so a Lantern Moss node yielded **Gloomleaf**. Fix: a node that authors its own non-empty `drops`
and has no explicit loot table now uses those drops directly (`resolveDirectDrops`) instead of the
generic per-category fallback. This makes every new herb yield its own item, and also **corrects
existing nodes** whose declared drops differed from the fallback (e.g. a Mooncap node no longer
mis-yields gloomleaf, a Darkstone node no longer mis-yields copper). Nodes with an explicit
`lootTableId` are unaffected.

## Validation

```bash
find . -name "*.js" -print0 | xargs -0 -n1 node --check      # ALL JS OK
```

### Results
- **JS/JSON:** all pass.
- **In-browser (no-cache server, deterministic overworld), zero console errors:**
  - 5 herb defs + 5 item defs present; Blackroot `rareNodeXpBonus=20`, vendor `sellValue=32`.
  - Auto-placement via the generateMap hook seeds all **18** herbs (5/4/4/3/2) with no manual call.
  - Every placed node is on walkable terrain and in the correct region: Lantern Moss in Dead Lantern
    Outskirts (18–41 tiles from town), Thornberry/Gloomcap in Bramblefen, Wispbloom in Stone Hedge
    (≥40 tiles off the waypoint), Blackroot in Gloamroot Depths (119–252 tiles from town).
  - **Idempotent**: re-running the placer adds 0.
  - **Zone-exclusive**: 18 herbs in the `dark_woods` grid, **0** in the `mossfang_cave` grid.
  - **End-to-end gather** (constructed harvests): each node drops its own herb, with correct
    first-discovery tiers and the rare-node bonus:
    - Lantern Moss → "Lantern Moss x2. First discovery! +15 EXP." (29 XP)
    - Wispbloom → "Wispbloom x1. First discovery! +30 EXP." (66 XP)
    - Blackroot → "Blackroot Herb x1. First discovery! +50 EXP. Rare node +20 EXP." (127 XP)
  - Blackroot `levelRequired=7` (the Phase 19 hard gate blocks under-level gathering).

## Known limitations
- `respawnSeconds` is a single midpoint value; the plan's min–max randomization is deferred to Phase 24.
- Gloomcap's declarative `allowedRegion` is a pseudo-label (`shaded_woods`); it is region-agnostic in
  code (placed in the Bramblefen/Gloamroot forest rings). This scan run placed all 4 in Bramblefen.
- Reachability of every node vs. dense props is best-effort via `isWalkable`; Phase 26 (collision +
  readability cleanup) will do a final sweep.
