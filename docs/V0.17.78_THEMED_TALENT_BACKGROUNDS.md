# V0.17.78 — Themed Talent-Tree Spec Backgrounds

**Type:** UI / UX (talent panel)
**Previous version:** V0.17.77
**New version:** V0.17.78

Each specialization column in the talent panel was a flat dark panel. It now has its own **atmospheric
themed background**, and it applies to **all 14 classes / 42 specs automatically** — no per-spec art
required.

---

## What each spec column now shows (`systems/talent-system.js` → `renderTalentPanel`)

A `specBackground(spec)` helper builds a layered background from the spec's **own colour** plus a stable
hash of its id (so every spec reads distinct):

1. a top **radial colour glow** (hash-positioned),
2. a **tinted diagonal gradient** fading to near-black,
3. a subtle **SVG fractal-noise grain** for a painterly feel,
4. over a dark base, with a spec-tinted **border** and an inset **vignette**.

Behind the tiles sits a large, faint (11% opacity), blurred **signature-ability emblem** — the spec's
capstone/representative spell icon (real in-game art) — so each tree carries a thematic watermark (a bolt
behind Assassin's Boltweaver, a dagger behind Nightblade, a flame behind Wizard's Pyromancer…) with
**zero new assets**.

Backgrounds are applied via JS after the HTML is built (the noise data-URI contains quotes that would
break an inline `style=""` attribute). Header / tier / tile text gained shadows to stay readable over the
busier backdrop; the emblem sits at `z-index:0` behind the content (`z-index:1`) and is clipped by the
column's `overflow:hidden`.

## Image-ready — drop in real painted art anytime

If a spec defines a **`bg`** field (an image url/path) in `data/talents.js`, `specBackground` layers that
image in under a tinted veil. So the hand-painted per-spec scene art from the design mockups can be added
per spec **without any code change** — e.g.:

```js
{ id: 'assassin_boltweaver', name: 'Boltweaver', color: '#8a5aa0',
  bg: 'assets/talents/boltweaver.png', /* … */ }
```

## Verification (in-browser, port 8899)

Every column renders 2 gradients + noise grain + a distinct spec-tinted border + a signature emblem
icon. Confirmed across multiple classes (Assassin purple/green/blue, Wizard fire-orange/frost-blue/
arcane-purple); the emblem sits behind the content; no console errors. (Full-page screenshots time out on
the live canvas loop, so verified via computed-style inspection.)

## Note

This themes every spec **procedurally by colour + a signature emblem** — it is atmospheric, not literal
painted scene art. The `bg` hook above is the path to the fully-painted look once art files exist.
