# V0.18.37 — Silk Web Cavern web coverage

**Report:** *"Silk Web Cavern looks exactly the same, nothing is covered in webs at all! it looks generic as hell, i want webs everywhere like they belong there!"*

## Why V0.18.35 didn't show
The V0.18.35 "web density" update only **scattered a handful of web props** (`caveWeb`, `silkStrands`, `webCurtain`, …) — a few sprites in a large room read as "generic cave with a couple of cobwebs." The **floor and wall surfaces themselves stayed bare**, because:
- The per-tile silk floor overlay (`render/terrain-renderer.js` → `drawSilkWebCavernSurfaceDetails`) drew only **2–3 hairline strands at alpha 0.20 on ~56% of tiles** — essentially invisible.
- Only **room** tiles were flagged `silkWebCavern` (by `stampSilkRoomTileIdentity`); the **corridors between rooms got no webbing at all**, and **walls were never webbed**.

So the change that would actually make the cavern *look* webbed — the floor/wall surface render — was doing almost nothing.

## The fix (render surfaces, not just props)

### 1. Dense woven-silk floor mat — `render/terrain-renderer.js` `drawSilkWebCavernSurfaceDetails` (rewritten)
Every silk floor tile now draws:
- a **faint silk film** across the whole tile diamond so no bare cave floor shows through;
- a **lattice of strands** strung between the tile's four diamond corners (`north/east/south/west`) and a central hub. Because those corners are **shared with the neighbouring tiles**, the strands meet edge-to-edge and the webbing reads as **one continuous sheet across the entire floor**, not per-tile patches;
- an **orb-web hub** (concentric rings + radial spokes + a dew bead) on ~45% of tiles.
- Royal (pink), venom (green sheen), and cocoon tint variants preserved. Deterministic per tile via `hash2D`, so it never shimmers while the camera pans.

**Measured in-browser:** ~**9051 web pixels over a 3×3 tile patch** (near-full coverage) vs the old couple of faint threads.

### 2. Wall silk drapes — `render/terrain-renderer.js` `drawSilkCavernWallWebs` (new)
Cavern walls now get silk drapes: vertical drape strands falling from each exposed wall face's top edge partway down the face, plus a couple of horizontal binding threads. Called from `drawCaveWallDepthOccluder`, gated on the new `tile.silkWebWall` flag.

### 3. Web the WHOLE cavern, not just rooms — `systems/dungeon-system.js` `generateSilkWebCavernFloor`
Right after `stampSilkRoomTileIdentity`, a pass now flags:
- **every walkable floor tile** `silkWebCavern` (rooms **and** the corridors between them), and
- **every wall tile bordering the walkable floor** `silkWebWall`.

So the floor overlay and wall drapes cover the entire cavern — no bare corridors, no bare walls.

## Important: floors regenerate on entry
Dungeon floors are generated fresh each time you enter (`loadDungeonFloor` → `generateDungeonFloor` → `generateSilkWebCavernFloor`). The webbing will **not** appear on a floor you are already standing in — **re-enter the Silk Web Cavern (or descend/ascend a floor)** to see it.

2. Pan the camera → the webbing should stay fixed to the tiles (no shimmer/re-rolling).
3. Floor 3 (Queen's Loom) → webbing should take the pink "royal" tint; venom rooms should show the green sheen.
4. If it feels too heavy or too pale, the knobs are the film alpha (~0.11) and strand alpha (~0.42) in `drawSilkWebCavernSurfaceDetails`.
