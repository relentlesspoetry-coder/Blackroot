# V0.16.52 - Main Menu Rain Weather Report

## Summary

Added animated main-menu rain as a menu-only visual layer while preserving the existing Blackroot main-menu layout, logo, floating class sigils, button routing, modal routing, and in-game weather behavior.

## Root cause / implementation approach

The in-game rain visuals were owned by `systems/weather-system.js`, but the main menu had no safe way to render that visual language because the menu is DOM-based while gameplay weather is canvas-based. The implementation exposes a small shared `DR.Weather.drawRainLayer()` primitive from the existing weather owner and uses it from the canonical Blackroot main-menu UI owner through a non-interactive canvas layer.

## Existing rain/weather helper reuse

Reused the existing weather particle math by exposing `drawRainLayer()` from `systems/weather-system.js`. The main menu supplies a menu-safe rain visual configuration and viewport; it does not change `weatherState`, climate rolling, world-zone weather suppression, or in-game weather timing.

## Render layering details

Main-menu layering now follows this order:

1. `#logoSplash` dark background and logo pseudo-elements
2. `.blackrootMenuRainCanvas` with pointer events disabled
3. `.blackrootMenuClassSigils`
4. `.logoSplashCard` menu buttons
5. `.blackrootMenuModalBackdrop` and active modal panels

Rain opacity and density are reduced automatically while a modal is open so Login, Create Account, Settings, and Patch Notes remain readable.

## Performance notes

The menu rain layer uses a single persistent canvas and deterministic particle drawing. It does not allocate particle arrays every frame. The animation loop stops and clears itself when the game enters `gameStarted` state.
