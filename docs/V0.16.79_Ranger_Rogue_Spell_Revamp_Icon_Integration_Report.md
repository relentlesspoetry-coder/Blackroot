# V0.16.79 - Ranger + Rogue Spell Revamp and Icon Integration Report

## Source files used
- `Ranger Spells(1).txt`
- `Rogue Spells(2).txt`
- `Ranger Spell Icons(1).zip`
- `Rogue Spell Icons(1).zip`

## Spell data updated
- `data/spells.js`: replaced the Ranger and Rogue Level 1-20 runtime spellbooks with the uploaded design-driven names, unlock levels, Focus/Energy costs, cooldowns, ranges, cast times, durations, damage fields, DoT fields, marks, traps, stealth, poison, bleed, evasion, execute, and capstone metadata.
- `data/classes.js`: set Ranger to Focus and Rogue to Energy using the existing numeric mana/resource pool field for compatibility with the engine's resource plumbing.

## Behavior code updated
- `systems/spell-system.js`: added clean owning-system resolver paths for Ranger and Rogue spells instead of adding bottom-of-file patches.
- Ranger support added for Hunter's Mark, trap limit tracking, trap trigger resolution, bow/mark damage bonuses, Focus labels, tracking/camouflage buffs, Kill Zone Focus, Apex Predator, and True Hunt.
- Rogue support added for Energy labels, stealth/vanish/smoke buffs, poison stacks, poison detonation, bleed application, positional behind-target checks, Shadowstep, evasion/counter readiness checks, Phantom Blades echoes, execute thresholds, and Silent Execution setup bonuses.
- `systems/ui-system.js`: Ranger now maps to the existing Focus resource color/label path, and Rogue continues to map to Energy.

## Icon assets added
- `assets/spell-icons/ranger/`: copied normalized Ranger icon files and wrote `manifest.csv`.
- `assets/spell-icons/rogue/`: copied normalized Rogue icon files and wrote `manifest.csv`.
- `systems/spell-system.js`: registered `RANGER_SPELL_ICON_PATHS` and `ROGUE_SPELL_ICON_PATHS` in the existing `CLASS_SPELL_ICON_PATHS` registry.

## Rogue icon mapping
Rogue icons matched the revamped spellbook by spell name and level. All 20 were mapped directly.

## Engine support notes
- Focus and Energy use the existing numeric `mana`/`maxMana` fields internally for compatibility; all player-facing spell labels use Focus Cost or Energy Cost.
- Ranger trap placement is resolved through the spell owner as a valid hostile-path trigger with a tracked two-trap active limit. It does not introduce a duplicate trap engine.
- Rogue poison stacks use the existing status-effect stack/tick system and Deathdose consumes those stacks.
- Positional Rogue bonuses use existing actor facing vectors where available.
- Save migration was not required because Ranger and Rogue spell names remained stable and editor/runtime drafts refresh from canonical spell data.
