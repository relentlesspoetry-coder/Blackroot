# V0.21.1 — Tile Scale Restored At The True 2:1 Ratio

**136 / 68 / 45**, replacing V0.21.0's literal-spec 96 / 48 / 32.

## The problem with the literal numbers

Measured in V0.21.0 and reported rather than hidden:

| | tile edge |
|---|---|
| old build (104/58 @ zoom 1.28) | 76.21px |
| V0.21.0 spec-literal (96/48 @ zoom 1.0) | **53.67px** |
| | **−29.6%** |

That contradicts the same specification's own requirement — *"Tile footprint: larger visual tiles than
your current low-detail pass."* The document's prose and its numbers disagree with each other.

V0.21.0 followed the numbers and surfaced the conflict rather than silently picking one. This version
resolves it in favour of the prose: a high-detail art style whose subject matter is *further away* is
self-defeating.

## What 136 / 68 / 45 preserves

| property | value | note |
|---|---|---|
| ratio | **exactly 2.000 : 1** | the part of the spec that matters; old build was 1.79:1 |
| tile edge | **76.03px** | vs the former 76.21px — a **0.24%** difference |
| `ELEV_STEP` | 45 | ratio to tile height **0.662** vs spec's 0.667; old build was 0.483 |
| zoom | 1.0 / 0.85 / 1.35 | unchanged, still the spec's |

The taller cliffs V0.21.0 introduced are **kept**, not undone — that part of the migration was a
genuine improvement and is independent of the scale question.

## Derivation

Solving for a true 2:1 tile that reproduces the previous apparent size at zoom 1.0:

```
previous edge = hypot(104/2, 58/2) × 1.28   = 76.21px
2:1 constraint: W = 2H, so edge = hypot(H, H/2) = H × √1.25
H = 76.21 / 1.1180 = 68.17  →  H = 68, W = 136
edge = hypot(68, 34) = 76.03px             (−0.24%)
ELEV_STEP = 68 × (32/48) = 45.3            →  45
```

## Why this had to be settled before Phase 2

Phase 2 authors the waypoint shrine against a tile scale. Art produced against the wrong one has to be
redrawn. Settling it now costs three constants; settling it later costs the asset.
