Dream Realms V0.12.72 — Audio Event Coverage + Dynamic Music Pass

Scope:
- Expand audio beyond base footsteps/combat into gameplay, UI, loot, quest, mercenary, and meditation events.
- Add a semantic event layer so systems can request named audio events instead of hardcoding SFX file names.
- Add more music states while preserving standalone procedural fallback audio.

Runtime behavior:
- Game.playAudioEvent routes semantic event IDs into SFX playback.
- Town music is used near the starting camp.
- Cave ambience remains active in caves.
- Combat pulse music takes priority while the player is in combat.
- External WAV assets are used when fetchable; standalone HTML still uses procedural WebAudio fallback.
