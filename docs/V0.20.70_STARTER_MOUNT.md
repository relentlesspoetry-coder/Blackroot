# V0.20.70 — Purchasable Starter Mount

## The gap

Every mount had to be **tamed**. Even the cheapest entry point (Webling Skitterer / Dusk Bat) requires
level 3, bait that drops in the woods, and a beast beaten below its health threshold.

A level-1 character therefore had **no route to a mount at all**, which left the whole system invisible
until well into the zone.

## What was added

**`item_drovers_whistle`** — a 120-silver consumable stocked by `shop_camp_basic_goods` (the camp
quartermaster) — grants **`mount_camp_packbeast`**, a camp-broke pack boar.

## Three constraints, each enforced rather than assumed

### 1. The creature must be a real authored mob (Item 7.J)

`beastKey` names `briarBoar`. The pack boar is the *same species*, broke to harness — not an invented
creature with invented art.

### 2. It must not shadow the tameable wild Briar Boar

Two mount definitions now share one `beastKey`, and `MOUNT_BY_BEAST_KEY` answers *"which mount does taming
this wild creature grant?"* — a question a vendor mount is not an answer to.

```js
if (def.taming) MOUNT_BY_BEAST_KEY[String(def.beastKey).toLowerCase()] = def;
```

Only tameable mounts are indexed. **Verified by taming a wild boar afterwards**: it granted
`mount_briar_boar` (the wild mount) while the vendor mount stayed owned. Without the guard the pack boar
would have shadowed it and made that beast untameable.

### 3. Buying must not advance taming rank

Rank gates the rarer beasts, and buying is not an act of handling. `tamedCount()` now counts only mounts
that *have* a taming block; a new `ownedCount()` covers the collection total.

After buying one and taming one: **owned 2 · tamed 1 · rank Novice Handler.**

## Plumbing — one ownership path

The use-effect goes through `mountSystem.unlockMount` via a new `unlocksMountId` branch in
`item-compiler-system`, so a bought mount is recorded, saved and displayed **exactly** like a tamed one.
There is no second bookkeeping path to drift out of sync.

Two guards followed from the new shape:
- the bait validator (V0.20.66) skips mounts without a taming block, since a vendor mount has no bait
- the panel shows *"Sold by the camp quartermaster"* for a locked vendor mount rather than taming
  requirements it does not have

## Balance

| | value |
|---|---|
| price | **120 silver** — dearest item in the camp shop (iron breastplate: 84) |
| speed | **1.18×** — the *slowest* mount in the game (tamed briar boar: 1.32×) |
| rank contribution | **none** |

A starting point, not a shortcut past the system.

## Note on a typo caught in review

The item's icon colour was first written as `'#c9b externalName'` — a paste artefact that would have
produced an invalid fill. `node --check` passes it happily (it is a syntactically valid string), so it was
caught by reading the written file back rather than by the syntax gate. Worth remembering that
`node --check` validates grammar, not meaning.
