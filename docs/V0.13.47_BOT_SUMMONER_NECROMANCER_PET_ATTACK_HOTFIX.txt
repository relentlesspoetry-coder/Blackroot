Dream Realms V0.13.47 — Bot Summoner + Necromancer Pet Attack Hotfix

Bug fixed
- Summoner and Necromancer bot pets could remain idle because pet targeting did not directly consume the bot owner’s current combat target and stale attack state was not cleared when owner combat ended.
- A living bot pet whose entity registration became stale could be replaced instead of relinked, leaving ownership and update participation inconsistent.

Systems changed
- Pet AI now validates hostile targets centrally and rejects players, bots, mercenaries, pets, quest NPCs, non-combat actors, ambient actors, dead actors, and invalid coordinates.
- Bot-owned pets resolve targets in this order: owner combat target, enemy attacking owner, party primary target, enemy attacking player/healer/support, then a nearby already-engaged hostile while the owner is in combat.
- Bot AI now publishes and clears its current target IDs and explicitly returns its pet to autonomous assist/follow state when combat ends.
- Pet combat stores owner, ownerId, target, targetId, and truthful AI intent. It clears stale target state on death/invalidity and follows its owner after combat.
- Ranged shard pets require clear line of sight; undead melee pets require melee clearance. Blocked attacks request the existing companion combat pathing and never apply damage through the blocker.
- Pet attacks continue to use playAttackAnimation and damageEntity, preserving damage numbers, combat death, loot, threat, and combat-log behavior without a second combat loop.
- Existing extreme-distance pet catch-up remains unchanged. No new teleport behavior or manual pet command UI was added.
- Bot pet maintenance now relinks owner data and re-registers the same living pet in the entity update list instead of duplicating it.
- Dead stale pets are removed before replacement. Old bot snapshots continue to recreate pets through the existing class summon logic.
- Party XP attribution now credits a non-party bot owner when its pet lands the killing blow; party-owned pet kills retain normal party reward distribution.

Save/load compatibility
- Save build metadata is V0.13.47. Existing save schemas and botPetId fields are unchanged.
- Pet targets remain transient and are cleared/reacquired rather than persisted.
- If a bot pet is not serialized as an entity, Summoner/Necromancer class AI recreates it through existing summon data after load.
- Existing linked living pets are normalized with the current owner reference, owner ID, zone, and entity registration.
