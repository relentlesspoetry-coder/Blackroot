## Root Cause: Player HUD Transparency
The V0.17.31 HUD unification made the Player HUD use the same class-card language as the Party HUD, but the Player HUD surface had only translucent overlay layers. Transparent areas in class-background art allowed the world to show through, making the top player panel too transparent and visually underweighted.

## Root Cause: Missing Player HUD Class Background
The class resolver was being called, but the Player HUD backdrop did not have a solid panel base beneath the themed class layers. The class art was present but visually lost against the world because the background stack lacked an opaque bottom layer.

## Player HUD Background Compositing Changes
- Restored a high-opacity panel base on `#hud.compactCharacterHud`.
- Added an opaque bottom layer to the `.hudClassBackdrop` background stack.
- Kept class background art and class emblem watermark visible above the base.
- Preserved foreground layout, HP/resource bars, class icon placement, and text hierarchy.

## Class Background Resolver Usage
The Player HUD still uses `applyClassHudVisualStyle()` and `classBackgroundVisualSpec()` to resolve background art by the current player class. No class-specific hardcoding was added.

## Party HUD Regression Verification
Party HUD card layout and command rows were intentionally not redesigned in this phase. The existing companion cards continue to use their current `unifiedHudFrame compactPlayerHudFrame` renderer and class backdrop treatment.
