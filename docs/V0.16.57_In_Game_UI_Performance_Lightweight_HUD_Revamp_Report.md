# V0.16.57 - In-Game UI Performance / Lightweight HUD Revamp Report

## Summary
Revamped only the always-visible in-game Action Bar and Character HUD into compact, low-paint CSS/DOM layouts. The action bar remains bottom-centered and the character HUD remains top-left anchored. Gameplay behavior for hotkeys, slot clicks, Auto Attack, cooldowns, EXP, Meditate, HP, Mana, player name, class, level, and class emblem is preserved.

## Root Cause of the Lag
The previous action-bar PNG integration had already been removed from the active HUD path, but the in-game HUD still carried expensive always-visible styling from prior UI passes: large shadow stacks, glow-heavy pseudo-elements, conic/radial decorative gradients, active-slot filters, cooldown height transitions, and repeated hotbar DOM querying/class churn during combat HUD updates. Those costs are paid while combat is active, when responsiveness matters most.

## Action Bar Changes
- Reworked the active `#hotbar.abilityBar` owner CSS into a compact low-paint layout.
- Preserved bottom-center anchoring.
- Preserved 10 slots, numbered `1` through `0`.

## Character HUD Changes
- Reworked `#hud.compactCharacterHud` into a smaller top-left anchored layout.
- Reduced portrait/emblem footprint from the oversized presentation to a compact readable frame.
- Preserved player name, class, gender, level, HP, Mana, and class emblem display.
- Removed heavy decorative pseudo-elements, conic gradients, large glow stacks, and HUD bar gem decorations from the active character HUD.

## Removed Heavy Rendering Work
- Removed/disabled active action-bar decorative pseudo-elements from the V0.16.51 hotbar override block.
- Removed large always-visible HUD shadow stacks and active-slot filters.
- Removed cooldown height transitions from action slots.
- Disabled `backdrop-filter` for the always-visible character HUD and hotbar.
- Added containment to reduce layout/paint invalidation scope for the character HUD and hotbar.
- Cached hotbar cooldown/readout nodes to avoid repeated query/create work during combat updates.
- Cached class emblem rendering so full UI updates no longer rewrite the same emblem styles every time.
- Reduced full combat UI update cadence while keeping fast HP/Mana/EXP/cooldown updates responsive.

## Action Bar Input / Hitbox Verification
The existing `.actionSlot` DOM nodes, `data-action-slot`, and `data-spell-index` attributes remain intact. The owning `bindActionBarInput()` handler still delegates pointer/click/touch/keyboard activation from `#hotbar` to the existing slots, so no duplicate input system was introduced.

## Performance Notes
This pass targets HUD paint and DOM update overhead rather than gameplay simulation. The action bar and character HUD now use simpler backgrounds, smaller dimensions, fewer shadows, no active CSS filters, no transition-driven cooldown animation, cached hotbar runtime nodes, and lower full-update frequency during combat. Fast HUD updates still refresh HP, Mana, EXP, target HP, cooldowns, and Auto Attack state.
