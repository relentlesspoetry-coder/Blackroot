# V0.19.2 — Dialogue States 5 (Failure) & 8 (Post-Completion) (Roadmap Item 1)

Item 1 requires eight dialogue states. V0.19.1 cleaned the spoken lines; this models, wires and
authors the two most player-visible missing states — and both were *already reachable in game*, just
saying the wrong thing.

## The gaps

**State 8 — post-completion.** An NPC you had already helped replayed their turn-in line forever.
Three call sites all rendered `completedText` for the re-talk:

- `systems/npc-system.js:1245`
- `systems/event-system.js:615`
- `systems/quest-system.js:1192`

So NPCs never acknowledged the changed regional conditions the roadmap asks for.

**State 5 — failure.** The **only** failable quest in the game (`quest_fourth_member`, the escort —
the sole quest carrying `onFail`) had no authored failure dialogue. `systems/quest-system.js`
hardcoded the line in the escort-death branch:

```js
game.log(`${e.task.name || 'The survivor'} is dead. She does not come back.`);
```

— a gendered pronoun baked into system code.

## New fields + wiring (fallbacks keep every existing quest working)

| field | used at | falls back to |
|---|---|---|
| `postCompletionText` | all three post-completion re-talk sites | `completedText` |
| `failureText` | escort permanent-failure branch | the old generic line |

## Authored

- **`failureText`** for `quest_fourth_member`, naming the permanent consequence (no Quartermaster;
  Iren's last name unaccounted for). **State 5 is now 100% covered** — that quest is the game's only
  failable one. *Verified:* every quest carrying `onFail` has `failureText`.
- **`postCompletionText`** for **8 quests**, reflecting the changed world rather than repeating the
  thank-you — a carter following the lanterns in after dark, Liora's coins turning face-up, the
  bandits pulled off the north road, the stone ring gone quiet, Iren burying the five at the treeline,
  the Quartermaster counting antivenom.

## A bug this pass found

`quest_fourth_member`'s spoken `inProgressText` contained **raw map coordinates**:

> "Escort the survivor safely back to the camp **(100,100)**. She is slow and badly hurt…"

Developer terminology in an NPC's mouth — which V0.19.1's word-list validator **missed**. The line is
rewritten in the survivor's voice, and `validateQuestDialogue` gained a coordinate pattern
(`\(\s*-?\d{1,4}\s*,\s*-?\d{1,4}\s*\)`) plus the two new spoken fields, so it now scans **6**.

## Verification

- `node --check` clean; JSON valid; **0 console errors**; suite **PASS** with 0 dialogue issues.
- The validator **provably** catches an injected `(100,100)` on the correct quest/field, then restores
  clean.
- `cloneQuestForRuntime` uses a full `cloneJson`, so `onFail` and the new fields survive into runtime
  state — **checked, not assumed** (a field-dropping clone would have silently broken the permanent
  failure flag).
