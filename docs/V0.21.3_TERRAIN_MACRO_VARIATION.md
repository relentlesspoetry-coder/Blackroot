# V0.21.3 ŌĆö Art Direction Phase 3 (Partial): Macro Ground Variation

## The finding matters more than the feature

Call counts per frame, instrumented:

| | cold cache | warm |
|---|---|---|
| `buildTerrainChunk` | 9 | 0 |
| `drawTile` | 34 | 34 |
| **`drawGroundDetail`** | **0** | **0** |

Visible terrain is built inside `buildTerrainChunk`, into cached per-elevation layer surfaces.
`drawTile` handles only ~34 edge-case tiles. **`drawGroundDetail` never runs.**

So the high-poly grass / dirt / cave-floor detail passes it dispatches to are dead in normal play ŌĆö along
with the hardcoded `hash2D(x, y, 7028) < 0.60` gate at its two call sites (`drawTile`,
`drawCaveWallDepthOccluder`). They appear to have been dead since chunk caching landed.

A first version of this change was written at those call sites, measured as **nothing** (signal 0.635
mean ╬öluma against 1.269 drift ŌĆö half the noise floor), and was **reverted**. The code was correct; the
path never executes.

**Two wrong hypotheses eliminated first**, recorded so they aren't retried:
`buildTerrainChunk` does **not** swap `DR.runtime.ctx`; the detail call sites are **not** inside the
chunk builder.

## The feature

The ground fill in `buildTerrainChunk` is two lines:

```js
layer.context.fillStyle = material.base;
layer.context.fillRect(px - 0.5, py - 0.5, scale + 1, scale + 1);
```

That is literally the *"large flat-color tiles"* the spec objects to ŌĆö every tile of a material gets one
identical colour. `drawChunkMacroVariation` now lays a tonal wash into the chunk layer immediately
after it: **darker in troughs, warmer in peaks**, so sunlit ground reads warmer rather than merely
brighter.

Strength comes from `terrain.macroVariation` ŌĆö shipped in V0.20.96, read by nothing until now.
`terrain.decalDensity` remains unread, because its only consumer is the dead path above.

**Low frequency is load-bearing.** Sampled at 1/9th tile, so the wash drifts over ~9 tiles and
neighbours differ by ~2% alpha. This file had already learned the lesson ŌĆö the depth-darkening buffer's
own comment records that a flat per-tile fill showed a hard diamond edge. A per-tile random here would
have repeated a known mistake.

## Stated honestly

Layer-wide tonal **spread did not increase** ŌĆö sd 24.551 ŌåÆ 24.511, **ŌłÆ0.2%**. That spread is dominated
by *material* differences (grass vs dirt vs stone), which a ┬▒9% wash cannot move.

The mechanism is proven to be in the right place and running. It is **not** proven to visibly break up
repetition. That is a judgement for the eye, and the strength is one profile value
(`terrain.macroVariation`, currently 0.35) away from being tuned.

## CORRECTION (audit, same day)

This report and V0.21.3's release notes claimed the dead detail passes meant *"some of the terrain
fidelity from earlier versions may never have reached the screen."* **That claim is wrong.**

A full audit instrumenting 63 terrain draw functions found the live equivalents:

| function | calls per chunk rebuild |
|---|---|
| `drawCachedHighPolyGrassFloorDetails` | **1,237** |
| `drawCachedHighPolyDirtFloorDetails` | **643** |
| `drawCachedHighPolyWaterLayerDetails` | 172 |
| `drawCachedShoreContour` | 138 |

When chunk caching was introduced, **cached variants were written and the originals were left behind**.
So `drawGroundDetail` and the three `drawHighPoly*FloorDetails` it dispatches to are **superseded
duplicates**, not lost work. The grass and dirt fidelity does reach the screen ŌĆö via the cached path.

Dead code worth removing; **not** missing visuals. The correct framing is a cleanup, not a rescue.

Of 48 functions that never ran, most are simply **context-gated** rather than dead ŌĆö cave and silk
content (`drawCaveEntrance`, `drawSilkCocoon`, `drawCaveStalagmite`, ŌĆ”) with the player standing in
Dark Woods, combat VFX that only fire in a fight, and player-state passes like
`drawTerrainMeditationResponse`.

Genuinely unexplained and still worth checking individually: `drawTileBevels`,
`drawTileMicroDetails`, `drawGroundDecalLayer`, `drawTerrainScatterDetails`,
`drawTerrainEdgeTransitions`, `drawOrganicTerrainBlend`, `drawEdgeStrip`, `drawLedgeLip`,
`drawBrokenEdgeDetail`, `drawLegacyShoreDepthEdge` ŌĆö terrain surface and edge helpers that ought to run
on ordinary ground.
