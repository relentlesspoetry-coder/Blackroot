# V0.20.79 — Mount Speed Ladder: the Floor Was Too Low to Feel

Reported: *"the mount speed doesn't seem to be faster than walking at all."*

## First: it was not a bug

Worth recording, because the instinct on a report like this is to go hunting for a dead multiplier — and
that would have been a wasted version. The wiring is intact end to end:

```
mount-system.js:88   game.mountSpeedMultiplier = () => this.speedMultiplier()
game.js:2644         speed *= Math.max(0.5, Number(this.mountSpeedMultiplier()) || 1)
```

Measured through the **real movement call** — `tryMoveActorSubstepped`, the same path `game.js:2989` uses
for keyboard movement, not a recomputation of the stat:

| | tiles/sec | measured ratio | authored |
|---|---|---|---|
| on foot | 2.1283 | — | — |
| Black Wolf | 2.4262 | **1.140** | 1.14 ✅ |
| Thorn-Crowned Stag | 3.9373 | **1.850** | 1.85 ✅ |

Drift **0** on a repeated on-foot control arm (2.1283 both times).

### A note on how this was measured

The first two attempts returned **0 tiles moved in every arm** — once from injecting fake key state, once
from calling `tryMoveActorSubstepped(entity, distance, options)` when the real signature is
`(entity, ux, uy, distance, options)`. Zero in *every* arm is the same false-negative shape as the canvas
sampling bug: it looks like a finding and is actually a broken harness. Reading the call site at
`game.js:2989` and copying it exactly is what fixed it.

## The problem was the authored value, and it was mine

The starter was set to **1.14** to honour *"make it a little slower than normal tamed mounts"* — the tamed
floor was 1.18, so 1.14 satisfied the letter of the request.

In practice that is **+0.298 tiles/sec: one extra tile every 3.4 seconds.** Imperceptible.

Compounded by `PLAYER_WALK_SPEED_SCALE = 0.5` from V0.20.47, which halves walk speed *before* the mount
multiplier applies — so every mount's absolute gain is half what its percentage implies. The two requests
were made three versions apart and their interaction was never checked.

`PLAYER_WALK_SPEED_SCALE` deliberately left at **0.5**. The halved walk speed was itself a request, and
raising it would change the feel of all travel rather than just riding.

## Invariants preserved

- The Black Wolf is **still the slowest mount in the game**.
- It still sits **below every tameable mount**, so taming keeps its purpose — the reason 1.14 was chosen
  in the first place survives the change.
- The **ceiling does not move**; the fastest mount is exactly as fast as before.
