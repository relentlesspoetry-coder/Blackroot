# V0.18.90 — Stat Breakdown Tooltips (Roadmap Item 14, inspectability)

Roadmap Item 14 asks for a **Stat Breakdown UI** so the player can inspect where each stat comes from
(base / level / equipment / skill / set / buff / final). This version delivers the per-source
breakdown on the character sheet, reusing the pipeline audited and documented in V0.18.89.

## The inspector — `getPlayerStatBreakdown()`

New `Game.prototype.getPlayerStatBreakdown()` (`systems/inventory-system.js`, right after
`recalculatePlayerStats`) returns the **per-source raw stat contributions** from the **same sources in
the same order** the aggregator uses:

```js
{
  sources: {
    trainer:   { ...npcTrainingBonuses },
    talents:   { ...talentStatBonuses() },
    equipment: { ...sum of equipped item.stats + armor },
    set:       { ...sum of fully-equipped set bonus.stats }
  },
  final: { attack, hp, mana, defense, armor, speed }   // derived values, for reference
}
```

It **recomputes nothing and mutates nothing** — it mirrors the aggregator's inputs and reads the
already-derived final values. So the breakdown is always faithful to what combat uses (single source
of truth: the same sources as `recalculatePlayerStats`).

## The UI

The character sheet's **Damage, HP, Mana, Armor and Speed** stat cards now carry a `title` tooltip
built from the breakdown (`bdTitle` in `systems/ui-system.js`):

> HP breakdown — base class + level scaling, then: Equipment +98 · Set Bonus +10. Final 367.

It **honestly separates** base/level derived scaling from the summed per-source contributions, rather
than pretending the final is a plain sum — `computePlayerDerived` applies base class values, level
growth, and a scalar, so equipment/set/talents/trainer are the raw additive inputs, and the tooltip
labels them as such while naming the final derived value. `statCardMarkup` gained an optional 4th
`title` element; stat cards without one are unchanged (no visual/behaviour change elsewhere).

## Verification

- `node --check` clean; JSON valid; boots with **0 console errors**.
- `getPlayerStatBreakdown()` on a full-Wanderer-set stub correctly buckets:
  - **equipment**: rarity-scaled item stats (e.g. `hp 98` — the four blue pieces' scaled HP, the same
    value the aggregator reads),
  - **set**: the aura `hp 10 / speed 0.03` (unscaled, straight from `bonus.stats`),
  - **trainer / talents**: empty (none on the stub),
  - and reports the derived finals (`hp 367`, etc.).
- Read-only: no gameplay or combat math changed; reuses the V0.18.89 pipeline.
