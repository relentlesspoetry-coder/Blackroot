## Stamina implementation notes
The engine still stores class resources in the existing `mana` / `maxMana` actor fields. Fighter uses that numeric storage as Stamina to avoid save-format churn. Player-facing resource labels are handled through `spellResourceLabel()` / `spellResourceShortLabel()`, which now return `Stamina` / `STA` for Fighter.

`systems/combat-system.js` gives Fighter Stamina distinct regen rates:
- 8 stamina/sec in combat.
- 18 stamina/sec out of combat.

## Momentum implementation notes
Momentum is status-based and stat-impacting through combat timing:
- Status id: `fighter_momentum`.
- Max stacks: 5.
- Duration: 5 minutes.
- Each stack contributes +2% attack speed.
- `getPlayerSwingDelay()` applies the Momentum attack-speed bonus through the existing auto-attack timing owner.
- Perfect Masterstroke consumes up to 5 stacks.


## Heavy Riposte support note
The current engine does not expose a dedicated dodge/parry event stream. To make Heavy Riposte gameplay-real without adding a separate combat system, `systems/combat-system.js` now grants a short `Riposte Ready` status from the combat damage owner when a Fighter receives a direct enemy attack and a parry-opening roll succeeds. This enables Heavy Riposte through the existing status/spell gating path and does not add a shield requirement or taunt behavior.

## Savage Momentum support note
The existing damage owner already rolls critical hits. Fighter critical hits while `Savage Momentum Watcher` is active now trigger the existing Fighter buff path and grant Momentum through the combat damage owner.
