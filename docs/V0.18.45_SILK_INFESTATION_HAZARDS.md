# V0.18.45 — Silk Web Caverns: Infestation & Hazards

The "Bug Fixes & Updates – Web Silk Caverns" 5-item request, on the V0.18.44 baseline. **No layout/config changes** — `SILK_WEB_FLOOR_CONFIG` (Floors 1–3, V0.18.43/44) is untouched. All work is surgical, inside the owning modules.

## 1. Web Density & Variety
> "Add significantly more spider webs, with many stretching all the way to the ceiling. Introduce varied web sizes — extremely thick, 3-tile-wide webs alongside smaller ones."

- **Placement** (`systems/dungeon-system.js` → `placeSilkCeilingWebsAndJunk`): the single `scatter(..., 'ceilingWebColumn', big?3:pocket?1:2)` call was replaced with a per-room loop placing **more** columns of **mixed sizes** — `big ? 6 : pocket ? 2 : 4` per room. Roughly every third column in a big/wide room is flagged `webSize:'thick'`; the rest alternate `'small'` / `'medium'`.
- **Render** (`render/object-renderer.js` → `drawCeilingWebColumn`): now reads `obj.webSize`:
  - `thick` — a `spread` of **2.7×**, **26** vertical strands (35% thicker lines) on a soft **filled silk sheet** ~3 tiles wide, 14 ladder rungs, a 10-thread ceiling fan, and 1–2 extra egg sacs. This is the "extremely thick, ~3-tile-wide" curtain.
  - `small` — `spread` 0.62, 6 strands, slightly more translucent: a thin wisp.
  - `medium` (default / legacy) — unchanged look (spread 1, 10 strands).

## 2. New Hazard: Egg-Spawned Baby Spiders
> "Place tiny baby spiders inside some of the eggs. If a player walks over these eggs, the spiders will crawl out and immediately attack."

- `placeSilkNestEggs` now tags ~half the eggs with `babyEgg:true` and renames them **"Twitching Spider Egg" / "Twitching Royal Egg"** (brood rooms lean ~2/3 twitching). Non-twitching eggs keep the old "Spider Egg Clutch" behaviour.
- `spawnBabySpidersFromEgg(game, floor, x, y, count, options)` gained an `options.aggressive` path: each baby gets `aggro=true`, `aggroTarget = game.player`, and a widened `leashRadius` (≥45) so it **immediately chases** rather than idling.
- `smashSpiderEgg`: `const guaranteedBabies = obj.babyEgg === true;` — twitching eggs **always** hatch (3–5 babies), others still roll (45% → 2–3). Both spawn with `{ aggressive:true }`. Log: *"The egg bursts open – N baby spiders scuttle out and attack!"*
- (Egg-break animation + tiny baby-spider models were added in V0.18.38/41; this makes the hatch guaranteed, aggressive, and — via Task 3 below — actually visible.)

## 3. Bug Fix: Descending Spider Model (was invisible → bots attacked nothing)
**Root cause (not a symptom patch):** the game keeps two entity lists. `setActiveEnemySet()` builds `game.entities = [...enemies, ...allies]` as a **snapshot** that the renderer iterates (`game.js` render loop), while the AI iterates `game.enemies` directly. Runtime-spawned enemies (the ceiling dropper, egg babies, boss adds) were pushed only to `game.enemies` / `game.dungeonEnemies` — so they were **AI-active but never in the render snapshot**, i.e. invisible. Bots pathed to and swung at an enemy with no model.

**Fix:** new helper, applied at the owning spawn sites:
```js
function registerRuntimeSilkEnemy(game, add) {
  if (!add) return add;
  if (Array.isArray(game.dungeonEnemies) && !game.dungeonEnemies.includes(add)) game.dungeonEnemies.push(add);
  if (Array.isArray(game.enemies)        && !game.enemies.includes(add))        game.enemies.push(add);
  if (Array.isArray(game.entities)       && !game.entities.includes(add))       game.entities.push(add); // <- the missing one
  return add;
}
```
Applied to `makeBabySpider` (which the ceiling dropper and egg babies both go through) and the boss `summon_adds`. The ceiling dropper already had a silk spider model via `drawSilkWebSpiderModel`; it simply was never reached because the actor wasn't in `entities`. Now it draws the moment it lands.

## 4. Balance: Descending Spider Frequency
> "Reduce the frequency at which this happens, so it remains a rare and impactful event."

- `silkDropTick` reset raised from `13 + rand*16` (~13–29 s) to **`60 + rand*75`** (~1–2.25 min). Initial delay `45 + rand*30`.
- Ambient dropper cap in `maybeDropCeilingSpider` lowered from `>= 4` to **`>= 2`** concurrent.

## 5. Boss Room Entrance Overhaul
> "Implement a single, giant, thick web that seals the entire entrance from wall to wall and floor to ceiling. This barrier must only trigger once the player fully enters the boss room, preventing early escapes."

- **Wall-to-wall, no gaps** (`startSilkBossRoomSeal`): the old fixed `s = -5..5` perpendicular band could miss the edges of a wider/offset opening. It now sweeps the **full** perpendicular width (`±(maxRoomHalf+6)`) at each of 4 depth slices (`d = 0..3`) and seals **every** open `CAVE_FLOOR` tile it finds — so the plug spans the throat wall-to-wall and is thick enough that you can't corner-cut around it. The 3 spinning crawlers are now spread across the *actual* measured entrance width (`sMin..sMax`).
- **One giant floor-to-ceiling web** (`drawBossWebSeal`): height raised to floor-to-ceiling (`168·grow`) and `halfW` widened to 32, with more drape strands, so neighbouring seal tiles **overlap into a single continuous curtain** instead of reading as separate tiles.
- **Only once fully inside** (`updateBossSeal` + new `playerFullyInsideSilkBossArena`): the seal no longer starts the instant the boss engages at the threshold. It waits until the player is within the inner **~68%** of the arena bounds (i.e. past the outer doorway ring), so it can only slam shut once you've committed — no more "walk in, it seals behind you at the edge, you slip back out."
