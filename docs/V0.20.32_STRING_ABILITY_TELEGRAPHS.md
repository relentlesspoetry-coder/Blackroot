## 1. String-ability telegraphs — the V0.20.31 follow-up, closed

V0.20.31 telegraphed the **data-driven** enemy AoE (`executeDataAbility`). The **string-format**
abilities on the **Ashroot Horror** miniboss still applied damage/status instantly via a
`centerTargets` loop:

| ability | was | now |
|---|---|---|
| **Ash Cloud** (r2.2) | instant damage + Ashen | telegraph → 0.5s → damage + Ashen |
| **Death Bloom** (r1.8) | instant damage + Withered | telegraph → 0.5s → damage + Withered |
| **Blightwood Pulse** (r3.0, self) | instant Vulnerability | telegraph → 0.5s → Vulnerability (status-only) |

All three route through a shared `telegraphAoe` helper into the **same `enemy_aoe` pending-impact
queue** V0.20.31 built. `ashroot_crush` (a melee crush) and `cinderroot_snare` (single-target snare)
stay instant, as before.

**Cleanup:** the now-unused `centerTargets` local was removed. Its helper `ashrootHostileTargets` is
now an **unreferenced module function** — left in place to keep this change surgical, noted as a
follow-up cleanup.

## 2. Enrage conditions & boss phases — investigated, reported

The spec's boss requirements list *"distinct phases where appropriate"* and *"enrage conditions."*
**Neither exists as a system:**

- **0 enrage fields authored** anywhere (`enrageTimer`/`softEnrage`/`berserkAt`/`hpThreshold`/… all
  absent).
- **No boss combat-phase system.** Every `phase` reference in code is unrelated: entity `living`/
  `dead`, the silk boss **walk-in entrance** animation, fishing-bobber and portal state, decoration
  scatter offsets.
- **Only ad-hoc per-enemy HP-threshold behaviours** exist — the Ashroot Horror heals once at 48% HP,
  and one other enemy has an HP-ratio check. No timers, no phase transitions, no enrage.

**This is a feature build plus design decisions, not a dead-field fix.** "Distinct phases where
appropriate" needs deciding *which* bosses get phases and *what* the phases are; "enrage conditions"
needs a timer/threshold system and per-boss tuning. That's the same class as the channel-implementation
and projectile-hand-side items: real design work, not a quick correction. **Reported for the user to
decide the scope rather than speculatively built.**
