# V0.18.33 — Portal always returns to camp

Follow-up to V0.18.32 (walk-through portals). User:

> "portal is still glitched its supposed to return you to camp, not a random
> location."

## Root cause
`tryCastPortalSpell` (`systems/portal-system.js`) implemented a **two-way swap**
keyed on `isPortalCampArea`, which counts anywhere within `CAMP_RADIUS = 22 tiles`
of the camp centre as "at camp":

```js
const atCamp = this.isPortalCampArea(this.player); // true within 22 tiles of camp
let destination = this.ensurePortalCampArrivalSafe(); // camp
if (atCamp) {
  destination = { ...runtime.lastWildernessLocation }; // <- a STALE anchor
}
```

22 tiles is a large radius, so casting **anywhere near camp** took the `atCamp`
branch and set the destination to `lastWildernessLocation` — a leftover anchor from
a previous excursion. Walking through then dropped you at that old spot: the
"random location" instead of camp.

## Fix
A cast portal now **always** leads back to camp; the return portal that opens at
camp still brings you back to exactly where you cast it:

```js
const atCamp = this.isPortalCampArea(this.player);
let originTravel = this.capturePortalLocation(this.player.x, this.player.y);
const destination = this.ensurePortalCampArrivalSafe(); // always camp
if (!atCamp) runtime.lastWildernessLocation = { ...originTravel };
// ... originPortal label 'Portal to Camp', destinationZone 'overworld'
```

The `atCamp → lastWildernessLocation` destination swap (and its "no wilderness
anchor yet" early-return) are gone. `originTravel` is unchanged, so the return leg
(`applyPortalLocation(link.originTravel)`) still returns you to your cast spot, then
the link collapses — one round trip.
