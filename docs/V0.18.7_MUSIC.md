# V0.18.7 ‚Äî New Music (Menu / Dark Woods / Silk Web Caverns)

Three real recorded tracks applied to their in-game contexts and wired into the
audio system (`systems/audio-system.js`).

## Files (in `music/`)
| Track | File | Change |
|-------|------|--------|
| Dark Woods (overworld) | `dark_woods_loop.wav` | **Replaced** the old loop with the new recording |
| Silk Web Cavern (dungeon) | `silk_web_cavern_loop.wav` | **New** |
| Main Menu theme | `menu_theme_loop.wav` | **New** |

Filenames follow the existing `*_loop.wav` convention so the audio system loops
them. All three are valid RIFF/WAVE and serve correctly over http.

## Wiring (`systems/audio-system.js`)
- **`MUSIC_MANIFEST`** gains `silk_web_cavern` and `menu_theme` entries (these are
  also preloaded at boot alongside the other tracks).
- **Dark Woods** needed no code change ‚Äî the overworld scene already resolves to
  the `dark_woods` manifest id, which now points at the new file.
- **Silk Web Cavern** ‚Äî `audioSceneForCurrentState()` now returns the dedicated
  `silk_web_cavern` scene when `currentZone` is a cave/dungeon **and**
  `dungeonSystem.state.active.dungeonId === 'silk_web_cavern'`. Every other cave
  keeps the generic `cave_ambience`. Combat still overrides with `combat_pulse`.
- **Main menu theme** ‚Äî before a character enters the world there is no "zone"
  scene (`updateAudioSystem` only runs once `started`), so:
  - new `Game.prototype.playMenuMusic()` loops `menu_theme` while on the logo
    splash / character screens (respects the music volume + mute settings);
  - it is triggered from `unlockAudioSystem()` on the **first user gesture** that
    unlocks audio (browsers require a gesture before audio can play), and refuses
    once `started` is true;
  - `playMusic()`'s async-load retry guard was generalized so the menu theme ‚Äî
    which is not a zone track ‚Äî still starts once its (~11 MB) buffer decodes;
  - on world entry, `updateAudioSystem` crossfades the menu theme out to the zone
    scene automatically (no explicit stop needed).

## Verification (in-browser, served over http, 0 console errors)
- All three wavs return HTTP 200 with valid RIFF/WAVE headers.
- Silk Web Cavern dungeon scene ‚Üí `silk_web_cavern`; a different cave ‚Üí `cave_ambience`.
- On the menu (pre-start), unlocking audio ‚Üí `menu_theme` decodes and plays.
- Entering the world hands `menu_theme` off to the zone track (observed
  `town_lantern` at the town spawn); `playMenuMusic()` returns `false` after start.

## Notes
`file://` standalone builds still fall back to procedural WebAudio when the files
can't be fetched (unchanged behavior); served (http) builds play the real tracks.
The tracks are looped as-is ‚Äî if a seam is audible on loop, they can be trimmed to
seamless loop points later.
