# V0.18.56 — Silk Web Caverns: the web columns are randomly mixed

Your note: *"all those need to have varieties, they can have any of those things, 1 thing, 2 things, mixed things, all things — we want variety, so randomly make them all mixed."*

Showcase: [V0.18.56_column_variety.png](V0.18.56_column_variety.png) — 8 columns at different tile positions (no explicit seed), each with a different mix.

## The two parts
### 1. Independent feature rolls
Before, `drawCeilingWebColumn` gave every **thick** column the same full set (nest + bones + cocoon + eggs + spiders), and tied the rest to size — so thick columns all looked alike. Now each of the 5 "infested" extras rolls **independently**, regardless of size:

| extra | chance | notes |
|---|---|---|
| egg sacs | ~58% | 1–3, size-scaled |
| woven web nest | ~38% | size-scaled |
| bones at the base | ~50% | occasionally a bigger cluster |
| a cocooned body | ~50% | `drawWebCocoon` |
| crawling spiders | ~75% | count varies |

So a column can carry **none, one, two, or all** of them in any combination. Item sizes scale to the column (`sizeF`), so small "wisp" columns get lighter versions rather than a full-size cocoon.

### 2. Distinct seeds (the bug that hid the variety)
Columns/piles/nests are placed **without** a `_propSeed`, so their PRNG seed `st = obj._propSeed || 1` collapsed to **1** — every instance shared one random sequence and rendered **identically**. That's why the previous "variety" all looked the same.

Fix: `drawCeilingWebColumn`, `drawWebbedBonePile`, `drawWebJunkPile` and `drawGiantSpiderNest` now fall back to a **tile-position seed** when `_propSeed` is absent:
```
st = obj._propSeed || (floor(obj.x)*374761 + floor(obj.y)*668265 + k)
```
Objects get `x,y` from `placeSilkObject`; the **tunnel** column placement (`systems/dungeon-system.js`) now also stamps `x,y` on its objects. So every column, pile and nest is unique.

No gameplay changes.
