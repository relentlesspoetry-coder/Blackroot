# V0.16.75 - Fighter Model Revamp Report

## Summary
Revamped the Fighter visual model using the existing canvas/procedural class identity renderer. The old Fighter-specific runtime path read as a plate-and-shield combatant; it now renders as a fast light/leather two-handed bruiser with a massive greatweapon, fur mantle, leather scale harness, wrapped arms, wild hair, scars, war paint, belts, pouches, trophy chain, and battle-worn details.

The uploaded phase prompt referenced an older V0.16.72 -> V0.16.73 target, but the supplied current baseline was V0.16.74. This patch therefore advances the current build to V0.16.75 rather than reverting version metadata.

## Old Fighter Model Location
The old active Fighter player/class renderer lived in `render/class-identity-procedural-model.js` inside `drawPremiumFighterModel()`. It called the older plate-oriented torso, shoulder, leg, arm, sword, and shield helpers.

## Old Visual Code Replaced / Disabled
The Fighter render path no longer calls:
- `drawPremiumFighterTorsoPlateMesh()`
- `drawPremiumFighterShoulderPlateMesh()`
- `drawPremiumFighterLegPlateMesh()`
- `drawPremiumFighterArmPlateMesh()`
- `drawPremiumFighterShieldArm()`
- `drawPremiumFighterSideShield()` from the active Fighter path

Those helpers were not deleted because Paladin and shared fallback rendering still rely on some of the old plate-compatible primitives. They are no longer active for the Fighter model path.

## New Fighter Model Implementation

## Direction and Animation Coverage
The implementation remains inside the existing direction-aware Fighter model owner, so it inherits the current direction system: north, northeast, east, southeast, south, southwest, west, and northwest where available. The revamp covers the existing states used by the renderer: idle, walk, attack, hit flash, death transform, meditation/rest, fishing, swimming, and cast/ability aura hooks.

## Player / Bot / Mercenary Coverage
Player Fighter uses the class identity renderer directly. Fighter Bot and Fighter mercenary variants that share the class identity renderer inherit the new model. The generic standing fallback in `render/entity-renderer.js` was also updated to remove Fighter shield/heavy-plate fallback visuals and use a greatsword/no-offhand profile.

## Renderer / Mesh Note
BlackRoot is currently using the existing canvas/procedural renderer for this class model path. No true 3D mesh pipeline was added. The high-poly brief was translated into layered sharp procedural shapes, silhouette changes, direction-aware weapon placement, and readable accessory detail.
