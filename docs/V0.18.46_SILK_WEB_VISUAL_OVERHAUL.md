# V0.18.46 — Silk Web Caverns: Web Visual Overhaul

The environmental/visual half of the "Bug Fixes & Updates – Web Silk Caverns" request, on the V0.18.45 baseline. **No layout/config changes.** The four hazard/bug-fix items on that list (egg-spawned baby spiders, the descending-spider model, its reduced frequency, and the wall-to-wall boss entrance seal) were already implemented in **V0.18.45** and are unchanged here — this version is the four *visual/environmental* items.

A captured showcase render of the new elements is at [`docs/V0.18.46_showcase.png`](V0.18.46_showcase.png) (top row: floor woven orb-webs incl. royal; middle: thick/medium/small columns with nest + base bones, large/small bone piles, junk pile; bottom: solo web nest, skull, femur, ribcage, bone cluster).

## 1. Web Visual Overhaul — webs read as thick, woven silk
> "Completely redo the appearance of all webs. Ceiling and floor webs must look like thick, realistic spider webs that are visibly woven together, replacing the current low-quality look."

- **Floor** (`render/terrain-renderer.js` → `drawSilkWebCavernSurfaceDetails`): the per-tile orb-web hub was concentric ellipses + straight spokes. It's now a **real woven orb** — 8 iso-squashed radial spokes tied together by a genuine **capture spiral** (~2.2 turns) with **bright silk junction beads** on the outer ring and center. The front lattice is a touch thicker (1.15 px) and denser (skip threshold 0.94), with bright beads where threads knot at the hub and mid-edges. Reads as woven silk instead of loose threads.
- **Columns** (`render/object-renderer.js` → `drawCeilingWebColumn`): added a **diagonal cross-weave** pass (threads tying the verticals together) plus **junction beads** scattered along the strands, so the floor-to-ceiling columns read as woven, not parallel.

## 2. Ceiling Web Nests
> "Place large web nests inside the thick webs that stretch down from the ceiling, adding depth and a sense of active infestation."

- New `drawWebNest(cx, cy, r, silkRGB, rnd)`: a silk bulb with **concentric woven rings + 11 radial spokes**, a **dark funnel mouth** (radial gradient) where the spider lurks, and **clustered egg sacs**. It bulges partway up (`0.42–0.6` of the height) inside **every thick column**, and occasionally on medium columns.

## 3. Webbed Bone Piles — recognizable bones
> "Scatter both large and small piles of bones… heavily wrapped in webbing. The bones must be distinct and recognizable as actual bones — not random shapes or blobs."

- New **bone primitives** (shared, reusable):
  - `drawBoneSkull` — cranium dome + two **eye sockets** + **nasal cavity** + a **jaw with teeth** + a dome highlight.
  - `drawBoneFemur` — a slightly bowed shaft with **knobbed epiphyses** (two lobes at each end → the classic "bone" silhouette).
  - `drawBoneRibs` — a spine with four **curved ribs** off each side.
  - `drawBonesCluster` — composes crossed femurs + a skull (+ ribs + a second skull when large).
- New object type **`webbedBonePile`** (`bonePileSize: 'large' | 'small'`) via `drawWebbedBonePile`: the bones sit on a silk-caked mound and are then **heavily wrapped in webbing** (vertical strands + horizontal binds + bright silk junction nodes) drawn *over* them.
- The old `drawWebJunkPile` had squiggle-bones (a curved line + a 4px circle "skull"); those were replaced with the recognizable `drawBoneFemur` + `drawBoneSkull`.

## 4. Web Columns & Mixed Clutter
> "In big open caverns and tunnels, create thick columns of webs stretching from ground to ceiling. Mix the webbed bone piles into these columns and the surrounding web structures."

- `placeSilkCeilingWebsAndJunk` (`systems/dungeon-system.js`) now scatters **`webbedBonePile`** objects across **every non-safe/treasure room**: **large** piles in the big caverns (×2), **small** piles everywhere (×3 big / ×1 pocket / ×2 otherwise), interleaved among the existing thick web columns (a `phase` offset was added to `scatter` so the piles spread out rather than stack). `placeSilkObject` already searches a radius-9 for a free floor tile, so piles and columns never collide.
- The thick columns themselves **tangle a webbed bone cluster at their base** (drawn in `drawCeilingWebColumn`), directly satisfying "mix the bone piles into the columns."
