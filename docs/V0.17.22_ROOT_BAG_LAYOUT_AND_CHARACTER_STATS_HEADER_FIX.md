## Rotling Root Anchor Fix Details
The previous root VFX used the generic world projection and drew tall wrapping curves over the target after entity rendering. On humanoid targets this read as a waist/body cage. The fix publishes a rendered `_rootVfxFootAnchor` from the entity renderer and makes `drawRotlingRootEntangleEffect()` prefer that foot/base anchor. The root tendrils now grow from the ground contact point and are shortened to grip ankles/lower legs.

Root/entangle render depth was also lowered so the effect behaves as a ground/feet effect instead of an over-body overlay.

## Bag Renderer / Icon Layout Changes
The bag renderer still uses the canonical `itemIconHtml()` resolver. Filled bag slots now request a dedicated `bagSlotItemIcon` class so the owning CSS can size the same canonical icon output for bag slots without adding a separate bag-only icon map.

## Bag Text Overlay Changes
The previous bag presentation used a side-text grid. The new layout makes the icon fill the slot and places only minimal overlay text over the image:
- slot number badge at the top-left
- stack/quantity badge at the top-right
- item name in a small bottom overlay strip

Rarity borders, tooltips, right-click, double-click, and drag handlers remain unchanged.

## Model Preview Background Cleanup Details
The Character model preview fallback no longer uses the green-tinted panel treatment. The preview uses the canonical class background variable when available and a dark neutral fallback if it cannot resolve. The backdrop overlay was retuned to dark framing rather than green wash.

## Character Stats Header Layout Changes
The stats header now uses a three-column layout:
- large class emblem on the left
- name and level/gender/class identity in the middle
- Gear Score card aligned on the right

The Gear Score formula and stat calculations were not changed.
