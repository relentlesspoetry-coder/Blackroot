# V0.20.20 ‚Äî The Rogue Had No Knife (Roadmap Item 6)

## I recommended this wrong, twice

I told the user: *"the engineering is genuinely done and the work is just craft ‚Äî author `vfxStyle`
across six classes."*

**Both halves were false.** Checking before authoring is the only reason it didn't become a hundred
lines of dead data.

### Correction 1 ‚Äî the mechanism wasn't built

`spell.vfxStyle` was read in **exactly one place** ‚Äî `spell-system.js:1021`, the Fighter's own
resolver ‚Äî out of **thirteen class resolvers**.

Two other `playAttackAnimation` call sites **hardcoded `'slash'`**, both inside **`resolveClassSpell`**
‚Äî the generic fallback every class without a bespoke path falls through to:

```js
if (spell.kind === 'melee' || spell.kind === 'meleeDebuff') {
  this.playAttackAnimation(p, t, color, 'slash');   // ‚Üê ignores vfxStyle
```

**That is the exact `id === 'titan_chop' ? 'slam' : 'slash'` bug V0.20.0 fixed for the Fighter ‚Äî still
alive one function away**, making `vfxStyle` inert for everybody else.

Both call sites now read the authored style ‚Äî **the melee branch and the charge/gap-closer branch**.
Both, or a style would work on a class's normal swing and vanish on its charge: the kind of seam that's
worse than no feature.

### Correction 2 ‚Äî the scope wasn't six classes

Only **four** classes have `melee`/`meleeDebuff` spells at all:

| class | melee spells |
|---|---|
| Fighter | 10 (9 styled, V0.20.1) |
| **Rogue** | **11** |
| **Paladin** | **4** |
| **Warden** | **3** |

The casters use `spawnBolt`/`spawnRing` ‚Äî a **different mechanism entirely**. The Wizard's resolver
calls `spawnRing` **10 times**: that's the Assassin's pre-V0.18.29 problem, and it needs a **build**,
not a field.

## The real find ‚Äî the Rogue never drew a weapon

`resolveRogueRevampSpell` **never called `playAttackAnimation` at all**. Its VFX calls are `spawnRing`
√ó4 and `spawnStatusPulse` √ó1.

**Eleven melee abilities** ‚Äî Backstab, Garrote, Silent Execution ‚Äî spawned a ring on the ground and
dealt damage with **no strike drawn**. The only melee class in the game with no melee animation. The
class whose entire identity is a blade in the dark had been killing people by standing near them.

Fixed in its resolver, placed **before** the damage so the strike reads as causing it.

## Authored ‚Äî 10 styles, all existing silhouettes, no new renderer work

| spell | style | why |
|---|---|---|
| Backstab ¬∑ Perfect Counter | `thrust` | a stab into an opening; a counter isn't a swing |
| Sap ¬∑ Stonehand Strike | `crush` | a blackjack and a stone fist are blunt |
| Twin Fang | `claw` | two daggers **are** the twin-arc silhouette |
| Flurry Cut | `wild` | many cuts; the crossing off-balance arcs |
| Killing Cut ¬∑ Silent Execution ¬∑ Divine Verdict | `execute` | gated on a dying target ‚Äî a decision |
| Judgment | `slam` | a hammer brought down overhead |

### Deliberately left baseline

Quick Cut and Radiant Strike **are** the plain opening swing. And **Bleeding Vein, Garrote, Deathdose,
Briar Lash, Thornsnare, Holy Rebuke** have **no honest silhouette** in `DR.SLASH_STYLES` ‚Äî a bleed, a
wire and a poison are not shapes this renderer knows. Forcing one would be decoration, not
communication.

## Verified by what reaches the renderer ‚Äî and it caught me mid-flight

The first run showed **all 7 Rogue styles as `(none)`** ‚Äî the resolver never animated, so **I had
authored dead data exactly as feared.** I removed them, fixed the resolver, restored them, re-tested.

| | |
|---|---|
| spells drawing their authored style | **12/12** |
| Rogue | thrust ¬∑ crush ¬∑ claw ¬∑ wild ¬∑ thrust ¬∑ execute ¬∑ execute |
| Paladin / Warden | slam ¬∑ execute ¬∑ crush |
| unstyled spells | correctly draw `slash` |

**One apparent failure was my test.** Killing Cut drew nothing ‚Äî because it's an **execute** and
correctly refused a full-health dummy (*"requires the target to be at or below 30% HP"*). At 10% it
draws its `execute`. **The code was right; the fixture was wrong.**

## The lesson, restated

**Don't ask whether the data is there. Ask whether anything reads it.**

I have now written this sentence in four separate reports and been caught by it twice more since. The
only defence that has ever worked is asserting that a value **reaches the thing that consumes it** ‚Äî
not that the field exists, not that the code looks right.
