# V0.18.73 — Crafting Redesign & Centered Windows

> "Crafting UI is still ugly it needs a lot of work, i want a way cleaner UI, also any windows the player opens by pressing a button should always open in the middle of their screen only."

## 1. Crafting window redesign (`systems/crafting-system.js`)
Replaced the inline-styled boxes with a **one-time scoped stylesheet** (`#dr-crafting-styles`) and a proper card layout:
- **Panel**: rounded gradient window, a **sticky draggable header** (title + station subtitle), styled Close button.
- **Recipe cards**: each recipe is a card with —
  - a **coloured item-icon tile** (from the output item's `icon` family/glyph/colour),
  - the **result name** + the **effect/stats** line,
  - **colour-coded material chips** per ingredient — **green** when you have enough, **red** when short — each showing `have/need`,
  - a **meta** line (skill/level/craft-time; turns red with the reason when it isn't craftable),
  - a clear gradient **Craft** button that greys out when disabled.
- **Category tabs**: proper pills with an active gradient state and per-category counts.

## 2. Windows open centered (`systems/ui-system.js` → `installMovableUiWindows`)
`prepareWindow` now attaches a per-window **style MutationObserver** that **re-centers a `.gameWindow` whenever it transitions hidden → visible** (via the existing `game.centerUiWindow`), overriding any previously dragged/stored position — so every button-opened window opens in the **middle of the screen**.
- Dragging while open still works — the observer only fires on the **open transition** (re-entrancy guarded by a `uiCentering` flag).
- **Opt-outs**: the full-screen world map overlay and the **profession/skills pair** (keeps its own side-by-side centering), plus any panel with `data-no-auto-center`.
