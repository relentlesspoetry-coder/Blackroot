# V0.16.46 - Main Menu, Character Preview, and Bot Party Roaming Polish Report

## Summary

Implemented the requested scoped phase only:

- Polished the canonical Blackroot main-menu hero presentation.
- Reframed the Character Stats paper-doll model viewport and preview fit math.
- Improved autonomous bot party sizing and route/objective distribution so bots split into smaller roaming parties.

No unrelated gameplay systems, zones, combat rules, class abilities, inventory contents, or save/account flow were changed.

## Root causes found

1. **Main menu logo visibility**
   - The canonical `#logoSplash::before` hero logo owner still had a strong violet radial overlay and dark vignette directly over the logo art.
   - The menu card padding placed the action stack high enough to cover too much of the logo.
   - Floating class sigils used slow, long-duration movement constants, making the icons feel nearly static.

2. **Character Stats model viewport**
   - The paper-doll viewport had improved CSS sizing from earlier passes, but the renderer still used aggressive fixed scale boosts and a low foot-origin anchor.
   - Tall class models, hats, robes, and held props could still sit too low or crop near the viewport bottom.

3. **Bot party clustering**
   - Autonomous bot parties used the full player-party cap as their party size, allowing large bot clumps.
   - Party members and separate parties could select the same nearby quest/hunt objective and route anchor from camp.
   - The system had autonomous party structures, but no strong party-level route distribution for different parties.

## Bugs fixed / improvements made

- Brightened the Darkroot logo presentation and reduced the purple haze over the hero art.
- Moved the main menu button stack lower so the logo is more visible.
- Increased floating class icon drift distance, speed, and mid-animation movement.
- Enlarged the Character Stats model preview column while keeping the existing stat/equipment information intact.
- Adjusted paper-doll model scale and bottom-origin math so the full model is better contained while still filling the viewport.
- Reduced autonomous bot parties to smaller groups.
- Added party-level route offsets and objective caching so different bot groups split toward different roaming/hunting/gathering areas.
- Added per-member formation offsets around party objectives to reduce stacking.
