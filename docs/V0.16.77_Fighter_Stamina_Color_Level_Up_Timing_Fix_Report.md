## Resource Bar Color Ownership
Resource bar runtime ownership is in `systems/ui-system.js` and the canonical HTML stylesheet.

Added shared resource helpers in `systems/ui-system.js`:

- `actorResourceType(actorOrClass)`
- `actorResourceLabel(actorOrClass)`
- `actorResourceBarKind(actorOrClass)`
- `applyResourceBarType(barNode, fillNode, actorOrClass)`

These map player-facing resource presentation by actor/class/resource type:

- Mana -> blue `mp`
- Stamina -> warm amber/orange `stamina`
- Focus -> green `focus`
- Energy -> yellow `energy`

## UI Panels Updated for Stamina Color Mapping
The shared mapping is now used by:

- Player frame resource bar.
- Mercenary resource bar.
- Unified Party / Companions resource bars for Fighter Bot / Fighter mercenary entries.

Fighter Bot and Fighter mercenary rows now render their resource label as `Stamina` and use the stamina bar styling when their class/resource identity resolves to Fighter/Stamina.

## Level-Up Popup Timing Ownership
Combat level-up popup state, update, and render ownership remains in `systems/ui-system.js`.

Updated timing:

- Old total duration: `5000ms`
- New total duration: `3000ms`
- Old fade-out window: `700ms`
- New fade-out window: `600ms`
- Fade-in window: `250ms`
- Steady phase: approximately `250ms-2400ms`
- Fade-out phase: `2400ms-3000ms`

Popup text was updated from `Fades after 5s` to `Fades after 3s`, and the countdown bar now drains over the 3-second duration.
