# V0.16.41 In-Game UI Root Fix and Revamp Pass

## Summary
Updated Blackroot from V0.16.40 to V0.16.41 for the requested in-game UI bug-fix and revamp phase only. The pass repaired shared DOM/CSS ownership for the action bar, generated icons, bag bar, bag window, item tooltips, player HUD, and world/party context menus.

## Root Causes Found
- The action bar renderer was already inserting icon containers, but the shared stylesheet did not constrain generated image icons inside hotbar slots, allowing large spell art to overflow.
- Generated item/spell/bag icon markup existed, but the shared icon CSS was incomplete, leaving bag icons and item/spell glyphs visually inconsistent or unreadable.
- The previous bag dock rule intentionally stacked the bag bar above the Menu button; this no longer matched the required bottom-right layout between the action bar and Menu button.
- Bag window cells inherited generic slot rules with insufficient fixed grid sizing for inventory item cards.
- The HTML tooltip renderer existed in `systems/inventory-system.js`, but there was no complete top-level tooltip presentation/z-order rule for item cards.
- The world context menu existed in DOM and gameplay routing, but it had no complete fixed-position/z-order/theme CSS, and right-click handling bypassed NPC menus by directly interacting.

## Bugs Fixed
- Necromancer and other class spell icons now render as clipped/scaled thumbnails inside action slots.
- The action bar remains horizontal, bottom-center, and laid out 1 through 0 left-to-right.
- Auto Attack remains on key 1.
- Bag bar is anchored to the bottom-right row between the action bar region and Menu button on standard desktop widths, with responsive compression on smaller widths.
- Bag bar slots now show readable bag artwork, including empty bag placeholders.
- Bag window item slots now use uniform fixed card sizing and readable internal text/icon layout.
- Item hover tooltips have a complete themed floating presentation and high z-order.
- Player HUD now has a compact class-emblem/vitals layout matching the current Blackroot theme.
- Bot/NPC context menus now render above gameplay UI and route NPC right-clicks through the context menu instead of silently opening interaction panels.
