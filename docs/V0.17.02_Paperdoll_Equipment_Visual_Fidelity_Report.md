# V0.17.02 - Paperdoll Equipment Visual Fidelity Revamp Report

## Phase
Paperdoll Equipment Visual Fidelity Revamp Pass. The uploaded brief targeted an older V0.16.98 -> V0.16.99 sequence, but the active baseline was V0.17.01, so this implementation advances the project to V0.17.02.

## Paperdoll visual ownership
Paperdoll visual ownership remains centralized in `render/paperdoll-equipment-renderer.js`. No duplicate paperdoll renderer, equipment system, weapon resolver, or late overlay pass was added. The existing `drawBodyLayer(ctx, rig, layer, arg)` entry point remains the owner used by the humanoid and race renderer draw pipelines.

## Equipment slots upgraded
The pass upgraded the visible slots already supported by the existing paperdoll renderer:

- `cape` / cloak / back item through `drawBack()`
- `legs` through `drawLegs()`
- `feet` / boots through `drawLegs()`
- `chest` / robe / armor through `drawTorso()`
- `shoulders` through `drawTorso()`
- `hands` / gloves / bracers through `drawArmGear()`
- `head` / helm / hood / hat through `drawHeadGear()`
- `amulet` / `charm` body jewelry through `drawTorso()`
- ring / earring jewelry through the existing `drawJewelry()` path
- weapons/offhands remain handled by the already-upgraded canonical weapon helpers in the same file

Slots that are not currently surfaced to the paperdoll renderer by the equipment model were not forced into a new slot system.

## Draw helpers added or updated
Added owner-local lightweight visual helpers:

- `rarityKey()` / `detailTier()`
- `classThemeName()`
- `materialKind()`
- `armorTheme()`
- `drawRivet()`
- `drawBuckle()`
- `drawShortStitches()`
- `drawRarityRunes()`
- `drawMaterialNoise()`
- `drawLayeredBand()`

These helpers are pure Canvas 2D shape/stroke helpers and do not allocate images, canvases, or external assets per frame.

## Class themes
Class identity is applied through item text, class name, armor category, material classification, and rarity accent. The renderer now distinguishes major material themes including cloth, leather, metal, holy, arcane, nature, bone/necromantic, and ritual/shamanic gear. The pass does not change class models or class roles.

## Rarity accents
Rarity accents are derived from existing item rarity metadata only. The pass adds sharper runes, brighter trim, gems, buckles, and stitching for higher rarity tiers without changing rarity values, item stats, or loot behavior.

## Equipped item overrides preserved
The existing equipment visual resolution remains intact:

1. Equipped items are resolved through `slotItem()` / `resolveVisualSlotItem()`.
2. Runtime equipment instances are hydrated from `DR.ITEM_BY_ID` / `DR.ITEM_DRAFTS` through `hydrateVisualItem()`.
3. Equipped item visuals override class defaults where valid.
4. Class default visuals remain available for weapon/offhand fallback when equipped data is missing.
5. Unknown items still use neutral item/category fallback logic, not white sticks or a global crossbow fallback.

No save fields or item data were changed.

## Direction-aware layering and animation anchors
The pass preserves existing direction-aware draw order:

- back/cloak layers render behind the torso
- legs/feet follow the existing leg anchors
- torso/robe/chest gear follows chest and pelvis anchors
- shoulder pieces attach to shoulder anchors
- gloves/bracers follow elbow/hand anchors
- headgear follows head anchors
- weapon/offhand layers retain the existing `weaponDepthLayer(rig)` back/front split

This preserves back-facing weapon/body occlusion, arm anchors, weapon anchors, and the prior fixes for white-stick placeholders, global crossbow fallback, and stretched auto-attack arms.

## Protected systems
No changes were made to item stats, rarity values, drop rates, loot logic, inventory behavior, equip/unequip rules, combat stats, class balance, spell behavior, class base models, save format, or action bars.

## Known limitations
Some full-body class-specific renderers intentionally suppress the generic paperdoll overlay to avoid drawing a second model over a bespoke silhouette. The pass documents and preserves that safety behavior rather than forcing unsupported slots into those renderers.
