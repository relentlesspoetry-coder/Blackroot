# V0.18.40 — Silk Web Cavern floor webs actually render (+ heavier walls)

**Report:** *"The floors still aren't covered in webs and the walls still look like shit. The dungeon still doesn't feel like a spider home."*

## Why the floor kept looking bare (the real root cause)
The dense floor webbing added in V0.18.37/V0.18.38 **never actually rendered.** The tell in the screenshots was that the **walls** had web drapes but the **floor** was bare — same file, different render path.

- The Silk Web Cavern floor is **chunked terrain**: `render/terrain-renderer.js` `isTerrainChunkGroundTile()` returns `true` for `CAVE_FLOOR` tiles, so the floor surface is drawn from a **cached chunk bitmap** (`drawTerrainChunks`).
- For those chunked ground tiles, `drawTile()` runs only in `depth-underlay` / `occlusion-cap` mode and **returns early** (before the per-tile surface-detail block at the bottom of `drawTile`, which is where `drawSilkWebCavernSurfaceDetails` is called). So the floor mat was effectively **dead code for the actual floor**.
- The **walls** rendered web because cave walls go through the separate `drawCaveWallDepthOccluder` pass, which is **not** chunk-bypassed — which is exactly why only the walls ever showed webbing.

## Fix 1 — dedicated floor overlay (`render/terrain-renderer.js` + `game.js`)
New `renderSilkWebFloorOverlay(minX, maxX, minY, maxY)`: iterates the **visible** silk `CAVE_FLOOR` tiles and draws the woven-silk mat (`drawSilkWebCavernSurfaceDetails`) **directly over the chunk-cached terrain, every frame**. It's wired into `game.js` immediately after the terrain passes and before the entity/object pass — the same slot the V0.18.34 resource ground layer uses — so the webbing sits over the floor but under actors. Gated to `currentZone === 'dungeon'` and the active dungeon being `silk_web_cavern`.

**Verified in-browser:** with a faked silk-cavern zone and a 5×5 silk floor, the overlay drew **~25229 web px** (dense coverage). The floor mat itself (film base + back layer + dense front lattice + orb hubs + thin dirt paths from a coherent noise mask) is the V0.18.38 render — it just finally runs.

## Fix 2 — heavier walls (`render/terrain-renderer.js` `drawSilkCavernWallWebs`)
Rewritten to read as caked-in web rather than lightly draped: a **top-weighted gradient silk sheet** over each face, **14 thick vertical drapes**, a **diagonal cross-weave**, **6 horizontal binding threads**, dangling silk beads, and **orb webs in both top corners**. Verified **~9383 px** for two faces (was ~5773).

## Important: floors regenerate on entry
Dungeon floors are generated fresh each entry, and the tile `silkWebCavern` flags (which the overlay reads) are set during generation — so **re-enter the Silk Web Cavern** (or change floors) to see the fully webbed floor.
