# V0.16.53 - Action Bar UI Asset Integration Report

## Summary
Integrated the provided PNG as the canonical in-game action bar shell under the stable asset name `Action Bar UI`. The existing action bar DOM, hotkeys, spell slots, EXP meter, cooldown state, Auto Attack state, and Meditate button remain the live gameplay owners; the PNG is used only as the visual frame/background.

## Implementation approach
- Copied the uploaded action bar artwork into `assets/ui/Action Bar UI.png`.
- Registered the asset metadata in `systems/ui-system.js` as `Action Bar UI` so code and docs use a stable readable name rather than an upload/hash filename.
- Updated the canonical `#hotbar.abilityBar` owner markup with `data-ui-asset="Action Bar UI"` and `data-ui-asset-file="Action Bar UI.png"`.
- Reworked the existing action bar CSS owner so the PNG is drawn as the shell and existing child elements are positioned as live overlays.
- Preserved the original `#hotbar` input event delegation in `systems/spell-system.js`; no duplicate input path or action bar renderer was added.

## Asset integration notes
- Canonical asset name: `Action Bar UI`
- Canonical asset file: `Action Bar UI.png`
- Runtime path: `assets/ui/Action Bar UI.png`
- Natural size: `2039 x 676`
- The PNG is used as the static art shell only. Gameplay data remains runtime-rendered by the existing DOM/action bar systems.

## Overlay mapping details
- `#hotbar.abilityBar[data-ui-asset="Action Bar UI"]` uses the PNG as a contain-fitted background with the original aspect ratio preserved.
- `.abilitySlots` is absolutely positioned over the slot-frame region of the PNG.
- Each `.actionSlot` remains the clickable runtime slot and carries its original `data-action-slot` / `data-spell-index` state.
- Hotkey labels, icons, spell names, cooldown fills, readouts, and active outlines are positioned inside the visible PNG slot frames.

## EXP bar mapping details
- `.actionXpMeter` is mapped to the top track region of the asset.
- `#xpFill` remains the live EXP fill and scales left-to-right based on current player EXP.
- `#xpText` remains a live text overlay.

## Slot hitbox alignment notes
- Slot hitboxes remain the existing `.actionSlot` nodes.
- The slot row is derived from the same asset-scaled container as the PNG background, keeping pointer/click bounds aligned to the visible slot frames.
- Keybind handling remains owned by the existing hotbar input delegate.

## Meditate button hitbox alignment notes
- `#steamDeckMeditateBtn` is positioned over the visible Meditate region of the PNG.
- The button keeps its existing runtime behavior and event binding.
- The button uses transparent styling so the PNG shell remains the visual source while the live button remains clickable and keyboard-accessible.
