## Bottom-Space Cleanup

- Removed the forced `height: min(900px, calc(100vh - 24px))` from the Character window.
- Removed the sheet's forced `height: 100%` and made the panel wrap its actual content.
- Retained a viewport-safe maximum height and scrolling fallback so smaller windows do not clip real content.
- Changed `characterStats` from filling arbitrary remaining height to sizing from its rendered sheet.

## Top Repeated-Section Cleanup

- Removed the render-time header crest and duplicate player name/level/gender/class summary.
- Kept one compact static `Character` title and the existing close control.
- Removed now-dead header identity/crest/title-block styling from the owning Character window rules.
- The complete character identity remains in the Character Stats panel where it already accompanies gear score.

## Container Sizing and Layout

- The two-column Character Model / Character Stats structure remains unchanged.
- Reduced header minimum height from 54px to 38px and reduced close control size from 50px to 36px.
- Tightened outer padding and header spacing without changing equipment or stat card contents.
- The existing responsive single-column fallback remains available for narrow viewports.

## Character Model Class Background

- `renderCharacterPanel` now applies the existing shared class visual variables to `characterPanel`.
- The model stage reads the canonical `--party-class-bg` variable through a dedicated background layer.
- A dark gradient and 48% opacity keep the background behind the paper-doll model and preserve gear visibility.
- No Assassin-specific or other per-class branch was added.
